# <span>关于本周集团例会上讨论的是否该增加预算审核层次的问题，我觉得是一个如何在现有业务中推进erp管理的大课题，以下是我的一些思考。</span>
# <span>事前控制与事后惩戒</span>
- <span>缺陷管理和报表管理都是事后管理和惩戒，往往损失和危害已经造成。</span>
- <span>现在我们面临的情况是：业务干部都是有经验的老资格，业绩突出，用事后惩戒的方式往往处罚起来投鼠忌器，或者不痛不痒，而对公司管理的损失已经造成。同时问题往往需要很长一段时间才能被彻底解决，一但换新人又会发生。</span>
- <span>我个人看法应该是尽量通过系统进行前置管理：如果不按要求完成相应管理动作，业务在系统上就无法走下去，这样才能有效地利用系统要求大家强制按流程执行。</span>
- <span>这样也能够快速将现有流程复制到新业务板块，而不是从头再来教育一遍，公司还要付出一定的学费，同时耗费很多时间。</span>
# <span>针对预估、预算、结算环节的控制点增加</span>
- <span>针对预算的审核，本来我考虑除了增加研发和研发负责人两级审核外，如果与预估毛利率差距在30%以上的，还应跳到当地总经理处审核。如果造成巨额亏损的（如5万/团以上），要跳到集团业务监控专人处备案核实。</span>
- <span>预估必须在开团时完成，否则无法完成开团动作，保证在开团时就有一个预估毛利率出来，便于管理层掌握下一季度的财务预测。</span>
- <span>预算如果无法在出团前完成的，应通知到研发负责人，某个研发如果每月有三次无法按时完成结算的情况，应通知到总经理。某个分公司如果当月有20%的团无法准时完成预算的，应通知到集团层面业务负责人，在下月的集团例会上应做出解释。</span>
- <span>结算与预算有毛利差异的，应该跳到研发负责人处审核；如果有20%毛利差异的，应跳到总经理处审核。超过100%毛利差异的，应跳到集团业务监管处审核。</span>
- <span>结算完成后如还有金额调整的，应选择“机损调整”、“客诉赔偿”等对应标签，毛利减少达到一定金额的，应抄送集团关注。</span>
- <span>按照以上做法，可以保证我们全国的团队预估、预算、结算环节都纳入事前统一管理，有问题及时解决，而不是事后再来追究责任，这时损失可能已经发生了。</span>
# <span>关于广州提出不需要更多控制点的意见</span>
- <span>我昨天会后跟任霞通了电话，广州的看法是认为她们欧洲产品经理有十多年的经验，同时忠诚度也得到了验证，所以多一层审核意义不大。但这还是依赖特定人员的管理。</span>
- <span>据我所知，从2019年的财务数据来看，广州年底毛利数据有大幅调整，而北京系统结算数据与财务数据基本吻合，是否表示广州在erp过程管理上还有较大空间，需要加强控制？</span>
- <span>如果拓展到中东非、日韩等新业务，或者假设欧洲骨干疫情期间有大幅流失的情况，广州表示那肯定是需要增加审核层次的。</span>
# <span>参考携程业务管理模式的问题</span>
- <span>管理颗粒度的不同：以我的体会，携程业务各环节的管控跟上海公司的精细度是有很大差距的，这也是邹雨过去携程后欧洲业绩能有较大提升的原因。</span>
- <span>管理范围的不同：携程的自营业务主要还是集中在上海，各地因为占比低所以被要求完全服从上海的管理模式。但华程的分公司（上海、广州）业绩占比很高，要如何做到加强管控又不扼杀当地活力，需要更多听取分公司的合理管理建议。</span>
# <span>目前是较好的推进时间点</span>
- <span>疫情期间增加业务控制点，可以让大家在业务量小的时候逐步习惯规范的操作模式，在平时业务旺季要加强流程管理会遇到很多反弹。</span>
# <span>结语</span>
- <span>通过erp系统第一次有机会让华程集团业务运行过程能够全面、即时、准确地呈现在集团层面的雷达图上。</span>
- <span>原则上目前应大力推进系统控制点的设置，目标是一杆子插到底，把整个集团的业务细节都纳入管控范围。这样形成的管理体系将是我们华程的核心资产。</span>
