# <span>对华程未来的一些基本设定：</span>
- <span>华程定位于一家提供优质旅游产品的旅游公司，我们的工作重点是产品研发、品牌推广和渠道开拓。</span>
- <span>我们的售卖渠道包括但不限于线下门店、OTA、电商平台、个人旅游顾问、异业合作伙伴等。我们只提供旅游产品和旅游服务，对终端消费者的主要预订服务工作由渠道合作伙伴完成。</span>
- <span>在可预见的未来，我们公司的产品研发中心还是将分布在华东、华北、华南，个人认为这是华远/华程的价值和优势，所以才能在集团总部设于北京的情况下，在华东和华南市场获得优势地位。以携程旅游自营品牌的发展历程来看，始终无法走出上海市场，无法在华北、华南、华西、华中市场形成足够的影响力，个人认为与携程自营的产品集中化管理模式有直接关系。</span>
- <span>客户规模：在未来可预见的时间内，华程稳定合作的B端合作伙伴数量规模预计在“千”的规模，小几千到大几千，但不太可能超过“万”。</span>
# <span>平台客服的局限分析：</span>
- <span>平台客服人员素质长远来看无法有效提高，因为我们对他们的岗位职责设定就是从事简单重复的工作，不可能支付较高的薪资。但我认为旅游行业B端销售跟房产中介、保险中介类似，销售过程和客户服务过程是无法分割的，一个销售人员如果服务热情，是可以获得B端客户的好感，从而获得二次销售的机会的。我们应该是想办法通过信息技术手段来简化或自动化一些服务过程，从而提高销售人员的效率，但不应该把这两个过程切分到两个不同部门，个人认为这会造成两个部门之间的互相推诿和指责，影响客户的服务感受，从而影响业绩。</span>
- <span>平台统一客服的价值在于通过牺牲服务的个性化，以服务的标准化来换取客户对服务的统一感受，同时降低成本。但这种标准化服务的效率优势体现在大规模的客户数量上，比如百万级别，这样才能分摊成本。如果可预见的客户规模有限，就会导致服务成本无法分摊而成本过高，同时客户也得不到更优质的个性化服务。</span>
- <span>平台客服因为需要较大规模优势才能有效管理和分摊成本，所以在面对华程这种分区域进行产品研发的模式时，会产生管理上的困难（各区域产品不同），沟通成本很高。</span>
- <span>平台客服为了提高效率，会采取随机抽订单服务的模式，一个事情还可能分几个人跟进，会造成客户的不便。如果是随机交易的终端消费者，这还能接受，但换成长年稳定合作的渠道伙伴时，这种不便会造成渠道伙伴较大的不满。</span>
- <span>平台客服集中在上海，面对各地不同渠道客户使用方言的情况，目前很难储备足够的各种方言客服人员。跟渠道伙伴用方言沟通，能快速拉进双方感情。</span>
# <span>结论</span>
- <span>公司的未来在于信息化，这毫无疑问，市场上所有公司都这么说，关键在于怎么实现的路径，现实情况是成功转型的公司百里挑一。而目前这种通过分割岗位、流程来提高效率的做法我认为方向是不对的，随着信息技术的进步，应该是更多的职责可以集中一个岗位上得到快速判断和决策，让一个岗位可以尽量为一个完整的工作职责负责，这才是本质上效率的提高。对销售这种需要主动性、专业能力和决策能力的岗位来说，尤其如此。</span>
  - <span>德鲁克在《管理的实践》第21章“人事管理是否已告彻底失败”中就指出了所谓科学管理的误区：科学管理强调分解岗位流程和设定标准动作，但这只是第一步，重要的是把这些连续流程动作整合到一个岗位上，才能充分发挥员工的价值和主动性，而不是每个岗位只负责一个动作。</span>
# <span>以前完全由销售面对客户时，客户有任何问题销售部门一定是先把客户安抚下来，内部的问题回来再来处理。现在把面对客户的销售过程和服务过程分成两个部门负责，一旦发生客户抱怨，到底是哪个部门要负责，大家一定会互相指责，从目前大家一上来就要划分清楚工作内容和责任范围就可以预见到，即使强行要求销售出面去安抚，某些销售肯定会把责任推给内部客服部门，因为他感觉无法掌控客服与客户的沟通方式和内容。这种情况下公司利益肯定是受损了而不是提高了。</span>
# <span>从这次的决策过程来说，我个人也有一些看法：据我了解，这么重大的业务流程调整决策并没有通过5人管理层开会形成正式会议决策再推行，更不要说开扩大会议以听取更广泛的意见（我是前周才听说此事）。如果只是听取流程技术部门单方面的意见就开始推行，个人认为这只会造成技术流程部门和业务部门之间更加严重的隔阂和对立，并不会有利于公司快速推进信息化的整体目标的实现。技术流程部门负责人一直抱怨业务部门不配合系统推进工作，认为他们不作为或有私利，我个人认为这种对同事的看法从职业精神角度来说是非常不妥的。以我对上海公司团队的了解，他们以前虽然没有系统，但流程管理是非常严谨的，团队同事落实管理流程的态度也是一板一眼的，我个人认为是比携程业务管理要精细的。技术流程和业务部门之间应该还是互相尊重，倾听和解决对方的关切，而不是从道德层面去指责对方。这种道德指责是无法证伪的，也是无解的。</span>
