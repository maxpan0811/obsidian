# <span>领导，关于我们昨天沟通的几个问题，当面沟通时可能有点凌乱，这里我再做一个梳理。</span>
- <span>关于组织结构的优化</span>
  - <span>重点是压实经营责任</span>
    - <span>欧洲团的收益取决于每个团是否能收满：做10个团如果每个团不能达到定价人档，还不如做5个团超过定价人档赚钱，因为我们的机票承担着巨大的风险。这代表着我们欧洲业务在微观层面并不具备规模效应，需要每个团每个团去努力，这是我这几年观察下来我们欧洲业务的收益管理关键，我们必须基于这个前提来做经营决策。</span>
    - <span>每个经营团队必须具备高度的责任心和业务能力才能做好业务，其中责任心还是在首位，韩晓莉的成长经历可以充分说明这个道理。</span>
    - <span>某个区域、某个产品的经营结果必须落实在一个具体的人身上，这个人必须签署相应的经责书，而不能有两个或两个以上的人来承担同一个责任，这样其实就是无人承担责任。</span>
    - <span>对有航点的出发城市，可以认为是有客户需求的，就应该有相应干部承担市场开发经营责任，集团应该直接关注到这个市场的经营情况，单独核算并把相关的经营数据在每次月会上单列进行分析讨论。</span>
    - <span>后续我们如果考虑追求更高的收益而不是规模，更应该把经营责任压实在每个小团队身上，这样才能充分发挥每个小团队的积极性，在每个区域市场充分挖掘每个团的潜力，提高收益。</span>
    - <span>经营权责到人才能更好地培养经营干部。</span>
    - <span class="bold">综合以上，我建议我们的组织结构优化应该是朝着更好地压实经营权责到人的方向走，而不是相反。</span>
  - <span>阿米巴经营的导入</span>
    - <span>我这一年多来在盛和塾学习，发现阿米巴经营比较适合我们做提高经营收益的管理，阿米巴经营就是要求把经营责任压实到尽可能小的经营单位，对其进行独立经营核算并反馈给相应经营干部，以提高每个经营单位的收益。有反馈才会有改善。</span>
- <span>关于您和郭总关心的几个问题：您和郭总提出的几个问题确实在目前经营过程中是存在的，但我觉得具体的解决办法是可以商榷的。</span>
  - <span>提高经营效率</span>
    - <span>重点还是加强ERP系统的使用以减少经营过程中的“跑冒滴漏”，这可以相当程度提高净利。</span>
      - <span class="bold">我认为我们的ERP系统在使用上存在很大问题，因为并没有和我们的管理流程结合。老板不用ERP数据做管理，下面的人肯定不会好好用系统，因为老板的关注力在哪里，下属们的工作重心就在哪里。</span>
      - <span>团队结算工作：</span>
        - <span>渠道返佣的录入：我们北分是要求把所有团队涉及的费用都录入到每个团里，其中最容易遗漏的就是给代理商的返佣。比如OTA基本都有2%的返佣，其他线下大客户也有类似的返佣需求，我初期是要求销售手工录入，后面是请朝阳帮忙开发了一个针对特定客户录入返佣政策的功能。北分在我要求了一段时间后，基本能做到每月月底ERP系统内的团队核算报表数据与财务报表数据几乎一致（除去财务自己的一些做账调整项）。在管理电商团队期间我也是这么要求的，但晓莉一直不太理解，我走了之后应该上海电商就没有这样执行了，我听说在疫情期间与同程结账时发生几十万的返佣争议，应该就是这个问题。广州公司2019年年底我听说也发生了渠道返佣到年底大幅调整影响利润的情况，这都是没有严格每团录入渠道返佣的问题，如果我们每个月都在盯系统数据，问题就不会积累这么久。</span>
        - <span>预算和结算的差异：我们系统内其实是可以设置预算和结算利润差异在多少比例时就自动跳到上一级审批的功能，我不知道其他公司有没有在用，我北分是要求设置在1%的差异就要到我这里审核，因为假设毛利率在10%，1%的结算差异就是导致10%的净利润不见了。通过这种严格审核，就会发现哪些产品经理存在问题，有可能是工作细心度、工作能力，甚至是诚信的问题。</span><span class="bold">乔立明那个私自调整团队利润的事就是我在审核利润差异时发现的，</span><span>我相信严抓这一块全国一定会有很多问题暴露出来。</span><span class="bold">我建议可以让集团总裁助理（林必智个性和能力都适合）来协助总裁</span><span>严格监督团队预估、预算和结算中存在的差异问题，</span><span class="bold">把系统设置为每个阶段利润差异在多少比例以上就必须跳到总裁助理处审核，这样既不影响区域经营团队的经营主动性，又能充分保证集团对具体经营过程异常的监控。</span><span>另外产品定价调整和订单价格调整也可以这样做，超过多少比例的价格调整就会跳到集团层面审核，这样就可以充分起到监督的作用。</span>
        - <span>预估和预算的差异</span>
          - <span>之前大家嫌上线的团太多都要做预估太麻烦，因为很多团其实都不会成，只是先录入产品放着。其实可以让大家把自己预估能成的团先从预估阶段做起，这样也可以事后看看大家的预估成团能力。</span>
          - <span>预估能成的团与实际成行的团做比较，可以看出我们预估的价格和人档是否能做到，如果经常是到最后收客价格相比预估价格大跳水，那就是需要改进的。</span>
      - <span>制定年度预算：我要求我的团队做每年业务预算时销售端和产品端的数字必须能够对齐，</span><span class="bold">同时产品端必须具体到产品线，销售端必须具体到客户。</span>
        - <span>因为产品线和客户在系统内都有相应的历史数据，在系统内做明年预算时可以直接参考。</span>
          - <span>产品经理</span>
            - <span>已有产品线：在自己现有的产品线上直接填明年不同月份的团数、人档数、预估卖价、预估毛利等数据，而今年和去年该产品在不同月份的实际团数、成团人数、实际成交价和实际毛利也都自动带出，供产品经理参考。</span>
              - <span>已有产品线如果预估明年产量下降必须写出原因。</span>
            - <span>新产品线：产品经理简单填写想做的新产品名称和简要描述，在明年不同月份填入预估团数、预估人档、预估卖价和预估毛利。</span>
          - <span>销售</span>
            - <span>已有客户：基于该客户今年和去年的实际各产品销售量，填入相应明年预估的各产品销售量。如果明年公司有增加新产品，也要预估该客户在新产品上的产量。</span>
              - <span>如果已有客户预估明年产量下降必须写出原因。</span>
            - <span>新客户：在新客户导入的月份填入预估各产品线的销售量。</span>
            - <span>通过这种方式可以保证系统内客户信息数据的真实性，大客户在第二年如果销售发生较大下降也可以通过实际和预算的差异第一时间发现。</span>
        - <span>以上是携程多年总结出来行之有效的预算制定方法，通过固化在ERP系统内可以让所有业务人员严格按照公司要求思考、制定预算，并通过在第二年实际业务运行过程中的数据差异，发现问题，提高预算制定能力。</span>
      - <span>机票申请决策工作：机票是我们经营中最大的风险，但整个机票采购决策过程中缺少监控。</span>
        - <span>机票的申请量必须是当地销售负责人、相应产品经理和相应票务经理一起商量后决策的结果，如果三方无法达成一致，则由上一级主官拍板决策。这个决策方法论必须是固化在ERP系统内部。</span>
        - <span>在填写每季度机位数量时，系统应自动带出之前两年同期该目的地区域的机票实际使用量供申请人参考，具体到航司、日期、价格和开票量。</span>
        - <span>机位申请表由各区域产品经理具体填写（和销售商量后），对应票务经理、分公司的票务总监审核通过后，由分公司总经理最后在系统内签字认可（当地销售负责人可以会签）。如果机票政策风险超过一定金额（按每张票）的申请，可以跳到集团票务总经理处复核。</span>
        - <span>机票的申请量和实际使用量差异应该作为重要的运营参考数据，每月复盘会分析，以提高机票申请的精准度。</span>
      - <span>销售客户信息管理</span>
        - <span>对系统内销售客户的关键信息设置必填，并加上数据校验。</span>
        - <span class="bold">最关键的是要有定期的客户信息真实性抽查，派人打电话抽查客户数据的真实性，或者去门店现场核实。也就是必须形成管理闭环，大家才会真正使用ERP。</span><span>这个抽查工作可以在集团层面来完成，可以与之前的产品异常数据审核放在一起。</span>
    - <span>以上是我在过往工作中对ERP系统使用的一些实际做法和设想，我认为ERP是我们公司的核心竞争力，因为可以固化我们行之有效的管理经验和实际经营数据，做得越久越有价值。以后我们在国内各个城市扩展业务，增设新城市的业务负责人，就不用一个个苦口婆心地教方法，只要严格按照系统流程操作就行。我相信欧洲的航点可能在未来中国主要大城市都会有，我们经营的颗粒度必须跟上这个速度，而这种扩张最大的难点就在于经营风险（机位和降价）的把控。</span>
    - <span>ERP系统还是要与我们实际的管理流程相结合，</span><span class="bold">老板要自己一直用ERP数据做实际管理，这样的话ERP就会变成管理过程的一部分，自然使用度、准确度就上来了。</span><span>我这几年最大的困扰是和技术产品部门的沟通问题，在薛薛管理之后基本我的技术需求都被打回来了，很多想法没法落地。但我认为ERP对生产效率的提高是实实在在的，并且时刻在积累有价值的经营数据。在目前的管理结构下，特别需要老板亲自介入技术和业务的整合过程，才能真正发挥ERP系统的作用。</span>
  - <span>控制人员的快速增长</span>
    - <span>新业务：新的产品板块建议先打样做出实际业绩再大规模加人。</span>
    - <span>老业务：严格卡人效，人效到了再逐步加人。</span>
  - <span>提高产品创新研发能力和效益</span>
    - <span class="bold">创新产品采取指导性计划而不是指令性计划：</span>
      - <span>产品是否适销对路应该由市场决定而不是我们自己决定，否则容易错失市场机会。</span>
      - <span>广州过往的产品规划也不是全部都成功的，如果全国限制由一个团队做产品规划，那这个团队的能力就是我们集团的天花板了。类似计划经济的问题。</span>
      - <span>要相信大部分公司的经营干部都是希望把业务做好的，也是有学习能力的，好的产品指导计划如果有实际业绩，大家肯定是愿意跟进的。但如果采取指令性计划，要求大家只能在一个产品筐里选产品，会影响大家的经营灵活性，导致经营责任不清。</span>
      - <span>我们需要做的是不断树立好的产品经营典型，鼓励各经营单位跟进，对一直不能学习跟进的经营干部逐步淘汰。</span>
    - <span>对产品创新采取奖励而不是限制处罚的方式，因为创新是无法计划的。</span>
      - <span>对当年产出优异的创新产品可以在集团范围内评选“年度最佳产品“和”年度最佳产品经理“，予以相应重奖。</span>
  - <span>产品品质控制</span>
    - <span>后端控制供应商的选择范围：必须在集团产品委员会认证过的供应商清单里选择。</span>
    - <span>前端客户反馈收集：通过行程助手100%回收客户反馈，内部设立面对消费者的QA岗位，统计相关数据、分析问题。</span>
# <span>结论：</span>
- <span>我建议在我们这个业态中应该不断缩小业务独立核算单位（在核算上变小不代表组织结构变小），压实每个小团队经营干部的经营责任，提高收益压低费用，以提高整体经营利润。</span>
- <span>通过ERP系统把整个经营过程规范化、透明化，减少生产损耗，积累经营数据，不断提高团队的整体运营效率和水平。</span>
