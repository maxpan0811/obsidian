# 《左晖》读书笔记---潘渤

---

郭总，我也谈一下我对这本左晖访谈录中关于企业基础能力建设的体会：P122提到，链家内部每三年都会有一个大的变化，比如2007年是数据库，就是楼盘字典、2011年是线上真房源、2014年开始建全国网络、2017年做平台。我注意到，这里面并不是简单地说我搞了一个什么东西出来，而都是强调解决了一个什么问题：楼盘字典、线上真房源、全国网络、平台等等。可能过程中有一些相应配套的软件、硬件、制度、投资等方面的措施，但重点是围绕解决问题。

结合到我们华程的实际情况，我觉得我们也要制定每年的问题解决计划，这个计划不仅仅是每年要开发几个系统，而是一整套包含业务、技术、制度甚至投资的组合动作，是每年要真正彻底解决一些问题，实现能力的提升。这个计划要分解到月、周，每个部门需要完成哪些事情、做到什么标准，每次集团例会上review进度，没有完成的必须说清楚问题，如果发现执行不力的，就换人，不管是什么级别的干部。有这样一个明确进度表后，就可以避免一些部门间无谓的互相指责，领导层也能随时把握到真实进度。

我们现在强调信息化建设是重中之重，那以我的理解来整理一下我们这几年信息化建设的进展（想要解决的问题）。可能不是很完整和有盲点，主要是从我个人的视角来观察公司的信息化进展。

* 库存分配问题：大家开始通过ERP下单、线上实时分配库存。从人工占位变成了系统占位，以前在上海旺季抢库存大家都要打起来了，现在不会了。
* 产品质量数据收集问题：通过行程助手，客人旅游过程中的不满意见后台可以实时掌握、及时解决，导游的质量评估可以量化考核。
* 行政事务管理问题：通过OA系统，所有出差、报销、请假等事项实现在线管理。
* 电子分销问题：

+ 2019年推出了翔龙小程序，北京公司推的最积极，结果说是to C的，暂时搁置；目前在试用EBK的PC版，但使用场景与门店实际情况有较大差异，大范围推广使用可能有问题；新的小程序据说6月份出来。未来可能还是需要通过app的形式最终解决问题。
+ 主要是实现客户在线查询产品信息、下单占库存、查询签证进度、查询促销活动、查询账户余额、社交分享等功能。未来可以考虑把微信账号和EBK打通，学习很多其他服务行业采用微信机器人的方式更方便客人自助服务。比方说客户在我们微信公众号输入产品ID数字，就能快速获得未来三个月该产品的价格、班期、库存数据，会大大提高客户查询的便利度和满意度。门店也可以通过输入客人护照号来快速查询客人送签进度。
+ 门店管理：通过手机的定位功能，EBK可以搜集我们活跃客户的地域分布，了解我们渠道的实际覆盖度，指导销售有针对性地开发新的零售门店或加强现有门店合作；也有利于集团层面掌握我们各地真实的门店数量和活跃度，进行定期检查。

* 产品预算管理问题：

+ 目前erp里预算、结算录入已经做得比较规范了，但缺少产品预估环节，无法在产品上线时就大概评估可能的收益情况；
+ 预算和结算的差异很大，没有严格的审核制度和追责的管理制度；
+ erp和财务系统没有完全打通，业务看的ERP数据和财务看的报表还是两本账。
+ 我一直认为产品预算管理是目前可以最大程度提升我们盈利水平、防控业务风险的环节，但目前预算和结算感觉主要还是一个数据录入环节，并没有真正发挥出管理控制和评估的作用，需要有更进一步配套的管理制度跟上。

* 消费者数据管理问题：

+ 不管以后我们是继续to B还是要to C，跟终端消费者直接互动是必须做的事情，我们系统里有大量消费者信息和资料，但并没有一个完整的组织形式，分布在各处。
+ 系统里现有信息数据的真假存疑，同时并没有严格规范新数据录入的准确性。
+ 消费者数据对市场推广、产品开发的价值贡献渠道没有确立。
+ 现有关注了行程助手服务号的11万粉丝要怎么有效利用起来，并提高这些客人的复购率。

* 突破新的旅游顾问渠道：

+ 首先业务端要搞清楚哪里可以发展旅顾，旅顾的痛点是什么，后面才是系统配合问题。

* 突破新的异业合作渠道：

+ 首先业务端要搞清楚哪些异业渠道有更多的合作意愿和潜力，他们的痛点是什么，再考虑做系统和客服。

何总评价我们这几年信息化工作推进慢，我理解不是说我们各部门同事不努力，其实大家都做了很多工作，特别是技术流程部门，做了很多看不见的基础工作，增强了系统的易用性和稳定性。但从整体上来看，我们很多新的构想都没有实现，或者实现得不如预期，并没有在管理上产生价值。我总觉得信息化是个系统工程，不仅是技术部门的事情，业务和管理跟不上的话，是达不到我们预期的效果的。

综上，我是建议在集团层面围绕解决具体问题，设定每年的工作计划并分解到具体时间、具体部门，定期评估进度，这样就能比较清晰地看出问题卡在哪里，到底是哪个环节有问题，也能有针对性地做出调整和推进。通过这种有组织、整体推进的方式，才能保证我们信息化目标的完成，真正让信息系统提升我们的公司管理水平和盈利水平。

> > > >
> > > > 1.左晖的厉害之处：有能力构建起一个庞大的线下服务网络；有能力把线上和线下结合起来；成功地让链家从一个自营业务变成
> > > >
> > > > 了贝壳这样的一个平台公司。
> > > >
> > > > 2.一个场景：随着组织变大，掌握的资源越来越多，永远有个命题，就是这事你干不干，有一个判断的方法：这个事情是不是足够
> > > >
> > > > 大，跟我们的使命是不是一样？我们到底要解决什么问题。长期利益角度的价值排序：客户、员工、股东。
> > > >
> > > > 3.做难而正确的事，很多成功是时代造就的，并不是自己有多能干。组织比较好的状态是轻盈，内部没有什么博弈，大家高度统一，
> > > >
> > > > 实际很有质量。要创造价值，要选择较难的路（有挑战）。
> > > >
> > > > 4.在组织面前，把自己当成客体来看，就会少点主观的东西；用品质赢得正循环；先纵后横（先做业务，再做平台），产业互联网跟消费互联网有很大差别。
> > > >
> > > > 5.平台发展的三要素平衡：规模、效率、品质。做平台要规模（流量）必须短期内获得足够大的势能，形成比较大的规模。同时，效率
> > > >
> > > > 和品质不能坍塌式的下滑。-----
> > > >
> > > > 6.做任何垂类，线上和线下都非常重要，线上的人获得线下的的能力比线下的人获得线上的能力要容易，线下很难打败线上。做小而美的
> > > >
> > > > 公司容易，做大而美的企业难。品质的规模化复制，从具象到抽象，真正能走过的人很少。---高度抽象能力。
> > > >
> > > > 7.竞争思维是低效率的。关心用户要远胜过关心对手，让自己变强＞让对手变弱。真正强的团队就是承认先进、学些先进、赶超先进。
> > > >
> > > > 8.选人与授权：怎么挑选CEO，足够聪明，抽象能力足够强；坚韧；有格局。保持高频沟通，充分授权决策。人性的本质需要被激励。
> > > >
> > > > 9.长期主义：做好了准备，做这些事情今天得不到回报，明天得不到回报，可能要后天甚至大后天才能得到一点回报。所谓做好准备
> > > >
> > > > 的核心在于钱要足够厚。大多数人在关键事情上的判断错误都是把别人的事当成自己的事情，对我们来说，核心在于我们自己怎么样。
> > > >
> > > > 自己的事情是确定的，担心大环境的不确定影响自己的确定吗？
> > > >
> > > > 10.对于一个组织来说，招人选人是最大的事情，员工不要其他，公平就足够了。
> > > >
> > > > 11.重公司，慢公司。从线下往线上走，从慢往快走，从重往轻走，从低频往高频走。中国的线下企业，资源依赖性比较多，掌握一些
> > > >
> > > > 核心资源，获得竞争优势，但是通用能力（基础能力），比如战略、组织、IT、信息化能力不够。线下从0到1是容易的，从1到N是难的。

> > > > 线上的确是从0到1难，但是进门以后就没那么难了。
