一份研究报告指出，大部分 Chrome 插件根本没什么人用
=============================

[产品](https://www.ifanr.com/category/product)

13 小时前

2

评论

作为目前市占率超过三分之二的浏览器应用，Google Chrome 在这个领域的主导地位基本不可撼动，而丰富好用的扩展插件也是我们愿意选择它的理由之一。

为了调查这些插件对用户权限的调用，安全公司 [Extension Monitor](https://extensionmonitor.com/blog/breaking-down-the-chrome-web-store-part-1) 对 Chrome 应用商店中超过 18 万个扩展应用进行了跟踪监测，还挖到一些有趣的数据。

![](.evernote-content/1CA50817-3544-4FBB-AF92-61392FA3743C/1809230F-6319-41F9-969F-565448E0891D.jpg)

▲ 排名第一的 Cisco Webex Extension 是思科 Webex 视频会议系统的组成部分，用户量超过 2000 万。

从分析结果来看，大部分用户只会使用几个头部应用，有 13 个扩展插件的安装量突破了 1000 万大关，基本都是我们耳熟能详的。

比如知名的广告屏蔽插件 Adblock，视频通话服务 Skype、Google 翻译插件、语法检查服务 Grammarly、「油猴」Tampermonkey、网页转 PDF 插件 Adobe Acrobat、购物优惠券服务 Honey 等。

![](.evernote-content/1CA50817-3544-4FBB-AF92-61392FA3743C/1F83EB10-B86D-4FF4-B6AC-980588D76B5D.jpg)

此外，像 Google 照片、环聊、密码管理服务 LastPass 等几个插件，也均有 680-900 万左右的下载量。

由此也能看出，多数人对 Chrome 插件的需求都十分明确，主要集中在广告拦截、通讯、购物、安全、密码管理器等几个领域，而几个特定应用也已经瓜分了大部分用户。

![](.evernote-content/1CA50817-3544-4FBB-AF92-61392FA3743C/2B18C2A9-49C9-45AC-BD36-8B24FF938493.jpg)

但繁荣的一面也到此为止了，抛开这些热门插件不谈，Chrome 扩展插件的生态只能用死气沉沉来形容。有近 87% 的插件安装量都低于 1000 次，更有约一半的插件安装量都低于 16 次。

另外，下载量为 0 的插件也有不少，数量为 19379 个，占总量约 10%；还有近 25540 个插件只拥有一个用户。

![](.evernote-content/1CA50817-3544-4FBB-AF92-61392FA3743C/66CC6AA5-A7B5-4C35-987A-DCFE171487D9.jpg)

Chrome 应用商店的付费插件（包含单次付费和应用内付费）表现也不算理想，大部分付费插件的安装率都低于 2.6%，其中 35% 的付费插件根本没有人下载过。

不过，这类付费插件的数量也不多，仅占总量的 8.9%。

![](.evernote-content/1CA50817-3544-4FBB-AF92-61392FA3743C/32FF1F6F-83FA-4DE5-9E57-8D9629AF83B4.jpg)

目前，付费插件中表现最好的是「IE Tab」，安装量达到了 410 万，占据付费插件安装总量的三分之一。这是一款能够在 Chrome 浏览器中使用 IE 渲染引擎，专门访问那些只能在 IE 浏览器中才能访问的网站。

同时，还有另外 4 款下载量超过 40 万的付费插件，分别为快速创建并制造流程图的 Lucidchart Diagrams、匿名代理插件 anonymoX、甘特图项目管理应用 Gantter Project Management 和泼辣修图编辑器。

![](.evernote-content/1CA50817-3544-4FBB-AF92-61392FA3743C/B29223CB-6BCF-4C25-AB46-1F2A0AA9BCC4.jpg)

▲ 我个人在用的一些 Chrome 插件，大部分是工作需求

总得来看，虽然 Chrome 在浏览器领域获得了巨大的成功，但扩展插件的使用率却存在很严重的两极分化情况，无疑给众多插件开发者们泼了盆冷水。

不过，PC 浏览器在移动互联网时代逐渐失势已是不争的事实，如何让 Chrome 浏览器实现从「浏览器」到「操作系统」层面的转变，无疑是 Google 未来需要考虑的问题。
