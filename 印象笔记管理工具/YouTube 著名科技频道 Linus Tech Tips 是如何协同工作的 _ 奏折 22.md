**@Fairyex：**在我们平时浏览视频网站的过程中，会发现每个分类下都有一些非常庞大的频道。他们一般是这个分类粉丝最多的频道，往往也是更新速度最快的频道。

如何保持这种速度的更新，同时保持住视频质量，是每一个从业人员都非常想知道的事情。就在最近，[Gamer Nexus](https://www.youtube.com/user/GamersNexus/about) 就对 YouTube 上著名科技频道 [Linus Tech Tips](https://www.youtube.com/user/LinusTechTips) 进行了一次[深入报道](https://www.youtube.com/watch?v=kdOBFu-mYpw)，了解他们是**如何通过高效的工作协同流程**，达到每周同时更新 4 个高质量节目与直播的程度。

![](.evernote-content/34D88F3F-6335-42BC-884E-D524FBD3A456/31445CE0-E8A5-42BF-935E-74209A638539.jpg)

Linus Tech Tips 是 YouTube 上著名的科技频道，Linus 是其拥有者，频道专注于科技产品评测，科技产品开箱和科技产品新闻。频道拥有近千万粉丝，并且 Linus 同时还运营着 Techquickie、TechLinked、Channel Super Fun 等数十万粉丝的频道，视频总播放量接近 50 亿次。

下面就是他们稳定高效地产出高质量节目的秘诀（LMG 表示 Linus 工作室，约 30 人）：

### 选题阶段

首先在每周的选题会议上每个人（6–7 人）都会提出视频创意选题。Linus 本人通常会提前提出，作者们在一周内提出他们的想法。接着 Linus 就会将不同的选题分给不同的作者去写稿，基于他们每个人不同的专业性和擅长领域。

而作者们通常不是只写一个稿。因为有产品购买或者等待厂家邮件回复的时间，所以作者们通常都会在不同的主题之间切换来写。

### 视频缓冲区

分配选题后，作者们就就可以申请购买材料等必要的步骤，同时在等待时完成自己正在进行的项目。Linus 本人偶尔会检查选题列表，删除已经完成的项目，并且添加项目有关的资料。

由于需要运营几个频道，同时工作室正常双休，所以工作室每天至少要制作出一个视频，并且随着各种大会的举办而增加，Linus 也会同时主持几个视频节目。这对工作室的效率有很大要求，现在他们尝试设置 10 个视频缓冲区，保证任何时间都有 10 个视频可以上传。但有时候碰到团建或者周末，视频缓冲区就会缩小到 1 个视频。

当其他人休假的时候，Linus 自己就会负责最少两个视频脚本。LMG CEO Nick 表示：

最理想的情况下工作是在一个块中完成的，一个下午有 2 至 5 个脚本评论，第二天根据这些评论拍摄 2 至 3 个视频。摄影师在设置下一个场景时 Linus 本人在拍摄上一个场景是很常见的情况。

工作室的目标是每周拍摄大约 6 个视频，不包含任何长期项目。比如他们去年年底制作的 1500 美元电脑系列就是在拍摄每集前几周讨论脚本，但 Techquickie 频道就每周要讨论三次脚本。

### 每日科技新闻

同时进行的还有每天都要更新的 tech link 科技新闻。负责人 Riley 每天早上会发布昨天拍摄好的科技新闻视频，然后检查论坛和其他常见科技网站来获取新闻选题。

Riley 会收集 20 至 30 个选题，将它们组织成三个主要故事和 5 个快速故事，在下午 1:00 前把脚本写好，接着就会通知摄影师准备拍摄。

拍摄在下午 1:30 至 3:00 之间完成，拍摄完成后 Riley 会将原文件传输给剪辑 Dennis 并在 5:00 之前完成编辑，视频上传后会立即发布，这样的流程每天都会重复。

### 日程管理和流水线调度

视频创建的过程另外一个重要的部分是**日程管理和流水线调度**，产品经理 Ed’sol lago 负责处理日程安排和与每个团队成员沟通视频制作进程。沟通后保持或者设定截止日期，并且应对延迟拍摄、各种障碍与选题更换。

### 讨论脚本

回到主要拍摄项目的流程中，每个主要的项目 Linus 都会和脚本作者一起熟悉产品和脚本。Linus 通过利用过去的经验看待产品，向作者提问并对视频形成初步印象。接着 Linus 会和作者一起逐行阅读脚本，改变措辞以保证符合自己的声音和语气，并检查质量与缺点。

在这个阶段中，**如果 Linus 认为产品缺少一些必要的测试，他会将作者稿打回去完成额外的测试，以保证第三方产品评测质量**，确保评测考虑到了所有角度。

Linus 还会和作者辩论，有强烈的意见分歧时要么挑战作者，要么要求作者挑战他。Linus 认为将多个声音合成一个评论能够确保内容更有完整的观点，如果团队不同意产品的某个部分，他们会将观点添加到脚本中，并显示为每个人的意见，为观众提供更多的选项。

整个讨论过程可能持续 30 至 60 分钟，有时当团队的内容质量不足时 Linus 会亲自编写脚本填补空白。但这个过程非常困难，因为不时会有需要中断处理的其他工作。

最后 Linus 或者 Nick 会将作者的建议标题确定为最终标题。**标题的选择会从相关性、浏览数、过度煽动或耸人听闻间寻找到平衡**，确定脚本后，作者团队会利用一些措施来帮助看提词器时读取重点，并将内容分块以便更好地融入视频。

### 布置现场和拍摄阶段

接下来就到了拍摄的时候，LMG 有多套可以匹配不同场景的设置：卧室、生活场景、厨房场景、录屏场景、TechLink 场景等。

确认拍摄场景后就开始布置照明，每个场景都已经准备好了基本的照明，可以使用电源开关和手机应用调整。但视频团队可能在特殊情况下重新安置一些灯，按照拍摄场景复杂程度可以使用 10 分钟到一个小时去布置灯光。为了控制混响和回声，工作室还用黑色多孔喷涂泡沫贴在墙壁上，工作室还使用可以移动的墙壁，使每个拍摄配置都可以组合，大大提高了拍摄场景角度。

团队接下来还要决定是否使用三角架等方式让拍摄尽可能的简单，通常使用三脚架拍摄 A-Roll（主要视频素材），而摄影师则使用背包和支架等拍摄。LMG 询问了摄影师这套背包 + 支架 + RED 摄影机 + 监视器 + 麦克风配置使用的劳累程度，并得到了「很有挑战性」的回复，所以摄影师有时候会轮换不同的人。

在 LMG 中，摄影师不是「袖手旁观」的，有时候他们会打断拍摄并表示自己要近距离拍摄某个物体。这样子对后期编辑人员也更好，因为他们可以很容易在片段中找到 B-Roll（过渡素材）。被拍摄人员有时候也会被摄影师要求更换位置来获得更好的拍摄角度，而不是连续拍摄一个镜头。

并且 Linus 和摄像师可以自行判断何时停止录制，以免录下太多无趣或者无用的文件。使用 20:1 30fps 8k 分辨率录制一秒消耗 65MV 的储存空间，8:1 时一秒更是可以消耗 162MB 的空间，因此如果摄影师连续拍摄不停止，那么一个视频项目就可以很容易达到 1TB 以上容量。

摄影师必须知道什么内容是无用的，所以脚本讨论完成后 Linus 还会带着作者和摄影师一起查看脚本，这里也可以进行新一轮的修改。

### 剪辑视频

在拍摄完 A-Roll 之后（或者之前），作者将会为视频编辑注释每个文件，包括哪里可以找到相关的旧剪辑素材，哪里应该插入哪种 B-Roll 镜头，以及哪里应该插入哪种图表。视频编辑和视频拍摄是不同的人，虽然有时候特殊的项目，比如强烈个人风格的就会由同一个人拍摄和剪辑，因为他清楚知道所有有趣和无趣的地方。

通常一段视频有两个人负责剪辑，一个人负责把所有的素材放入服务器并将视频排好分给一个剪辑师。剪辑师使用带有场记板（就是拍电影时常见的 Action 然后啪一声响那个板子）的素材轻松同步音频和视频，检查作者留下的注释确保所有素材都完整。

使用场记板的好处有很多，不仅可以根据声音同步音频和视频，场记板上还可以记录很多信息，由于是视频素材开头，**所以场记板会出现在文件浏览器的视频略缩图里面**。这样剪辑师只需要轻松一扫就能清楚地知道所有素材信息、排序、大概内容了。

场记板也可以帮助减少数据损失。根据场记板上的信息，如果视频出现任何错误和声音异常等都可以追溯到拍摄的 SD 卡与音频设备，任何有缺陷的设备都可以定位。

### 储存问题

一旦视频多了起来，储存也是非常令人头疼的问题。LMG 的做法是项目完成后文件夹保持不变，删除本地完成的最终主副本并移动到本地服务器上，特别旧的内容会从本地服务器里面删除并备份到 Google 网盘，所有的前期素材也会被保留。

### 质量控制

视频剪辑完成后，质量控制的最后一步也就开始了。主编辑 Taran 会对所有的视频负责最终的编辑和质量控制，以保证它符合 LMG 的视频质量标准，比如有放相关视频卡片（YouTube 支持视频叠加层）的空间，确认输出视频质量以及脚本准确性，镜头与主题是否匹配。

### 最终审核和发布

这个时候视频已经制作完成了，但它还需要 Linus 与 Nick 最终审查一遍，两人对视频是否符合 LMG 核心价值观有最终决定权。这一步通常在 Adobe Timeline 中完成，确认无误后根据销售团队的需求插入片头片尾的赞助商镜头，上传到 YouTube 并发布。

### Arch Linux 正式登陆微软应用商店

**@Fairyex：**微软最近宣布将在自家的预览版 Windows 系统里面内置完整的 Linux 配置环境 WSL2。但在它到来之前微软应用商店已经上架了数款 Linux 发行版，包括最新的 Arch Linux。

![](.evernote-content/34D88F3F-6335-42BC-884E-D524FBD3A456/BB8CB1AF-7530-495B-9931-B74776F6CF3B.png)

Arch Linux 商店截图

WSL 是微软内置在系统里面的 Linux 子系统（目前内核版本 4.4.0–17763-Microsoft），它让安装包括 Arch Linux 在内 Linux 发行版的难度，从之前的重装系统级别，变成现在的应用商店下载应用的级别。用户只需要像下应用一样下载打开即可使用 Linux 环境，并且可以很方便地同时使用 Windows，避免了原本双系统切换的麻烦。

![](.evernote-content/34D88F3F-6335-42BC-884E-D524FBD3A456/DE99DE44-462A-4F39-84AB-609E712263A4.png)

WSL 支持的发行版

虽然现在 WSL1 只能支持部分的 Linux 功能，但即将更新的 WSL2 拥有完整的系统内核，用户可以在上面实现所有 Linux 能够进行的操作 —— 包括桌面环境。而且 WSL 使用的是经过优化的虚拟化环境，比起普通虚拟机运行 Linux 效能成倍提高。

![](.evernote-content/34D88F3F-6335-42BC-884E-D524FBD3A456/C51F93F5-3F09-46FC-BBD1-038E0B6B1644.png)

在 Windows 上执行 pacman 命令

现在你可以轻松在 Windows 上拥有 apt 包管理（Arch Linux 是 pacman 包管理）、SSH 终端、运行 GUI 窗口程序（比如 VS Code）了，更多的操作可以参考少数派文章[「在 Windows 上用 WSL 开发的操作体验指北」](https://sspai.com/post/47719)。这样说来，未来 Windows 将有很大潜力成为最著名的 Linux 发行版。

### 在手机上也能画画吗：Procreate Pocket 3.0

![](.evernote-content/34D88F3F-6335-42BC-884E-D524FBD3A456/D1E5F67C-B755-4780-BF4A-40415B612C37.png)

**Procreate Pocket 3.0（2019–05–13）**

[下载链接](https://itunes.apple.com/cn/app/procreate-pocket/id916366645?mt=8)

**@Minja：**去年 App Store 年度榜单中有一款我难以理解的应用：Procreate Pocket，一个只在手机上画画的工具，而且不能和它的 iPad 版同步。Procreate for iPad 的名气倒是够大，但从 Procreate Pocket 不到 4 分的 App Store 评分也可以看出来，大家对于口袋版不是特别买账。正好前几天 Pocket 发了 3.0 大版本，跟着 iPad 版的节奏把今年的重磅功能都加了上去，我们就赶这个机会聊聊 Procreate Pocket 和手机绘画。

画画这件事，要么较起真来狠下功夫，要么图个好玩随便涂涂，两头都有对应的工具：要专业可以在 iPad 上使 Procreate、SketchBook 或 Concepts，图个乐呵拿 Evernote 或自带备忘录也行，iPhone 4 流行那会儿就有一股「指绘」热。

我用原生备忘录画的草图

Procreate 在手机端使的劲儿大家都能看出来，这次加上的几个功能都在追着 iPad 版：

3. 导出 GIF，在手机上直接制作表情包。
4. ……

![](.evernote-content/34D88F3F-6335-42BC-884E-D524FBD3A456/29158044-6030-44FA-ADCB-D2FA13FA1694.png)

这些功能出现在 iPad 版 Procreate 里的时候，每次 Twitter 上都是一片叫好声。由于我关注的很多画师都在用 Procreate，每逢它更新就不得不临时屏蔽所有相关消息。不过把这些高级功能搬到手机上，这事儿就有点尴尬，毕竟屏幕空间的不足和 Apple Pencil 的缺席都让 Pocket 从硬件底子上来说就不太行。说到底，iOS 设备上没有 Apple Pencil 的绘画，都还只是在 2011 年的水平上小打小闹。

如果哪天 Procreate Pocket 能够和 iPad 版的 Procreate 打通，而不是在区区几寸的手机屏幕上孤军奋战，那时候才最值得入手。

这两项功能需要在 Beta 套件中下载使用，需要先在「套件中心」启用「测试版」，然后在「Beta 套件」中下载它们。

### 在电脑上找回忘记的 Wi-Fi 密码

**@Fairyex：**我们手头都免不了有一堆设备连接着家里的 Wi-Fi，随着使用时间越来越长，我们对家里无线网络也更加习以为然。可是，一旦需要添加新设备，或者家里来了客人，很多人就会碰上忘记 Wi-Fi 密码的情况。

虽然可以进入路由器后台查看，但忘记 Wi-Fi 密码往往意味着忘记了路由器管理密码。如果你家里有连接过无线网络的 Windows 电脑，我们可以通过简单的两行命令从电脑上提取 Wi-Fi 密码，免去这种情况下需要重置路由器的麻烦。

方法非常简单，首先右键点击电脑右下角的开始图标，选择 `Windows PowerShell（管理员）` 打开命令行窗口。

![](.evernote-content/34D88F3F-6335-42BC-884E-D524FBD3A456/F6E0551E-A751-4DE4-9AB6-BEA6F58300AB.png)

打开命令行窗口

接着输入 `netsh wlan show profiles` 列出所有连接过的无线网络，如果已经知道无线网络的名字这一步可以省略。也可以通过 `netsh wlan show networks` 查看所有可用的无线网络。

![](.evernote-content/34D88F3F-6335-42BC-884E-D524FBD3A456/54C088C1-BF7E-4A81-9BF2-73F68D8FCF87.png)

命令执行效果

接着使用命令 `netsh wlan show profile name="这里填 Wi-Fi 名字" key=clear` 就能在弹出来的配置文件信息 → 安全设置 → 关键内容这里看到密码啦。

![](.evernote-content/34D88F3F-6335-42BC-884E-D524FBD3A456/737A0907-C4DA-4EBD-99BB-50367A969FF2.png)

命令执行效果

### macOS 技巧：把任意文件变成模板

**@Minja：**很多文档的样板基本已经确定，比如论文、申请表、固定栏目的题图…… 它们都可以当作模板来使用。但每次使用这些文件时总要先复制一份以免破环原来的模板，在心理上就多了一件事。

其实 Finder 里有一个很实用的**文件样板**技巧，藏得比较深，在文件的详细信息里面。勾选文件详情中的**「Stationery pad（样板）」**，再打开这份文件时就会原地创建副本文件，此时编辑的不再是样板文件，而是刚刚创建出来的副本。

![](.evernote-content/34D88F3F-6335-42BC-884E-D524FBD3A456/FD1DCCCD-F945-4930-98FE-BB375E4FFADC.png)

设置 Stationery pad 样板文件

我自己就吃过直接改模板的苦头。有次填入境申请表时，我习惯性地打开原文档就写，递交时才发现格式错误，保险起见只好又问旅社讨了一张新的。申请表就是典型的模板，可以长一个心眼，先设置成 Stationery pad 再使用，免得到时候哪里格式出问题都查不出来。

各类模板文件很适合通过 Stationery pad 设置保护起来，毕竟双击打开、直接编辑是最符合我们直觉的，Stationery pad 在保证操作方便的同时，也避免破坏原模板。

### CloudApp 关闭 HTML 上传后，还可以这样分享文章

**@Minja：**有些文章，我们写完后想分享给别人看，此时创建一个预览网页是比较周到的选择，不用操心排版，各种设备上都能阅读。Hum 曾经在《[一个优雅地分享本地文本的方法](https://sspai.com/article/42866?series_id=9)》中介绍过 CloudApp，它可以借助捷径把 Markdown 文件变成网页，连搭页面的麻烦都免了。

![](.evernote-content/34D88F3F-6335-42BC-884E-D524FBD3A456/DFAEE2B7-1F0C-4D1C-8C80-013966FFC8F4.jpg)

CloudApp 可以优雅地分享文章 图 / Hum

不过 CloudApp 出于清理垃圾网页考虑，从去年年底开始就慢慢关闭 HTML 文件上传功能，这周我发现捷径中也经常出现链接不可用的情况，估计也用不了多久了。换用其他服务的话，我希望保持 CloudApp 的一些优点：

1. 可以拿 Markdown 文件当原材料
2. 可以和捷径配合

在这些需求指引下，我找到了另一种思路：**把文章临时放到博客上。**

![](.evernote-content/34D88F3F-6335-42BC-884E-D524FBD3A456/3E7D33DF-2D99-411F-B94F-D1E839CCEA2D.png)

通过博客分享文章

我用的博客服务是 Bitcron，它可以和 Dropbox 连接、把网盘里的 Markdown 文件渲染成网页。这就连通了 Markdown 原稿和预览网页；Dropbox 又是捷径原生支持的网盘服务，快速操作也可行。其他能够直接渲染 Markdown 文件的在线服务应该也可以，比如免费的 GitHub Pages。

总的来说，**创建、管理和删除网页，就简化成了网盘文件管理。**其中涉及的操作比较基础，都是一些增删文件的步骤。

![](.evernote-content/34D88F3F-6335-42BC-884E-D524FBD3A456/F0369554-AF4C-4E15-BA45-147CA539F04A.png)

在捷径里查看、删除网页

博客的可玩性比 CloudApp 要高，喜欢自定义样式或者搞动态效果的读者可以自己研究。到这里还没结束。我平时用 Drafts 打草稿，所以把捷径和 Drafts 串联了起来，通过 URL Scheme 把草稿推给捷径、丢进 Dropbox 顺路获取分享链接，完成了「写作 - 渲染 - 分享」的一条龙。

![](.evernote-content/34D88F3F-6335-42BC-884E-D524FBD3A456/382C8DFE-C6DC-448B-B308-701159A81BBD.gif)

「写作 - 渲染 - 分享」一条龙

我有段时间习惯把改稿建议渲染成网页再发给作者，因为很多作者和我在微信上面联系，太长的消息不适合阅读，还是给一个链接、在网页里面慢慢读比较舒服。边打电话边在 Drafts 里记录，聊完一个网页发过去，一次沟通也就圆满结束。

![](.evernote-content/34D88F3F-6335-42BC-884E-D524FBD3A456/AE5A8431-C79E-4BCF-9003-718F032CD782.png)

通过网页分享改稿建议，阅读更方便

不过博客解决法还不够完善。CloudApp 的**私密性**比较好，分享出去的链接没有规律，不拿到链接就无法查看文章；而博客上的文章都汇总的主页上，实际上算公开内容（尽管知道这个博客地址的人不多），只能分享一些公开的、无伤大雅的内容。比如刚才提到的改稿建议，现在就不适合传到博客了。不管怎么折腾，我个人还是更怀念 CloudApp 便捷。

### 网站：支持在线预览的 CSS 模板库

**@沨沄极客：**当我们通过 Markdown 编辑器生成一个 PDF 或是 HTML 网页时，总是希望有更好的显示效果。这些显示效果就是由 CSS 样式实现的。如何获取优质的 CSS 就成了一个难题。

macOS 上的一款工具 Marked，可以将 Markdown 导出为样式丰富的 PDF。开发者收集了大量的 Marked CSS 文件，放在了 [Github 资源库](https://github.com/ttscoff/MarkedCustomStyles) 中，作为 Marked 自定义模板库供大家免费下载使用。

![](.evernote-content/34D88F3F-6335-42BC-884E-D524FBD3A456/C07C902A-9BF3-4692-9F2B-C91E2A06E43C.png)

在线预览 CSS

但时间一长，作者发现只是分享纯 CSS 文件无法快速浏览效果，用户下载后仍然需要花时间去一个一个尝试。所以他开发了这个**支持在线预览 CSS 效果**的模板库网站：[Marked Custom Style Gallery](https://ttscoff.github.io/MarkedCustomStyles/generate_examples/styles.html#default_styles/antique)，这也是今天要推荐的网站。

你可以在网页的左上角选择 CSS 主题文件，然后快速预览当前 CSS 的效果，如果觉得不错可以下载这个主题。这些 CSS 主题文件不仅可以用在 Marked 里，也可以用在任何支持自定义 CSS 的 Markdown 编辑器中。

![](.evernote-content/34D88F3F-6335-42BC-884E-D524FBD3A456/2D0427E2-23E6-4683-B43E-F42C0E6B7F1C.png)
