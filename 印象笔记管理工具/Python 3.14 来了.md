如果你以为 Python 的发展已经放缓，那就大错特错了。Python 3.14 已于 2025 年 10 月 7 日发布，对于初学者和高级用户来说，这一版本是一个充满实用魔法和深度工程的奇妙组合。这是一次让你重新思考 Python 能力的更新。

让我们深入了解一下这些功能、特点以及改变游戏规则的特性，因为这不仅仅是一个小版本的更新，而是朝着现代开发者渴望的方向迈进的一大步。

Python 的惊人新能力
-------------

### 自由线程 Python (PEP 779)

忘掉你对全局解释器锁（GIL）的旧有认知吧。Python 3.14 的主要创新是自由线程——终于正式、稳定地实现了，这是并发爱好者的梦想。

* 为数据科学和异步 Web 服务器提供强大动力。
* 更简单、更安全的多线程应用程序。
* 这些底层变化将在未来几年内影响到各个库。

![](.evernote-content/8A3CC70C-670F-449E-BADC-C26BB113E2F4/25A6C7BA-20E1-4FDD-BF30-D50AB99852E2)

### 模板字符串字面量 (PEP 750)

将 f-string 升级一下，就得到了 t-string。

* 轻松格式化和处理自定义字符串：`t"Hello, {name:upper}!"`。
* 可用于领域特定语言（DSL）、配置文件，甚至是本地化。
* 对于处理文本或配置的人来说，这是一个无痛且高影响力的特性。

### 标准库中的多个解释器 (PEP 734)

是的，你现在可以在一个进程中启动多个解释器——为工作负载提供沙箱环境，隔离插件，让复杂应用程序更加健壮（当插件崩溃时也不会那么可怕）。

那些重要的低调升级
---------

* \*\*延迟注解评估 (PEP 649)\*\*：在复杂系统中，注解的行为终于变得可预测了——这对于类型检查器、数据类和框架来说是个好消息。
* **语法高亮**：PyREPL、unittest、argparse、json 和 calendar 的命令行界面（CLI）因为内置的颜色而变得更加生动（也更易读！）。
* **Compression.zstd**：对 Zstandard 的原生支持——现在你的数据可以通过网络快速传输。
* **加强的 UUID 模块**：新增 UUID v6–8，以及更快的 v3–5 创建速度。
* **零开销调试**：与 CPython 的外部调试器配合使用：将你的工作流程连接起来，而不会出现性能下降。
* **改进的错误消息**：新的措辞和精准提示，帮助你更快地定位问题。
* **pdb 的远程调试**：可以连接到正在运行的 Python 进程——适用于云应用、批处理作业等。

兼容性、安全性和新的二进制文件
---------------

* **JIT 编译器（实验性）**：默认用于 macOS/Windows/Android 版本，让 Python 在真实世界的基准测试中感觉更快，尤其是对于新的应用和工具。
* **Android 可嵌入二进制文件**：在手机、物联网设备或信息亭设备上运行完整的 Python——设计上便于携带。
* **HMAC 内置**：通过 HACL\* 提供正式验证的加密功能，让你的安全关键型应用得到了微妙但重要的提升。

开发者和构建工程师的重大变化
--------------

* **安装管理器大修（Windows）**：转向更现代化的安装程序，升级更顺畅，依赖跟踪更精准。
* **Sigstore 签名**：不再使用 PGP；现在你可以通过 Sigstore 进行验证，这是一个更透明的现代软件包安全解决方案。
* **C API 的改进和移除**：为嵌入式开发者提供新的接口——同时对不安全的遗留入口点进行积极的弃用。

Python 3.14 的与众不同之处
-------------------

![](.evernote-content/8A3CC70C-670F-449E-BADC-C26BB113E2F4/FF31AD39-C147-4776-81A6-C9479A33D813)

* 现在，如果你试图在 `finally` 块中使用 `return`、`break` 和 `continue` 等函数，将会抛出错误——在重构旧代码时需要留意这一点！
* 发布版本不再使用 PGP 签名；只剩下 Sigstore。
* 许多遗留的 C API 特性和模块已被弃用或移除。请仔细检查你的构建脚本。
* 在多核和服务器密集型环境中，脚本运行更快。
* 更智能的错误报告，便于快速修复和新员工入职。
* 现代化的安装流程：为 IT、企业团队和分布式开发团队节省大量时间。
* 为区块链、分析和大数据传输提供原生、安全的压缩（Zstandard）。
