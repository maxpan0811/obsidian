先回答苹果 Mac 系列怎么办，几乎是毫无悬念的开启 **Time Machine** 就可以恢复了。

前提是，你平时有开着自动备份的习惯

**Mac 党**用 Time Machine ，应该就和中国人用筷子一样自然，不是吗？

**但是如果是 Windows，怎么办呢？**

毕竟，很多小伙伴还是工作生活在 Windows 的光环（或阴影）下的。

难道，万一出了事，重要的资料说没就没了？ 就没法好好备份，好好恢复了吗？

当然不是，还是以火箭君为例：

**先说，最终目标**
-----------

目标是：**物理上摧毁任何一台火箭君的工作电脑，都不会影响或极少影响火箭君的正常工作和已有资料查阅。**当然前提是你也要有一套有效的备份方法。

下面看看 火箭君怎么做的，希望抛砖引玉，多少给大家一点启发。

**首先，记住 2 件事**
--------------

**（1）常用电脑永远只是文件的临时容器**

你的重要数据一定存储在另一个地方。

**（2）备份 是 “多久” 和 “多快” 的取舍**

* • **“多久” 意味着，你的数据多久备份一次，不会影响你的工作**

这个回答可以是 “每分每秒” 都要备份，一秒都损失不起； 也可以是 “每周一次”，爱咋咋地。

* • **“多快” 意味着，从你意外失去资料到恢复资料，你愿意等多久**

回答可以是 “老子一秒也不能等”，也可以是，等个 10 天 20 天也无妨。

*（顺便，火箭君恢复 Mac 用了 6~7 个小时，你没有看错，这才是 Time Machine 的威力，让你不知时间都去了哪里）*

但如果，你不想取舍。既要 “每分每秒” 的备份，又要 “老子一秒也不能等” 的恢复。

可以直接拉到文末，火箭君也给你准备了方案，就是费钱。

**看看火箭君 怎么取舍的**
---------------

**从云开始**

**方法：云同步 + 云产品**

**优点：**轻量不敏感的文件一般可以存储在云上，可以最频繁的存取，每分每秒的备份。

**缺点：**文件不能体积太大

如果是 笔记文字类的内容，我们采用：

印象笔记

老牌的云笔记，几乎适合备份你所有的笔记记录

![](.evernote-content/8318A0A0-ADA4-4C2F-8A55-581428113BE7/2428B576-4012-45CA-88E9-0836653B710B.jpg)

OneNote（配合 OneDrive）

记录形式比印象笔记多样，微软出品，值得称道的是，同步速度比以前明显提升。

如果是 不特定类型的文件，我们采用这三款：

![](.evernote-content/8318A0A0-ADA4-4C2F-8A55-581428113BE7/79593B8A-E93B-487B-88D3-42D94F697FF5.jpg)

iCloud , 私人数据，跨 Mac /windows 平台备份的首选

不是**说好 Windows 下的工具**吗？

你没看错，**iCloud 有 Windows 版的**，而且效果不错。

只要有 Apple ID 即可使用。

如果你也用 iPhone 和另一台 Mac 设备，那天然的备份了你的手机和 Mac 数据（手机照片之类），而且能在 Windows 下访问。

坚果云 （工作数据，多人协同使用）

有很好的权限控制，增量备份

以及文件历史版本控制（协作场景下历史版本尤其重要）

**适合团队和多人协同使用。**

![](.evernote-content/8318A0A0-ADA4-4C2F-8A55-581428113BE7/883383AD-E690-4C3C-8139-9B655CCFC628.jpg)

Reslio Sync（自建云）

Reslio Sync, 几乎囊括了云同步的所有优点，就是需要你自己动手部署，还要有一台公网服务器。 一般人可以止步于此了。

**加一重保险**
---------

**方法：移动硬盘（或 NAS） + GoodSync （或 SyncToy）**

**优点：**数据自己可控，隐私无忧

**缺点：**需要定期执行，数据恢复和实际需要之间会有时间差距，这就是 “多久” 的问题。

我们采用：

![](.evernote-content/8318A0A0-ADA4-4C2F-8A55-581428113BE7/0AC01FA5-1A92-41B0-8FDB-D94C80009E7A.jpg)

GoodSync（老牌的备份软件）

GoodSync， 支持各种存储设备和网络协议。能够 “单向”“双向”“增量” 同步你的数据。

一般如果纯粹定期备份的目的，只要 “单向” 即可。

![](.evernote-content/8318A0A0-ADA4-4C2F-8A55-581428113BE7/9E820458-64BF-4D8E-B5A1-9255A375CB56.jpg)

SyncToy（微软出品，免费的）

和 GoodSync 差不多，就是 N 年前微软就停止更新版本了。

**再加个保险**
---------

**方法：** **Windows 也是自带功备份功能的哦。**

![](.evernote-content/8318A0A0-ADA4-4C2F-8A55-581428113BE7/1277FBD3-5F3A-42DA-BFD3-D0F37833F87B.jpg)

在 “控制面板” 里面可以打开设置

这个用来防止大灾难发生。你就理解成 Windows 的 “时光机” 吧。

缺点也很明显，必须外置存储设备，一般速度也不会很快，只能作为最后的手段。

**其它可能的 “加个鸡腿” 的选项**
--------------------

火箭君能想到的就是 **组建 RAID （磁盘阵列）**了。

这才是 “高频 + 高速” 的方案，很可能是目前科技下的最好选择了，在企业的应用比较广泛。

不过，考虑到跨设备的信息保存最终还是要依赖云同步和传输，其实 RAID 的意义也不是那么大了。

但对于前面说的 “又重要，容量又大，更新又频繁”，“老子不想等就要恢复” 的情况，

RAID 就是为这个存在的。

RAID 组建这个话题过于专业，我们可以后台私聊，不浪费广大群众宝贵的注意力了。

Archivism. [SYrLO]

两三年前关注这个问题时也想找一款本地用的同步软件（同步是强调两边都可以作为工作目录，而不是像 Windows Backup 那样备份侧是一堆备份格式文件），最好能像 Dropbox 一样增量同步、静默运行。试用了几个软件后选择了 [FreeFileSync](https://link.zhihu.com/?target=http%3A//www.freefilesync.org/)。

* • 建立任务后 FFS 会先对两边进行扫描比对，确认需要同步的文件后再进行复制、移动（同一侧内）或删除。对于经常有较多移动操作的场合（如整理照片和视频，就是题主所说的移动了位置但没有新增的情况）优势明显。  

  ![](.evernote-content/8318A0A0-ADA4-4C2F-8A55-581428113BE7/776E1A9F-12DD-4924-918F-132AC031D1AC.jpg)
* •  还可以将同步任务存成批处理文件（.ffs\_batch）。添加到计划任务里的话，虽然不像 Dropbox 那么即时，每天同步一次或者按需同步也够用了。速度方面，我的同步任务大概 11 万文件（两侧加起来 22 万文件），第一阶段扫描耗时 40 秒，剩下就是 copy 文件的时间了（移动操作速度很快）。  

  ![](.evernote-content/8318A0A0-ADA4-4C2F-8A55-581428113BE7/514E08C4-B487-4707-8EBB-A62B9A99D0AC.jpg)
* •  一点提醒：FFS 会在同步目录下生成一个 sync.ffs\_db 文件，应该是用来存储所有文件的路径、hash 等信息的。如果这个文件被删除或损坏的话，再同步时就无法处理前述的文件移动位置的情况。我在回答问题截图时，FFS 还是 5.x 版本，在确认官网地址时发现 8.0 都出了。欣然升级后，却发现原来的 sync.ffs\_db 文件格式无法兼容。这时如果再执行同步的话，就会导致已经移动的文件被重复复制。
