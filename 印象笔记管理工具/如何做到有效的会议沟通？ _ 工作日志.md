| 本文为付费栏目文章，您已订阅，可阅读全文 |

只要你在一个团队里工作，就免不了开会。在 Power+ 1.0 中我曾写过一篇《[如何开一场高效的会议？](https://sspai.com/post/43763)》，列出了会前、会中、会后的一些注意事项，并穿插了一些硅谷大型科技公司的开会诀窍。

不过，会议最终的目标其实就是八个字 —— **相互理解，达成共识**。要做到这一点，会议沟通就是重中之重。这一篇，我打算就会议沟通这个话题，具体展开分享一些心得。

会议沟通的四大难题
---------

试想一下这四种场景，是不是在你的工作中也历历在目：

1. **我不知道你们在说什么**：中午 12 点，你正打算出去吃饭了，突然间同事 A 跑过来拉你去参加一个讨论会。你走进会议室，看着白板上已经密密麻麻写了一大堆东西。整整前十分钟，虽然你人站在会议室里了，但整个状态都处在一个断片的状态，内心 OS 我都不知道你们在说什么，要我来干嘛？
2. **我听不懂你在说什么**：作为一名内容运营，你希望搞一个年底盘点活动，统计大家一年来的阅读情况。于是你拉来研发开会，刚说了你的想法，研发就开始发问了：这个页面的统计是静态还是动态的？如果是静态的数据，需要数据仓库那边先提前做好 ETL 持久化，或者去年的 API 看看能不能拿出来改改…… 你感觉听懂了每一个字，却不知道他到底在说什么。
3. **我说的东西你没有听懂**：还是上面那个例子，假设你现在转换身份成为了那名研发，当你说出拿去年的 API 改改后，运营马上反馈说：不行不行，不能用去年的东西，今年的设计风格完全和去年不一样。你打断了运营，说：你没听懂我的意思，用去年的 API 和今年的设计风格没关系。
4. **我不知道到底要做什么**：作为广告公司的策划，上周在客户那开会评选最终版的两个方案，对方品牌部的老大喜欢方案 A，市场部的老大喜欢方案 B，讨论到最后也没有明确的结论。结果等到下周一，对方又跑来问：上次讨论的东西细化到什么程度了？什么时候可以再看一稿？你内心苍凉，都不知道到底该用哪一稿。

会议沟通中的这四个老大难问题，该怎么改善呢？

会前会后的沟通，简单却容易被忽视
----------------

深入地去看这四个沟通难题，不难发现，「我不知道你们在说什么」是典型的会前沟通不到位，而「我不知道到底要做什么」，则是疏漏了会后沟通。

要解决会前会后的这两个沟通问题，其实并不难，它们都属于要解决非常简单，但却总被忽视的问题。**这其实不能怪参会人员，而是会议组织者和决策者的责任**。他们必须更新自己的观念：完整的会议沟通，并不仅仅只发生在会议的那几个小时里，而一直延展到了会前和会后，都是沟通的一部分。

在《[如何开一场高效的会议？](https://sspai.com/post/43763)》中，就提到了议程是会前沟通必不可少的一个环节，会议组织人应该事前整理好一页纸，将以下内容事前分享给所有参会人：

```
- 会议的时间、地点、参与人；
- 本次会议希望达成的目标；
- 当前已经掌握的背景资料和信息；
- 将会议的几项重要议题列成一个清单；
- 针对每项议题，提供当前已有的方案、决定或要求每个人针对议题做出的准备；
- 针对每项议题，预估所需要的会议时间；
```

而在会后，会议组织者和决策者应当尽快确认会议结论和相关执行人，把达成的共识落实到文字记录，这样才能开始执行。在撰写会议结论时，可以应用 SMART 原则，即每一条结论中，都应该包含五项元素：Specific（具体）、Measurable（可衡量）、Achievable（可达成）、Relevant（与目标的相关性）和 Time-bound（有时间限制）。例如：

```
- 模糊的会议结论：上线文章收藏功能
- 应用 SMART 原则的会议结论：在下周五前（T-时间限制），为了方便读者管理优质文章（R-目标相关性），产品研发同事已经在会上确认能够实现（A-可达成）在每篇文章的底部（S-具体）增加收藏功能。为了稳定性，先暂时给 20% 的用户开放灰度（M-可衡量）。
```

会中沟通：应该掌握的沟通技巧
--------------

如果说会前会后的沟通，主要是会议组织人和决策者的责任，那么做好会中的沟通，则是人人有责了。无论是「我听不懂你在说什么」还是「我说的东西你没有听懂」，会议中最让人痛苦的感受就是这种鸡同鸭讲的感觉。

我在多年的工作中积累总结了六条会议沟通的小技巧，许多适用的范围也不仅仅局限于会议沟通，而可以运用在更广义的沟通上。

这些技巧并不难学习，只要稍加练习都可以掌握。许多谈论沟通技巧的书籍和经验分享，都认为你首先应该具备什么样的性格，才能掌握出色的沟通技巧。但我并不以为然，没错，如果你想成为一名出色的演讲者，伟大的布道者，去鼓舞人心，去扭曲现实，的确依赖于你的性格。然而，**如果我们的目标是成为一名优秀的沟通者，去更好地相互理解，达成共识，那么要做到这个程度，无关乎你的性格**。

这六条人人都可以掌握的会议沟通技巧是：

1. **统一语言**
2. **使用「翻译器」**
3. **越不熟越是笨蛋**
4. **复述的力量**
5. **放大你的音量**
6. **识别关键因素和权重**

我们把这六条技巧，一条一条来具体说说。

### 1. 统一语言

![](.evernote-content/C0605612-868A-45A6-9C30-0C807C0E89CB/5B672F71-A2EE-47B6-9627-858DDEAA93F0.jpg)

统一语言

人类为什么会有不同的语言？在宗教传说中，人类曾经尝试联合起来建造通天的巴别塔，上帝为了阻止人类，创造了不同的语言，从而使人类之间无法沟通，由此挫败了建造巴别塔的雄伟计划。可见，语言是多么的重要，如果语言不通，沟通的基础便不复存在了。

然而，其实你仔细想一想，上帝为了阻止人类而创造不同的语言，可能是多虑了。因为，即使我们说着同一种语言，很多时候我们也无法正确地理解对方。**语言很多时候是模糊的，不同的人，对同一个名词，往往会有不同的理解。**

这种情况，在远程开会，或者跨团队、跨职能合作时，往往会更加突显。例如，不同公司的研发人员在会议上讨论工期，A 公司常说的工期指的是单纯的开发时间，而 B 公司的工期则是指从开发到测试上线的完整周期。大家可能口头上都在说着工期，但指代的意义是完全不同的。

因此，在会议的一开始，你可以专门设置一个流程：名词解释时间。如果这次的讨论会用到一些有特定含义和概念的「专有名词」，可以先花五分钟时间，在白板上，或者事前发出的议程中给出定义和解释，让所有参会的人理解，这次会议上提到的 XX，实际上指代的是什么东西。

### 2. 使用「翻译器」

![](.evernote-content/C0605612-868A-45A6-9C30-0C807C0E89CB/E5219148-3CA7-4606-8D75-891C2629B613.jpg)

使用「翻译器」

不过，总有时候还是会出现「你说的每一个字我都懂，但连在一起不知道是什么意思」的情况。这也很正常，毕竟，不同职能不同岗位，其掌握的专业技能天差地别，很多时候都会出现类似本文第一部分中内容运营和研发沟通的场景。

这就需要你找好会议上的「翻译器」。**这个翻译器，其实指的就是沟通的中间人，就像会说双语一样，他可以帮你把你的话，翻译成对方听得懂的语言**。因此，运营和研发开会时，可以拉上一名产品充当翻译器；即使是研发内部开会，如果是跨前端和后端的项目，有一名全栈工程师在场，也能更好地协调两方的沟通。

以本文开头提到的运营和研发讨论年底盘点活动，当研发说出「这个页面的统计是静态还是动态的？如果是静态的数据，需要数据仓库那边先提前做好 ETL 持久化，或者去年的 API 看看能不能拿出来改改……」，产品就可以充当翻译给运营解释：静态和动态是指盘点的数据是不是提前计算好，不会再更新了。一般这种活动都是用静态的数据，你要指定好一个时间范围和统计的口径，然后让数据部门的同事，提前跑好存到数据库里面。

### 3. 越不熟越是笨蛋

![](.evernote-content/C0605612-868A-45A6-9C30-0C807C0E89CB/4B7225B0-0E6E-43CE-A752-47D641302BCA.jpg)

越不熟越是笨蛋

我们掌握的信息按照通识程度，可以分为三级：

* 自识（Self Knowledge）：我自己知道
* 共识（Mutual Knowledge）：我们各自都知道
* 常识（Common Knowledge）：我们不仅各自都知道，还知道对方也知道

**沟通的许多误解，就源于把自识错当成共识，甚至是常识**。上面说到的「统一语言」和「使用翻译器」，其实就是要我们审视哪些东西是「自识」。不过，一个显而易见的问题是，会不会矫枉过正，把一些是常识的东西，也拿出来反复讲？

在实践中，可以运用「越不熟越是笨蛋」的法则，**即跟你开会的人关系越疏远，你就越可以放心地假设他是一个什么都不知道的笨蛋**。在第一次开会的时候，不要害怕啰嗦，不要担心对方是不是已经知道了，就是事无巨细地讲，并且提前声明，如果对方觉得你啰嗦了，让他来直接打断你，不用觉得不好意思。

### 4. 复述的力量

![](.evernote-content/C0605612-868A-45A6-9C30-0C807C0E89CB/8CF37E3D-4925-4D82-B654-BEDB071CDC7D.jpg)

复述的力量

网络流行语说人类的本质是复读机，而在开会的时候，如果每个人针对关键问题，都能成为一台「复述机」，倒不一定是件坏事。注意，是复述机（Paraphrase），而不是复读机（Repeat）。你不必刻意去还原对方说的一字一句，而是转化成自己的语言进行描述。

为什么我们要做一台复述机？**因为复述有助于理解，有助于打破表面的和平**。

什么叫表面的和平？在开会的时候，一个人说了一个观点或结论，往往没有人站出来反对或异议，那么发言人自然认为大家都接受了他的意思。但这种接受很可能来源于误解，我们在接收处理信息的时候，很多时候本能地会经历这样的心理活动：他说的这个没太听懂…… 但我猜应该这样理解…… 嗯这样的话那和我想得差不多…… 那就没问题。这就叫表面的和平，大家看着相安无事，很可能只是因为每个人自己内心把对方的话给合理化了，但合理化之后的意思，可能是和表达者有出入的。

而通过复述，可以是你复述别人的话，或者让别人复述你的话，总之，大家把内心的理解活动过程表达出来，往往就可以帮助对方找到理解上的偏差。

作为聆听者，你可以使用这样的句式来开始复述：「我重新用我的语言表达一下你刚才说的，你看看是这个意思吗？……」。

而作为表达者，如果你希望确认对方真正理解了你的意思，就需要让对方来复述。值得注意的是，在让他人复述时，往往会给人带来命令的感觉，使人产生本能的反感。你可以从语气和表达上，来缓解这一状况，试着使用如下的句式，如「我刚刚说的这些，你能帮我再总结一下吗」，「我希望你能帮我从技术语言的角度，再跟大家说一下这个」……

### 5. 放大你的音量

![](.evernote-content/C0605612-868A-45A6-9C30-0C807C0E89CB/F5335E88-FB0B-475E-8C2B-FA7ECEB0663A.jpg)

放大你的音量

前面四个技巧的侧重点都在于会议沟通的相互理解，那么最后两个技巧，则侧重于达成共识。为什么在开会的时候，说同样一件事情，同事 A 说有用，你说却好像没人听见一样？因为你的声音太小了，需要放大。

当然，这里说的放大你的音量，并不是抬高嗓门，加大分贝，而是找到你的扩音器，也就是**先小范围达成共识，从你一个人发声，变成多个人发声**。

例如，在跨团队的会议时，如果你需要对方团队配合工作，在会前先跟对方团队的 leader 达成共识，那么会议就会轻松得多。又比如，如果老板决定公司要抓紧上线某一个项目，你在开项目启动会时，如果说「我们这几个周末都需要加班」，这个音量就很小，反弹却很大，但你其实可以让老板和你一起发声，变成「老板决定这几个周末我们大家要辛苦一下，赶一赶进度」。

当然，也不要滥用这个借力的技巧，如果你迷恋上了这种方法，总是狐假虎威，那到最后破产的也是你的个人信用，久而久之就没有信任你了。

### 6. 识别关键因素和权重

![](.evernote-content/C0605612-868A-45A6-9C30-0C807C0E89CB/C8D3C460-E939-429F-AC14-CBC9D2189C69.jpg)

识别关键因素和权重

会议有时候陷入胶着状态，是因为大家有了分歧，话说不到一块去，有时候甚至还会上升到争吵。如果我们不能很好地处理分歧，往往最终各说各话，不欢而散。我在几年前读到这篇《[如果争论不欢而散，那么必有一方是虚伪的](https://www.guokr.com/article/38244/)》文章，对我触动很大。

简单概括一下的话，就是说，**两个理性的人讨论问题，如果双方能坦诚地考虑所有因素 (x1, x2……xn) 和权重 (a,b……z)，那么最终他们将达成一致的结果 f (x)**。

```
f(x)=ax1+bx2+……+zxn
```

我们每个人在思考问题时，都可能只从自己的角度出发考虑，而疏漏了某些因素，导致得出的判断和别人不一样。例如支付宝运营部门希望好友间能互相查看彼此的年终盘点，促进活跃；但风控部门提出这造成了隐私安全风险，不能做这样的活动。

当大家聚在一起开会讨论，每个人拿出不同的影响因素之后，我们其实应该把新获得的信息，重新代入 f (x) 去计算一下，得到新的结果。然而在现实中，很多时候我们都是在自说自话，你不停地强调因素 x1，我不停地强调因素 x2，而不会停下来重新计算一下这个 f (x)。

因为，想要在会议上达成共识，很重要的一个沟通方式转变就是：**在遇到反对意见时，先不要直接想着反驳，而是给自己一个缓冲时间，试试把对方的反对意见纳入考量之后，你得出的结论还和原来一样吗？**

当然，会存在着一些时候，即使双方对于所有应该考量的因素 x 都达成了共识，但每个人的权重依然是非常主观的。例如，要不要接保健品广告时，对公司收入的提升、对公司品牌的潜在风险、对用户的信任度等等，都是值得考虑的影响因素。销售部门可能认为收入是最重要的，应该接；公关部则认为对公司的品牌风险是巨大的，不能接。

这种时候，双方的沟通就很难继续下去了，事实上也不应该继续下去。如果我们已经真诚地互相倾听，彼此考虑了对方提出的影响因素和权重，在认真思索后，依旧无法达成共识。这时候，只需要记录下来分歧点，交由相关的决策人做决定即可。

---

以上，就是会议中沟通的六个小技巧。回顾一下本文，核心是：

* 有效的会议沟通，就是促进我们去「相互理解，达成共识」。
* 会前会后沟通是会议组织者的责任，它们做起来很简单，却容易被忽视。会前给议程，会后按 SMART 原则整理会议结论；
* 会议中的沟通技巧，人人都可以掌握，有些更可以用在广义的沟通上。这六条小技巧是：
  1. 统一语言
  2. 使用「翻译器」
  3. 越不熟越是笨蛋
  4. 复述的力量
  5. 放大你的音量
  6. 识别关键因素和权重
