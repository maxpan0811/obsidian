用听写命令打造「增强版 Siri」 | 实用技巧
========================

| 本文为付费栏目文章，您已订阅，可阅读全文 |

语音操作，最酷的往往不是「帮我放歌」这种炫技操作，而是工作结束时随口一句「今天就到这里了」，电脑就能听话地关上屏幕，目送你离开办公桌。

很遗憾，即使是休眠屏幕这种小事，Siri for macOS 也无能为力。除了嗷嗷待哺期望苹果出马，我们还有一个选择：**用 Automator 听写命令自己打造一个增强版本的 Siri**。

Automator 是 macOS 的预装自动化工具，能够通过语音控制电脑，自带了大量现成模块，并且支持脚本和第三方自动化拓展。借助 Automator，我们不难做到：

1. **跑动作**：任何 Automator 动作，都可以通过语音启动，换壁纸、切主题是只需一句话。
2. **跑脚本**：任何 Automator 支持的脚本，都可以用语音触发，休眠、更新 Mac App Store 应用不再手动点。
3. **执行第三方自动化动作**：间接运行 LaunchBar、Keyboard Maestro 等工具的动作，进一步拓展语音操控的范围。

语音控制 Mac 休眠

注意，本文我们只研究 Siri 做不到的事情，诸如切歌、查天气这些功能，完全可以交给现有的工具去做，无需重复造轮子。

听写命令的基础使用
---------

先说一下，我们的目标是：**完全不动手，只靠吩咐「触发词 + 命令」，就能控制 Mac**。有没有「那个谁，你去打个水」的感觉？听写命令的确和日常说话很像。事实上，你愿意把 `那个谁` 当作触发词也可行。

考虑到不是人人都熟悉 Automator，第一个例子就从「切换主题」开始，这个动作只涉及最简单的 Automator 模块，我们得以把精力集中在语音命令相关的设置上。

### 第一步：创建命令

首先新建一个 Automator **听写命令**，从左侧的资源库里找到「更改系统外观」模块，拖至右边，一切设置照旧，不用改动。好了，关于 Automator 模块的部分就说完了，来看语音命令吧。

![](.evernote-content/182360BE-88A6-4811-A825-FA3E7118198C/342BA469-C2E1-4706-844A-A1D6700B564F.png)

切换主题命令

上图青色框内的两处设置比较关键，相当于语音命令的钥匙：

1. 填写**「****听写命令****」**，即命令的触发词，不过目前还不支持中英文混用。
2. 勾选**「命令已启用」**，可以直接开启当前这条听写命令。

一番配置完毕，保存上面的命令（名字最好和听写命令触发词一样，免得自己都搞不清）。命令有了，但还不能做到「君子动口不动手」，需要移步系统设置。

### 第二步：开启真正的语音控制

先在「键盘 - 听写」里开启「听写」，同时勾上「使用优化听写」。这两步的意义在于为听写命令提供**识别基础**，特别是后一步，实现了**离线语音控制**，一边儿的 Siri 只能空羡慕了。离线识别需要下载数据包，大概 1GB 上下。

![](.evernote-content/182360BE-88A6-4811-A825-FA3E7118198C/28FDF64E-55E3-4F9B-BB93-757AE166481A.png)

开启听写功能

刚才的「听写」还包含了语音输入文字的功能，而我们现在关心的只是语音命令，所以还要再去「辅助功能 - 听写」里面勾选**「启用听写关键词短语」**，并输入自己喜欢的触发词，这才能完全摆脱键盘。做完这步以后，就准备 Hand Free 吧。

![](.evernote-content/182360BE-88A6-4811-A825-FA3E7118198C/70687B42-62A9-426F-B8F9-FE02AC125256.png)

启用听写关键词

触发词需要和听写的语言一致。别觉得 `嘿麦克` 很土，你大可字正腔圆地来一句 「hā mak」，电脑也能识别出你心中的「Hey Mac」。这就是那 1GB 的数据包在发挥作用。想要再酷一点，还可以顺带勾上「命令被识别时播放声音」，有种下完命令后电脑「嗫」一声就去执行的满足感。

### 进阶：更新 Mac App Store 应用

除了系统自带的命令，你装上的第三方命令行工具也能通过听写命令来启用。基本用法和上一节一样，但是要在整份脚本的开头加上**环境变量**，给 Automator 指指路，免得它找不到第三方命令：

```
PATH=$PATH:/usr/local/bin/
```

如果你的命令非常简短，只有一行，也可以把它和环境变量写在一起：

```
PATH=$PATH:/usr/local/bin/;你的命令
```

### 高级：命令 Keyboard Maestro 进入工作模式

除了 Automator 内部的动作，听写命令还可以启动第三方工具的功能，只要对应的应用提供了任意一种接口：

1. **脚本接口**：AppleScript 和 Shell 脚本都可以。
2. **Automator 动作**：直接集成了 Automator 模块，使用更直观。

先来看利用脚本的例子。@Hum 在 Power+ 1.0 里介绍过「[用 Keyboard Maestro 一键进入工作状态](https://sspai.com/article/40784?series_id=9)」的动作，简单说就是一次性打开工作需要的所有 App，并且把它们放到该放的位置。这个动作用语音来触发同样合适，而且颇有仪式感。

听写命令里只用到一个模块：**运行 AppleScript**。接下来的脚本更简单了，你真正需要写的只有 `你的 Macro 名字` 一处。

```
tell application "Keyboard Maestro Engine"
    do script "你的 Macro 名字"
end tell
```

之所以将听写命令和 Keyboard Maestro 组合起来，**不仅因为酷，更重要的是 Automator 本身做不到或做不好一些事，有必要引入别的工具**。比如上面的动作就涉及了移动窗口位置的操作，在 Automator 手工写代码不知道要花多久，借助 Keyboard Maestro 则只是点几下鼠标的事。

### 高级：命令 LaunchBar 一键备份

通过语音控制第三方自动化工具有两种情况：第一种就是 Keyboard Maestro 这种情况，属于「求救型」；第二种听上去比较懒：**捡着现成的动作就用**。用第三方自动化工具的人，本来就不常用 Automator 吧 1 ？既然都有现成动作了，直接借个模块来触发，总比去 Automator 里重写一遍方便，而且**需要修改的话也只用改一处**。

后一种的典型，**就是用听写命令触发 LaunchBar 动作**。用到的模块只有「[Perform Action](https://www.obdev.at/resources/launchbar/help/Workflows.html)」，填上 LaunchBar 动作名就行了。

小结
--

Automator 语音指令是 OS X 时代留下来的礼物。尽管现在已经有了 Siri，但是涉及高度自定义的指令时，仍然是 Automator 一展身手的机会。

制作语音指令时，先不用指望靠它控制一切，只在适合的时候设几条；另一方面，也不用限制自己的想象力，除了 Automator 自带的一些功能，还可以和第三方工具连通起来，进一步拓展语音指令的功能。
