# 利用 NFC 实现自动记录时间 - 少数派

---

利用 NFC 实现自动记录时间
===============

在上两周的文章《[用 Tasker 实现时间记录自动化](https://sspai.com/post/56699)》中，我给大家讲述了时间记录中容易碰到的三个主要问题 —— **记录步骤繁琐、容易忘记记录、碎片时间记录不准确**。并教大家如何利用 Tasker 配合时间记录服务的 API，实现时间记录的自动化，解决时间记录容易遇到的问题。

但是，除了使用手机 app 以外，在**现实世界**中我们也有许多像是读书、开会、学习等不同的工作内容需要记录时间；且不同于手机 app 的时间记录，记录现实事件的过程中，上面提到的三个问题会更加明显。

这篇文章我们就来了解如何利用好现代手机都有的 NFC 功能，配合 NFC 标签与自动化应用 Tasker，实现手机 app 之外的时间记录自动化，并且还能根据日程、位置等条件，实现自动开启。

需求分析
----

我们主要需要解决以下两个需求 / 问题：

1. 当我开始做某一件事情的时候，不会忘记开始计时任务。同理，当我结束做某一件事情的时候，不会忘记结束计时任务。
2. 在此基础之上，尽量实现自动开始 / 结束计时任务，即使不能实现自动，也要有一种最简便高效的方法开始 / 结束计时任务。

要实现现实世界事件与手机事件时间记录的关联，我们需要一个手机与现实世界的媒介，NFC 标签就是个很好的选择。

> 近距离无线通信（英语：Near-field communication，NFC），又简称近距离通信或近场通信，是一套通信协议，让两个电子设备（其中一个通常是移动设备，例如智能手机）在相距几厘米之内进行通信。—— 维基百科

我选择的是型号为 NTAG213 的 NFC 芯片，只有不到 2 厘米的长宽，144 字节的容量（支持读写），非常适合我们用来制作自动化标签。

![](.evernote-content/6F1D5297-F4F5-4945-8924-5EB1A3EC7D4A/9FC7FEDC-BB82-489F-A914-CAA17F93B818.jpg)

我购买的 NFC 芯片

以我自己的工作流程为例，每周二和周四我都需要分别参加 Power+ 奏折选题与文章排期会。自动化任务制作好之后，我将 NFC 标签与少数派 Logo 贴纸结合贴到了桌子上，这样每次开会之前我只需要用手机轻轻扫一下标签， Tasker 就会判断今天是周几，并选择好任务标题，然后自动开始记录时间并打开企业微信；当会议结束之后，轻轻用手机扫一下标签，Tasker 判断目前有时间记录任务正在运行，就会自动停止任务。

只要把标签贴在顺手显眼的地方（你甚至可以把 NFC 标签藏在便签下面，便签上面写上提示），就几乎没有发生过忘记记录时间的情况。

很多朋友可能会认为无线充电底座是一个非常方便贴 NFC 标签的地方，**但是千万不要将 NFC 标签贴到无线充电底座上！**因为标签中包含金属，会激活无线充电底座的异物检测功能导致无法无线充电，当无线充电底座没有异物检测功能时更是会将中间的标签迅速加热，有起火燃烧的危险。详细可以查看我的文章《[各种好玩的无线充电设备值不值得买？](https://sspai.com/post/53977)》中的相关内容。

制作过程
----

提示：以下所有过程都需要开启手机的 NFC 功能。

### 写入任务标识到 NFC 标签

我们拿到手的 NFC 标签（以下简称标签）都是空白的，为了让 Tasker 知道我们扫描的是哪个标签，我们要给不同的标签写入不同的内容。

我选择了安卓上比较老牌的一款 NFC 应用「[NFC Tools](https://play.google.com/store/apps/details?id=com.wakdev.wdnfc)」来读写标签，打开 NFC Tools 后靠近标签应用会自动显示该标签的所有信息（类型，技术，序列号，剩余容量等等），在「写入界面 - 添加记录」中选择文本类型并输入你喜欢的内容（只是用来区分标签，和其他标签内容不一样就行），最后点击写按钮再靠近标签即可将内容写进去。

![](.evernote-content/6F1D5297-F4F5-4945-8924-5EB1A3EC7D4A/41A271B7-1AE9-43DB-8073-EE3186B880ED.png)

写入内容到标签过程

NFC Tools 功能非常强大，除文本以外还支持将网址、搜索结果、社交网络账号、视频、文件、图片、地址位置、蓝牙、Wi-Fi 网络、邮件、联系人等多种类型信息写入标签，甚至本身也支持简单的自动化，感兴趣的朋友可以深入探索一下。

### 实现计时任务

接下来我们要将目光转移到 Tasker，在判断 NFC 标签之前，我们来制作开始与停止计时任务。具体步骤和我前两周的文章中建立这两个任务的方法一样，大家可以直接查看《[用 Tasker 实现时间记录自动化](https://sspai.com/post/56699)》中创建开始 / 停止计时任务部分。

不过由于我们要将计时任务的名称交给接下来新建的 NFC 内容检测任务来决定，所以**要跳过变量定义「% TASKNAME」这一步（第三步）：**

这一步不用做

### 根据不同 NFC 标签内容触发不同任务

有了计时任务，接下来就要检测手机扫描标签时的动作，获得标签内容，并判断不同标签内容从而设定不同的计时任务名称。

**第一步：**在 Tasker 配置文件界面中点击右下角的加号，选择「事件 - 网络 - NFC Tag」（真是奇怪的分类…），在弹出的界面中直接按返回键返回。

![](.evernote-content/6F1D5297-F4F5-4945-8924-5EB1A3EC7D4A/4D1BF2C9-2AD2-46F4-BF96-502117D3F614.png)

添加检测事件

**第二步：**在弹出的任务列表选择中选择「新建任务」，输入你喜欢的 Tasker 任务名字（比如 NFC 判断）点击确定。

![](.evernote-content/6F1D5297-F4F5-4945-8924-5EB1A3EC7D4A/22D73BF2-6416-43EA-B3EA-6246DDFF9BE0.png)

新建判断任务

**第三步：**在任务编辑界面中点击右下角的加号，选择「网络 - NFC tag」，在弹出的界面中我们可以看到「% nfc\_payload」，是储存读取到的标签内容的变量名，在这个界面直接按返回键返回。添加这一部内容的目的是让 Tasker 将读取到的标签信息写入变量中。

![](.evernote-content/6F1D5297-F4F5-4945-8924-5EB1A3EC7D4A/2C6F8DAF-FE18-4459-AAFD-5963086DBF7E.png)

添加标签信息

**第四步：**点击右下角的加号，选择「任务 - If」，在弹出界面中左边输入框输入「% nfc\_payload」，右边输入框输入你刚刚写入标签的内容（比如开始会议），接着返回。这一步是为了判断我们扫描的是哪个标签。

![](.evernote-content/6F1D5297-F4F5-4945-8924-5EB1A3EC7D4A/44C58FB3-A415-445D-886B-B4DE6F4997EA.png)

添加判断信息

**第五步：**任务编辑界面点击右下角的加号，选择「变量 - 变量定义」，选择「变量 - 变量定义」，在编辑界面名称输入「% TASKNAME」，往里面输入你喜欢的计时任务名称。这一步的目的是根据我们的判断，设定计时任务的名称。

![](.evernote-content/6F1D5297-F4F5-4945-8924-5EB1A3EC7D4A/D29BAA74-B5D7-47FD-B82E-0E754F23B8E6.png)

设定计时任务名称

**第六步：**返回到任务编辑界面，点击右下角的加号选择「任务 - 执行任务」，在弹出的界面点击名称右边放大镜选择我们在上面「实现计时任务」中建好的 Toggl 计时任务返回即可。这一步是为了实际执行计时任务。

![](.evernote-content/6F1D5297-F4F5-4945-8924-5EB1A3EC7D4A/1AEF45D8-198D-4CFB-B96E-597E0C4C21DE.png)

添加执行计时任务

这样我们就完成了一个标签的判断，当我们同时拥有多个标签时，可以再次添加第四步到第五步，只不过要第四步选择「任务 - If」变成「任务 - Else」并点击 If 右边的加号。每一个标签内容都创建一个 Else If，并起不同的计时任务名称。

![](.evernote-content/6F1D5297-F4F5-4945-8924-5EB1A3EC7D4A/0374E24A-3FAD-4803-8362-78DBF2A08952.png)

添加 Else If

**第七步：**返回到任务编辑界面，点击右下角的加号选择「任务 - End If」，这样我们就结束所有 If 判断，整个判断执行流程变成了一个有头有尾的 If 判断块。

![](.evernote-content/6F1D5297-F4F5-4945-8924-5EB1A3EC7D4A/83D7E9AE-5392-4961-B74D-7638C1A56EF1.png)

添加结束判断标志

我们还可以在每个 If 最后添加一个「Toast 提醒」（具体方法可以参考上一篇文章），内容写上「已扫描到 XXX 标签内容」之类你喜欢的内容。

到了这一步，我们已经成功地实现根据扫描不同 NFC 标签执行不同的计时任务了。但现在我们还差重要的一步，那就是**再次扫描标签时停止计时任务。**要实现这个效果，我们需要给建好的 Toggl 计时任务添加一个 If 判断，当没有计时任务运行时才开始计时任务，当有计时任务运行时自动停止当前计时任务。

### 停止计时任务

返回到 Tasker 主界面，在任务列表中找到 Toggl 计时任务点击进去，接着：

查看 [Toggl API 文档](https://github.com/toggl/toggl_api_docs/blob/master/chapters/time_entries.md)发现通过 `https://www.toggl.com/api/v8/time_entries/current` 可以获取当前运行的计时任务，通过 Httper 测试后发现当没有计时任务时该请求会返回 `{"data":null}`，这样我们就可以通过 If 判断请求返回值有没有包含 `null` 知道当前有没有计时任务。

![](.evernote-content/6F1D5297-F4F5-4945-8924-5EB1A3EC7D4A/F058D714-893A-4AAC-A16D-539EE2502B46.png)

Toggl API 文档相关内容

**第一步：**点击右下角的加号，选择「网络 - HTTP Request」，按照 API 文档的形式填写请求：

Method 选择「GET」。
URL 填 `https://www.toggl.com/api/v8/time_entries/current`。
Headers 填
`Authorization:%AUTHCODE
Content-Type:application/json`

超时设置短一点，勾选「信任任意证书」，返回即可。

![](.evernote-content/6F1D5297-F4F5-4945-8924-5EB1A3EC7D4A/8928851F-FBEA-48C8-BFF4-9C94D2380604.png)

设置请求

**第二步：**点击右下角的加号，选择「任务 - If」，在弹出界面中左边输入框输入「% http\_data」，右边输入框输入「null」，中间选择「匹配正则式」，接着返回到任务编辑界面。

![](.evernote-content/6F1D5297-F4F5-4945-8924-5EB1A3EC7D4A/05AFEBC9-CB22-4C00-8B34-1E48CA42E1E9.png)

添加判断

**第三步**：在任务编辑界面长按选择刚刚创建的两个任务，选中后把两个任务拖动到顶部。

![](.evernote-content/6F1D5297-F4F5-4945-8924-5EB1A3EC7D4A/968AC775-5572-4BC3-B27B-1D38CF83463B.png)

拖动任务到顶部

**第四步：**点击右下角的加号选择「任务 - Else」并返回，接着点击右下角的加号选择「任务 - 执行任务」，在弹出的界面点击名称右边放大镜选择我们在上面「实现计时任务」中建好的 Toggl 停止计时任务返回即可。

![](.evernote-content/6F1D5297-F4F5-4945-8924-5EB1A3EC7D4A/14D22E99-0BDC-48F7-9C98-555F201D20FD.png)

添加 Else 与停止计时任务

**第五步：**添加一个 End If 结束判断，再在合适的地方添加一个 Toast 提醒（比如 XXX 计时任务已停止）。

这样我们就完成了一个最简单的**判断任务**，扫描不同标签执行不同内容，当有计时任务正在运行时自动停止。回顾我上面提到的工作内容，细心的朋友可能已经发现目前制作的任务没办法实现我说到的「根据周几自动判断开的什么会」，下面我来说说如何给 If 添加**高级判断**，通过日程来确定任务名称。

高级条件判断
------

以开会为例，我们点击 NFC 判断任务中判断内容为开始开会的 If，点击条件最右边的加号，你会发现我们又多了一个判断条件：

![](.evernote-content/6F1D5297-F4F5-4945-8924-5EB1A3EC7D4A/D3FE3C4E-E3F8-4F45-824A-8508BAA83BE2.png)

再添加一个判断条件

点击中间的「与」则可以选择运算符，运算符决定上下两个条件最终结果的规则：

选择「与」则上下两个条件同时成立时才成立；
选择「或」则上下两个条件只要有一个成立就成立；
选择「Xor」则上下两个条件结果不同时才成立；

在开会例子中，我们需要「当标签内容为开始开会，且今天是星期二」才开始计时任务，所以运算符选择默认的「与」。

第二个判断条件左边的输入框输入「% DAYW」，右边的输入框输入「星期二（这里的内容会根据手机系统语言有所变化，英语是 Tuesday）」，返回任务编辑界面把任务名称改成「开会 奏折选题」即可。

我们再复制添加一个相同的 If 判断，把星期二改为星期四，任务名称改为「开会 文章排期」。这样即使扫描同一个标签，也能实现周二和周四自动开始不同计时任务了。

![](.evernote-content/6F1D5297-F4F5-4945-8924-5EB1A3EC7D4A/A866F34F-76F4-4FD8-97BF-F9F725003D74.png)

添加判断与最终效果

### 如何找到想要的变量

在所有输入框右上角都有一个五角形图标，点击它可以查看 Tasker 内置的变量以及它储存什么值。如果在这里面找不到我们想要的，也可以谷歌搜索「Tasker + 关键词」（比如 Tasker 获取星期几，我就是这么找到 % DAYW 的）来查找内置变量名。我们可以利用里面的变量制作更高级的功能，比如自动设置日历事件标题为计时任务名称，只需要在计时任务名称变量设置里面填入「%% CALTITLE」就行了。

利用所学，发散思维
---------

配合上一篇文章提到的配置文件同时添加多个检测信息，与这篇文章提到的 If 高级判断，我们就可以创建符合自己需求的复杂任务。

比如很多人都会遇到的太过投入做某件事情，做完之后忘记停止时间记录的问题：还是以我开会为例，开会的时候一碰标签就开始记录，并设定会议状态变量为开始，关闭企业微信时，检查会议状态变量值，自动停止记录。也可以换成离开会议室等等标志会议结束的事件。

或者我们可以用 If 先判断日历事件标题里面是否包含「节」字（这样可以去除节日的全天事件），然后再决定是否要将日历事件作为计时任务的名称。

总结
--

今天的文章我写了如何利用 NFC 标签将手机与现实世界关联起来，并通过自动化应用 Tasker 捕捉判断标签内容，执行不同的自动化任务，实现现实世界时间记录的自动化效果。

按照我的经验，根据自身日常添加足够的配置判断，即使只有几个 NFC 标签，也可以实现完全覆盖日常事件，用手机轻轻扫一下搞定时间记录。

这篇文章之所以没有上手即用的配置文件（最复杂的部分上一篇文章[《用 Tasker 实现时间记录自动化》](https://sspai.com/post/56699)中已经有配置文件分享了），是因为每个人现实中需要记录时间的条件都大不相同，我希望大家跟着文章制作完成后，可以发散思维，在以后生活中慢慢添加完善自己的任务判断。
