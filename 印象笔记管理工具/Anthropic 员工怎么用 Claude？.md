你用 AI 写代码，写出来的东西总不是你想要的。你觉得是模型不行。其实大概率是你自己的问题——没想清楚就开始了。

前几天 Anthropic 有个叫 Thariq 的员工在 X 上写了篇文章，说他们 Claude Code 团队内部有套用法：写代码之前，先让 AI 帮你找盲区。

你想啊，你写 prompt 的时候，写下来的基本都是脑子里已经想清楚的那部分。剩下那些还没想透的、以为自己知道但说不明白的，全是盲区。

AI 一碰到盲区就只能猜。猜错了你改，改完还不对，死循环。

所以与其花时间在 prompt 上精雕细琢，不如先让 AI 帮你把看不见的坑挖出来。

第一招，让 AI 采访你。打开 Claude，第一句话别说需求，说"采访我，每次只问一个问题"。AI 每轮就抛一个问题给你，几轮下来你自己都更清楚了。

第二招，让 AI 出选择题。跟它说"给我几个完全不同的方案，我先看再选"。有些需求你确实说不出来，但看到具体的东西，立刻就知道对不对。

Thariq 自己用 Claude Code 也是这么干的，连剪视频都不直接说"帮我剪"。先让 AI 转录字幕、清理语气词、生成几个不同版本让他挑，再让 AI 教他调色。每一步都是先搞清楚再动手，不是上来就甩指令。

下次用 AI 写代码，别急着动手。让它先采访你，一次一个问题，把没想清楚的部分一个个问出来。几轮下来你自己都通透了，写出来的东西也就对味了。
