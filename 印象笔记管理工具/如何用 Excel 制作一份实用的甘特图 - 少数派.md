# 如何用 Excel 制作一份实用的甘特图 - 少数派

---

如何用 Excel 制作一份实用的甘特图
====================

在《[项目管理必备技能之：计划扑克](https://sspai.com/post/56853)》一文中，我介绍了如何使用打牌的方式，来进行项目规模的估算和共识的达成。在这篇文章当中，我们就继续围绕「项目管理」这个话题，来谈一谈另一个必备技能：甘特图的制作。我将通过自己的实践，在这篇文章中分享：为什么在众多制作甘特图的方法中，选择用 Excel 来制作，以及如何用 Excel 打造一份实用的甘特图模板。

为什么我们需要甘特图
----------

在讨论甘特图的制作之前，如果你还不了解甘特图，那么我们简单花一点篇幅来谈谈为什么我们需要甘特图。甘特图的名字看着不明所以，其实「甘特」两字是其英文名 Gantt Chart 中 Gantt 的音译，是因为这种图形化的表达方式，最早是由 Henry Gantt 创造的，就以他的名字来命名了。

如果你经手管理过一个规模稍大的项目，那么对于甘特图可能就不会陌生。要了解为什么甘特图在项目管理中必不可少，和它的形态是密不可分的，一般来说它分为两块区域：

* **左侧任务区：**会列出项目中所有任务和事项的名称，以及可以按需要添加任务的一些基本属性，如责任人、分类等等；
* **右侧图表区：**会按照每个项目的开始与结束时间，绘制横向的柱形图，从而形成一幅整体的图形，展示每个任务所占用的周期；

![](.evernote-content/B498788F-4DD2-4D93-B499-EF6E366734BA/B4490B74-7F88-4DEA-8DAE-F3EF6F79EC4F.png)

一个甘特图的样子

了解了甘特图的样子，就不难想到为什么它在项目管理中非常有用，主要有三点：

1. **计划与展示项目的全貌：**制作甘特图的基础，是对项目进行拆解，把大的任务分解成一个个更小的单元，以及明确任务的开始与结束时间、责任人、优先级等属性。通过制作甘特图，不仅可以迫使项目管理人员完成项目的计划，更可以让项目中的每一个成员，通过甘特图了解项目的全貌，从而有更强的目标感，而不再只是整个项目中的一个小螺丝钉。
2. **更清晰的时间周期规划：**甘特图通过横向的柱状图，展示了每个项目的开始与结束时间，相比起单纯的列出时间，这种图形化的展示对我们理解项目的周期更加友好。人是非常视觉性的动物，我在《[日历 App 的巅峰对决：Fantastical 2 vs Calendars 5 对比评测](https://sspai.com/post/27589)》一文中也提到过，Calendars 5 之所以能够作为 Fantastical 2 的一个重要补充，是在于它的视图的多样性。甘特图也是同样的道理，2019/11/11 - 2019/11/20 你可能需要反应一下这其中是 10 天，但通过甘特图的图形化展示，你能直观地通过图像，一眼知道这是一个刚好从周一开始，横跨一周半的任务事项。
3. **追踪任务的进度：**甘特图不仅仅可以用于展示项目的计划，在项目的实施阶段，甘特图也可以追踪项目的实际进展，并通过图形化的方式，与原先项目的计划进行对比，清晰地了解每个任务是提前了，还是如期完成，或者延后了。

为什么选择 Excel 作为制作甘特图的工具
----------------------

制作甘特图有许多种方式和工具。从这一点上来说，可以首先分为两个大的工具选择方向：专业软件，或是「万金油」Excel。

先来谈谈专业软件吧。像 Microsoft Project、OmniPlan 这样的工具，都能够用于制作甘特图。除此之外，越来越多的在线服务工具也开始支持甘特图，例如 Worktile、轻流等等。它们有些功能的确非常强大，有些可以生成非常美观的甘特图，的确也是不少项目经理的首选。

![](.evernote-content/B498788F-4DD2-4D93-B499-EF6E366734BA/3172E3C8-21E6-4E88-AAED-E126D7B607A7.png)

使用 Microsoft Project 制作甘特图

不过，弊端也同样明显：

* **不利于分享和修改：**这些专业软件大多是付费的，而且显然，它们并不是那一类人人必备的软件。因此，当你想分享项目甘特图时，往往只能另存为图片，别人拿到手后，只能看不能改。在项目管理越来越强调团队协作的今天，专业工具反而变成了一种无形的门槛。
* **自定义程度不够：**这类工具的功能的确非常强大，例如项目的基线制定、变更追踪、任务估算…… 光看名字，你都不一定知道这些是干什么的。但是功能的强大却并不一定等同于你的需求。相反，它们的自定义程度往往不够高，导致很多时候只能由你适应于工具，而不是让工具为你服务。不管是 OmniPlan 还是 Microsoft Project，在甘特图中对不同属性的任务筛选，都不如 Excel 中的自定义程度高。

因此，在我的实践中，Excel 一直是我用来制作甘特图的首选。作为 Office 三件套之一，几乎不用担心 Excel 的普及性。同时，Excel 的自定义程度更高，只要花一点时间和精力，你完全可以按需打造出最适合你的甘特图模板。

不过，用 Excel 来制作甘特图，也有常见的三种套路：

1. 第一种是**朴实派**：他们只是利用了 Excel 的普及性，但整个甘特图完全靠手工绘制，十分麻烦。想像一下，如果你的甘特图有几十项细分任务，你要自己按时间轴，拉选不同的区域，进行几十次填充上色操作。
2. 第二种是**图表派**：如果你搜索 Excel 制作甘特图的教程，网上[分享的方法](https://www.jianshu.com/p/d9261f9923fb)，大多都是维护好源数据后，使用 Excel 中的图表来生成甘特图。这种方法的操作步骤也十分繁琐，而且只侧重于展示图形，却忽略了甘特图中的任务列表，对任务的展示非常不友好。
3. 第三种是**模板派**：模板派非常简单，一次性搭建出一个好用的模板，每次有新的项目，只需要在模板中修改内容即可。这个模板可以按照我们真正的需求，进行量身定做。

本文接下来要分享的，正是我在项目管理的实践中，自建的一套非常实用的**甘特图模板**。

---------------------

正所谓授人以鱼，不如授人以渔。接下来我将结合自己的实际需求，来分享一下这个 Excel 甘特图模板是如何搭建出来的，以及我是如何使用它的。为了方便理解，大家可以先下载这份 Excel 模板，我模拟了一个使用我的 Excel 甘特图模板，来管理少数派 Mac 端开发的项目。

我将按照以下五个特性来一一介绍：

### 1. 自动填充颜色

使用 Excel 制作甘特图，相比于专业工具软件，最大的痛点便是每个任务对应的柱形图的绘制。如果你曾经当过我前文提到的「朴实派」，那种痛苦的经历一定还历历在目。特别是如果涉及到甘特图的修改，想想那种酸爽的感觉：在众多柱形图中，对应到要修改的那一行，先选中原有区域取消填充颜色，再拉选新的区域填充。这个过程本身也大大增加了操作出错的可能性。

其实，完全可以把填充上色这个步骤自动化。你只需要输入项目的开始时间与结束时间，然后，对应的日期区域就会自动填充好颜色。你可以试试在 Excel 模板中修改日期，后面的颜色柱形图就会自动发生改变。

![](.evernote-content/B498788F-4DD2-4D93-B499-EF6E366734BA/7062CDD0-DE11-4901-83B1-61A6FDEE2731.gif)

随着日期的改变，填充区域自动变更

要做到这一点，需要利用到 Excel 中的「条件格式」功能。条件格式的功能概括来说，可以实现：**一旦满足某个条件，自动给单元格填充相应的格式。**选中整个甘特图中可以被上色的区域，然后点击菜单栏「开始 - 条件格式 - 新建规则」，样式选择为「经典」，然后设置为「使用公式确定要设置格式的单元格」。

![](.evernote-content/B498788F-4DD2-4D93-B499-EF6E366734BA/74207298-036D-4653-935C-5DB597911D2A.png)

在 Excel 中新建一个条件格式

现在，让我们来思考一下，如何通过「条件格式」来实现单元格的自动上色。可以看到，虽然这份模板中分为了四种主要的颜色，但让我们从最简单的开始考虑：根据 F/G 两列「计划开始于」和「计划结束于」，如何能够通过填写这两个日期，便在右侧将这个任务的计划周期自动填上浅蓝色？

在右侧上色区域，比如 K 列的日期是 10 月 25 日，一共存在三种可能情况：

1. 10 月 25 日早于计划开始时间，则其不应该上色；
2. 10 月 25 日在「计划开始于」和「计划结束于」两个时间之内，则应该上色；
3. 10 月 25 日晚于计划结束时间，则其不应该上色；

那么，把这个逻辑转换成 Excel 中的公式语言，不难得出这个判断条件的公式应该写成：

```
=IF(K$7 >= $F9,1,0) * IF(K$7 <= $G9,1,0)
```

也就是说，K7 作为这一列的日期，F9 是这一项任务的计划开始时间，G9 是其计划结束时间。如果 K7 在这两个计划日期之间，这个公式将等价于：

```
=IF(K$7 >= $F9,1,0)，结果为真，返回1
=IF(K$7 <= $G9,1,0)，结果为真，返回1
=IF(K$7 >= $F9,1,0) * IF(K$7 <= $G9,1,0) = 1 * 1 = 1
```

看到这里，还需要补充关于「**条件格式**」的两个小知识点，方便你更好地理解：

1. **是否上色的判断：**当公式最终返回的值为 0 时，则不上色；否则，返回任何不为 0 的值，都会对该符合条件的单元格执行格式填充。因此，这个公式里面返回的 1 只是表达为 TRUE 的惯用表达，你实际换成别的数字也可以。
2. **设置区域的判断条件：**你仔细一想可能发现了，这个公式实际上指定了 K7、F9、G9 这三个具体的单元格。难道针对整个上色区域，都需要一个一个单元格指定吗？当然不用，当你在「条件格式」中，选中一片区域来创建规则时，Excel 会默认你填写的公式，是适用于这个区域的首行首列的单元格，然后，按此推广到整个区域内。这时候需要注意，比如所有填色区域的日期都在第 7 列，你需要在前面加上 $ 符号，这样，在运算别的单元格时，Excel 只会变动列，而不会变动行。

至此，你已经完成了第一个小飞跃：从手工填色，实现了自动填色。

### 2. 区分计划与实际

前面我们设置的条件格式，可以实现按照计划开始与结束时间自动填色。然而，在管理项目的过程中，实际情况往往与计划千差万别。甘特图不仅仅只可以用来计划项目，当然也可以追踪项目的实际情况。我们可以在甘特图中，添加「实际开始于」和「实际结束于」两列信息，来让甘特图展示每个任务的真实执行情况。

为了更好地对比项目的计划与实际情况，除了原有的淡蓝色表示项目的计划周期外，我们可以用三种颜色来区分实际进展：

* **深蓝色**表示在项目的计划期内；
* **深绿色**表示提前于计划开始进行；
* **深红色**则表示已经落后于计划的结束时间；

![](.evernote-content/B498788F-4DD2-4D93-B499-EF6E366734BA/586220D4-1F45-4437-9185-5154441554C4.png)

三种颜色表示项目的实际情况

要实现这三种颜色的填充，和上一小节计划周期的淡蓝色填充，从原理上来说是一样的，概括来话：

* **深蓝色**填充区域：所在列应该介于「实际开始于」和「实际结束于」之间；
* **深绿色**填充区域：除了符合深蓝色区域的条件，还需要早于「计划开始于」之前；
* **深红色**填充区域：除了符合深蓝色区域的条件，还需要迟于「计划结束于」之后；

![](.evernote-content/B498788F-4DD2-4D93-B499-EF6E366734BA/D54A69BF-91B1-4416-8F6A-1E049863A46F.png)

另外三种颜色的填充公式

如上图所示，需要特别注意的是：

* **优先级顺序：**试想，一个单元格可能被多条规则同时命中，例如，一个单元格既在计划周期内，也在项目实际周期内。但显然，一个单元格只能展示一种颜色，也就是说，这些不同颜色之间是有优先级的。在 Excel 的条件格式设置中，越排在上面的规则，优先级越高。按照上图的设置，同样在实际项目周期内，提前（深绿色）和延期（深红色）的优先级高于如期进行（深蓝色），而这三个颜色表示项目的实际进展，优先级又高于计划（淡蓝色）。
* **`IF($F9 * *$G9 * *$H9,1,0)`：**你可能注意到，前三条规则中，在最后都跟上了这样一个判断条件。也就是说，项目实际进展要填充颜色，必须要满足「计划开始于」、「计划结束于」、「实际开始于」三列都有值。如果去掉这一条件，你会发现，当项目只有计划而没有实际开始时，或者只是列了一个任务，连计划时间都还没有定时，之前的所有时间都会被错误地填充。

![](.evernote-content/B498788F-4DD2-4D93-B499-EF6E366734BA/C70C2ED0-2928-4A1E-90DD-48B7CC259BA4.png)

去掉这一判断条件会导致填充错误

### 3. 自动填充到今天

细心的你可能还发现了，上一小节的三个公式，还有一个元素重复地出现：`K$7 <= IF($I9="",$C$4,$I9)`。它是用来做什么的呢？

其实，这个部分是为了实现一个小需求：**让表格将未完成的任务进度，自动填充到今天。**我们以深蓝色填充公式为例，如果不添加这个部分，公式会简化成这样：

```
=IF(K$7 >= $H9,1,0) * IF(K$7 <= $I9,1,0) * IF($F9 * $G9 * $H9,1,0)
```

然而，它会有一个小问题：当任务的实际结束时间为空时，也就是 I9 对应的值是空的时候，`IF(K$7 <= $I9,1,0)` 的结果为 FALSE，返回 0，会导致单元格不填充。这就带来了一个很尴尬的问题：任务已经明明实际开始执行了，但是，因为还没有结束，所以，它在甘特图上不会有任务的显示，必须等到填写了项目结束时间，才会填充颜色。

显然，这对于项目管理过程中，用甘特图审视整体进度是非常不友好的一个体验。对于项目经理来说，这些已经开始的任务，在视觉上却和只完成了计划的任务一样，没有任务区别。在理想的假设情况下，对于这些正在进行中的任务，如果今天还没有标注实际结束时间，那么其实默认的含义就是：**今天仍在实际进行当中。**

因此，最合理的作法，是在甘特图中，自动将进行中的任务填充到今天为止。这也是 `K$7 <= IF($I9="",$C$4,$I9)` 这个公式的作用：如果 I9 的「实际结束」于时间为空，则返回 C4 单元格的值。而 C4 单元格的值非常简单：`C4 = TODAY()`，每次打开 Excel 的甘特图模板时，这个单元格永远会更新成今天的日期。

### 4. 可修改起始日期，以及是否包含周末

对于一个甘特图模板来说，在应用到不同的项目时，要大改的区域除了填写左侧实际的任务拆分，另一个经常改动的就是右侧图表区的日期情况。不同的项目有不同的开始时间，如果不加以处理，每次需要自己手工重新填写这个项目的开始日期，然后在 Excel 中横向拉动填充单元格。

显然，这个体验可以变得更简单，通过在 C2 单元格输入整个项目的开始时间，从 K2 单元格开始，都会以 C2 的日期作为基准值，然后按顺自动递增。

![](.evernote-content/B498788F-4DD2-4D93-B499-EF6E366734BA/4AFD1617-430C-4DBE-B59C-9B5ED805EEF6.png)

日期使用公式来决定开始日期，以及是否包含周末

这个日期自动化又会引出一个项目管理的常见需求：不同项目的时间包含范围是不一样的，有些紧急的项目，周末也要计算在内，而有些常规项目则只在工作日进行。因此，我们可以引入是否包含周末的设置，在 C3 单元格中，你可以通过下拉框来选择，项目时间段内是否需要包含周末。那么，以 L2 为例，单元格的公式就是：

```
=IF((WEEKDAY(K7)=6)*($C$3="否"),K7+3,K7+1)
```

也就是说，先判断 C3 中是否需要包含周末，如果不需要，且前一个单元格的日期，已经是周五，那么当前单元格的日期，就已经是在周五的基础上，往后再三天到下一个周一。这里需要注意的是，WEEKDAY () 这个函数可以返回每个日期对应的是星期几，不过由于默认以星期天作为一周的第一天，那么要表示周五的话，实际上 WEEKDAY ()=6。

### 5. 按需快速筛选

甘特图在用于项目的实际管理时，能够快速地筛选不同任务的状态，也是非常有必要的。我举几个最常见的场景：

* **为新任务制定计划：**在项目进行的过程中，总会发现有新的任务出现。这些新的任务，都需要指定相应的负责人，安排在项目的周期内等工作；
* **是否有空余的资源：**如果一个项目提前完成了，那么意味着原本分配在这个任务上的那个人，应该也空闲出来了。因此，找到提前完成的任务，可以重新安排人力资源的分配；
* **重点关注落后于计划的任务：**如果一个任务已经落后于原定的计划，则可能会阻碍后续的计划推进，也占用了本应该被释放出来的人力资源，这种延期任务，往往是项目管理中需要重点关注的；

这些类似的场景都是项目管理过程中的必要环节，本质上，你可以通过项目的计划开始时间、计划结束时间、实际开始时间、实际结束时间，这四个日期的比较，得出任务的不同状态。

在我的这份甘特图模板为例，我维护这些状态的公式是：

```
=IF(F9="","未计划",IF(H9="","未开始",IF(I9="",IF($C$4>G9,"已经延期","进行中"),IF(I9>G9,"延期完成",IF(I9=G9,"如期完成","提前完成")))))
```

虽然这个公式貌似看着很长，但其实逻辑上并不复杂，只是多个 IF 函数的嵌套，最重要的是我们罗列出项目中的各个状态，可以方便地通过 Excel 的过滤器功能，快速地切换到不同的关注视图。这份模板中，主要维护了这些状态：

* 未计划
* 未开始
* 进行中
* 已经延期
* 如期完成
* 提前完成
* 延期完成

使用 Excel 快速过滤不同的状态

这样，一方面你不用人工去维护这些不同任务的状态，只需要填写好相应的日期信息，就能自动填充好不同的任务状态。另一方面，你只需要使用 Excel 的过滤器，就能快速切换和聚焦不同的状态，更方便地分门别类地处理和追踪不同状态的任务。
