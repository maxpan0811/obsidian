我推荐的不仅是apps，更是一种态度!

![](.evernote-content/34F4E258-A45E-4462-8FEE-8178F1D4C90E/2F4E7F54-D532-4946-AB3A-4D52FCF8538A.jpg)

OS X 上好久没有出现好看又好用的应用了。前段时间上架的剪贴板应用「Paste」 倒是不错，只可惜9.99美金的售价着实不便宜。好在 twodollartues 发了次力，联合官网做了一次1.99美金的冰点活动。于是，是时候给大家介绍一下这款剪贴板应用了。

为了帮大家更好的去感受，所以我决定先带入一下各位观众酝酿一下情绪：

1. 你有因为在多个文档间粘来贴去经常弄混而感到崩溃吗？
2. 你有因为在剪贴板工具中文件名找历史记录而感到抓狂吗？
3. 你有因为同类应用繁琐的操作而感到苦闷吗？

如果你有以上「症状」，那我建议你一定要试试这款新上架不久的剪贴板应用。

剪贴板工具的出现本质就是为了解决问题，提升效率。所以，评价这么一款应用好不好我觉得需要先回答几个问题：

1. 核心功能如何。首先剪贴板功能得足够强大；
2. 操作效率如何。操作起来是否顺手、快速；
3. 颜值是否足够高。在 OS X 上，一款应用的 UI 是否足够美观得体会很大程度上影响用户在使用时的感受。

安装完成后打开「Paste」是一个默认的设置界面。这里有应用呼出快捷键、历史记录数量、开机自动运行、直接粘贴四个可设置项。

![](.evernote-content/34F4E258-A45E-4462-8FEE-8178F1D4C90E/36D54F7D-821D-43B5-8BC9-694CA76E2B12.jpg)

快捷键：默认是「+Shift+V」，使用了一段时间之后感觉这个设置还算比较合理，这也是左手区域操作键程较短且不太会被占用过的快捷键了，要找到现在要找出一个合理且没被占用的快捷键真心不容易的。

历史记录数量：默认是100条，觉得基本够用了。剪贴板的内容还是具备一定的时效性的，再多了其实意义不大。当然想要设置成无限制也是可以的。

开机自动运行：如果你依赖于剪贴板工具，这个建议一定勾上。

直接粘贴：建议打开，直接影响操作效率，这个后面会说到。

设置完成后使用快捷键「+Shift+V」直接呼出应用。下图可以看到「Paste」针对每一条记录来源都设计了一个非常漂亮的卡片，用源应用的 app icon 作为卡片的背景。这极大的增强了剪贴板记录的可识别性，也算是为提升效率做出的一个较大贡献。扫一眼就大概知道哪条记录是我刚才复制过来的。

![](.evernote-content/34F4E258-A45E-4462-8FEE-8178F1D4C90E/E7D82C6D-7E25-46CC-BF1D-47D335C04964.jpg)

针对每一条记录，点击右键可以直接调用或进行社交化分享。注意 「Paste Without Styles」 ，这是一个很有心的功能，经常文字处理的同学都懂的。

![](.evernote-content/34F4E258-A45E-4462-8FEE-8178F1D4C90E/0E1BDE0F-0E3C-47C3-91E5-32E989BE9E18.jpg)

记得之前提到的「直接粘贴」功能么？双击一下图片就可以直接粘贴到画布中了。这在需要多个素材来回调用时会显得尤为方便。

![](.evernote-content/34F4E258-A45E-4462-8FEE-8178F1D4C90E/604B94B7-9E63-43EB-9DD3-F507531C8907.jpg)

因为应用的历史记录排序方式是按照时间横向排序（被使用过的记录时间会自动更新排到最前面），当数量多起来的时候找寻会是一个比较麻烦的工作。所以「Paste」内置了一个搜索功能，可以针对文件名（或文本关键词）或文件类型过滤来提升查找的效率。

不过在我看来「Paste」更需要的是一个加星功能。我在写这篇文章的时候其实就频繁用到了好几个关键词。但由于他们的位置总是在发生变化，想要立刻找到会有一点点成本。如果有了加星功能，至少我可以将当前常用的几条记录固定筛选出来，这能让我的效率提升不少。

![](.evernote-content/34F4E258-A45E-4462-8FEE-8178F1D4C90E/58ED9C17-DA54-4C71-8094-21A757DF67FB.jpg)

最后一个细节，「Paste」默认将钥匙串里的信息排除在外了。如果你不需要某些应用中数据复制的时候被留下记录，可以在这里将他们排除。其实这个功能通常来说可有可无，「Paste」对它的增加可见其团队之用心。

![](.evernote-content/34F4E258-A45E-4462-8FEE-8178F1D4C90E/BA0B16B7-5850-43CF-AAEB-C23FB58A143B.jpg)

最后，回到前面的三个问题：

> 1. *核心功能如何。首先剪贴板这个功能得足够强大；*
> 2. *是否提升效率。操作起来是否顺手、快速；*
> 3. *颜值是否足够高。在 OS X 上，一款应用的 UI 是否足够美观得体会很大程度上影响用户在使用时的感受。*

1. 剪贴板记录保存记录足够多，支持搜索、视觉化卡片、排除规则等操作，功能完全满足大家的需求；
2. 快捷键支持、双击直接调用到当前应用，操作效率不错，如果再有一个加星的功能作为筛选就更好了；
3. 界面大家都看到了，还是挺漂亮的**。**

对我而言，这样的功能加上界面设计完全值得入手，当然前面提到的几个点如果能改进就更好了。

其实如果你有好的想法完全可以给开发者去写邮件，大部分活跃应用的开发者都非常注重用户的感受。前几天团队里一位设计师发现了 Sketch 一个比较扯淡的 bug，于是给 Bohemian 写了一封邮件。没想到对方几天时间内就发布了更新版本，其中就修复了这个 bug。回头我也给「Paste」写一封邮件去。
