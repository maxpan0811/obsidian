在 iOS 上找回桌面级文件重命名体验 | Power+ 定制
===============================

| 本文为付费栏目文章，您已订阅，可阅读全文 |

文件重命名是一个基础而普遍的需求：

* 熬夜写的文档，打上今天的日期，方便以后查找
* 微博上刷到一位插画师，想把他的作品标上作者保存下来
* 下载了一季美剧的字幕，更需要命名整齐才能和电影对上号
* ……

也许不会每天都有那么多文件和你过不去，但万一遇上了，有个能批量处理的工具就会舒服很多。在桌面系统上，重命名就是一件轻松的事，比如 macOS 上的 Finder，就具有添加前缀、编号、加日期等丰富的重命名功能 —— 当然，支持一次性处理一批文件。

![](.evernote-content/160D2400-3BBF-493B-AF35-3CD0338607AE/B4A4570C-1203-462F-A964-B0633983637E.png)

Finder 支持各种重命名方式

但是 iOS 上则不然：Files 不支持批量重命名，而相册干脆不支持修改文件名，一些明明挤地铁时就能处理掉的杂事，非得拖到回家开电脑。

**这篇文章，就通过捷径来找回 macOS 上的重命名体验。**

**捷径下载：**[文件重命名](https://www.icloud.com/shortcuts/5ac1e35db116458280f626a548e41fc9)

⚠️ 注意：如果用本捷径修改了 macOS 上制作的文件，可能导致 Mac 上的本地历史版本丢失，需在 iCloud 中找回。

使用说明
----

这个捷径的设计目标就是还原 macOS 上的重命名功能，最终它完整实现了 Finder 里的 4 种最主要的重命名样式：

1. **前缀 + 原文件名**，如 `banner - 付费栏目本周定制`。
2. **自定义文件名 + 文件编号**，如 `壁纸 1`。
3. **自定义文件名 + 创建日期**，如 `照片 2018-09-16`。
4. **替换原文件名中的文本**，如去除 `照片 2018-09-17` 中的 `照片`。

重命名捷径可以通过分享菜单或者 Drag & Drop 来运行，具体每种样式的使用步骤不一样，下面分开介绍。

### 场景一：添加前缀

1. 选择「添加前缀」
2. 输入前缀
3. 运行后可以选择删除原文件 / 图

添加前缀

### 场景二：添加编号

1. 选择「添加编号」
2. 输入文件名的通用部分
3. 运行后可以选择删除原文件 / 图

添加编号

### 场景三：添加日期

1. 选择「添加日期」
2. 运行后可以选择删除原文件 / 图

添加日期

添加日期的选项不需要输入文本，捷径会直接在原文件名后加上当日日期。

你可以对 **当前日期** 变量进行自定义，设置喜欢的格式（本文所提供的捷径默认是 ISO 8601 的通用格式）。

![](.evernote-content/160D2400-3BBF-493B-AF35-3CD0338607AE/6F571B90-397E-4E08-AAAB-121DBA67215D.jpg)

调整日期格式

### 场景四：替换文本

1. 选择「替换文本」
2. 输入要替换的文本
3. 输入替换后的文本
4. 运行后可以选择删除原文件 / 图

文本替换

如果需要删除原文件名中的文本，可以把第 3 步空着不填。因为用「空文本」替换原文本，就能实现删除效果。

不管选择哪种样式，**重命名结束后，捷径会自动判断你输入的是文件（Files）还是照片（Photo media），并保存回对应的应用。**另外，对于所有集成进 Files 的应用，其中的文件都是可以进行批量重命名处理的，Dropbox、Documents 甚至 Working Copy 用户也可以享受这个捷径。

制作思路
----

重命名捷径看着样式丰富，其实真正的难度不在复杂的步骤嵌套，而在于一些细节处理：

1. **批量保存**：我们需要一次性保存所有文件，而不是一个一个保存。要达到批量存储的目的，需要用**添加到变量**为批量保存做准备。
2. **自动保存**：这个捷径支持 Files 或相册中的文件 / 图片。我们想要在重命名后自动把文件 / 图片保存回对应的应用中。这里的关键步骤是 **获取类型**。

整个捷径很长，但用不着完整地解析一遍，只需要搞懂上面两个难点，剩下的就是搭积木一样的简单工作了。

![](.evernote-content/160D2400-3BBF-493B-AF35-3CD0338607AE/325F595F-8CB0-4C43-87CD-E907944BC4B5.png)

搭积木一样来制作捷径

和 macOS 上的情况有所不同，由于缺乏快捷键，在 iOS 上我们要么用分享菜单，要么拖拽处理，都不如桌面端那么干练。所以在设计这个捷径时，我以「减少操作步骤」为目标，可能捷径本身看上去有很多重复操作，比如「要求输入文本」就分开用了 3 次，就是为了避免在文件添加日期时弹出没有出场必要的输入框。

### 批量保存

为了便于理解，下面以最简明的「添加前缀」选项为例。

注意图片下方的 **添加到变量** 步骤，每次循环时它都会把刚重命名好的文件添加到「所有文件」变量中，循环一结束，所有处理完的文件就都在「所有文件」里了。

批量存储相关的步骤

明白了如何把每次重命名的结果放在一起后，就可以如法炮制其他 3 种样式的处理步骤，最后再用一个菜单把 4 个选项组合起来。

到现在为止，捷径的主体部分已经完成，我们接下来看看**如何把文件 / 图片自动保存到对应应用**。

### 自动保存

在和读者交流定制需求时，我了解到摄影师和摄影爱好者可能是最需要 iOS 批量重命名功能的：他们外出时可能会携带 iPad，但是拍摄和导入的照片多是 `DCS0001` 这类名称格式，整理起来不直观。相信即使不是专业人士，也不会喜欢整理手机上那堆 `IMG_0001`、`IMG_0001(1)`……

iOS 相册里的照片其实可以发送给捷径处理，然后储存回相册，刚才那些重命名操作是完全适用的。问题在于，这样就要用户手动选存到相册还是 Files，在一串选择命名方式、输入文本的操作后，多加哪怕一步也容易造成手滑存错位置。

想让捷径帮我们选择保存到 Files 还是相册，前提是**要判断准重命名后文件的类型**。通过**快速查看**步骤做个小测试，我们可以看到相册里的所有图片的类型都是「Photo media」 —— 不管是你拍的照片、手机截图还是网上下载的图片。

![](.evernote-content/160D2400-3BBF-493B-AF35-3CD0338607AE/65004833-F470-40FD-BD79-9ADC192EF66C.jpg)

相册中图片的类型都是 Photo media

接下来的事情的就简单了，我们加一个 **如果** 判断，检测到输入的文件类型是 Photo media，就保存相册；反之，则保存到 Files。

![](.evernote-content/160D2400-3BBF-493B-AF35-3CD0338607AE/25C39D11-26BA-41A4-8B4C-C8F70125436D.png)

判断文件类型相关的步骤

上图红色部分就是判断文件类型相关的步骤。

现在把上一节的重命名步骤和这一节的保存步骤组合在一起，整个重命名的捷径就完成了。

小结
--

设计本捷径的想法，源自和一位读者的交流。他在尝试 iPad Only 的工作方式时，发现 Dropbox 总是不分青红皂白把照片文件改成 `照片 2018-09-17` 的样式，整理归档时他需要手动把 `照片` 二字去掉。macOS 上批量去除文本很简单，但是 iOS 上却没找到解决办法。

通过这个文件 / 图片重命名的捷径，我解决了他的需求；扩展到其他重命名场景后，也补上了 iPad 工作流中的重要一环，部分重现了桌面级文件管理的功能。

遗憾的是，捷径并不算「无痛」的方案，由于我们用新文件替换了原文件，将会导致**文件本地历史版本丢失**，拿来处理网上下载的壁纸、插画倒是无妨，但如果是自己辛辛苦苦做出来的文档、工程文件，还是使用 Files 一个一个命名、或者干脆回到电脑上编辑更加保险。

「iPad x 生产力」这条路，仍然需要更多系统级的支持。
