万众期待的 iPhone 门禁卡，可能就要来了！

今年早些时候，欧盟与苹果在经过漫长的谈判后，终于达成了一项协议。

苹果将通过开放 NFC API 接口的方式，允许第三方支付平台调用 iPhone 的 NFC 模块完成数据交互。

此前，这一新闻最早经由欧盟官网进行公示。

现在，苹果终于在最新的官方新闻稿中，确认了这一即将到来的改变。

苹果表示，自 iOS 18.1 开始，开发者将被允许通过安全元件在 iPhone 的自有应用上完成 NFC 非接触式交易。

该支付流程将独立于 Apple Pay 和钱包应用，并通过开放后的 API 提供数据交互。

![](.evernote-content/84CCD644-38D9-472B-8CDB-C430AFA4A513/7D22DD28-94D7-4AA4-9C9F-F26EB3579FFA.png)

图片来源：Apple 官网

考虑到安全和隐私问题，这一全新的解决方案将基于 SE API，而非此前宣布的 HCE 来进行。

相比之下，NFC 在经过 SE 安全模块进行信号处理时，会提供更加安全的执行环境。

用户只需要打开 iPhone，接触 NFC 验证区域，并双击侧边按钮即可完成交互。

而本次最值得关注的，除了苹果对更多 iOS 原生应用的 NFC 支持之外，还首次允许第三方应用加入其中。

开发者们可以将这一解决方案整合到他们的应用中，与苹果签订相关协议后，就能完成 NFC 和 SE 的授权。

如此一来，用户只需在 iPhone 设置中将该应用的 NFC 权限设置为默认，即可完成与原生应用相同的 NFC 交互体验。

当然，以苹果的一贯策略来看，这一授权过程显然不会是免费的。

按照苹果的说法，开发者需要支付相关费用，并符合行业内监管标准后，才能访问相关 API。

![](.evernote-content/84CCD644-38D9-472B-8CDB-C430AFA4A513/4D035C4F-E696-4CBD-AD40-CAEF5D831C2F.jpg)

图片来源：9to5mac

至于这笔「费用」的具体数额，苹果并未在新闻稿中提供更多细节，不过多半只有两种可能。

其一便是参照「苹果税」，作为手续费，获得 API 授权的开发者将向苹果支付应用营收的 30%。

其二则是在审核阶段所支付的一次性费用，具体数额可能会根据开发者工作室的规模进行调整。

但无论哪种收费方式，结合外界对于 NFC 开放的需求，以及 iOS 开发者的规模来看，对于苹果来说都是一笔巨额收入。

苹果表示，NFC 和 SE API 将在今年晚些时候发布的 iOS 18.1 正式版中提供给部分国家和地区。

首轮开放的国家和地区包括美国、英国、澳大利亚、巴西、加拿大、日本和新西兰。

随后，这一权限还将陆续开放给更多国家和地区。

![](.evernote-content/84CCD644-38D9-472B-8CDB-C430AFA4A513/5FA4C384-565C-4DF8-80BF-EA3356E5E3AC.png)

总之目前来看，未来向国内用户开放也只是一个时间问题。

而随着 NFC 权限在更多种类应用上得到调用，值得关注的显然不仅是 iPhone 门禁卡本身。

包括基于 NFC 的车钥匙、酒店钥匙以及演唱会门票等功能，或许都将在未来向用户开放。

再加上近期支付宝大力推广的碰一碰支付，NFC 或许会就此成为移动支付的下一个主流趋势。

总之，一起拭目以待吧。
