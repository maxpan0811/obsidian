5月25号，上海。

IEEE国际电路系统研讨会ISCAS 2026的开幕日，华为半导体业务总裁**何庭波**站上台，30分钟，发了一个新东西 ——

**韬（τ）定律**。

全文翻一遍，第一句话最重要：

韬定律提出以**「时间（τ）缩微」**替代**「几何缩微」**，作为半导体与电子系统演进的新指导原则——通过逻辑折叠（Logic Folding）等技术，持续压缩信号传播时延，提升晶体管等效密度。

这事三个理由值得专门写一篇 ——

· 这是**中国半导体首次**对外发布产业演进指导原则，分量不同

· 它出现在EUV被卡、14nm是物理上限的当口，**背后是一份被制裁逼出来的工程作战手册**

· 它讲的「换维度算账」逻辑，**跟做AI工具、做Agent编排的人正在干的事，是同一件事**

---

一、韬定律到底是什么
----------

先把硬事实摆清楚：

**发布时间**：2026-05-25

**发布场合**：IEEE ISCAS 2026（电路与系统旗舰会议）上海站

**发布人**：何庭波（华为半导体业务总裁）

**核心命题**：以「时间缩微 τ scaling」替代「几何缩微」

**关键技术**：逻辑折叠（Logic Folding）

**产业实证**：过去6年量产**381款芯片**

**未来目标**：2031年高端芯片等效**1.4nm制程**密度

**首批落地**：2026秋季麒麟芯片，**首次采用逻辑折叠**

### 用大白话讲什么是 τ scaling

摩尔定律说的是**「越做越小」** —— 每18个月晶体管尺寸缩一半，密度翻倍。这是几何上的「小」。

韬定律说的是**「越走越快」** —— 不再纠结晶体管物理尺寸有多小，而是**盯着信号穿过电路要花多少时间**。把这个时间压下来，单位面积能做的事变多 —— 等效密度就上去了。

![](.evernote-content/D766EC67-52F6-4825-B7D8-62F5BFCB2C8E/3D46E37C-1EA1-4AD6-ACAD-E38E674B8585.png)

图1 · 几何缩微vs时间缩微 · 两套不同的「让芯片变强」路径

### 逻辑折叠到底是怎么干的

韬定律的核心技术叫**逻辑折叠（Logic Folding）**。一句话：

**逻辑折叠** = 把原本需要N个物理逻辑门并行才能完成的运算，**用一个物理门在N个时钟周期内时分复用完成**。物理面积没变，等效逻辑变多了。

![](.evernote-content/D766EC67-52F6-4825-B7D8-62F5BFCB2C8E/3516F0BC-4512-4302-AEB3-44FA304E7797.png)

图2 · 逻辑折叠原理：1个物理门 × 4时序 = 4倍等效逻辑

这个思路本身**不新**。

熟悉硬件设计的人都知道，1990年代FPGA领域就有「资源时分复用」的研究，时序逻辑设计教科书也讲了二十年。

华为做的事是**把这套技术系统化、工业化、做进真实的高端SoC**，并且配套了完整的器件 → 电路 → 芯片 → 系统四层协同优化体系。

这点是诚实评价的关键。韬定律的创新不在「发明了一种新物理」，而在**「把分散在学术界30年的换维度思路，凑成一套可量产的工程方法」**。

一句话定调：**韬定律不是物理学定律，是设计哲学。**它告诉你当物理尺寸走不下去的时候，**从「空间维度」搬到「时间维度」去算密度**。

二、为什么是现在
--------

韬定律的命名时间不是偶然。

2026这个时间点，中国半导体面对的现实是 ——

· EUV光刻机出口被卡，国内最先进可量产工艺停在**14nm一线**

· 想做5nm、3nm？物理路径走不通

· 但旗舰手机、AI推理芯片、智能汽车都**等不起**

这是一道**必答题**：

路径 A · 硬刚物理

→ 国产 EUV 自研，至少 8-10 年

→ 在此之前，旗舰产品空窗 5 年+

→ 商业 / 政治 / 国民情绪都扛不住

路径 B · 换维度

→ 用 14nm 物理工艺，做出等效 5nm-3nm 的逻辑密度

→ 系统级优化补设计代差

→ 这就是「韬定律」

选B不是华为「想到了什么聪明主意」，是**不选B公司活不下去**。

过去6年的381款芯片，是这条路径的真实成本和真实产出 ——

· **不是PPT**：381款 / 6年 = 平均每6天一款，规模化量产

· **不是只做AI芯片**：从手机SoC到汽车座舱、从智能音箱到工业控制器，覆盖千行百业

· **不是单点突破**：四层协同（器件 / 电路 / 芯片 / 系统）是系统工程

2026秋季的麒麟芯片是**第一个公开采用逻辑折叠的旗舰产品**。如果性能数据兑现，韬定律就不只是「华为的设计哲学」，会变成**整个国产半导体业的参考路径**。

到2031年，目标是**用14nm物理工艺做到等效1.4nm密度**。

如果实现，单这一项就够把摩尔定律的体感寿命延长10年 ——

在中国语境下，这相当于**买回了被制裁夺走的10年时间**。

三、半导体史上每次「换维度」都是被逼的
-------------------

回头看历史，韬定律不是「中国独创」，是**「被卡的玩家」共同的剧本**。

| 玩家 | 年代 | 被卡在哪 | 换的什么维度 |
| --- | --- | --- | --- |
| 日本 | 1980s | 被美方半导体协议卡DRAM | 换赛道做NAND Flash + 存储器件 → 东芝、铠侠存活至今 |
| 韩国 | 1990s | 被美方光刻技术限制 | 押DRAM + NAND两条线 → 三星 / 海力士做到全球前二 |
| 台积电 | 2000s | 卷不动IDM综合工艺 | 换赛道纯代工 + 工艺细分 → 拿下全球60% 先进制程份额 |
| 中国 | 2020s | EUV封锁 / 14nm物理上限 | 换维度：架构创新 + 系统级优化 + 时间复用（韬定律）→ 待验证 |

表 · 半导体史上四次「换维度」复盘

四次复盘里有三条共同规律 ——

· **被卡的玩家都尝试过硬刚**：花数年砸钱 + 政策，最终发现物理 / 政治都过不去

· **成功的玩家都换了一个维度**：日本换品类、韩国押宽度、台积电拆IDM做纯代工

· **时间维度复用从未真正被产业化**：上一次有人在硬件层认真做「时间换空间」，要追溯到1990年代的FPGA学术研究

华为做的，是**把第三条变成第四种「换维度」**的工业实践。

能不能像前三次一样让一家公司活十年、二十年？

得看2026秋季的麒麟、2027的车规芯片、2031的等效1.4nm —— 时间是最公允的裁判。

四、灵魂段 · 硬件时间复用 = 软件Skill复用
--------------------------

如果你做的不是芯片，是AI工具、是Agent编排、是SaaS —— 韬定律也跟你有关。

因为**它讲的「换维度算密度」逻辑，正是你正在干的事**。

![](.evernote-content/D766EC67-52F6-4825-B7D8-62F5BFCB2C8E/F7CF1E9D-B245-4A64-9CAB-B8F001324311.png)

图3 · 硬件「时间复用」vs软件「Skill复用」· 一套思路两个赛道

### 把硬件的话翻译成软件的话

硬件视角 · 韬定律：**1个物理门 × N个时序 = N倍等效逻辑**

软件视角 · Agent编排：**1个LLM × N个skill / N个agent调度 = N倍等效能力**

细看，**四个对应关系**一模一样 ——

【1】资源不够 vs 物理上限

硬件：14nm 物理工艺是上限

软件：LLM 参数 / 上下文 / 算力都有上限

【2】换的是同一个维度：时间

硬件：单个物理门在多个时钟周期复用

软件：单个 LLM 在多个 agent 调用里复用

【3】关键不在能力，在编排

硬件：怎么折叠 / 怎么调度时序，决定密度

软件：怎么拆 skill / 怎么编排 agent，决定能力

【4】最大瓶颈：调度复杂度

硬件：时序违例 / 综合工具瓶颈

软件：上下文丢失 / agent 调用风暴

如果你用Claude Code、用Codex、用Cursor，可能不自觉做过下面这些「软件版逻辑折叠」 ——

· **把一个大任务拆成多个skill串调**：本质是把空间维度的「能力堆叠」搬到时间维度的「能力串联」

· **用sub-agent并行加工**：1个主agent × N个分身时序 = N倍等效产出（这就是逻辑折叠）

· **用MCP把工具能力外挂到LLM**：模型参数没扩，能调用的工具维度变多 —— 等效模型变强

· **Plan-Execute-Clear工作流**：本质是给LLM强加一个时序框架，把单次调用拆成多个时钟周期

换句话说：**过去6年华为做的事，跟过去1年AI工具党在做的事，是同一种被资源约束逼出来的工程方法。**

差别只是 ——

· 华为做的硬件版叫**韬定律**

· AI工具党做的软件版还**没人给起名字**

这是我看这次发布最大的启发。

**一个值得思考的延伸**：AI行业现在还在迷信「更大的模型 / 更大的参数」 —— 这是「几何缩微」思路。

下一波突破可能不在模型本身，在**「编排维度」** ——怎么把1个不那么大的LLM复用出10倍能力。这就是软件版的韬定律。

五、不戴民族情绪滤镜的四维风险
---------------

讲完类比，得把客观风险摆出来。

对一项「定律级」发布最负责的态度，不是欢呼，是**把可能翻车的地方一条条点出来**。

| 维度 | 真问题 | 为什么 |
| --- | --- | --- |
| 学术 | 「定律」二字承重过重 | 韬定律本质是设计哲学，不是物理学定律。摩尔定律自己也是经验律，但国际同行命名争议曾让寒武纪、龙芯吃过亏。 |
| 工程 | EDA工具链跟得上吗 | 逻辑折叠依赖大规模时序仿真 + 综合工具。国产EDA在14nm以下还在追赶，能否支撑系统化逻辑折叠是真问题。 |
| 商业 | 等效1.4nm ≠ 真打台积电N2 | 等效密度只解决「装得下」，没说功耗 / 良率 / 单die成本。台积电N2是综合数据，韬定律目前公开的只有密度一项。 |
| 营销 | 民族情绪滤镜 | 「中国首提」标签易遮蔽技术细节。叙事可以民族，但技术评判得回归数据 —— 否则下次海外同行做了类似工作，自己反而尴尬。 |

表 · 韬定律四维风险拆解（非民族情绪滤镜下的工程评估）

### 每一维展开说一句

**学术维度**：「韬定律」三个字，跟摩尔定律、丹纳德缩比放在同一层级 —— 海外学界买不买单是问号。摩尔定律自己也是经验律不是物理定律，但它经过了60年的工业检验。韬定律目前6年。命名建议低调点，**让数据说话**。

**工程维度**：逻辑折叠需要的不只是芯片本身，还需要**能跑通的EDA工具链**。Synopsys、Cadence、Mentor都被卡了出口。华为内部EDA自研到什么程度、能不能扛住全国晶圆厂用 —— 这是真问题，比单芯片本身更卡脖子。

**商业维度**：「2031等效1.4nm密度」是营销话术里最容易被攻击的一句。等效密度 = 单位面积塞多少晶体管。但是 ——

台积电 N2 的真实优势项：

· 等效密度（可量化对比）

· 功耗（韬定律未公开）

· 良率（韬定律未公开）

· 单 die 成本（韬定律未公开）

光对比密度而忽略其他三项 = 单点比较。

真打 N2 还差至少这三个数据维度的公开验证。

**营销维度**：「中国首提全球产业指导原则」是一个非常容易被解读成民族叙事的标签。但工程评判应该**回归数据 + 长周期检验**。如果叙事先行、技术细节后置，反过来会伤害韬定律自己的可信度。

这四条说出来不是泼冷水，是**预先把可能掉的坑标出来**。

如果韬定律真的是十年大事件，那这四条到2031年都会被回过头来逐个验证 —— 我宁愿现在说在前面，赌它每一条都被工程数据反驳。

六、写在最后 · 给AI工具党的4条启示
--------------------

回到这个号的主战场 —— AI工具、Agent编排、智能体落地。

韬定律给AI工具党最有用的不是技术细节，是**4条工程方法论**：

### 启示1 · 被卡能逼出原创

华为被EUV卡6年，硬生生憋出一套新方法论。

Anthropic API限速、ChatGPT拒服国内 → 反而催生DeepSeek蒸馏、cline开源、openrouter中转、国产Coding Agent。

**被卡的人，往往才是真正长出新东西的人。**

### 启示2 · 换维度比硬刚资源强

物理工艺刚不过台积电 → 不刚，换时间维度算密度。

单模型参数刚不过GPT-5.5 → 不刚，**换编排维度做能力复用**。

做AI Agent时，与其纠结「我的模型够不够大」，不如想「我的skill拆得够不够细 / 我的agent能不能时序复用」。

### 启示3 · 真创新不一定是新概念，是新组合

逻辑折叠在FPGA学术界存在30年。Skill / Agent / MCP单独看都不是2026的新东西。

华为做的不是「发明逻辑折叠」，是**把它系统化、量产化、配套工具链**。

AI工具党也一样：你不用发明新算法。把已有的skill / agent / hook / MCP配套打通成一个工程化体系 —— 这本身就是创新。

### 启示4 · 「定律」二字慎用

华为给「韬」起了一个定律级的名字，是要承重60年工业检验的。

AI圈现在动不动就发「XX法则」、「YY定律」、「ZZ原理」 —— **大多数都活不过6个月**。

做工具、写文章，先做事再起名，让数据先长出来，再给它一个名字 —— 这比反过来稳。

「**韬光养晦，物盛则衰**」——《淮南子·原道训》

「韬」字本意是**「藏，蓄势」**。

华为用这个字命名定律，多半不是巧合。

被卡的这6年，藏的不是芯片，是**一整套换维度的工程方法**。

到2031年，这套方法能不能撑起等效1.4nm密度，能不能让国产芯片买回被制裁夺走的10年时间 —— 时间会说话。

对每个做AI工具的人来说，韬定律真正的价值，不在于它解决了多少硬件问题，

而在于它再一次证明：**当资源不够、维度走死，换一条维度往往比硬刚更有效。**

这是6年硬件实践给整个工程师群体的一份礼物。

哪怕你这辈子不碰芯片，做Agent、写工具、做SaaS —— 它依然适用。
