# 从初学者到进阶跑者，你应该考虑 Runkeeper

---

作为一名跑者，当你积累了一定的跑步里程，回望自己跑步生涯的时候，是否会出现这样一种想法：有没有一种能够更加直观，更加数字化地审视自己的跑步生涯的方式存在？

[Runkeeper](https://runkeeper.com) 正是这样一个存在，一个可以让我们的跑步数字化的工具。用 Runkeeper 累积里程是跑步数据的输入过程，其生成的统计数据则是跑步数据的输出过程。如此一个从输入到输出的过程，便形成了跑步数据的「处理流」。

数据输入：跑步记录
---------

数字化跑步的第一要义就是要跑起来，如果说 Runkeeper 作为我们的跑步记录工具负责数据输入的话，那我们的腿就是数据的创造工具了。打开 Runkeeper，首先纳入眼帘的便是「开始跑步」界面，作为我们数据处理流的起点，我们先来介绍一下这个界面。

![](.evernote-content/CE17ED2A-5BC4-4533-AF68-EB3201309C49/0C01E09A-23E6-48C9-9254-66CC8172A04E.jpg)

作为跑步 App 的 Runkeeper 在这个界面上做的很好，关于跑步前需要准备的一些关键设置都直接给出显示在地图下方：运动类型、训练计划（即目标设定，可以自定义目标，也可以设定为 App 给你提供的训练计划）、陪跑音乐来源（直接与 iTunes 或 Spotify 联动）和语音提示频率。比起再点开左上角的设置二级菜单在进行设置，这样设定体验起来方便了不少。

![](.evernote-content/CE17ED2A-5BC4-4533-AF68-EB3201309C49/9F7F74F4-340D-40A3-8387-1BEFBB1DDD72.jpg)跑步前可以挨个进行快速设定

需要注意一点的是，Runkeeper 默认的距离单位是英里，如果你是第一次使用，建议你先到左上角的 ⚙️ 按钮进行一番调整，之后便可一劳永逸了。

点击「去跑步」后，随着倒数，跑步记录过程便会开始。此时只会剩下三个卡片式的界面，中心卡片显示的是跑步实时数据，左右两个卡片分别显示地图轨迹和分段数据。由于 Runkeeper 未对大陆的 GPS 数据进行校对，所以定位和轨迹显示起来都很随缘，无法正常作为参考，后文的「数据再加工」部分会讲到解决办法。

![](.evernote-content/CE17ED2A-5BC4-4533-AF68-EB3201309C49/217E3AC6-2363-4C58-8347-1BDF5DA9F8D9.jpg)开始跑步后的三个卡片界面

如果对中心卡片显示的数据面板不满意，用户可以点击右下角的设置图标自定义数据的显示。结束跑步后，Runkeeper 会跳转到一个界面来进行本次跑步信息的补全，一次跑步数据的输入就算完成了。

![](.evernote-content/CE17ED2A-5BC4-4533-AF68-EB3201309C49/1833B8A8-098A-4FB1-BB8D-7EC0EDB12540.jpg)

数据输出：详细的跑步数据和数字大观园
------------------

作为数据，其实在输入之后还应该有一个处理过程，作为工具的 Runkeeper 帮我们完成了这些工作，所以我们只需要参考数据的输出即可，Runkeeper 中数据的输出主要有两个去向，一个是单次跑步的详细信息，另一个则是「数字大观园」。在「我」的界面，有这么两栏：活动和数字大观园，我们的数据输出的入口即来自这两处。

![](.evernote-content/CE17ED2A-5BC4-4533-AF68-EB3201309C49/F81654C8-9E55-40F1-A43B-E1D7BF606FC7.png)

点进「活动」，以列表的形式，所有的跑步记录会按照时间顺序排列，如果想进行对某类记录进行筛选，右上角的筛选设置可以帮忙进行过滤，其中提供了相当丰富的过滤规则。

![](.evernote-content/CE17ED2A-5BC4-4533-AF68-EB3201309C49/8969BF6F-294E-4227-B2BF-ABFB71583E5B.jpg)跑步列表和筛选器

点进具体某一次的跑步记录，便可以查看单次跑步的详细信息了。

![](.evernote-content/CE17ED2A-5BC4-4533-AF68-EB3201309C49/0C9FA74A-F24D-48C6-B05C-4A483E3454E4.jpg)

除去一些基本的数据，Runkeeper 在「图表」这一栏里提供了丰富的图表，以时间为 x 轴，展示了整个跑步过程中所有的数据走向。当你在其中任意一个图表上滑动手指时，多个图表会联动显示每个时间点的该数据的具体值，这样联动数据的做法有很多用途：

* 方便关注心率的跑者匹配自己心率和配速之间的关系
* 方便研究自己体力的消耗是怎样直观的随着配速走势的上下浮动而变化的
* 通过步频和步速的变化了解自己怎样能采取更好的成绩
* ……

![](.evernote-content/CE17ED2A-5BC4-4533-AF68-EB3201309C49/4A0C64A9-F8FE-40CC-881A-0B79D7179431.png)

单次跑步的数据价值在此体现：通过具体到某个时间点的身体表现的数据化，了解自己的身体状况和进步空间。

看完单次跑步的数据的统计，接下来就是进行长期累积后，跑步数据的一个大观了：数字大观园。通过右上角的按钮进行筛选，你可以查看自己从开始跑步算起，或是从近三个月算起的所有跑步数据统计。数字大观园提供了一种从很长的一个时间尺度上审视自己跑步的方法，看自己的配速是否一路高歌猛进，看自己的体重变化和每个月锻炼量的关系，以此作出调整，制定更好的跑步计划。

![](.evernote-content/CE17ED2A-5BC4-4533-AF68-EB3201309C49/0EA8DC86-AAAF-4C0B-9266-89788B965986.jpg)

数据再加工
-----

前文提到了 Runkeeper 在国内使用会遇到不可避免的地图 GPS 数据漂移问题，对于追求数据和谐完整的强迫症患者，有没有解决办法？如果你恰巧是 [Gyroscope](https://sspai.com/post/34227) 的用户，那么是有的。

Runkeeper 支持与 Gyroscope 数据共享，在关联好账号后，每在 Runkeeper 中完成一次跑步，相应的数据会自动同步到 Gyroscope 中去。每次跑完步后打开 Gyroscope，都会有提示你完成了一次跑步，点击查看详情就可以看到更详细的数据，这其中包括了不漂移的（甚至有点好看的）轨迹信息。

![](.evernote-content/CE17ED2A-5BC4-4533-AF68-EB3201309C49/4F7E84D0-C582-4149-888F-3E48D7795848.png)

如果你是长期的 Gyroscope 用户，那么你还可以在每周，每月和每年的总结生成后，看到一张张很酷炫的跑步轨迹地图。

![](.evernote-content/CE17ED2A-5BC4-4533-AF68-EB3201309C49/27590182-7D48-4B60-B29E-AEAB5D2D1798.jpg)

订阅制
---

Runkeeper 实行订阅制，若想完整的体验从数据记录到数据输出的过程，包括数字大观园等功能，需要付费成为 Runkeeper Go 会员。Go 账户一个月的定价为 ¥68，一年则是 ¥258。看到这你也可能注意到了 Runkeeper 的许多功能和本身的定价并不适合普通群众，更适合于有一定跑步里程积累的（数据越多，统计就越有价值）坚持型跑者，或是有参加比赛诉求的较为专业的跑者。如果你只是把跑步当作茶余饭后的一个运动消遣，Runkeeper 并不适合你，同类的跑步软件有许多更好的选择，甚至说你也可以不需要任何 App，只要出门跑起来就可以了。

如果你的需求比较简单，可以选择 Nike 出品的 [Nike+ Run Club](https://sspai.com/post/40331)。

关于这篇文章
------

本篇与其说是一篇测评，不如说是对跑步数据处理的流化尝试，把 Runkeeper 带入一个生活情景中——跑步，再加上一个需求：数据化的审视跑步，以此来探寻 App 功能的可能性。许多人可能会不理解这样的做法，认为这偏离了跑步的本质，掺杂了太多的无意义的「数字化」成分。其实在跑步数据处理流中，从数据创造者——我们跑者自己——出发，通过产生数据，记录数据，处理数据，再输出数据，通过查看这些数据，可以制定出更好的一套跑步方案，反作用于数据创造者之身，跑出更好的成绩。我认为这不仅是跑步数据处理流的价值，更是「数字化」理应起到的作用之一。

我们可以用数字化全副武装自己，即便是跑步这样古老的运动也可以披上数字化的外衣，也可以选择与数字化一切保持距离，穿上一双鞋，就那么去跑吧。数字化也好，脱数字化也罢，终究只是一种选择。
