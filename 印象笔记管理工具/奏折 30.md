### 剪藏：比 Notion 原生更强的第三方剪藏插件

![](.evernote-content/AF3A4C00-CF96-421A-B15D-4FD5D1595E6F/65C39AE3-3256-4734-B95E-D68C10141667.png)

**剪藏**

[Chrome 商店](https://chrome.google.com/webstore/detail/icidaiapcopkcgepepcncehhodfafidg)   [GitHub 链接](https://github.com/webclipper/web-clipper/releases)

#### 值得注意的更新内容

**@沨沄极客：**喜欢用印象笔记剪藏的朋友会发现，在不同页面下剪藏的文章会以各自形式显示在笔记页面，造成格式不统一。Notion 的 Block 结构让文章更规范，但由于 Notion 在剪藏方面做的实在是不太上心，很多网页的剪藏效果并不好，很多时候只能剪下来标题和网址，连内容都不翼而飞了。

所以最近我尝试用一款 Chrome 插件「剪藏」来取代它的官方插件。没错，这款插件的名字就叫「剪藏」。**它一口气支持了四种笔记工具，分别是 Notion、语雀、GitHub、有道云笔记**。初次使用 Notion 和有道云笔记时需要登录网页版，其余的则是通过 Token 实现。

![](.evernote-content/AF3A4C00-CF96-421A-B15D-4FD5D1595E6F/A1241988-A60B-49F0-972A-E7D06299FCC7.png)

支持多种账户、多种图床

「剪藏」最大的特点在于**能够自由选择剪藏格式**，提供了「书签、整个页面、智能提取、手动选取」这几种格式。其中智能提取、手动选取是使用频率最高的。Notion 对富文本支持比较差，经过智能提取之后，文字就会被转换为相对规范的 Markdown 形式。

但有时候剪藏不到标题怎么办？「剪藏」还提供了**页面编辑器**，不仅可以删除网页元素、看到剪藏后的效果，还能在**所见即所得的编辑器**中直接编辑文字，确保剪藏的质量。常用的添加一个标题、删除一些广告之类的问题，完全不在话下。

![](.evernote-content/AF3A4C00-CF96-421A-B15D-4FD5D1595E6F/4212E0B6-217E-4C30-9677-8F3AEB63D0E6.png)

所见即所得的剪藏编辑器

针对一些做了图片防盗链的网站，「剪藏」有一项功能可以在剪藏的时候**自动将图片保存到图床**。使用前需要绑定图床后使用，避免剪藏后图片失效的问题。

关于「剪藏」的开发方向，开发者表示石墨文档、为知笔记支持起来很简单，估计马上就可以支持了。而 Evernote 和 OneNote 比较麻烦。**未来还会将更多功能****做成扩展组件的形式**（比如 Send to Kindle）。非常期待它的改进。

你可以在 [Chrome 商店](https://chrome.google.com/webstore/detail/icidaiapcopkcgepepcncehhodfafidg)直接下载安装「剪藏」，也可以参考 [离线安装方法](https://www.yuque.com/web-clipper/help_cn/install_offline) 来离线安装。

### OmniFocus 3.4：连小众场景也帮用户考虑到了

![](.evernote-content/AF3A4C00-CF96-421A-B15D-4FD5D1595E6F/1716E4A7-4807-4BE4-8CDD-BF9D4F0F6F62.png)

**OmniFocus 3.4 for Mac、OmniFocus 3.3 for iOS**

下载链接：[macOS](https://apps.apple.com/cn/app/id1346203938?mt=12)、[iOS](https://apps.apple.com/cn/app/id1346190318)

#### 应用简介

苹果平台上久负盛名的任务管理工具，功能全面且稳定。

#### 值得注意的更新内容

**@sainho：**在最新的 OmniFocus 3.4 版本中（iOS 为 3.3 版本），新增了一项动作的状态属性：**已丢弃。**

具体来说，现在动作可以像项目属性一样，将状态设置为「已丢弃」。和已丢弃的项目一样，动作将变为不可用，只有在透视的显示选项设置为「全部」时，才能看到。

这个状态属性，适合分配给那些**时间上已经错过了、无法完成**的动作。

工作期间我需要定期在内网登记工作时间，于是记录了一项每周二、五重复的动作。但当我旅游时，就面临两难的抉择，这些在旅游期间自动生成的「登记工作时间」，我该将它打勾标为已完成，还是直接修改「推迟至」时间呢？前一种方案违背了「已完成」动作的含义；后一种手动修改既繁琐，还会影响动作的原始信息。

这是就轮到「已丢弃」出场了。将动作标为「已丢弃」，既符合它目前已经无法被完成的现实情况，也不会影响动作的原始设置。

最妙的是，如果你尝试对重复动作设定「已丢弃」，OmniFocus 会询问你是「彻底丢弃」，还是跳过当前项。如果你选择后者，它会在标记当前动作为「已丢弃」的同时，再和完成动作时一样，自动生成下一个重复动作。

应用到我的场景中，我就可以一路将日期为旅游期间的「登记工作时间」动作标为已丢弃，直到推迟至时间延后到有意义的日期。

![](.evernote-content/AF3A4C00-CF96-421A-B15D-4FD5D1595E6F/CB37F21E-DBD0-42E5-BFE9-A92A7E781264.gif)

丢弃重复动作

在 3.4 版本更新中，还对 OmniFocus 的 AppleScript 进行了更新，适配新的「已丢弃」属性。具体的 API 改变可以查看官方的[更新日志](https://www.omnigroup.com/releasenotes/omnifocus/3.4)。

需要注意的是，如果你最近活跃地使用 Omni Sync 进行同步，应用更新到 3.4 版本后，可以直接使用「已丢弃」属性。否则，「已丢弃」的图标会呈灰色，点击后 OmniFocus 会引导你手动进行一次数据库迁移。

### 任务管理工具 Agenda 集成了原生提醒事项

![](.evernote-content/AF3A4C00-CF96-421A-B15D-4FD5D1595E6F/1B4195B4-4DC4-4D57-8171-1F845D101059.png)

**Agenda 6.0（2019–07–04）**

[下载链接](https://apps.apple.com/cn/app/agenda/id1370289240)

**@Minja**：在 [奏折 28](https://sspai.com/post/55435) 中我们介绍过一个「任务、日历备忘录三合一」的应用 Re:Schedule，这次要介绍的 Agenda 可以说是前者的前辈，同样主打将几项任务相关的功能集合在一起。

Agenda 此次的主要更新点都围绕着**提醒事项集成**展开，在很多地方都和系统提醒事项实现了互通：

* 在备忘中直接**添加**提醒事项
* 在备忘中**修改**已有的提醒事项
* 通过 URL Scheme 从提醒事项跳回 Agenda
* 模仿自然语义为备忘设置**提醒**日期和时间
* ……

![](.evernote-content/AF3A4C00-CF96-421A-B15D-4FD5D1595E6F/EF2747A5-24FB-430D-98E6-DB0259BCE0A0.png)

通过 Agenda 创建提醒

对于有经验的用户来说，清单中的最后一项「自然语义」是个亮点，类似的设置方式在很多高级工具中都有出现，但不客气地说，Agenda 暂时只学了一个皮毛。它的语法用起来实际很糟糕。形似 `\remind(时间)` 的语法需要中文用户反复切换键盘和全角 / 半角，这种输入体验是很别扭的 —— 即便是英文用户，来回切键盘输符号肯定也不舒服。

虽然还不好用，Agenda 这次更新起码给任务管理工具提供了一个改良方向：**和原生提醒事项靠拢**。由于任务管理工具天天要用，粘性很大，已经用过一段时间老牌或原生工具的人，一般很难说换就换，于是 Agenda 为代表的这批新秀（Agenda 本身不算太新，也发布一年了）都面临着**迁移问题**：凭什么我要把数据迁移过去？搬家起来累不累？用得不爽了换回旧工具麻烦不麻烦？集成原生提醒事项就是一个很好的过渡方式，用户以前的数据都在，不用从零开始使用一个新工具。再考虑到 iOS 13 的提醒事项有了很大改动（可参考 [这篇文章](https://sspai.com/post/55237)），估计已经足够满足不少没有复杂需求的用户，像 Agenda 这样集成原生提醒事项的第三方应用就显得更加明智了。

#### 我的观点

这次草案的图片在游戏圈迅速传开，玩家和从业者们的槽点主要集中于各年龄段游戏不应出现内容，特别是 18 岁以上（18+）这部分的规定。

在一群人的狂欢之中我们也要理性看待这次的草案，首先草案的建立就值得肯定。先不管它内容是否适合，只要这套草案能够获得官方支持以及大部分游戏公司的配合，那么它就能够将成年人与未成年人做一个区分，避免现在「把大家都当成乖乖宝宝」的局面。

另一个比较大的批评点是草案对 18+ 游戏的内容限制或许苛刻，毕竟成年人已经拥有了完全的信息甄别判断能力。而且对比国际上最通用的 [ESRB](https://www.esrb.org/ratings/ratings_guide.aspx)（娱乐软件分级委员会）以及日本游戏分级标准 [CERO](https://ja.wikipedia.org/wiki/%E3%82%B3%E3%83%B3%E3%83%94%E3%83%A5%E3%83%BC%E3%82%BF%E3%82%A8%E3%83%B3%E3%82%BF%E3%83%BC%E3%83%86%E3%82%A4%E3%83%B3%E3%83%A1%E3%83%B3%E3%83%88%E3%83%AC%E3%83%BC%E3%83%86%E3%82%A3%E3%83%B3%E3%82%B0%E6%A9%9F%E6%A7%8B)（计算机娱乐分级机构），草案的 18+ 大概相当于这两个标准的青少年级别（12–19 岁）。

![](.evernote-content/AF3A4C00-CF96-421A-B15D-4FD5D1595E6F/8A5396DF-4C16-4977-9B16-23B5ABE53F79.png)

ESRB 分级标准

虽然草案的确把成年人当成了「祖国的向阳花」，但还是那句话，只要这套草案能够获得官方支持以及大部分游戏公司的配合，**包含限制内容的游戏就从以前的不能上变成了有适龄提示就能上**，相信以后根据实施效果限制的内容也会渐渐地放宽。人民网也重点表达了「不强制」与「征求意见继续修订内容」的态度。

### 微信解决了「停机怎么充话费」的难题

**@Minja**：出门在外，手机欠费停机时怎么充钱，是一个先有鸡还是先有蛋的问题。长久以来让很多人疑惑的是，停机时分明还有没用完的流量，却连一个充值页面都打不开。

最近微信联合三大运营商推出了「手机绿色缴费通道」，终于解决了停机状况下充不了话费的尴尬问题。停机时用户会收到缴费提醒短信，按提示打开网页即可缴费。只要保持移动流量开启，即便手机已经被停机，这一充值页面仍然可以正常打开。目前，这一功能已经在**两湖两广、四川和青海**推出，各地支持的运营商有所不同，其他地区今年稍晚推进，大家可以前往 [微信官方](https://mp.weixin.qq.com/s/DPxgnEcUaahZ5oWRJ6hKtQ) 文章中查看。

![](.evernote-content/AF3A4C00-CF96-421A-B15D-4FD5D1595E6F/11B329FA-6924-412F-BA00-B110671270DB.png)

中国电信的充值页面

这一停机缴费功能帮我们免去了四处蹭网或借钱充值的麻烦，不过也有人对安全性表示了怀疑。微信派公众号也给出了简明的原理解析，如果实在担心，可以把自己所在地的充值网址**保存为书签**，避免恶意链接的骚扰。

技巧：如何加速 iPhone 上的 Face ID
-------------------------

**@沨沄极客：**使用 iPhone X 和更新机型的朋友们可能会发现 Face ID 和 Touch ID 之间的区别，Touch ID 是一种触觉反馈，你能明确知道自己有没有按到 Home 键。而 Face ID 是一种视觉反馈，需要你对准摄像头，然后看到 的图标变成打开状态才能确认。

所以尽管 Face ID 的认证过程本身算是很快了，给人的感觉还是不如 Touch ID 来得快。还有一种情况，如果你经常戴着口罩或是墨镜出行，你会发现 Face ID 会比平时花费更多的时间来解锁。

如果你想让 Face ID 以更快的速度解锁，只需要关闭「需要注视亦启用面容 ID」这一选项，你就可以省下检测眼睛是否直视手机的时间，提升解锁速度，**几乎是抬起手机的同时就能看到正在打开的** 。

![](.evernote-content/AF3A4C00-CF96-421A-B15D-4FD5D1595E6F/1A8198A4-DC11-41D6-ADCF-6E94435D27FA.jpg)

关闭「需要注视」选项

这个方法可能有一定的安全风险，但也仅限于身边的人恶作剧。仔细想想，又有几个同事会在你睡午觉的时候从你口袋里掏出手机解锁呢。所以现在禁用这个选项来提升解锁速度，也未尝不可。

### 文章：《走向超级人工智能第三部分 —— 它们至今无法做到的事情》

**@Fairyex：**人工智能（机器学习）是最近在科技行业炒得大火的概念，下至普通的逻辑算法上至高端的深度学习 LSTM 算法都能被安上「人工智能」的名头，在这种铺天盖地的宣传下，大众也渐渐地习惯了「人工智能无所不能，能解决任何问题」的认知。但是，现在的人工智能并没有我们想象中这么厉害，真正在搞前沿技术的人士对于现在的形势认知也是比较理性的，比如前 MIT 人工智能研究所所长 Rodney Allen Brooks 就撰写了四篇主题论文来给大众科普正确的人工智能现状。

[文章](https://rodneybrooks.com/forai-steps-toward-super-intelligence-iii-hard-things-today/)

![](.evernote-content/AF3A4C00-CF96-421A-B15D-4FD5D1595E6F/B4E4C31C-CD7D-4A38-A421-F70C0604595B.jpg)

作者画像

今天我们推荐的就是其中的第三篇，这部分论文专注讨论了为什么人工智能在某些领域的应用迟迟得不到进步，而它跟智能又有什么区别，有什么东西是它无法完成的。

文章通过列举了人工智能无法拥有真正感知、真正的控制感、真正的阅读理解等七个大点，并针对每一个点阐述了人工智能（以及由它操纵的机器人）与智能标准（人类）在获得信息与处理信息上的差异，利用生动详实的例子来解释为什么这些差异无法消除，以及消除这些差异有多大的可能性。

![](.evernote-content/AF3A4C00-CF96-421A-B15D-4FD5D1595E6F/20C2112F-4F49-49A0-8E50-0F4EFAF139F4.png)

感知错觉图，两个格子的颜色其实是一样的

比如在第一节作者就用了这个常见的感知错觉图，来说明机器人目前离拥有真正感知的能力还有一大段距离。图里面所指的两个格子颜色其实是一样的，我们的眼睛因为有阴影的欺骗，会认为两个格子的颜色明显不一样。人类非常善于产生类似这种错觉，而目前机器则做不到 —— 它们通过取色识别图像，相同色值就是相同颜色。

这种错觉不是坏事，反而是真正智能拥有的特征。它让我们即使是很嘈杂的地方也能听清楚对方在电话里的声音，猜测对方的位置，让我们能够通过母语的推测出一个人在哪里长大。人工智能显然还不能与我们的世界联系起来。

除了没有真正的感知，现在的人工智能也无法完成抓东西、读懂一本书、诊断修复问题、将人的行为与周围环境联系、自己编写程序等对人类而言习以为常的事情。

读者阅读完这篇文章甚至这个系列文章，能够对人工智能和机器学习的本质有一个客观的认识。能够正确看待各种应用服务的宣传，也能让自己对未来世界的想象更详实。
