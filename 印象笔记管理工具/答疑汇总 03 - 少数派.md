答疑汇总 03
=======

不知不觉教程已经只剩最后三章，在上一章中，我们终于进入到自定义样式的领域。如果说之前我们是在像对着菜谱做菜一样学习 Ulysses，那么现在已经开始直接调整菜谱，以满足自己的需求了。

既然进入了自定义样式的领域，终于能回答读者 @bobb1208 之前提出的问题：

> 最近刚尝试用 Ulysses 撰写论文，但样式仿佛少了点。且不论可供选择的选择大概才十几种，修改样式（字体、行距）也很麻烦。请问你如何看待这个问题呢？是不是应该由 Ulysses 导出后再用 Pages/Word 编辑格式呢？

以这位读者为例，许多人的写作需求，或者说写文章的真正目的，其实是为了导出一份样式相对复杂的文档。在这个过程中，其实是蕴含了**写作和排版**两个需求。

Ulysses 作为写作工具，在「写」这个环境无可挑剔。而 Word 是字处理工具，它的本职工作就是排版。所以不用纠结在 Ulysses 里写完再导出用 Word 排版，是没有发挥 Ulysses 的作用，其实它们两者的分工完全不同，我认为**两者搭配起来使用其实是正常的流程**。

不过大家有没有仔细想过，其实所谓「排版」也是分步骤的。调整图片位置是排版，设置标题样式也算排版。在 Ulysses 中，通过自定义样式，其实可以完成类似设置文字格式、调整页眉页脚等任务。即**通过自定义样式，我们可以在 Ulysses 中完成相当一部分排版操作**（具体是哪些，我将在第七章具体介绍）。这正是我喜爱 Ulysses 的重要原因。

从使用流程的角度来看，其实 Ulysses 和 Word 应该是接力赛跑的关系：前一棒跑的快，后一棒就轻松一些。面对论文、调研报告等大型文档时，如果我们能在 Ulysses 里多完成一些步骤，之后在 Word 中的任务就轻松一些。

说完 Ulysses 与排版工具的关系，我们再来说说 Ulysses 与图片的那些事。前者是 Markdown 与传统文档之间的矛盾，后者则是 Ulysses 与网络文章之间的需要改进的地方。

读者 @三黄蛋 问道：

> 图床是什么？Ulysses 需要图床么？
>
> 为什么给别人 MD 格式的文件人家看不到图片？怎么才能给对方直接可以用 MD 编辑器编辑的带有图片的文件？

这些问题涉及到一些普通用户不常接触到概念，极易让初学者陷入疑惑。

Ulysses 是以 Markdown 为体验核心的工具。 Markdown 是一种纯文本文件，只能存储字符。图片、音频视频等多媒体，是不能存储在 Markdown 格式的文件中的。Ulysses 编辑器本身对它们做了特殊处理，才使得我们可以预览图片和视频。但是导出成 Markdown 文件后，就不能把图片放在文件中导出了。

那这么解决这个问题呢？思路大致有两种：**使用 Textbundle 格式文件，以及将图片变为网络图片**。

Textbundle 一种文件格式，它既能存储 Markdown 文稿内容，又能存储图片。在 Ulysses 的快速导出中可以选择将文件导出为 Textbundle 格式。在第一章我们对此有过介绍。其实 Textbundle 的原理也不复杂，用特殊的编辑器打开它，就能看到它其实是将 Markdown 和图片文件打包在一起。但是这就确保图片也能被导出。

![](.evernote-content/AC1507B9-BFD0-4DA4-BBA9-0A2C35183126/98CDFDE9-1E99-4B3E-A759-E67344562777)

Textbundle 文件的内部结构

既然 Markdown 文件只能存储纯文本，那我就把图片信息变成纯文本。当然不是直接转换图片，而是将图片变成网络图片。对 Markdown 熟悉的朋友肯定知道，如果我们想插入一张网络上的图片，就可以用 `![]()` 语法写成：

```
![少数派的图标](https://upload-images.jianshu.io/upload_images/5811594-c06c14af2fe9986b?imageMogr2/auto-orient/strip%7CimageView2/2/w/512)
```

后面括号中的链接，就是图片的链接。在 Ulysses 中使用插入图片功能，将这个链接填在 URL 一栏中，和使用语法标记实际是一样的。

那问题来了，对于网站来说，有专门的服务器来存储这些内容，但对于个人用户来说，该怎么把本地图片变成网络图片呢？答案就是图床。

图床简单来说就是**在网络上存储图片的地方**。当然由于这并算不上一个多大的市场，所以并没有企业专门提供图床服务。实际上也不需要，只要是能存储文件的云服务商，并且能获得存储图片后产生的链接，就能当作图床。

图床的选择、使用和设置是相对复杂的内容，限于篇幅这里不作展开。少数派上已经有了不少教程，这里贴出来，方便有兴趣的读者自行阅读。

从第五章开始，我们进入了读起来稍微费劲一些的自定义样式的学习。但是如果能很好的理解自定义样式的用法和使用场景，就能让 Ulysses 发挥远超 Markdown 编辑器的价值。

我们下一章见！
