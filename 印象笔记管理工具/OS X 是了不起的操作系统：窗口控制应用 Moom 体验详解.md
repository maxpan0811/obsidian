目录

窗口（Window）在操作系统中，不仅是用户界面（User Interface，简称 UI）的改革，也是多任务管理方式的创新。OS X 的原生窗口管理方式虽然强大且高效，这并不等于足够「人性化」，所以问题来了：**如何应对个别用户需要定制窗口尺寸及位置的需求？**就今年 WWDC 2014 的发布现状来看，OS X 短期内在这方面并不会有所改进，所以第三方辅助工具的作用依然重要。

Moom 就是「**窗口控制**」软件的代表。它的体积不大，安装后仅占用 7.3 MB 空间，与之成反比的是丰富的窗口自定义功能和略贵的售价（10 美元），不过好在与同类应用 BetterTouchTool、BetterSnapTool 等横向比较，Moom 不仅用户体验最好，而且简单易懂，没用过的小伙伴也能轻松掌握。

鼠标操作花样多
-------

在 Moom 开启的情况下，无需点击，只要将鼠标移动至此，你就会发现原本代表全屏的「加号键」会变成「放大键」，并同时显示屏占比菜单（按住 Option 键可以暂时还原）。在这之中，白色部分代表当前窗口；浅灰色部分代表临界区域（即当前窗口的尺寸已达到允许值的最小）；深灰色部分代表桌面剩余区域。

![](.evernote-content/5057414D-7489-49F0-B532-88CC89CB365A/A24FAC57-3F32-48A3-9719-B133E748D34F)

由于笔者设置的桌布大小（即格子数量）为 **12 x 12**，所以看上去很密集，若你的调整要求较低，那么将该值修改为 **6 x 6** 或 **4 x 4** 这样较大的尺寸即可。使用方法也很简单，直接在格子中拖拽即可。

![](.evernote-content/5057414D-7489-49F0-B532-88CC89CB365A/28851870-F784-4A65-9417-8AE7F28B6032)

第二套方案：**鼠标位置识别**。（按照下图中的设置）拖动窗口的过程中，当鼠标接触到屏幕边缘时，所在位置的不同会对窗口调整的结果带来影响：**Quarter Screen** 代表 1/4 屏幕尺寸、**Half Screen** 代表 1/2 屏幕尺寸、**Full Screen** 代表全屏。该功能的默认反应时间为 0.2 秒，拉动 **Delay** 进度条即可进行调整。

![](.evernote-content/5057414D-7489-49F0-B532-88CC89CB365A/8A1BA7AC-916D-44A2-850C-006D8121008E)

键盘模式效率高
-------

上文介绍的鼠标操作其实很简单对吧？键盘的使用也同样 Easy。输入组合键 ⇧⌘M 开启键盘模式（默认，可以修改），随后会出现一个半透明的窗口，这就是 Moom 的**感应器**（它会使当前窗口高亮）

默认组合键如下：

* 左半屏（⌘ 左方向键）
* 右半屏（⌘ 右方向键）
* 上半屏（⌘ 上方向键）
* 下半屏（⌘ 下方向键）
* 还可以快速占据全屏（空格键）

![](.evernote-content/5057414D-7489-49F0-B532-88CC89CB365A/9A30C7D7-E4BE-4638-B192-67F218DFF11C)

如果这样得到的窗口太规范，不符合你的个人喜好，那么笔者建议在 **Custom** 自定义中设置。这部分的选项比较多，带来的好处是功能也多，但设置起来较为复杂，这里挑选几个适合大众的方案进行介绍，其余的留给读者自行摸索。

![](.evernote-content/5057414D-7489-49F0-B532-88CC89CB365A/82C37E2B-DC25-4046-BCE5-401B1E734F05)

---

* Move & Zoom（移动和放大）

与鼠标部分提到的调整方式完全相同，只不过使用的工具由鼠标变为键盘罢了。这里注意一下，桌布大小在窗口最下方 **Define window sizes** 处修改，然后在上方桌布中选择需要覆盖的区域，并在右侧设置组合键。

* Resize（尺寸重置，单位为像素）

左侧的蓝点代表**原点**，即窗口的左上角（初始位置），右侧输入窗口所占像素面积，以整数为佳。当然，别忘了设置组合键。

* Grow（增加尺寸）

这个功能是方便用户往单个方向，按照设置好的单位像素，增大当前窗口的覆盖面积。左侧蓝点代表方向，右侧数字代表一个单位的像素点数。

* Center（画面置中）

它的作用是，在不修改窗口横向尺寸的条件下，将其快速移动到屏幕正中位，是个很常见的功能。这里有两个选项：**actual screen center** 及 **standard OS X center**，唯一的区别在于，前者按照实际屏幕大小，后者遵循系统偏好。

--
