我的电脑桌面上现在贴着一块看板。

不是壁纸图片，是活的：左上角是长沙今天的天气和未来七天的温度曲线，往下是本月日历和待办，再往下，晨报出刊到哪一步，公众号、微博、小红书和播客的粉丝与播放，网站访客，服务器内存，GitHub 上四个仓库的星标，**一屏排开**。

![](.evernote-content/001256C5-D378-4DD0-8A9C-1079BDE95EBE/CB4BAB31-4A29-4797-97EB-40B084B2208F.png)

每天早上开机，我不用打开任何网页和 App，要盯的**八类信息**已经在那了。

这块看板是我用 **Claude Code** 造的，**方法和代码都开源了**，文末有仓库地址。今天想跟你聊的不只是怎么装它，而是为什么我觉得，往后每个人都该有这么一块。

![](.evernote-content/001256C5-D378-4DD0-8A9C-1079BDE95EBE/E9BE73F7-6FF6-417A-BBE4-EDC679EB6A96.jpg)

01PART先看东西：一块钉在壁纸上的看板八类信息一屏

从左上角说起。

天气那格，现在多少度、体感多少度、几点后有雨，下面一条未来七天的温度曲线。它替我省掉的，是每天早上摸手机看天气 App 那一下。

日历和待办在第二排。待办能直接在墙上操作：单击打勾，双击改字，列表下面就是输入行。不用打开任何软件 -- 桌面本身就是软件。

再往下是我的生意区：晨报链路(我的公众号每天凌晨自动出刊，七个环节走到哪一步，七个点从左到右变绿)、公众号微博小红书加播客四个平台的粉丝和播放、网站的访客和留言、服务器的内存硬盘、四个开源仓库的星标和 PR。每一格，替我省掉一个要登录的后台。

这块看板有三个地方，跟你见过的效率工具都不一样。

它没有窗口可开，直接画在壁纸上。底层用的 Rainmeter，一个二十多年历史的免费开源桌面引擎，说白了，就是一支专门往桌面上画东西的画笔。画出来的东西跟壁纸长在一起：不占任务栏，不抢内存，重启电脑它自己回来。

数据全在本地。采集脚本跑在我自己的电脑上，抓回来的数字写在我自己的硬盘里，不经过任何第三方服务器。

后台消耗接近零。它不会每分每秒刷新 -- 数据拉齐了就停，晨报七个环节全绿了就睡，睡到第二天凌晨再醒。我特意让它把调度做成这样，毕竟它是来伺候我的，犯不上反过来烧我的电脑。

02PART它不是买来的产品，是长出来的说完就改好

先说件好笑的事。

就在写这篇稿子的今天早上，我看着看板右上角的"7/7 已出刊"，愣了一下：今天不是 7 月 11 号吗，怎么显示 7 月 7 号？

其实那是"七步全部完成"的意思，七分之七。我自己定的显示格式，把我自己绕进去了。

搁任何一个商业 App，这就是个"已知问题"：提交反馈，等下个版本，等不等得到看缘分。而我只是跟 Claude Code 说了一句 -- 下面那排点已经能看出进度了，右上角改成日期和更新时间吧。

几分钟后，墙上就是"**2026-07-11 · 更新于 08:00**"。

同一个上午，我还嫌它半夜也在探测服务器，浪费。又是一句话：晨报全绿之后就别探了。它改完顺手把桌面日历的重绘也省成了一天一次。

前一天晚上是待办：我想要"单击打勾、双击改字"的交互，说完，当晚就用上了。

这就是看板跟你手机里的 App 的物种区别。App 是造好的，你只能用；看板是活的，它长。更新频次、显示内容、加什么模块、什么颜色什么字体、哪个数据放上面哪个放下面，全由你定 -- 而且是说一句，改一版，不用会写代码，我全程一行没写过。

**不是你适应软件，是软件适应你。**

03PART软件高定，过去只有企业买得起1% 到人人

你可能会问：效率软件成千上万，就没有现成的？

我找过。通用的个人仪表盘是有的，还很火 -- GitHub 上这个品类最热的一个项目，**攒了三万多颗星**。但它们都是"选择题"软件：作者预制几十个模块，你从里面挑。我要看的公众号后台、我自己服务器上的晨报流水线，选项里永远不会有。

这不怪那些作者不用心。是任何作者都不可能知道，地球上的每一个"你"到底要看什么。设计得再精密的产品，也只能给你几套皮肤、几个面板；而每个人的工作和生活，是千变万化的。

想完全合身，只有定制。定制软件过去什么行情？企业级项目，几万起步，上不封顶。给个人定制一块看板？没有公司接这种单，接了你也未必付得起。

所以"软件高定"这件事，长期以来跟普通人没关系。

1这一两年，风向真的变了

去年二月，前特斯拉 AI 总监 Karpathy 造了个词，**vibe coding**，意思就是：你只管说想要什么，代码的事交给 AI，忘掉代码的存在。去年年底，这个词被柯林斯词典选成了 **2025 年度词汇** -- 它不再是小圈子黑话了。

今年五月，李彦宏在百度的开发者大会上说，未来的软件可能是一次性的，软件会变成快消品，日抛型软件。同一场活动，一个八岁的小朋友展示了他用 AI 做的校园共享雨伞 App，已经在学校真的用起来了 -- 他爸妈都不是搞 IT 的。这一条我最当回事：八岁小孩的爸妈都不懂 IT，这一句，比十份行业报告都有说服力。

就在前两天，7 月 9 号，OpenAI 把写代码的 Codex 并进了全新的 ChatGPT 桌面应用，免费用户也能用。巨头们都在抢着把"能干活的 AI"装进普通人的电脑。

工具是到位了。但说实话，还差一截。

风投公司 a16z 今年二月发过一篇文章，标题很直接：《大多数人没法 vibe coding》。他们的判断是，这波浪潮目前只覆盖"知道终端是什么"的**那 1% 人口**。a16z 自己重仓 AI，立场你打个折听，但这个观察跟我的体感一致。文章里有句话说得很透：

工具给你能力，产品给你结果。

2那我为什么还是选了「手册」

我造这块看板的时候，恰好卡在这两者中间，最后选了个中间态。

我原本可以让 Agent 把它做成开箱即用的软件，但深思熟虑后还是选择打包成 skill，因为可定制化才是看板最核心的价值。

Agent，就是"能自己动手干活的 AI"的正式叫法，Claude Code 是眼下最顺手的一个。而 skill，你可以理解成给 AI 的一本施工手册：里面写清了怎么搭看板、怎么写数据采集器、我踩过的坑怎么绕。装上这本手册，你的 Claude Code 就从"会写代码的 AI"，变成"造过看板的施工队"。

做成开箱即用的软件，我就得替你决定看什么数据 -- 那我就成了又一个"选择题"作者。做成手册，决定权还在你手里：它教的是"怎么给你造"；造成什么样，由你说。

顺带说一句，"一次性软件"眼下也有反方声音：有行业评论提醒，涉及钱、责任、要长期保存的数据，别随便交给随手生成的软件。我同意。但一块只读数据、跑在自己电脑上的个人看板，恰好是最适合自造的那一类 -- 坏了重造就是，数据一个字节都不会丢。

04PART如果你不是自媒体人医生律师老师

看到这你可能会说：你这八类信息，天气粉丝服务器，是自媒体人的配置，跟我有什么关系？

问得对。看板上摆什么因人而异，但"信息散落各处"这个病，几乎人人都有。我拿三个职业换了位想：如果这块看板贴在他们的电脑上，该摆什么格子。

1医生：把"报告出了没"钉在墙上

《内科学年鉴》2016 年的跟踪研究里，医生每陪患者一小时，就要还给病历系统**近两小时** -- 这两小时里有多少，是在互不相通的院内系统之间来回点、反复刷新查"报告出了没"?

换位想，一个管床医生的看板至少值得摆四格：

➤

**我的床位**：自己管的病人一列排开，床号加姓氏，每人一个色点 -- 红是危重，黄是新入院或术后 48 小时，蓝是今天出院。早查房前扫一眼，晨会交班要报的那组数字就在眼前；

➤

**报告追踪**：自己开出去的检查检验，已出几张、未出几张，结果异常的标红点。省掉一天里第 N 次回工作站、逐个病人翻"检查报告"页；

➤

**文书倒计时**：按病历规范，首次病程记录 **8 小时内**、出院记录 24 小时内必须写完，超时要被质控扣分。哪份还剩两小时，倒计时排最上，不用靠脑子记；

➤

**随访到期**：出院患者今天该随访谁。这份名单眼下多半躺在医生自己的 Excel 和微信备注里 -- 有医院统计过，上电子随访系统之前，出院电话随访率只有**三成**，漏掉的大头就漏在人肉盘点上。

2律师：一格"死线雷达"

律师的早晨，是在 12368 短信、手机日历、Excel 台账三个地方交叉核对"今天有没有庭"。而每个在办案子身上，还挂着一串刚性期限：上诉期 15 天、举证期限、保全后 30 天内起诉……错过任何一个都是执业事故 -- 真有律所因为没在上诉期内把判决书转交当事人，被法院判赔。行业报告说律师 8 小时里只有 3 小时能计费，剩下的时间去了哪？对期限、翻台账、追账。

所以律师看板的头两格，都该围着"别错过"转：

➤

**今日庭审**：今天和未来三天的庭，案子、法院、几点，两案撞期自动标红 -- 提前三天发现，还来得及申请调庭；

➤

**死线雷达**：全部在办案件里最近的五个期限，按剩余天数排，剩三天变红。这一格等于每天自动做一遍"漏没漏"的盘点；

➤

再加一格**案件流水线**(立案-举证-开庭-待判-上诉-执行，六个阶段各挂着几个案子)和一格**待收律师费**(按拖欠时长分段) -- 客户来电问进展、月底对账，都不用再翻台账。

3老师：三个小程序并成一眼

带过班的人知道班主任的一天：七点多到校查考勤，课间问三个科代表收作业，晚上在 30 多个工作群里翻找被@的消息。皮尤 2024 年的调查里，**84%** 的美国中小学教师说常规工时干不完手上的活；国内教育部去年十月发文减负，八条措施里白纸黑字写着"精简数据填报"。

一个班主任的看板，四格能接住大半：

➤

**今晨到班**:41/43 的大数字，下挂没到的两个名字，红点表示还没联系上家长。不少学校的考勤制度要求未到未请假 10 分钟内联系家长 -- 红点没消，就是还欠一个电话；

➤

**作业收交**：语数英三科各自的收交计数，未交的直接列名字。原来这是三个小程序各点一遍的事；

➤

**本周薄弱点**：最近一次测验得分率最低的三个知识点，数据就来自阅卷系统导出的那份 Excel。明天讲评课先讲什么，抬眼即得；

➤

**填报倒计时**:"防溺水截图还差 7 人，今晚八点截止" -- 要不要现在去群里催一嗓子，一眼有数。

三个职业，同一个底层逻辑：这些数据全都已经存在，只是散在五六个系统、小程序和群里；看板做的事，是把它们抠出来，按"你"的优先级钉在一面墙上。行业里不是没有垂直软件，但它们按"平均的医生""平均的律师"设计，而你不是平均数。

还有一层，比合身更硬：这些数据根本不能随便上云。患者信息，《个人信息保护法》明文列为敏感个人信息；学生成绩，多地教育部门明令不得在班级群里公布；律师对委托人的保密义务，写在《律师法》第 38 条里 -- 而云端 AI 的消费者版，默认可能拿你的输入去训练。把这些数据填进去，不是效率问题，是执业风险。

而这块看板的数据，从头到尾不出你这台电脑，展示时还可以只留床号和姓氏。对这几个职业，**数据留在本机**就是用 AI 的前提条件，没有第二种及格线。

05PART怎么上手60秒与门槛

先看结果：装好之后，你对 Claude Code 说一句"帮我搭个桌面看板"，一分钟内，一块带时钟日历的看板就贴上你的壁纸了。先让桌面亮起来，再慢慢加东西。

三步装好：

1

从 GitHub 把仓库下载到本地(clone，复制一份的意思);

2

给 Claude Code 装上这本手册(一条命令，README 里直接复制);

3

说出那句话。

接下来是它问你，不是你查文档。它先给你的电脑做个体检：缺 Rainmeter 会提示一键安装，Python 版本不对会直说。然后一次性问你五个问题 -- 城市、想看什么、刷新多勤之类，全部留空也行，每问都有默认值。

想看的数据在官方模块里(时钟日历、天气、待办、GitHub、网络延迟、服务状态，共六个)，直接装。不在？这正是它跟所有"选择题"软件分道扬镳的地方：你把数据源告诉它，它**现场给你写一个采集器**。你家里的 NAS(网络存储盒子)、你们单位系统的数据接口、甚至一份定期导出的 Excel -- 只要数据够得着，就能上墙。

装任何模块之前，它会把"要联哪些网、读写哪些文件、要不要密钥"念给你听，你点头才装。哪天某个数据源改版，格子变灰，你说一句"帮我修看板"，它自己读诊断记录去修。

1四个门槛，先摆在这

我不想把这篇写成广告。这东西现在还不是给所有人的：

➤

只支持 **Windows**(看板引擎 Rainmeter 是 Windows 上的老牌开源软件，免费);

➤

需要装 **Python**(采集脚本靠它跑，免费);

➤

需要一款 **Agent 编程软件**：我全程用的是 Claude Code(Pro 档 **20 美元一个月**，官网口径含使用权)，但这本手册不锁定它 -- 仓库里带了通用说明，换 Codex 或国内的同类产品，一样能照着手册干活；

➤

最大的门槛其实在心理：它跑在终端里，就是那个黑底白字的命令行窗口。一位不懂代码的美国小企业主写过他的头一周："每次它要动我电脑，我都先去搜一下那条命令，确认不会搞坏什么。"这个阶段大概一周就过去了 -- 他现在用它管收件箱、做课件、搭家庭预算表。

四条都能接受，两个入口在下面——源码开在 GitHub，进不去就走我网站宝藏页的国内加速下载，那个 skill 就在页首的自研出品区。

GET IT两个入口，随手取用GitHub 拿源码；打不开就走宝藏页的国内加速下载，顺带能翻翻我做的其他工具。

06PART乐高、PR 与时代红利一人一块积木

看板现在有六个官方模块。这个数字，我盼着它尽快过时。

整个 skill 是按乐高设计的：每个模块自包含，拷走就能跑。写好一个新模块 -- 邮箱、股票、你们行业的某个系统 -- 通过一个 PR(在 GitHub 上把你的改进提交给原仓库，审核通过就合并进去)贡献回来，所有人就都多了一块积木。模块从六块长到六十块，后来的每个人搭自己的看板，就容易一分。

这种"人人加积木"的玩法，过去只属于程序员社区，因为做一块积木的成本太高。现在积木是 AI 做的，你出的是想法和需求 -- 而需求，恰恰是普通人比程序员富余的东西。

最后说说我把它开源的两个初心。

一是想让你亲眼看到，AI 时代的变化已经快成什么样 -- Agent 干活跟 AI 聊天是两回事。聊天的 AI 给你答案；干活的 AI 给你东西：一块真的贴在你桌面上、自己刷新数据、坏了还能自己修的东西。这块看板就是个活案例。看完这篇，你要是也起了"让 Agent 替我干点什么"的念头，它就没白开源。

二是希望每个人都能拥有一块自己的看板 -- 一个独属于你、由你亲手通过 Agent 定制出来的软件，实打实地把工作和生活的效率提上来。这在过去要开公司、组团队才做得到，现在试错成本低到一个下午，迭代快到一句话。

提效不是句空话。就说今天：看板替我盯着数据，我把这篇稿子写完了。

它明天大概又会变 -- 也许多个格子，也许换个颜色。它跟着我长。

你的那块该长什么样，你说了算。也欢迎来评论区说说：你的职业，最需要一块什么样的看板？说不定下一个官方模块，就是照着你的需求写的。
