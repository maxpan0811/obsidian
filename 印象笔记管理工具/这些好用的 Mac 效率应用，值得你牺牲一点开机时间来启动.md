提起开机自启动应用，我们总会对它深恶痛绝。但在 Mac 上，有一系列优质的工具能够提升你的操作效率，牺牲一些开机启动时间允许它们自行启动非常值得，本文就来盘点那些值得开机自启动的应用。

如何管理开机自启动应用
-----------

在 Mac 上我们可以利用两种方法允许或禁止应用的开机自启动。一是在每个应用的设置选项中勾选或取消勾选「登录时启动/Start at Login」选项，另一个是在 macOS 的「系统偏好设置 - 用户与群组 - 登录项」中批量进行开机自启动应用的管理。

![](.evernote-content/9AAAAC51-B04A-4398-8A3B-E3B45C4BCD0B/EDE23DA0-FA56-427E-8904-65F6C6EA73C3)

Recents - 文稿启动器
---------------

「开机即待命」指数：★★★★★

[Recents](https://sspai.com/post/43754) 是一款文件启动器，在今年年初时被安利这款应用之后，我每天都会频繁地使用它来打开文件。简单来说，Recents 把每个单个应用的「最近打开的文稿」列表归集到了一起。使用快捷键「Command + Shift + 空格」来让 Recents 窗口显示，窗口左侧显示了最近的应用，窗口右侧则是某个应用最近打开的文稿，最早可以追溯到数个星期之前。打开文稿后，窗口自动隐藏，干脆利落。

![](.evernote-content/9AAAAC51-B04A-4398-8A3B-E3B45C4BCD0B/12E85ABD-E6CA-402C-B967-2F9EBBDF4F0B)

Recents 一改我们先前打开文稿的思路。我相信大多数人打开文稿的方式都是，以文稿名为线索，在 Finder 中或者使用 Spotlight、Alfred 等搜索器找到它再打开。Recents 可以让你以应用为线索，再找到特定的文稿，你不需要像在 Sptolight 中进行的输入操作，更不需要在层层文件夹中找寻，Recents 的优势显而易见。想要用 Recents 打开文稿所在文件夹也很简单，在使用的时候按住 Command 键，窗口就会显示文稿的具体路径，点击即可打开该文件夹。

在 Dock 栏重按应用图标，也可以快速显示该应用最近打开的文件，看起来和 Recents 大同小异，但实际上 Recents 远胜一筹。一来，Recents 支持不在 Dock 栏常驻的应用， 二来例如 Word 和其他多款高频应用均不支持 Dock 栏重按显示最近文稿的功能。基于我个人的使用体验，Recents 支持 Office 系列、iWork 系列、OmniGroup 系列、PDF Expert、Pixelmator Pro、IINA 视频播放器、MindNode、VMWave 虚拟机等等应用。

开发者对 Recents 的更新非常活跃，应用也完全免费。对于经常在不同程序中打开各种文稿的用户来讲，[Recents](https://sspai.com/post/43754) 是非常值得「开机即待命」的应用。

Xnip - 截图工具
-----------

「开机即待命」指数：★★★★★

论 macOS 最佳的截图工具，可能非 Xnip 莫属了。

使用 [Xnip](https://zh.xnipapp.com/) 截图，你可以直接在截图上标注、打码、高亮、加文字；你还可以制作「步骤标注」，当需要一步一步指导别人怎么操作时轻松输出图文并茂的教程；对于任意区域的截图，你还能让截图带上 macOS 原生窗口截图的阴影效果。Xnip 还有更高级更特殊的截图玩法：滚动截图，支持；多窗口截图，支持； 窗口跨屏截图，支持。总之，[Xnip](https://zh.xnipapp.com/) 是 macOS 必备的截图工具，大部分功能免费使用，也是是非常值得「开机即待命」的应用。

Paste、Copied - 剪贴板管理器
---------------------

「开机即待命」指数：★★★★★

几乎每天我们都会用到复制粘贴这最基本的操作。在我没用剪贴板管理器之前我对剪贴板最大的痛点是，刚复制的内容被新复制的内容覆盖，以至于不得不追本溯源找到之前的内容原地址再重新复制一遍。[Paste](https://sspai.com/app/Paste)、[Copied](https://sspai.com/app/Copied) 等等的问世，完美解决了这个痛点，它们会把所有的复制内容都缓存归集起来，你用快捷键「Shift + Command + V」就可以轻松访问历史记录，再次粘贴只要双击就行，也可以利用「Command + 数字」，实在是方便非常。

![](.evernote-content/9AAAAC51-B04A-4398-8A3B-E3B45C4BCD0B/B69D000C-BF64-4D75-A95A-9861F1894204)

更高级的操作，如 [Paste](https://sspai.com/app/Paste) 的「钉起来」功能，可以允许你将一些常用的内容常驻在剪贴板中，工作中偶尔要用到的后台网址、开票信息等等，都可以利用这个功能快速引用。剪贴板记录也支持文本搜索、密码信息自动忽略，多项功能考虑周全。总之，剪贴板管理器也是非常值得「开机即待命」的应用。

Bartender、Vanilla - 菜单栏应用图标收纳器
------------------------------

「开机即待命」指数：★★★★★

在 Mac 上使用越来越多的第三方应用之后，菜单栏上常驻的应用图标会越来越多，但实际上真正高频使用的可能屈指可数。如果你不想见到菜单栏冗长的应用图标，那么使用菜单栏应用图标收纳器 [Bartender](https://www.macbartender.com/) 或者 [Vanilla](http://matthewpalmer.net/vanilla/)，能让你的 Mac 菜单栏化繁为简。

它们支持你将菜单栏的应用图标分为好几组，按照需求来决定一款应用的图标是否需要常驻在菜单栏上。一是常驻应用分组，在绝大多数情况下，这些应用会显示在菜单栏上；另一部分则是隐藏应用分组，它们也在菜单栏上，但是默认隐藏的，你需要手动点击一下才会展现；还有永远显示和永远隐藏分组，顾名思义这些分组里的应用图标会一直显示或绝不显示在菜单栏中。

![](.evernote-content/9AAAAC51-B04A-4398-8A3B-E3B45C4BCD0B/1F2988B2-A2B9-4C30-893F-C4EFC0C30D70)

不管是对于小屏幕的 MacBook 还是外接大屏显示器，简化状态栏不仅能让电脑界面更加简洁，而且更利于你在这一条应用图标中轻松找到需要的那个。菜单栏应用图标收纳器也是非常值得「开机即待命」的应用。

Hazel - 文稿整理器
-------------

「开机即待命」指数：★★★★★

在 WWDC 2018 上，[macOS Mojave](https://sspai.com/post/44840) 新增了桌面堆栈分组整理功能，但实际上，应用 [Hazel](https://www.noodlesoft.com/) 早已实现了将同类文稿合并整理的功能，而且这一切都是自动进行的。

简单来说，Hazel 是一款自动化工具，你可以利用它自行创建整理规则，将符合同一要求的文稿放进同一个文件夹中。比如，将桌面上「创建时间超过三天」的文稿标记上「亟需整理」的红色标签以提醒你及时整理它们；将「下载」文件夹中以 .mp4 或 .mkv 为后缀的文件移动到「影片」文件夹中并标记绿色的「娱乐」标签；最实用的莫过于自动删除 QQ 或微信缓存文件夹内的大型文件。

![](.evernote-content/9AAAAC51-B04A-4398-8A3B-E3B45C4BCD0B/A00E06C3-3D71-418B-AFD0-78D529C3D61C)

总之，Hazel 提供了包括文件夹名称、创建时间、扩展名、标签、子文件夹等等在内的多种规则条件，经过巧妙的组合设计，你总能创建出符合自己要求的文件整理规则。文稿管理器 [Hazel](https://www.noodlesoft.com/) 也是非常值得「开机即待命」的应用。

推荐阅读：

* [Hazel：自动整理，让你的 Mac 井井有条](https://sspai.com/post/35212)
* [利用 Hazel 和 Automator 自动转换文件格式](https://sspai.com/post/33808)

Hazel 是一款收费软件（[下载地址](https://www.noodlesoft.com/)），五月初的时候发布了 4.0 版本，单独购买是 $32，Family Pack $49，从 3.0 版本升级需要 $10。初次下载可以免费试用 14 天，建议大家先试用再购买。

Manico - 应用切换器
--------------

「开机即待命」指数：★★★★☆

在使用 Mac 时，我们会频繁地在多个窗口中切换，除了使用鼠标点击外，我们还可以使用 Command + Tab 来切换当前显示的窗口。不过 Command + Tab 的问题在于，如果我们打开十多个窗口时（常有之事），用它来切换窗口就显得很不方便。[Manico](https://manico.im/) 就可以解决这个问题，它能让我们利用 Option + 数字来快速切换应用窗口，如果数字不够用，那么字母 QWER 也可利用。

![](.evernote-content/9AAAAC51-B04A-4398-8A3B-E3B45C4BCD0B/810C3765-B602-4E0B-9B52-B98C18078E1C)

Manico 还有多项周到考虑的功能，加入在「过滤列表」里应用不会显示在切换器中，Dock 模式和切换器模式分别对应 Dock 上的应用和 Command + Tab 应用，还有自定义快捷键、显示外观自定义、静默模式屏蔽 [Manico](https://manico.im/) 等等多想贴心实用的功能。应用切换器 Manico 也是非常值得「开机即待命」的应用。

推荐阅读：[简洁、高效、易用，Mac App 快速启动及切换工具：Manico 2.0](https://sspai.com/post/32457)

你可以从 [Mac App Store](https://itunes.apple.com/cn/app/manico/id724472954?mt=12) 和[官网](http://manico.im/)下载Manico，两者功能没有区别。

Tooth Fairy
-----------

「开机即待命」指数：★★★☆☆

AirPods 的热销，也让它在 iOS 和 macOS 互相切换时的不便愈发凸显。在 Mac 上，我们要先点击 Mac 状态栏上的音频或蓝牙图标，再选择 AirPods。[Tooth Fairy](https://itunes.apple.com/us/app/tooth-fairy/id1191449274?mt=12) 的巧妙之处在于，你只需要一个快捷键就能让 AirPods 和 Mac 快速连接。实际上不只是 AirPods，其他蓝牙耳机或者蓝牙设备，都是 Tooth Fairy 支持快速连接的设备。

![](.evernote-content/9AAAAC51-B04A-4398-8A3B-E3B45C4BCD0B/6BA59F69-910E-4D4E-9AC5-255A1719CA59)

Tooth Fairy 会将蓝牙设备的信息状态直接显示在 Mac 状态栏中。当设备处于连接状态时，它的图标是实心的，断开时则是空心的，电量信息也可以显示。以一个快捷键来取代两步点击操作，[Tooth Fairy](https://itunes.apple.com/us/app/tooth-fairy/id1191449274?mt=12) 的创新和实用也让它成为我心目中值得「开机即待命」的应用。
