的](http://lizhi.us12.list-manage1.com/track/click?u=d7d162204694c4d5a53462f09&id=49148dbaa8&e=9065a2420a)」，没想到一下成了近半年来阅读、转发量最高的「干货」文。看来大家都比较需要这种类型的文章，那废话不多说，我们再来一波。此外大家不要光看不回嘛~到 [博客](http://lizhi.us12.list-manage.com/track/click?u=d7d162204694c4d5a53462f09&id=ed2ec85437&e=9065a2420a) 留点评论给点鼓励，我们可以做得更好！    **▍****KeyCue**    上次提到快捷键是 Mac 的利器，用好了效率随时翻番。但对于初学者来说快捷键的数量可能太多，特别是刚从 Windows 转过来的话，光分清 ⌘、⌥、⌃ 这几个符号也得花点时间；更何况，除了系统快捷键还有各软件自己的快捷键呢。如果您也有同样的困扰，[KeyCue](http://lizhi.us12.list-manage.com/track/click?u=d7d162204694c4d5a53462f09&id=b614919e97&e=9065a2420a) 可以帮您随时查看当前软件的快捷键，不用死记硬背。        **▍****Fluid**    习惯了快捷键的话，还记得上次介绍过的 [CmdTap](http://lizhi.us12.list-manage.com/track/click?u=d7d162204694c4d5a53462f09&id=1ce599286a&e=9065a2420a) 和 [Manico](http://lizhi.us12.list-manage1.com/track/click?u=d7d162204694c4d5a53462f09&id=dbc5283538&e=9065a2420a) 吧？用它们来配合 [Fluid](http://lizhi.us12.list-manage1.com/track/click?u=d7d162204694c4d5a53462f09&id=4512ba8d37&e=9065a2420a) 这个软件，也可大大提高效率。Fluid 可以将网站打包成独立程序，这样你就可以为它们分配上快捷键，窗口切换更方便，适用于需要长期挂着好几个网站的用户。但 Fluid 的只能用 Safari 内核来处理网站，个人感觉略显笨重，也没法再使用 Chrome 的丰富插件。        **▍****滴答清单**    貌似大家对 GTD 工具的态度一直在变化，从一开始惊为天人感觉用了就能拯救世界，到现在又说不被工具绑架纸和笔照样 GTD…然后又开始讨论用哪个牌子的「手帐」做 GTD 更好…本文不说 GTD 好不好，因为笔者也只是用清单类工具比较多，从一开始的 Wunderlist 到现在越做越好的 [滴答清单](http://lizhi.us12.list-manage.com/track/click?u=d7d162204694c4d5a53462f09&id=9e9e1084f6&e=9065a2420a) ，感觉挺够用的，就没折腾别的了。    清单工具其实都很成熟了，选择也很多，至于为什么是滴答清单而不是别家，主要是因为它的「语音输入」功能真的很好用，日期时间识别又快又准，连笔者这个一直排斥微信语音的人也爱上语音输入。    然后在常驻任务栏的 Mac 版里刷新一下，任务便可轻松同步，再也不用鼠标在日历、时间上点来点去了。    **▍****Typinator 的变量功能**    上一篇我们介绍了 Alfred，稍微提到了一个功能叫「Snippets 文本替换」，能将用户输入的字符变为自定义的内容，比如输入「smlz」可以自动变成「数码荔枝正版软件老板以为自己看起来很帅其实并没有」字样。但 Alfred 的这个功能应该尚在起步阶段，较为简陋，缺陷之一便是无法「自定义变量」。所以如果您也经常需要接触固定的内容格式，比如月度报表，或许可以试试 [Typinator](http://lizhi.us12.list-manage.com/track/click?u=d7d162204694c4d5a53462f09&id=e17f49d5c0&e=9065a2420a)。    如下图所示，上新软件时笔者需要和团队其他小伙伴交接信息，文字格式基本不变，只是每次改动一点软件资料就够了。所以在 Typinator 里，我设置了如下模板：        使用的时候只需要输入「lzshangxin」字样，依次填写软件信息，便可生成完整的一段话。        当然 TextExpander 也是同类产品里的大牌，但是好贵啊……    **▍****用 Together 整理文档**    除了本地硬盘的个人文件，笔者还需要频繁使用储存在网盘的团队文件。因此以前我经常需要打开这个文件夹看一下，又打开另一个文件夹参考一番。[Together](http://lizhi.us12.list-manage.com/track/click?u=d7d162204694c4d5a53462f09&id=15b24ff535&e=9065a2420a) 则可以帮我用 iTunes 的方式管理散落在各处的文件——不管原文件在那里，它都可以把它们添加到同一个「虚拟目录」统一管理，比如下图：        「会员通讯模板」编辑的文件夹里，「运营与国内合作…」是老板的，「海报规格」是设计师的…而笔者作为协调的角色，三个都要经常看。现在我可以把它们都设置成「Favorite Items」，只要打开 Together 就可以查看。由于 Together 只是指向原文件而不需要将其复制到本地硬盘，因此假如原文件有变化，我这边也会随之更新。    最后是此前有人问过的，Mac 或者 iOS 平台听无损音乐哪个软件好。笔者自己用的是 [Vox](http://lizhi.us12.list-manage.com/track/click?u=d7d162204694c4d5a53462f09&id=26e5d275e8&e=9065a2420a)，颜值高、界面简洁有质感，格式支持也多，还能导入 Spotify 的资料。有需要的小伙伴可以试试。    最后感谢大家的阅读，下周见。 | | | | | |  | | --- | |
