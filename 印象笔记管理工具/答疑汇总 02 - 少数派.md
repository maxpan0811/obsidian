答疑汇总 02
=======

感谢各位对教程的持续关注，经过第一章鸟瞰式的对 Ulysses 的了解，第二章我们比较深入地讲解了，Ulysses 在写作时会用到的功能和技巧。各位一时觉得所有功能不能完全施展也无妨，当你们在写作时，遇到了相应需求或者问题，可以再回过来阅读相关小节。

第二章中我们讲到了 Markdown XL 之外的 Markdown 语法以及自定义语法。读者 @zixr365 就问道：

> Ulysses 默认是使用 Markdown XL的语法格式，但 Markdown 原生格式是否与更多的编辑器及网站兼容？对于这两种语法格式，你的建议是什么？是用 Markdown XL格式还是切换为 Markdown 格式？如果是保持用markdown XL格式，原因是什么？

不得不说，虽然 Markdown 的本意是简化我们在写作时的负担，但是各式各样的语法也着实给选择困难症造成了一定困扰。

首先可以确认的是，Markdown 的[原生语法](http://wowubuntu.com/markdown/)几乎能在所有支持 Markdown 的网站、编辑器中运用，比起其它语法兼容性更好。

一方面是因为它已经被大众接受，新出现的网页或者编辑器一般会考虑到用户的接受度。Bear 和 Ulysses 一度因为修改了原生语法中的一些标记，引起了一些用户的争议。另一方面原生语法是其它语法的基础，这意味着它和其它语法往往是包含与被包含的关系。即使 GitHub 有了 GFM 语法，你使用原生语法也是可以被正确解析的。

但以我个人意见，我仍然建议**在 Ulysses 中主要使用 Markdown XL 来书写**。最主要的原因是 Markdown XL 默认设置好了注解、删除、视频等标记，失去了这些标记，会让写作体验变的不完整。如果是自己在 Ulysses 中给 Markdown 语法加上这些标记，那又为何不直接使用 Markdown XL 呢？

如果文稿是需要运用到其它平台，例如 GitHub，可以在导出前使用「转换标记」功能就可以。保持在 Ulysses 中书写时使用一种语法，让自己尽快形成肌肉记忆，是可以切实提高输入效率的好办法。

第二章中还有一节的内容聚焦如何在 Ulysses 中整理文库。读者 @MilkShake羊 就问道：

> Ulysses 似乎没有 crosslink 的功能，就是从一篇 Sheet 里面打开另一篇 Ulysses sheet 再返回 ？

如果能用交叉引用的方式，就能以像蜘蛛网一样串起文稿来。印象笔记中就有笔记链接功能。

在 Ulysses 中也有类似功能。在文稿栏按住「⌥」键再右键点击文稿，就能看到「拷贝回调 URL」选项。在 iOS 中则是左滑文稿，选择「更多 - 分享」，就能看到「拷贝回调 URL」选项。这里的回调 URL，可以当作一般网页的链接看待，只不过点击它打开的是 Ulysses 中的一篇文稿。

回调 URL 用法也和网页链接一样，要么直接插入文章中，要么当作链接插入。值得注意的是，这个 URL 可以在本机中其它软件运用，而不限于 Ulysses，例如 Bear 等 Markdown 编辑器。

可惜的是，在 Mac 上并没有很好的办法，能在打开后跳转回原应用界面。而在 iOS 上我们可以借助 X-Callback 特性，尝试实现这个需求。
