苹果终于解决了我对 macOS 最大的不满之一， 从 OS X 10.4 Tiger 时代开始，macOS 就在我心中占有特殊的地位，相比 Windows 系统，它在美学和设计方面远远领先。

![](.evernote-content/A8165DCD-E85A-4608-A5EE-30F8A8B52295/A5EB6A38-517E-474C-BCA7-3ECE87F71EFF.png)

至今，我仍然因为许多原因更喜欢 macOS。不过，有一项特性，macOS 一直没有实现，而我也一直希望它能从 Windows 中借鉴过来，那就是窗口贴靠（window snapping）。

这是一个对我的工作效率至关重要的功能，直到现在，我才不得不依赖第三方工具来实现这一功能。

但现在，在 macOS Sequoia 中，我不再需要了。

终于等到了。

什么是窗口贴靠？
========

在 Windows 7 中，微软引入了一个功能，即将应用程序窗口移动到屏幕的边缘或角落，窗口会自动“贴靠”到那个位置。

一个窗口移动到屏幕的左侧或右侧，会自动占据屏幕的一半；如果窗口移动到角落，它会缩小并填满屏幕的四分之一区域。

Windows 11 对这一功能做出了进一步改进，使得创建各种窗口布局变得异常简单。

![](.evernote-content/A8165DCD-E85A-4608-A5EE-30F8A8B52295/3FB13A85-D4AF-46EB-BC83-4F3C17F650E8.png)

Windows 11 中的窗口贴靠功能。

但是，macOS 用户一直都没有享受到这一功能。

当然，我们可以将应用窗口平铺到屏幕的左侧或右侧，但这和 Windows，甚至是某些 Linux 发行版的体验完全不同。

![](.evernote-content/A8165DCD-E85A-4608-A5EE-30F8A8B52295/0CF1A7A8-EBA2-4BCA-BDAF-1971148612F2.png)

从 Windows 回到 macOS 后，我忘记了 macOS 不具备这个功能，坦率来说，我感到有些震惊。

真的吗？苹果竟然没有加入这个功能，而微软却有？

第三方解决方案
=======

幸运的是，在经过一些研究后，我发现了一个收费工具——Magnet，它可以为 macOS 带来类似的窗口贴靠功能。使用 Magnet，我可以通过键盘快捷键或将窗口拖动到屏幕的边缘或角落，像在 Windows 中一样进行窗口管理。

![](.evernote-content/A8165DCD-E85A-4608-A5EE-30F8A8B52295/8F07CBAD-149D-4F5A-856E-AE149797B6A3.png)

Magnet 当时花的钱是值得的。

自然，在我为这个工具付费后，我发现了一个免费的工具——Rectangle，它也能实现同样的功能。（真是让人哭笑不得。）但我一直在享受使用它。

不过，体验上与我在 Windows 中的使用感受略有不同，但它也能满足我的需求。

![](.evernote-content/A8165DCD-E85A-4608-A5EE-30F8A8B52295/ABE37BC4-A24C-41D7-BCC2-A7F8963F3CD3.png)

Rectangle 让窗口布局变得更灵活。免费的工具，表现不错。

不过，对我来说，更让人郁闷的是，为什么这一切甚至是必须的？苹果怎么这么多年都没有提供窗口贴靠功能？

感谢苹果，在 macOS Sequoia 中终于做出了改变。

像 Magnet 和 Rectangle 这样的第三方工具确实帮了我很大的忙，我确实也在想，随着这一功能成为 macOS 的原生功能，它们将如何应对？它们能否找到差异化的竞争点，保持相关性？我希望它们能做到。

终于可以轻松布局窗口了！
============

虽然苹果在简化窗口布局这方面来得有点晚，但迟到总比没有强。

虽然我非常喜欢 macOS，但在 Sequoia 之前，它处理窗口管理的方式让我非常讨厌。

通常，我会觉得操作系统之间抄袭是件可笑的事，因为它们经常这样做。但在这种情况下，我很高兴苹果终于意识到了这个问题。

能够将窗口平铺并贴靠到边缘或角落，对于我这种同时处理多个任务的人来说，是一项非常重要的功能。

我一直很难只专注于一件事（这可能是我应该去反思的地方），如果没有窗口贴靠功能，管理多个应用窗口确实有些困难。

所以即使这个想法最初并不是苹果提出的，我也很高兴它最终在苹果的平台上得以实现。

![](.evernote-content/A8165DCD-E85A-4608-A5EE-30F8A8B52295/461E0D53-D635-4E63-8244-639E74FE73F0.png)

所有 macOS 用户，欢呼吧！本期分享到这了，下期我们再见！！！
