有过好几次，别人问我要「文章阅读清单」，我都很热心地把 Reading List 里的链接一一发给他。但事实证明，这样不仅对发送链接的人来说浪费时间，接收的人读起来也不方便，链接还容易丢。最后，好心却办不成事。

![](.evernote-content/9E356A56-D9F2-441A-A86B-F768B646EBA7/5B72A5AA-2C61-4040-8179-A0917CA43D59.png)

通过微信发送的文章，八成是没人读的

所以，我一直想要一种快速**从浏览器分享文章阅读清单的方法**，最好能做到：

* **快速**，能够一键导出清单
* **轻量**，不需要专门安装软件
* **自由**，不需要注册任何帐号，方便接收的人直接查看
* **自定义**，可以只导出自己想要的页面，可以自己调整导出的格式和样式，甚至还能排版美化一下……

最近 macOS Mojave 渐渐普及，新系统的限制迫使 [Marked](http://marked2app.com/) 作者 Bretta [宣布停止更新一款 Safari 拓展](http://brettterpstra.com/2018/11/04/the-future-of-the-tablinks-safari-extension/)，而它的功能恰恰就是我想要的「导出 Safari 标签页」。听到这个消息，我一面感叹相见恨晚、初识它就失效了；另一方面，我决定写个脚本来满足前面提出的需求。

**通过脚本制作清单，就不用再一条一条手动发送，发消息的人省力，收消息的人也可以统一保存链接。**当然，光溜溜的脚本用起来很不方便，所以我还准备了 Automator 和 LaunchBar 版本供大家取用。

动作使用
----

清单制作动作有 Automator 和 LaunchBar 两个版本，最后效果是完全一样的，都会导出 Markdown 和 HTML 两个版本的，不过使用过程略有不同，下面分开介绍。

Automator 版所有 Mac 电脑通用，下载后解压、双击安装，动作会自动出现在系统右上角的服务菜单里。使用方法是：

1. 点击菜单栏左上角 - Safari - 服务 - 导出阅读清单 1 ；
2. 选择需要导出的标签页，可以按 `⌘Command - A` 全选；
3. 输入文件名、选择保存位置，默认名是 `Safari 阅读清单`，保存在桌面。

Automator 版效果演示

LaunchBar 版本主要是启动方式不同，后面的流程和 Automator 版一样。

1. 点击菜单栏左上角 - Safari - 服务 - 导出阅读清单；
2. 选择需要导出的标签页，同样可以按 `⌘Command - A` 全选；
3. 输入文件名、选择保存位置，默认名也是 `Safari 阅读清单`，保存在桌面。

LaunchBar 版效果演示

这两个动作背后是同一套 AppleScript 脚本，第一眼看上去可能有点复杂，但其实都是一块一块的「代码块」，并不难理解，而且可以直接复制下来当模板，用到其他 AppleScript 脚本里去。

### 雏形：导出所有标签

来审视一下「导出浏览器标签页」这个需求，各种疑问一下子炸开来：

* 要收集哪些数据作为素材？
* 导出来的清单要放到哪里？
* 怎样生成 Markdown 格式链接？
* 我是 Chrome 用户，我是不是没戏了？
* ……

你看，每一个疑问貌似都没有清晰的解决方案。我相信，太多的未知数，正是让很多人打退堂鼓、不乐意写 AppleScript 的原因。现在我们不妨倒推一下，理理思路：

1. 最终要把 Markdown 链接放到一个文本文档里
2. 要生成一堆 `[标题](URL)` 格式的链接
3. 要收集每个标签页的标题和链接
4. 要挑一个支持 AppleScript 的浏览器

为了避免混乱，浏览器我们挑 Safari，Chrome 部分稍后再讲；其余步骤都会有一段儿对应的 AppleScript 代码来解决。我们直接来看最后的代码，它长这样：

#### 调整链接排版效果

总的来说，调整排版效果的所需的知识逃不出 Markdown 那点语法，所以会用 Markdown 的读者现在就可以自行发挥。作为示例，我们来改改列表的显示效果。留意上节代码的粉色部分，其中组合出 Markdown 链接的代码部分是这样的：

```
(t as string) & ". [" & (item t of nameList) & "](" & (item t of urlList) & ")"
```

它的大意是，对于每个标签页，我们都提取出标题、链接，然后配上一些括弧，并在最开始加上循环的次数作为前缀，从而形成一个有序列表，其中每一项都是这样的格式：

```
序号. [标题](链接)
```

不难看出，这些 AppleScript 代码的背后是 `1. []()` 这条 Markdown 语法。比如说第一个标签页是少数派官网，那么最终的 Markdown 链接就会是 `1. [少数派 - 高效工作，品质生活](https://sspai.com/)`。现在，如果你想要无序列表，那么就可以把 `(t as string) & ". ["` 换成简单的 `- [`，就能生成无序列表版本的清单了。

### 改进：只导出自己想要的页面

刚才实现的效果比较基础，能够一次性导出所有的标签页。不过有些页面是不需要给对方看的（比如正在浏览的购物页面），总不能为了导出清单而把正在看的网页关了，事后手动去 Markdown 文件里删除也不方便。

**要是能在导出清单前，加一个任选标签页的步骤就好了。**

这也可以实现，只需要进行一点点改动。我们继续利用刚才生成的两个列表，不过这回还要引进一个新的 AppleScript 知识点：**从列表中选择**。

### 优化：自动转换成 HTML 格式

刚才我们导出的都是 Markdown 或 TXT 格式文件，它们看起来倒还整齐，而且基本上哪儿都能打开。不过我们还有一种选择，**那就是 HTML 文件，其中的链接可以直接点击打开**。如果接收清单的一方愿意，更推荐这种导出格式。

![](.evernote-content/9E356A56-D9F2-441A-A86B-F768B646EBA7/3C3C3FDB-73EA-4931-8244-B9C8FDCFB7F9.png)

转换成 HTML 的效果

要导出 HTML 非常方便，只需要在刚才的基础上加一行代码，就能把 Markdown/TXT 文件转换过去。不过为了以后的便捷，我们需要先做一点准备：**安装 [Pandoc](http://www.pandoc.org/)**。它是一个命令行工具，擅长各种文档间的格式转换。这篇文章里我们姑且大材小用一回，先不管更高级的用法，只用 Pandoc 来生成 HTML。

![](.evernote-content/9E356A56-D9F2-441A-A86B-F768B646EBA7/8EB3CA31-A8EB-496E-BEC8-1D29F1FDB731.jpg)

Pandoc 支持的格式多到夸张

安装 Pandoc 的命令是：

```
brew install pandoc
```

Pandoc 中转换 Markdown 到 HTML 文件的命令很简明，像这样：

```
pandoc -f markdown -t html -o HTML文件路径 Markdown文件路径
```

英文好的读者，可能已经不自觉地读出来了：从（from）Markdown 转换到（to）HTML…… 打住，搞定安装工序后，加速回到我们的脚本，按 AppleScript 的格式在最后加上一行：

```
do shell script "PATH=$PATH:/usr/local/bin/;pandoc -f markdown -t html -o ~/Desktop/Safari\\ 文章阅读清单.html ~/Desktop/Safari\\ 文章阅读清单.md"
```

`do shell script "你的命令"` 这串代码的作用是调用 shell 命令，可以看出，从 shell 到 AppleScript 还是有不少细节变化的 [[1]](https://sspai.com/post/52151#fn:1 "see footnote")，但本文先不深究，我们以后会有专门介绍 AppleScript 的系列文章。

现在再运行脚本试试，你就可以获得 Markdown 和 HTML 两份文件。

![](.evernote-content/9E356A56-D9F2-441A-A86B-F768B646EBA7/98BE4748-AC11-48DB-91BC-441B7F037B34.png)

后记

对我而言，导出浏览器阅读清单的需求其实一直都在，但是还没痛到非解决不可；直到发现市面上最好的插件（之一）已经挂了，我才下决心自己做一个。发到社交网络上之后，一些有类似需求的朋友也给了正面反馈。
