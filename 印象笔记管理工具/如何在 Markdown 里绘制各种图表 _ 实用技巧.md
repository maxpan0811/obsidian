如何在 Markdown 里绘制各种图表 | 实用技巧

图表，包括各种流程图、时序图、甘特图，相比大段的文字，用于更强的视觉效果，拥有「一图胜千言」的效果；而 Markdown，则是一种简约的标记语言，可以用几个特殊符号实现快速排版。

你有没有想过，Markdown 还可以用来画图表？

![](.evernote-content/A26F6343-EF65-4489-9BE2-889314C0C08A/197431FB-45DF-43F2-B0EE-A1BF2AA07C42.png)

用 Markdown 画图表

用 Markdown 画图表，听上去有点像在炫技，其实这是一种很轻量的画图方式，绘制快速、易于修改；另一方面，大量免费工具对于「Markdown & 图表」的支持，也降低了我们「打字画图」的门槛。这篇文章里，我将和大家分享一些常用图表的绘制方法。

本文写作环境为 Windows 10 1809，所用编辑器为 Typora，版本 0.9.62 (beta)。以 Typora 为例，在 Typora「偏好设置 → Markdown 扩展语法」中打开「序列图、流程图和 Mermaid 选项」即可开启相关绘制功能。

为什么要用 Markdown 画图
-----------------

诚然，市面上有强大的 Visio、Omni 家族，还有免费、在线的 ProcessOn、draw.io 等等优秀的流程图、时序图绘制工具，那我们为什么还要学习利用 Markdown 来绘制这些图表呢？

我主要青睐 Markdown 语言本身带来的特点 —— **轻量和易修改**：

1. 比起那些拖拽建立图标的工具，**Markdown 更轻量，只靠打字就能制作图表。**在我的想法意识流过程中突然出现「闪念」时，用 Markdown 可以快速抓住灵感。
2. **用 Markdown 绘制图像，后期对图像进行更改的成本会很低。**比如需要修改图中的几个错别字，那不必打开电脑、开启软件、找到图片源文件，而可以直接在 Markdown 文件中修改几处代码，在手机上也可以进行。

具体到工具方面，支持用 Markdown 画图的编辑器也不少，从免费大众的印象笔记、Typora，到经过配置的 Visual Studio Code 和 Sublime Text 都支持了时序图 Sequence、流程图 Flow Chart 等的绘制，有的甚至直接集成了 Mermaid 渲染引擎，可以让你创作更多个性图表。

接下来，我会具体介绍：**如何利用简洁的 Markdown 语言，来高效的实现对时序图、流程图、甘特图等的绘制。**

通用代码块
-----

首先，利用 Markdown 绘制时序图、流程图和甘特图的过程，本质上都是在 Markdown 中嵌入代码，然后编辑器利用渲染引擎渲染出相应的图片。Markdown 中代码块是通过下面的语法进行插入的：

```
​```代码语言
代码主体
​```
```

上面「代码语言」能够让编辑器知道代码块中的内容需要怎样进行渲染，这样可以让编辑器对代码主体进行高亮处理。如果遇到时序图（sequence）、流程图（flow）等的关键词，编辑器还会将代码主体渲染成相应的图像。这就是我们利用 Markdown 绘制图像的原理。

时序图 Sequence
------------

通常，Markdown 编辑器的时序图渲染引擎是 [js-sequence](https://bramp.github.io/js-sequence-diagrams/#syntax)。时序图的关键词是 `sequence`，来看个例子

```
​```sequence
Alice->Bob: Hello Bob, how are you?
Note right of Bob: Bob thinks
Bob-->Alice: I am good thanks!
​```
```

上面的例子经过渲染之后，是这个样子的：

![](.evernote-content/A26F6343-EF65-4489-9BE2-889314C0C08A/E7C3F5D4-D91A-46D5-8A07-03C3B9408046.png)

最基础的时序图

可能有人要说，这效果怎么比文章一开头简陋了那么多呢？具体的样式在每个编辑器里都有不同，开头展示的是印象笔记的效果，讲解代码时为求简明，委屈大家看看最基础的样式。

时序图在本质上是一个循环，其**基本元素包括标题 `Title`、参与者 `participant`、箭头 `->` 和标签 `Notes`**。经过渲染之后，每个参与者 `participant` 由两个元素块构成，一条线段将两个元素块连接，这样就可以方便的用箭头表示他们之间的转换关系了。

![](.evernote-content/A26F6343-EF65-4489-9BE2-889314C0C08A/76677518-4FF9-48E7-AAF5-A23C447252D8.png)

时序图的代码分解

### 标题和箭头的语法

```
​```sequence
Title: Here is a title
A->B: Normal line
B-->C: Dashed line
C->>D: Open arrow
D-->>A: Dashed open arrow
​```
```

**在时序图基本语法中：**标题 `Title` 放在第一行，之后是参与者与用箭头标识的参与者之间的相互关系。

![](.evernote-content/A26F6343-EF65-4489-9BE2-889314C0C08A/3BFD106A-80BE-4FE8-8C67-9EF664E7D308.png)

时序图代码和效果之间的关系

箭头有四种形式：常规箭头、虚线箭头、开口箭头和虚线开口箭头。分别通过：`->`、`-->`、`->>`、`-->>` 来表示。

![](.evernote-content/A26F6343-EF65-4489-9BE2-889314C0C08A/4DDCE787-7B46-4ADF-BF4E-DF8B978C3F6A.png)

时序图中的箭头效果

### 标签语法

```
​```sequence
# Example of a comment.
Note left of A: Note to the\n left of A
Note right of A: Note to the\n right of A
Note over A: Note over A
Note over A,B: Note over both A and B
​```
```

![](.evernote-content/A26F6343-EF65-4489-9BE2-889314C0C08A/D5A8ED85-3746-43E4-9027-90C0BF22FF6E.png)

标签代码和渲染出来的效果

在时序图中，我们可以通过「标签」这个组件来给我们的某个参与者或是某些参与者进行标注。标签可以放在参与者的左侧 - `Note left of A`、右侧 - `Note right of A` 或者浮于参与者上方 - `Note over A`。当然，标签也可以同时浮于两个参与者上方 - `Note over A,B`。

![](.evernote-content/A26F6343-EF65-4489-9BE2-889314C0C08A/B3A487AF-D759-473A-ADE2-655DD177DA00.png)

标签的渲染效果

流程图 Flow Chart
--------------

想要说明一个复杂流程，比如捷径的思路、文章的主线，可以试着绘制一副流程图。

通常，编辑器渲染时序图所用的渲染引擎是 [`flowchart.js`](http://flowchart.js.org/)，和绘制时序图一样，我们不需要专门研究引擎，只要把关键词指定为 `flow` 就可以开始画图了。流程图的代码部分相对复杂，我们先下面这个例子开始：

```
​```flow
st=>start: Start
op=>operation: Your Operation
cond=>condition: Yes or No?
e=>end

st->op->cond
cond(yes)->e
cond(no)->op
​```
```

上面这部分代码，我们可以看到流程图的渲染代码 \*\* 大体分为两个部分：

1. 元素定义部分：定义整个流程图中的节点元素。
2. 元素连接部分：将各个节点通过箭头连接，以表示整个图表的流向。

![](.evernote-content/A26F6343-EF65-4489-9BE2-889314C0C08A/88CCE8BE-5354-4520-8E12-999A5330951D.png)

元素定义和元素连接

经过渲染之后，我们可以得到以下这样的基础流程图：

![](.evernote-content/A26F6343-EF65-4489-9BE2-889314C0C08A/399708BB-6D4C-4984-AC03-EC7DB3730031.png)

最基础的流程图

不过这样的图过于单薄，看起来就像计算机课本第一单元的内容。我们接着看看其中的元素，进行深一级的绘制。

### 第一部分：元素定义

我们从第一部分开始，可以看到，一个流程图的基本元素有六大类别：

* `start`：流程图开始
* `end`：流程图结束
* `operation`：操作
* `subroutine`：子进程
* `condition`：判断条件
* `inputoutput`：输入输出

![](.evernote-content/A26F6343-EF65-4489-9BE2-889314C0C08A/3727019B-8375-4634-AF0C-9C6C27A35B32.png)

元素定义的代码分解

元素定义部分的基本代码结构是这样的：

```
tag=>type: content:>url
```

把这段代码拆开来看看，每一部分都可以自己定义：

* `tag`：就是作为第二部分连接元素时候引用我们定义元素时的名称，这个 `tag` 需要独一无二。
* `type`：就是我们流程图的六大基本元素类别，选择元素种类填入即可。
* `content`：是我们流程图元素中所显示的文字。这里的冒号与文本之间一定要有个空格。
* `url`：是元素所带有的超链接

这样就是我们元素定义部分的基本语法了

### 第二部分：元素连接

第二部分就是将我们在第一部分定义的元素，通过想要的方式进行连接。

![](.evernote-content/A26F6343-EF65-4489-9BE2-889314C0C08A/0A23400A-19DA-451B-B4C6-1644E6505025.png)

元素连接的代码分解

在这一部分，连接十分简单，就是利用 `->` 来连接我们在第一部分定义的那些元素，元素利用 `tag` 来区分。这里面需要注意的是：

* 对于每个「判断条件元素」 `condition`，都会有 `yes` 和 `no` 两个分支，比如上面代码中的 `cond(yes)` 和 `cond(no)`。
* 每个元素可以制定分支走向，默认是 `向下` 的，也可以用 `right` 指向右边，比如下面这个流程图中，我们可以将「子进程元素」 `subroutine` 放在右侧显示。有些扒网页的捷径非常复杂，想要说明的话可能就需要加上子进程来补充。@熊猫小 A 写过一篇很硬核的 [捷径文章](https://sspai.com/post/47070)，是说给 GIF 加进度条的，其中就用到了比较长的流程图来帮助读者理解制作思路，这样的图也可以用代码来画。

![](.evernote-content/A26F6343-EF65-4489-9BE2-889314C0C08A/12FD37ED-9176-49B2-B52C-49733C767003.png)

甘特图 Gantt
---------

对时序图、流程图熟悉的我们，对甘特图可能会有点陌生。**甘特图是来源于软件工程的一个概念，是一种用来表示一个项目的进度等与时间概念相关的条形图。**项目管理工具 OmniPlan 就可以把日历呈现为甘特图的样式，便于我们掌握进度。

![](.evernote-content/A26F6343-EF65-4489-9BE2-889314C0C08A/9CAE70EF-34C5-43B1-859C-CB23C6C85AE3.jpg)

OmniPlan 是经典的甘特图工具

目前来说，支持甘特图渲染的编辑器（比如 Typora 和印象笔记）几乎用的都是 [Mermaid 渲染引擎](https://mermaidjs.github.io/)。实际上，Mermaid 渲染引擎同时也支持对流程图和时序图的绘制。虽然看起来利用 Mermaid 绘制甘特图的语法有点复杂，但是分解成模块之后每个模块都十分好写。下面我主要介绍一下具体**如何利用 Mermaid 绘制甘特图**。

### 基础语法

我们还是从基础的语法开始。首先举个栗子

```
​```mermaid
gantt
    title A Gantt Diagram
    dateFormat YYYY-MM-DD
    
    section Section I
    Task I                      :a1, 2019-01-01, 30d
    Task II                     :after a1, 20d
    
    section Section II
    Task III in Section II      :2019-01-12, 12d
    Task IV in Section II       :24d
​```
```

![](.evernote-content/A26F6343-EF65-4489-9BE2-889314C0C08A/41EAD3EB-4508-4FCA-9D62-4CE15144B3C1.png)

甘特图效果

上面就是一个甘特图的基础格式和渲染效果。可以看到，一个甘特图的代码部分主要是如下的结构：

![](.evernote-content/A26F6343-EF65-4489-9BE2-889314C0C08A/E6991D90-13DD-4B70-AB18-BEEAAD7602B3.png)

甘特图代码分解

具体来说，首先定义代码块语言为 `mermaid`，然后在代码内容第一行声明 `gantt` 即定义了一个甘特图的代码框架：

```
​```mermaid
gantt
    ...
    具体甘特图的代码
    ...
​```
```

在代码框架下，具体的甘特图是这样定义的：

* 第一行定义甘特图的标题：`title: 甘特图标题`
* 第二行定义甘特图日期显示格式：`dateFormat: YYYY-MM-DD`

之后，先定义每个部分（区域）的 `section 区域名称`，然后定义每个部分的具体任务 `任务名称: 任务状态, 任务标志, 开始时间, 持续时间`。

甘特图的基础语法就是这样，这样画出来的图片已经有了雏形。接下来我继续详细说说 **日期** 和 **区域（任务）**的定义方式，把甘特图画得更完善一些。

### 日期格式（dateFormat）

日期在甘特图中是重点元素，相应的，它的语法也比较完善。Mermaid 渲染引擎底层是利用 `Moment.js` 渲染日期和时间的，其格式可以参考 > [Moment.js 处理字符串格式日期的文档](http://momentjs.com/docs/#/parsing/string-format/)。具体的，以今天（2019 年 1 月 20 日，下午 6 点 30 分 10 秒）为例：

* 年份：
  + `YYYY`：四位年份 - 2019
  + `YY`：两位年份 - 19
  + `Q`：季度 - 1（范围为 1 至 4）
* 月份：
  + `M`、`MM`：月份 - 1 、01（两位即补 0）
  + `MMM` 或 `MMMM`：月份全称 - January（会根据系统语言进行调整）
* 日期：
  + `DD` 或 `D`：日期 - 20
  + `Do`：日期（序数词表示）- 20th
* 天数：`DDD`，表示一年的第几天 - 20
* 小时：
  + `H` 或 `HH`：24 小时计时法 - 18
  + `h` 或 `hh`：12 小时计时法 - 6
  + `a` 或 `A`：AM、PM（上午或下午）- PM
* 分钟、秒：
  + `m` 或 `mm`：分钟 - 30
  + `s` 或 `ss`：秒 - 10

利用上文介绍的日期标识符，在其中用「-」或「/」进行连接就可以组合出我们想要的日期格式。为了方便理解，我们不具体介绍详细的日期实现方式，直接给出几种常见的日期定义格式的例子：

* `YYYY-MM-DD`：常见，也是上文例子中使用的格式，即 2019–01–20
* `YY/MM/DD`：即 19/01/20
* `HH:mm:ss`：即 18:30:10

在甘特图日期部分这样填写就可以。

### 区域代码（Section）

定义好甘特图的标题和日期格式之后，我们就可以定义一个又一个的区域（Section）了。所谓区域可以理解为一个项目，下面容纳了一系列小任务。

```
section 区域名
任务 I         :done, a1, 2019-01-01, 30d
任务 II        :active, after a1, 20d
```

每个区域的代码如上面所示，即：首先定义一个区域名，然后定义区域中的一个又一个任务。下图是一个简单的区域结构示意：

![](.evernote-content/A26F6343-EF65-4489-9BE2-889314C0C08A/F861CBB2-B475-4BCC-B387-7163EB418A0F.png)

区域代码分解

每个任务的定义也有规律可循：首先定义任务名称，之后用冒号衔接，接着定义 **任务状态、任务标志、开始时间和持续时间**。下面是一个简单的任务结构示意：

![](.evernote-content/A26F6343-EF65-4489-9BE2-889314C0C08A/1D6FE165-709C-462C-98D8-8A53B9169497.jpg)

任务有很多属性可以自己设置

* **任务状态** 可以填入下面三个值：
* `done`：表示任务完成
* `active`：表示任务正在进行
* `crit`：表示任务属于紧急任务
* **开始时间** 和 **持续时间**：按照日期格式进行填写。比如 `2019-01-01, 30d` 表示 2019 年 1 月 1 日开始并需要 30 天完成的任务

其中任务状态、任务标志都可省略，只要填写好任务的开始时间和持续时间就可以在甘特图中渲染出任务本体。

值得一提的是，我们可以利用一个任务接着另一个任务的特性，省去某个后续任务的开始时间。比如上面例子中的「任务 I」后面紧接着就要进行「任务 II」，我们可以给「任务 I」赋予标志 `a1`，在「任务 II」里用 `after a1` 来表示二者的衔接关系。

小结
--

那么这篇文章到这里就结束了。值得一提的是，最后一部分 Mermaid 渲染引擎要比我介绍的这一小部分功能要强大得多，同时还支持利用自定义的 `CSS` 对样式进行更改，这些复杂的内容请参考 [Mermaid 的开发文档](https://mermaidjs.github.io/)。

同时，上文讲到的流程图、时序图的语法适用于使用上文提到的两种渲染引擎的编辑器（例如 Typora），而印象笔记的 Markdown 全部利用 Mermaid 引擎来渲染流程图和时序图，需要注意的是：Mermaid 对时序图和流程图渲染的语法和上文介绍的语法有小小的区别，如果大家有兴趣自己尝试的话，还请注意一下。
