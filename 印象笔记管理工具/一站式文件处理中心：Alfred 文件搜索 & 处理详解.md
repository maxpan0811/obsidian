一站式文件处理中心：Alfred 文件搜索 & 处理详解
============================

| 本文为付费栏目文章，您已订阅，可阅读全文 |

统一、符合习惯的操作，是许多人喜欢启动器工具的原因。以 Alfred 为例，它可以独立完成从搜索、打开、预览到自动化、批处理等操作，堪称文件搜索与处理的瑞士军刀。

当然，功能众多也给 Alfred 带来了使用门槛，很多人可能只知道基础的搜索和文件开启功能。为了让大家都能享受到 Alfred 的强大，本文将按照通常的使用流程与频率，介绍 Alfred 的文件相关功能：

1. 文件搜索：方便的快捷搜索模式。
2. 文件处理：找到文件后的打开、复制、拖拽、动作（Action）等功能，详细说明各种可选的文件处理方法以及不同情况中的最佳方法。
3. 文件导航：它提供了搜索之外的文件抵达方式，可以和文件搜索配合使用。
4. 文件缓存：这一功能可以方便文件的批量处理。

为方便大家理清各部分功能以及它们之间的切换方式，这里提供一幅涵盖本文主要内容的流程图。刚开始使用 Alfred 的读者可能对图中部分功能感到陌生，不妨先挑选感兴趣的进行阅读。

搜索处理文件的最佳实践
-----------

对于很多人来说，搜索文件就意味着放下手头的事情，跑到 Finder 里翻找一通。

* 下载文件后，你会怎么去打开它？大概是手掌从键盘移动到鼠标或触控板，然后控制光标移动到下方 Dock 栏，点开下载文件夹，打开下载的文件。
* 需要给朋友发送个文件，你会怎么做？可能是找到并打开文件夹，拖拽文件到微信中，然后关闭文件夹。
* 写文章时想看下某张截图确认内容，你又会怎么做？也许是打开截图文件夹，打开或者空格预览截图文件。
* ……

使用 Alfred，完全不必再那么麻烦。我们可以直接在任何应用中召唤出 Alfred 窗口，搜索、查看、拖拽和后续处理一条龙完成，尽量减少对当前工作的打断。具体怎么做，让我们一起来看。

Alfred 提供了方便高效的文件搜索方法「Quick Search（快捷搜索模式）」：唤出 Alfred 输入框，直接按下空格键会输入一个单引号'，然后输入搜索词就可以得到结果列表。或者也可以直接输入单引号 '，只是空格键更加方便。

![](.evernote-content/66C8F121-575E-4E63-B923-E321DDF94208/510A1C38-0863-4E91-9AEF-44F7FAD01D48.png)

快捷搜索模式

搜索结果列表中，一行由文件图标、名称以及路径组成。使用上下方向键 ↑↓ 选择需要的文件，按下 回车键 即可打开文件，而使用 ⌘Command-回车键 可以在 Finder 中显示，也就是打开所在文件夹。这可能就是大部分用户用到的功能。

但 Alfred 可不仅仅如此。除了上下方向键，我们还可以直接使用每一行右侧的快捷键 ⌘Command-数字，比如用 ⌘Command-3 打开第三个文件，相当于选择第三条后按下回车键。特别是当需要的文件在列表中比较靠后时，这样显然更加快捷。

我自己更加习惯的方法，则是使用快捷键 ⌃Control-N 和 ⌃Control-P 来模拟上下方向键，选择文件。尤其是将 ⌃Control 与 ⇪CapsLock 键位互换后，只需要动动手指就可以做到。不像方向键的使用需要移动整个手掌。

小技巧：文字编辑快捷键

⌃Control-N 和 ⌃Control-P 快捷键在互换 ⌃Control 与 ⇪CapsLock 键位后用起来非常方便。macOS 默认支持一套用于移动光标和编辑文本行的快捷键，它们只是其中的两个。无论是 macOS 本身的 Spotlight 功能，Finder 中重命名、搜索等输入场景，文本编辑（TextEdit）等内置应用，Alfred、Ulysses 等安装的应用，只要是文字输入或编辑的区域，都部分支持这套快捷键。

* ⌃Control-A：移动光标至行或段落的开头，对应于 Home 键。
* ⌃Control-E：移动光标至行或段落的末尾，对应于 End 键。
* ⌃Control-F：将光标向前移动一个字符，对应于左方向键。
* ⌃Control-B：将光标向后移动一个字符，对应于右方向键。
* ⌃Control-P：将光标上移一行，对应于上方向键。
* ⌃Control-N：将光标下移一行，对应于下方向键。

更加完整的快捷键列表可以看 [官方文档「Mac 键盘快捷键」的文稿快捷键部分](https://support.apple.com/zh-cn/HT201236#text)。

我们当然不可能只需要打开文件以及所在的文件夹，不时还要复制文件、复制路径、移动发送等等。这些操作在 Alfred 可以通过一套快捷操作完成：

* 复制文件路径：直接按下 ⌘Command-C 即可，需要在终端里进入某个文件夹时挺方便的。
* 预览文件：按下 ⌘Command-Y 快速预览（Quick Look）文件。
* 拖拽文件：可以直接拖拽结果列表中的文件，到文件夹中。和在 Finder 中一样，拖拽时按下 ⌘Command 是移动，按下 ⌥Option 是复制。也可以拖拽到应用中，比如搜到文件后直接拖入微信发送给他人。
* 复制文件名：连按三个快捷键 ⇥Tab-⌘Command-A-⌘Command-C。其中，按下 ⇥Tab 会在输入框中自动补全文件名，⌘Command-A 用来全选输入框中文字，使用 ⌘Command-C 复制。我常常用来复制记不清的应用和文件名，省得打字、避免出错。这算是个组合出来的非官方小技巧。

现在就尝试下，搜索需要的文件，直接拖拽到微信或者其他地方，不再需要「跋山涉水」地穿过一层层文件夹，简直不要太方便。

熟悉 Spotlight 的读者可能会说，Spotlight 也可以做到预览文件、打开所在文件夹（Command-R）、拖拽文件之类的。稍安勿躁，上面这些不过是 Alfred 正餐前的开胃小吃。

Action 窗口

Spotlight 或 Finder 在有些地方不太灵活，比如文件的打开方式比较死板。在需要编辑图片时，我会用 PhotoShop 打开图片文件，而不是直接双击用预览（Preview）应用打开。通常的做法是鼠标右键单击文件，在菜单中选择「打开方式（Open With）」，如果子菜单中没有 PhotoShop 这些需要的应用，那还要点击底部的「其他」，在弹出窗口中寻觅应用手动选择。这种让人无语的情形并不少见，因此始终想要更懒一点的我表示拒绝。Alfred 中，显示 Action 窗口，选择 Open with，然后输入应用的部分名称后回车就可以随时选择打开方式了。整套操作可以全部在键盘上完成，熟悉这个流程后，用起来比读这句话还要简单。

选择应用打开方式，还只是 Alfred 功能的冰山一角。在 Alfred 结果列表中，按下右方向键就可以打开 Action 窗口，这里列出了多种用于处理文件的内置动作。我们同样可以使用上下方向键选择，或输入 Action 名称搜索，而后按下回车键执行。需要返回之前的文件搜索结果列表时，可以使用 ⎋Esc 键。

Action 列表中，除了常见的打开、删除、复制、移动等功能，还有这些值得试一试：

* Open with：用指定应用打开文件。偶尔需要以非默认应用打开时挺方便。
* Find Similar：查找相似文件。执行后，会打开 Finder，用选中项的文件名搜索。因此，无论是文件名或文件内容存在相似性的都会被检索出来。
* Recent Documents：适用于 App 文件，当选中项为其他文件类型时不会显示在 Action 列表中。用来显示 App 的最近文档。不适用于 Microsoft Office 以及其他非原生 App。
* Browse Folder in Alfred：适用于文件夹，在 Alfred 的文件导航窗口中浏览这个文件夹。后文中有更具体的说明。
* Copy Path to Clipboard 与 Open Terminal Here：复制文件路径、在终端中打开文件夹。对于常用终端工作的情况比较有用。

另外，其中的 Copy to、Move to 两个 Action，需要详细说明下使用方法。以 Copy to 为例，回车执行后，打开目的地文件夹选择窗口。上方的文件名称与路径一目了然，下方右侧是文件预览图标以及文件信息，左侧是文件夹列表。可上下移动选择，或者输入文字在所有文件夹中搜索。因为搜索范围很大，所以难免会出现多个重名的文件夹，这时可以看窗口最下方的一行文字，它就是选中的文件夹路径。

![](.evernote-content/66C8F121-575E-4E63-B923-E321DDF94208/8BE52E71-0C71-4655-8573-14343C3ECFA9.png)

Copy to 窗口

除了这些内置的 Action，我们也可以编写 Alfred Workflow 添加自定义的 Action，比如我曾编写的一个[用于设定 App 语言的 Workflow](https://github.com/mpco/AlfredWorkflow-App-Language-Switcher)：仅适用于 App 文件，选择 Action 列表中的「Switch Language to」回车运行，显示存在的语言，可以选择临时以某语言启动，或固定设置为某语言并启动。

Action 中的许多功能挺不错，但是当文件就在桌面上或者打开的 Finder 窗口中时，再去唤出 Alfred → 输入搜索 → 上下选择 → 右方向键进入 Action 界面，未免太傻了。其实可以在 Finder 中选中单个或多个文件后，直接使用快捷键 ⌥Option-⌘Command-反斜线(\)  打开 Action 界面，此时操作的对象就是 Finder 内选中的文件。

让搜索更加精准
-------

仅仅通过文件名等信息进行搜索是难以应对所有情况的。事实上，很多时候我们并不知道所需文件的名称，可能是想找到含有一些内容的文件，或者是记得之前附上的标签等。

Alfred 提供了四种搜索方式，对应四个关键词 open/find/in/tags，允许用户在各种情形下都可以按图索骥。

前文中介绍的按下空格搜索（快捷搜索模式），其实就等同于输入 open 搜索词，通过文件名、注释等元数据信息匹配搜索。而 find 关键词与 open 的不同之处，仅在于 find 按下回车键后是在 Finder 中显示。在我看来，倒不如使用快捷搜索，按下空格输入搜索词，然后使用 ⌘Command-回车键 在 Finder 中显示。

in 搜索词 用来搜索文件内容，找出包含这个词的文件。然后，同样是回车键打开，⌘Command-回车键 在 Finder 中显示。

tags 用来按照标签搜索文件。输入 tags 标签名称 显示带有这个标签的文件，输入部分标签名称也可以搜索到，比如 tags re 列出 red 标签的文件。小技巧：tags+两个空格，显示所有带标签的文件。

即使有了这些不同的搜索方式，在很多时候搜索文件也像是「大海捞针」，因为 Alfred 默认的全局搜索范围几乎包括所有的用户文件。如果我们能自己设置这个范围，那么或许可以让「这片海」刚好包含自己常用的那部分文件。

![](.evernote-content/66C8F121-575E-4E63-B923-E321DDF94208/46B95B7E-D8C4-4D64-BC33-B14EEE59B101.png)

搜索范围

恰当的搜索范围，让我们既能找到需要的文件，也可以减少无用的干扰项。打开偏好设置 Features 内的 Default Results 面板，中间部分就是 Search Scope（搜索范围）。Alfred 中搜索应用与文件，就在这个范围中。默认包括这些：

* 用户目录（/Users/ 用户名 /）中的文件夹，基本上用户的文档都存在这里。特地排除了资源库文件夹（/Users/ 用户名 / Library/），因为其中装的是系统与应用的设置与资源文件，一般不会用到。偶尔需要时也可以直接通过后文中的文件导航功能获取。
* iCloud 文件夹（~/Library/Mobile Documents/）。
* 各处的 Application 文件夹，存储应用文件。
* 各处的 PreferencePanes 文件夹，存储系统偏好设置面板。

像连接的移动硬盘、额外划出的磁盘分区、挂载的网盘等，如果需要将其纳入搜索范围，我们可以点击右下角的加号 + 添加，或者直接拖拽文件夹到列表中。需要删除某一项时，选中后点击减号 -，或者按下 backspace（退格键）就行。列表中的默认项，除非你明白它代表什么，否则不要轻举妄动。

![](.evernote-content/66C8F121-575E-4E63-B923-E321DDF94208/8E2C5E6E-22E2-4188-B6D8-5EE63E028C4D.png)

屏蔽文件夹

屏蔽一些文件夹，不让它们中的文件出现在搜索结果中，可以保护隐私，也可以更加精细地调节搜索范围。比如说，上面的 Search Scope 包含应用文件夹 /Applications，但是安装 Adobe 家族应用会在其中生成的各种子文件夹，包含一些基本用不到的应用和文件。我们可以在「系统偏好设置 - 聚焦（Spotlight）- 隐私面板」中添加这些子文件夹，这样既不影响搜索应用，又可以屏蔽其中那些不需要的文件。这个设置同时对 Alfred 和 Spotlight 两者的搜索结果起效。

文件导航
----

搜索是找到文件的主要方式，但通过文件夹层级逐步定位也必不可少，在知道文件位置而不清楚文件名的情况下相当有用。这种情况还很常见，比如说，作图或者写稿时，知道需要的文件存放在素材文件夹中，可就是记不清文件名了，唤出 Alfred 使用文件导航功能就可以直达目标。有时下载文件后，我犯懒不想移动手掌去用鼠标点击，同样可以在 Alfred 中列出下载文件夹内容，按需获取。

![](.evernote-content/66C8F121-575E-4E63-B923-E321DDF94208/41DFE820-1DEB-4108-9543-FF60B0091DA7.png)

文件导航窗口

文件导航功能，使用方法很简单，在 Alfred 输入框中输入路径就可以了。此时，窗口布局与 Action 窗口类似，左侧是当前路径下的文件列表。右侧是选中文件的预览。最下方一行小字，显示选中的文件名。如果文件名过长，在列表中无法完整显示，可以看这里。

左侧的文件列表，与之前一样也是使用上下方向键选择。也可以输入文字在列表中动态筛选需要的文件，实际上，这才是更推荐，我个人更常用的方式。筛选时，需要输入文件名中以空格分隔的文字或单词的开头部分，是不是有些绕？看下面的示例：

* 文件名「Machine Learning」，需要输入「mac」或者「lea」等开头字母，输入「ine」是找不到的。
* 文件名「我怎么读书 阅读笔记 0530」，需要输入「我怎么」或「阅读」，输入「读书」是找不到的。

  ![](.evernote-content/66C8F121-575E-4E63-B923-E321DDF94208/E7C200CA-0674-4371-8F33-13B7C0EBAC16.gif)

  动态筛选

但是，一般情况下我们偏偏只是知道文件名的一部分，而不能准确记得开头文字，Alfred 的开发者也考虑到了这一点，加入了 星号通配符 功能。比如用「\*ine」搜索出「Machine Learning」，「\* 读书」搜索出「我怎么读书 阅读笔记」。文件名中的数字，则可以直接搜索，不需要考虑有无空格分隔以及在什么位置，比如输入「30」找到「我怎么读书 阅读笔记 0530」。

右侧预览区域中，下方显示相应的文件信息。如果选中的是文件夹，这里有创建日期和修改日期，如果是文件，还会加上文件大小、默认的打开应用。

![](.evernote-content/66C8F121-575E-4E63-B923-E321DDF94208/2C4F7D57-CF32-45B9-8CFB-FDCC78435DF5.png)

右侧区域中的选项

上面的动态筛选确实好用，但是吧，能不能允许我更懒点儿（得寸进尺）。比如说，对于「下载」文件夹、「截图」文件夹，大部分时候我需要的都是其中最新的那个文件。要是可以按照日期排列，最新的文件放在列表最前，那么文件导航功能中进入这些文件夹，默认选中的就是需要的，不必再手动选择了。感谢 Alfred，这可以做到。

先别急，让我们接着看窗口的右下角，有两个小图标。左侧是一个箭头，标明右侧列表的排序方式：顺序↓或逆序↑。点击图标就可以切换，也可以使用快捷键 ⇧Shift-⌘Command-S，毕竟 Alfred 与键盘操作更搭。右侧是一个齿轮 ⚙️️ 图标，点击或使用快捷键 ⇧Shift-⌘Command-逗号(,) 打开一个菜单，其中含有这些选项：

* Sort by Name：按照名称排序
* Sort by Creation Date：按照创建日期排序
* Sort by Last Modified Date：按照最后修改日期排序
* Folders at Top：置顶显示文件夹
* Sort Descending：逆序排列，快捷键 ⌘Command-⇧Shift-S
* Hide Preview Panel：隐藏右侧预览区域，快捷键 ⌘Command-⇧Shift-I

像上面提到的「下载」文件夹，我就会选择「按照创建日期排序」。充分利用这些选项就可以让我们在左侧列表中更轻松地找出需要的文件。

文件导航功能的主体部分，到此已经介绍完毕。接下来算是「餐后酒」，附上几点小技巧，也十分可口。

知识点：查看刚刚使用过的文件

刚刚关闭 Finder，又需要打开刚才的文件夹，查看其中的另一份文件，这种情形相信各位或多或少都遇到过。在使用 Alfred 文件导航功能时出现类似的情况也毫不意外。不过，这里我们不必再重复输入路径，可以直接使用快捷键 ⌥Option-⌘Command-斜线(/) 打开文件导航窗口，显示上一次浏览的文件夹。比如说，上次在文件导航窗口中浏览 ~/Downloads 后关闭，那么用这个快捷键打开还会是相同的位置。也可以在 Alfred 中输入关键词 previous，选择「Browse Previous Path（浏览之前的路径）」并按下回车键，效果与快捷键相同。

还有个文件导航十分有用的情况，就是获取不在搜索范围内的文件。前文中，我们通过设置，可以将 Alfred 搜索范围调整为包含常用的文件，减少干扰项，改善搜索结果。那么偶尔需要范围之外的不常用文件，就可以输入路径找到。另外，一种特殊的情形是，App 包内部的文件始终是无法搜索到的，同样需要通过文件导航访问。

### 结合搜索与导航

文件导航功能，让我们可以获取搜索无法抵达的文件。可是，输入路径，尤其是路径比较长时，貌似还是有点麻烦对吧？作为经常使用的功能，这些许的不便也应该消除。

将之前介绍过的搜索与导航结合起来，就能降低导航功能的操作量。

简单分析下，使用文件导航功能时，文件所在的文件夹是知道的，那么或许可以搜索文件夹，然后通过某种方式在文件导航窗口中直接显示它里面的文件？这样就不用输入一大串路径了。

![](.evernote-content/66C8F121-575E-4E63-B923-E321DDF94208/C4167CAB-1108-4FB8-88EC-70248D2FD615.gif)

搜索转为导航的演示

确实可以，而且很简单。Alfred 中搜索文件或文件夹后，只需按下 ⌘Command-下方向键 就可以打开文件导航窗口。如果在搜索结果列表内选中的是文件夹，那么文件导航窗口中就会显示这个文件夹内的文件列表。比如说，需要打开「文稿」文件夹中的某个文件，直接唤出 Alfred，敲下空格后输入 docu，选择「文稿」文件夹，按下 ⌘Command-下方向键 可列出其中的文件。也可以使用右方向键打开 Action 窗口，选择「Browse Folder in Alfred」有同样的效果。如果选中的是文件，文件导航窗口显示的则是它所处的文件夹。

搜索与文件导航相辅相成。我们可以搜索后使用快捷键打开文件导航窗口，进入相应的文件夹，也可以在文件导航窗口中使用搜索动态筛选出需要的文件。熟练掌握两种方式，合理运用，会让我们的工作更加高效。

批量处理
----

至此，我们知道了在 Alfred 如何搜索、定位文件，以及使用快捷键和 Action 操作处理。不知道你有没有发现，这些都是在与单个文件打交道。而批量操作多个文件也是工作生活中不可缺少的。这一节我们来看看 Alfred 的文件批量处理功能：文件缓存（File Buffer）。

前文中其实已经谈到，在 Finder 内选中文件后，使用快捷键 ⌘Command-⌥Option-反斜线(\) 就能打开 Action 窗口。此时，执行任意的 Action，对象就是这些选中的文件。它们会显示在 Action 窗口右侧预览区域，光标移动到文件图标上方会显示对应的文件名和路径。这是一个很好用的批量处理小技巧，适合于需要处理的文件就在桌面上，或者在打开的文件夹中时。

![](.evernote-content/66C8F121-575E-4E63-B923-E321DDF94208/CB5E5978-6C26-4B5D-AD51-2F3BDC6D9B95.gif)

批处理快捷键

不过这个快捷键只能在 Finder 中使用，有没有办法不离开 Alfred 就进行文件批处理？当然可以。下面来说更常规的方法：文件缓存。

![](.evernote-content/66C8F121-575E-4E63-B923-E321DDF94208/4B91CFE9-3B71-40C5-BEDB-6C84AB2746F1.png)

文件缓存

在 Alfred 中，无论是搜索应用和文件时，还是处于文件导航窗口中，都可以使用快捷键 ⌥Option-上方向键，将选中项加入缓存中。已加入的文件在列表中会被标记出来，以作区分。

这些待处理的文件就放在 Alfred 输入框上方的「置物架」里。光标移动到其中的文件图标上，会显示文件的名称、路径以及三个快捷键提示：

* ⌥click to remove：按住 ⌥Option 点击文件图标，从缓存中移除该文件。
* ⌥⌫ to remove all：快捷键 ⌥Option-退格键(backspace) 用来移除所有缓存。
* ⌥→ to action all：快捷键 ⌥Option-右方向键 用来打开 Action 窗口，批量处理这些文件。

除了用快捷键 ⌥Option-上方向键 添加文件，还可以使用 ⌥Option-下方向键，两者不同之处在于 ⌥Option-下方向键 在添加当前这一行后，会自动选中下一行的文件，用来连续添加非常方便。

默认情况下，在使用 ⌥Option-右方向键 打开 Action 窗口并执行后，缓存就会清空。只要不执行 Action，即使关闭后再次打开 Alfred 窗口，文件缓存也依然在那里。因此，对于分散在多处的文件，我们可以在 Alfred 中分别多次搜索、定位，添加至缓存，然后一次性地批量处理。

总结
--

Alfred 提供了两条抵达文件的方法：搜索、文件导航。两者相互穿插，灵活切换。在找到文件后，丰富的快捷键让我们可以方便地打开、复制、获得信息，以及使用 Action 处理。批量操作也不能少，文件缓存功能补上这一块。我们还可以编写 Workflow 添加自定义功能的 Action。

「让搜索更加精准」小节，介绍了如何按照标题、内容和标签搜索文件，以及如何设定全局搜索范围。其实，工作中还有一类需要精准搜索的情况，就是在指定的文件夹及其子文件夹内，搜索指定类型的文件。毕竟很多时候我们会高频地使用「素材」、「资料」之类文件夹的文件。这也需要利用 Workflow 实现。

说到 Workflow，那就是 Alfred 中又可以好好聊聊的另一个方面了。掌握了它，Alfred 能做什么，由我们说了算。之后，我还会写一篇详解 Workflow 使用与编写的文章。
