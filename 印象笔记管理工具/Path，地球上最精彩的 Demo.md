# Path，地球上最精彩的 Demo

---

[![](.evernote-content/31E7C0EA-FCA0-42D5-BB31-EF51A0FDF3AB/085FB941-7527-476C-BF8F-F6F58CFDC0B7.jpg)](http://www.ifanr.com/63022/path)

[Path](https://path.com/) 正在掀起一股流行。喜欢用数字纪录生活的用户，无论你是来自 Twitter、Facebook、Foursquare 或者 Instagram ，你都应该试试 Path ，因为它实在是一个精彩的客户端。

曾经有人怀疑：[Path 是否值得 Google 出价 1亿美元](http://techcrunch.com/2011/02/02/google-tried-to-buy-path-for-100-million-path-said-no/)，请看看 Path 是如何来回答这类疑问的。

### 不缺概念

Path 出自前 Facebook 员工 Dave Morin 之手，它打着 “反社交” 的旗号出现在社交分类应用中。面向 50人的小圈子——这个产品描述让 Path 看起来很有性格。

50人小圈子的概念来源于亲近关系在互联网社交中的稀缺性，因此 Path 鼓励人们将重要时刻分享给私密的圈子。这种概念有其成功的一面，用户们的确在斟酌交友的对象；而失败的一面在于，这个力求小众的产品果不然地遇到了用户量不大的问题。

### 不缺体验

如果有一类 App 称的上是艺术品，那么 Path 必定属于其中的佼佼者。我们曾经从侧面表述过 Path 在用户体验上的设计，当时的作者用了一个我很赞同词语：[人性关怀](http://www.ifanr.com/53504)。

新版的 Path 在交互上的改进大致分为两个部分：事件如何触发，触发的呈现形式。在事件如何触发这块我们看到了“左右滑动” 对屏幕利用效率上的帮助：向左滑动，打开好友列表；向右滑动，打开控制菜单。在触发的呈现形式上我们看到了 Path 团队对细节的雕琢：点击“+” 后蹦出各类型分享图标，而同时 “+” 变成了 “x” 号，自然而然就成了关闭按钮。同样的细节还体现在图标在出现和消失时的动画。

如果说一个 App 要体现人性关怀，那么它一定是让用户和程序产生了一种微妙的沟通。而 Path 的用户体验过程就充满了这种沟通。

### 不缺功能

Path 从一开始的图片分享到现在的图片，文字，音乐，地理位置，乃至作息时刻，实现了功能上的拓展。多功能满足了多元化的需求，但也会让 Path 的形象有所改变。难怪用户抱怨到 “Path 从单纯的照片分享变成了一个普通的轻博客”。

功能，决定了一个产品是什么。例如： A 类产品具备 A 种功能，而 AB 类产品具备 AB 两种功能。当 A 类产品扩展到 AB 两种功能的时候，你还能把自己作为 A 类产品看待吗？

我们可以将这种变化看作是转型，正如 Dave 所述，Path 变成了你的数字日记本。Path 团队通过一个版本升级实现了产品转型，从社交网络上的反馈来看，Path 的用户群也成功地随之转型。

### 让我们回到之前的问题，Google 要什么？

[从这次 Google 产品改版中](http://www.ifanr.com/59651)，我们认识到 Google 需要一个熟悉用户体验的设计团队。冰冷的用户体验，始终是这家技术公司的附加产品。尽管这种风格曾使它看起来很 Cool ，但当旗下的服务成为大众基础服务，Google 如果继续保持冰冷只能让人觉得他们对用户不够重视。

Path 团队看起来的确是不错的选择，有概念，注重体验，而且也懂得驾驭更多的功能。Google 收购 Path 团队，我举双手同意。

### 为什么 Path 非要被收购？

Path 一路获得约 1100万美元的投资，投资人需要 Path 创收，而 Path 的盈利方式真的很少。

* 程序收费？可以。但用户量能超过 1100万吗？我怀疑。
* 相机收费滤镜？已经这么干了，知道蹩脚的竞争对手会怎么干吗？抄袭你的功能，然后将之免费。
* 广告？可以，但很可能不会。Dave 表示他不想让广告去破坏用户体验。
* 收购团队？可以，Path 团队的背景很好，活儿做的也漂亮。

如果有朝一日，Path 团队被收购，让我们再回头看看 Path 的变化过程，它像不像是一个最精彩的 Demo 呢？
