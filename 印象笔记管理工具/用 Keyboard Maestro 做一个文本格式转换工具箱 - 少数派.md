# 用 Keyboard Maestro 做一个文本格式转换工具箱 - 少数派

用 Keyboard Maestro 做一个文本格式转换工具箱
===============================

文本格式转换是一类很杂的需求，就拿拷贝网页链接来说，写文章时常常用到 Markdown 格式链接，分享给朋友时更多使用 `标题：链接` 的简单格式，如果要去网站后台排版则可能需要 HTML 格式的网址…… 常见思路是为每个格式转换需求创建一个自动化动作，然后绑定快捷键。

但是，同类的动作多了，对应的快捷键数量也会随之增加。试想一下，如果我们为不同的文本格式转换操作设置了 `⇧Shift-⌘Command-C`、`⌥Option-⌘Command-C`、`⇧Shift-⌥Option-⌘Command-C`、`⌃Control-⌘Command-C` 等键位， 要用的时候八成会被一堆键位搞混头脑。更糟的是，拷贝链接还只是诸多文本格式转换场景的其中之一，再算上获取网页标题、拷贝文件路径等其他场合，全靠快捷键只会更吃力。

这篇文章中，我们将通过 Keyboard Maestro 来打造几个**文本格式转换工具箱**，把相似的一批动作放在一起，从而减少记忆快捷键的负担。在此基础上，我们还会为不同的场景设置不同的工具箱，满足常见的文本格式转换需求。

用 Keyboard Maestro 制作工具箱
------------------------

在《[给快捷键减压](https://sspai.com/post/44678)》（Power+ 1.0）一文中，文刀提到了通过 Keyboard Maestro 缓解快捷键压力的思路。Keyboard Maestro 中有个 Palette（浮窗）功能，可以把一批动作汇集在一起，通过统一的快捷键呼出。我们的工具箱其实就是 Palette，**只需要记一个快捷键**，就能呼出整个工具箱。

![](.evernote-content/12511DCD-1264-41DD-9F08-D20CA3E06468/6FB9F1AD-4CD2-468B-A153-5817AD6BB445.png)

两个常用动作组

上图是我在使用 Safari 浏览器和 Finder 时各自常用的一组格式转换工具箱，分别用于拷贝不同样式的**网页链接**和**文件路径**，基本涵盖了日常工作中的格式转换需求。

考虑到不是所有读者都熟悉 Keyboard Maestro 的操作，这里先对途中频繁出现的一些名词进行说明，正文还是会尽量使用中文：

* Macro：即一个自动化动作
* Group：容纳一堆 Macro 的文件夹
* Palette：一个包含了一组 Macro 的浮窗
* Trigger：触发方式，本文的 Trigger 都是快捷键

下面，就分 Safari 和 Finder 两个场景来分别介绍两组 Keyboard Maestro 动作。

获取不同格式的网页链接
-----------

不少人都遇到过这样的情况：一边开着参考网页，一边开着编辑器写东西，此时需要把链接粘贴进编辑器里。于是你只能停下来，切到浏览器，磨蹭到想要的标签页，拷贝地址，再切回编辑器里粘上。虽然链接就在手边，但是非得去复制一下再切回来，其实挺打断工作。

除了写作，还有其他从网页获取链接的场景，并且有着不同的格式需求：

* **获取 `标题：链接` 样式的文本**：给朋友发送文章链接时我习惯附上标题，方便对方不点开链接也大概知道文章主题是什么。
* **获取 Markdown 格式链接**：刚才已经讲过，这一格式常常用于写作。
* **获取网页标题**和**获取网页链接**：由于 Ulysses 会自动识别 Markdown 链接符号并转换格式，直接粘贴 Markdown 链接反而不方便，于是我做了这组动作快速分别获取标题和链接。

这些从网页获取各种链接（以及标题）的需求，我都汇集在了一个 Keyboard Maestro 动作组中。

![](.evernote-content/12511DCD-1264-41DD-9F08-D20CA3E06468/33E018F8-9FE0-4621-B4B9-AFCBD21BFAB3.png)

从 Safari 获取链接的动作

它们的使用流程是：

1. 按下快捷键 `⌥Option-⇧Shift-⌘Command-C`。虽然看上去很长，但这串键位实际上很好记，反正把左下角那三颗键全按下去准没错，而且用不容易和其他快捷键撞 1。
2. 按下所需项目的首字母（比如想要标题就按 `t`，表示有 Title 的这一项），选中对应动作。
3. 从弹窗中选择需要获取内容的网页（都是当前打开的标签页），选中后转换好格式的内容就粘到了当前光标下。

![](.evernote-content/12511DCD-1264-41DD-9F08-D20CA3E06468/54C9C88E-04E7-4AEF-A4B0-4AA7AD00C513.gif)

拷贝网页链接

我们注意到，按下快捷键后没有直接运行动作，而是出现了一个浮窗供我们选择具体要使用的动作。这种浮窗只在**快捷键产生冲突**时才出现，所以也叫**冲突 Palette**。我特意把关于 Safari 的辅助粘贴快捷键全设置成同一个，为的就是调出冲突 Palette，从而把功能类似的几个动作合在了一起。

![](.evernote-content/12511DCD-1264-41DD-9F08-D20CA3E06468/698A850D-3A57-47C1-9450-B697A4B3AEA4.png)

组内动作的快捷键是一致的

脚本最初来自 Dr.Drang，原理不难，就是获取网页链接并整合进一个列表，然后你想要哪个就选中哪个。如果对脚本感兴趣，可以读读这篇文章《[如何快速复制粘贴浏览器标签页的链接](https://sspai.com/article/42230?series_id=9)》。我对脚本做了一些改良，演变成了上面一系列的动作。

拷贝各种格式的文件路径
-----------

同样是一个文件，不同场景下我们会用到它们不同格式的路径，比如：

* **常规路径**：就是平时复制文件时得到的路径，在 Finder 里按下 `⇧Shift-⌘Command-G` 并输入常规路径后，可以直接跳转到对应的文件或文件夹。向别人介绍一些系统隐藏文件夹时可以附上常规路径，方便对方快速打开这些文件夹。例如：
  `/Users/apple/文件 A`
* **终端路径**：不能直接在 Finder 中拷贝，因为名称含有空格、`~` 等特殊字符的文件无法被终端直接识别，一般要把文件拖进终端才能完成「转译」（在特殊字符前加上 `\`）操作。规范的终端路径长这样：
  `/Users/apple/文件\ A`
* **文件 ID**：常出现在 AppleScript 中，用于处理文件，中间用 `:` 相连。例如：
  `Macintosh HD:Users:apple:文件 A`
* **GitHub 链接**：GitHub 是我日常分享自动化动作的主要方式，由于云端文件和本地 Git 仓库中的文件位置一一对应，所以分享时可以拷贝本地路径、再改改前缀，不用打开浏览器：
  `https://github.com/用户名/仓库名/文件 A`
* ……

上面这些拷贝需求，要么没有容易记忆的操作方式，要么干脆无法直接实现，这也难怪 Totoal Finder、Command One 等第三方文件管理工具会把「快速拷贝路径」这种原生功能作为亮点来宣传。

![](.evernote-content/12511DCD-1264-41DD-9F08-D20CA3E06468/45DA80F3-3DCC-4CD3-8BFA-96F4754C7BEF.png)

Total Finder 把剪切功能当成了卖点

如果只是为了各种拷贝操作，并不值关掉系统保护机制去装上面提到的几款 Finder 增强工具。我们还是用 Keyboard Maestro 来解决，为 Finder 打造一个专用**路径格式转换工具箱**。

![](.evernote-content/12511DCD-1264-41DD-9F08-D20CA3E06468/EBB99CF6-FE5A-4228-A730-58C91A413159.png)

Finder 格式转换工具箱

### 创建 Finder 专用工具箱

先来创建一个动作文件夹，容纳一系列 Finder 中的拷贝动作。由于其中的动作都是 **Finder 专用**的，没必要出现在其他场合，所以我们需要在动作文件夹中设置**「Avaliable in these applications: Finder」**，只让这些动作在 Finder 里出现。

![](.evernote-content/12511DCD-1264-41DD-9F08-D20CA3E06468/416358A9-983A-432B-B59A-60F9ECBEB573.png)

限制动作只在 Finder 中出现

至于其中的动作，为了统一快捷键，可以和设置 Safari 动作时道理一样，把所有拷贝相关动作快捷键全设置成 `⌥Option-⇧Shift-⌘Command-C`，在使用时就可以展开**冲突 Palette**。

![](.evernote-content/12511DCD-1264-41DD-9F08-D20CA3E06468/8E2971F6-D437-4445-B5F7-86B560B8695A.png)

设置统一的快捷键

由于刚才已经限制过 Finder 路径相关动作的出现场合，它们就不会在无关的地方冒出来。这样一来，使用 Finder 以外的应用时，我们就避免了无关动作的工作。而获取 Safari 链接的动作组则没有设置应用限制，这是因为我们可能在 Ulysses、Byword 或者网站后台等不同地方使用链接，没有必要限制应用范围。

### 通过文本替换进行格式转换

稍微观察一下几种路径，可以发现，它们的主要差别在于分隔符和前缀。所以，Keyboard Maestro 可以先获取常规路径，然后做一些**文本替换（Search and Replace）**得到其他 3 种格式的链接。

下面，我们通过「Copy as ID（拷贝 ID）」动作熟悉一下文本替换操作的用法，掌握之后，拷贝终端路径和 GitHub 链接的动作也可以如法炮制。拷贝 ID 的步骤是：

1. 模拟快捷键 `⌥Option - ⌘Command - C`，拷贝常规路径
2. 将剪贴板中的 `/` 替换成 `:`
3. 在替换好的内容前加上 `Macintosh HD` 前缀

![](.evernote-content/12511DCD-1264-41DD-9F08-D20CA3E06468/C650B390-03B0-4DBD-9651-EFE6A97EF748.png)

拷贝 ID 的动作

其他两个动作，要做的文本转换操作分别是在特殊符号前加上 `\` 进行转译，以及在本地路径前换上 GitHub 仓库链接前缀，都不需写代码，实现起来很轻松。

做好整组动作后，拿一个文件做测试，各种拷贝动作得到的结果分列如下：

* ID：`Macintosh HD:Users:apple:Scripts:Automator:校验 MD5 值.app`
* 路径：`/Users/apple/Scripts/Automator/校验 MD5 值.app`
* 终端路径：`/Users/apple/Scripts/Automator/校验\ MD5\ 值.app`
* GitHub 链接：`https://github.com/BlackwinMin/sspai-sample-script/blob/master/Automator/校验%20MD5%20值.app`

![](.evernote-content/12511DCD-1264-41DD-9F08-D20CA3E06468/9A014625-5428-4CF9-92AD-9C1C9ED9573C.png)

不同格式的路径拷贝结果

Finder 里常用的拷贝动作就是这几个，通过 Keyboard Maestro 统一之后，比使用分散的自动化动作进行格式转换要省力不少。

小结
--

一个简单的 `⌘Command-C` 无法满足不同人的需要。「复制粘贴」这个看似简单的操作，在具体场景里可以衍生出各种变体，涉及不同的格式转换。

拷贝 Safari 链接、复制特殊格式的 Finder 路径…… 当这些零零散散的需求数量变多之后，就可以用 Keyboard Maestro 统一解决，为不同场景打造各自的工具箱。
