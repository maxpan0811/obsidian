从 Alfred 到 uTools
-----------------

那么 uTools 又有什么值得称道的呢？

### Spotlight + 小程序

如果说一众类 Spotlight 应用有什么共同点的话，我一定会说它们都像极了搜索引擎，不管是 Spotlight 还是 Alfred，不管是 Wox 还是 Listary，并不是说搜索引擎的这种模式出了问题，而是其功能插件的展现形式仍然受限于搜索引擎「一个输入框 + 一个列表」的形态就显得过于单一了。

![](.evernote-content/233F8392-7F8F-4CEC-A5DB-9E259F0EB40A/A0D84351-E8BA-4554-A8CA-614D02B006D2.png)

搜索引擎大合集

深入使用过 Alfred 的人都能体会到，Alfred 的插件或者 Workflow 都必须是列表形态，诚然，「搜索」这个场景占了 Alfred 插件场景的绝大多数，但即便如此，列表这样的形式所能展现的内容局限性还是很大。

就以常用的「翻译」这一场景来看一下两者的差异。

![](.evernote-content/233F8392-7F8F-4CEC-A5DB-9E259F0EB40A/D4933794-AB5C-4CCF-ACFA-E6483C8C0378.png)

Alfred 翻译

![](.evernote-content/233F8392-7F8F-4CEC-A5DB-9E259F0EB40A/7D8F2BF6-437E-4FDF-B3B5-B61652C40D69.jpg)

uTools 翻译

可以看到，相比较 Alfred，uTools 的翻译插件带来了更丰富的内容，不仅可以在同一个插件页面轻松地切换翻译源，还可以在翻译结果中进行二次查询。

对于翻译语言的指定，uTools 可以通过直观的按钮选项来切换，而 Alfred 中只能通过类似「en>zh」这样的命令参数形式来指定，对于单词发音这样的额外功能，Alfred 插件靠的是一个不看插件介绍页面就不知道的快捷键来操作，而 uTools 的发音按钮就这样简单直观地摆在显眼的地方，触手可及。

uTools 强大的界面展现能力再次发挥了作用，计算器插件可以轻松地保留多次计算的结果以便对照，而 Alfred 只能逐次计算不同的算式。

uTools 色值

你一定知道我说的小程序是什么意思了，在 uTools 中，所有的插件都拥有独立的界面，不再拘泥于列表的一行短小精悍的文本和固定大小的方形图标，一切都是自由的，从界面到操作形式，从内容到交互，完全把主动权交到插件的手上，这样的表现形态，像极了小程序，尽管小程序依托于微信，但小程序不再受限于公众号的「一行底栏，三个选项」，更自由的创作，自然也迸发出别样的生机。 而这，也是我在 uTools 中能看到的。

可以说，拥有了独立界面的插件，早已不再是一个只拥有某一方面功能的简陋的「输入 / 搜索框」，而是一个完整的「应用」，再加上 utools 赋予插件强大的系统交互能力，utools 插件正在无限接近于一个正儿八经的工具软件。

### 「更聪明」的列表选项

对于 uTools 的插件来说，uTools 拥有一个强大的特性: 识别当前所在的应用。 这也是我非常喜欢的一个功能，设想这样一个场景：我在使用 Chrome 的时候想要生成当前网址的二维码，但是我在使用其他软件的时候，完全用不上这个功能。

Alfred 会怎么做呢，它会毫无保留地让我查询到所有符合关键词的插件，而 uTools 可以替你把与当前应用无关的插件选项过滤掉。

### 全面的跨平台支持

对于非 macOS 用户来说，没有一款可以媲美 Alfred 的 Spotlight 应用，一直是一个遗憾，在 Windows 上，最为著名的 Wox 早已停更多年，后继无人，在 Linux 上，也仅有 Albert 在偶尔更新，但其插件的数量寥寥可数。

Alfred 只支持 macOS，但 uTools 接受全平台的爱，三大操作系统全面支持，不管你换到了什么操作系统，都可以无缝体验到 uTools 给予你强大的功能支持。

uTools 的未来
----------

在 uTools 的新年里，还有很多不尽如人意的地方，在与 uTools 开发者交流的过程中，也了解到一些关于 uTools 接下来的开发方向。

### Workflow

作为 uTools 众多插件开发者中的一员，在深度参与到 uTools 的插件开发后，我感受到插件开发对于普通用户来说是仍然是有一定难度的，自由表达的插件形态自然也伴随着对插件开发者能力的高要求，想要自制一款稍微完整的插件，你需要懂得一整套正统的前端开发，包括 Javascript、UI 框架、nodejs 等，相比较 Alfred 简单易懂的可视化 Workflow 编排，难度实在是不小。

所幸 uTools 开发团队也了解到这一痛点，属于 uTools 自己的 Workflow 已经在路上了，相信很快就能面向用户开放，虽然还不知道具体的形态，但从 uTools 一贯秉承的高质量和高质量来看，一定不会让人失望。

### 对列表搜索场景的支持

Alfred 的优点同样也是 uTools 的缺点，uTools 在插件界面自定义上做了很多工作，但是在传统的「搜索框 + 列表」的场景却缺乏支持，开发者需要自行实现列表组件，对搜索类型的插件开发增加了不小的难度。

uTools 开发团队也看到了这个问题，在之后的版本中将会就搜索场景的插件推出统一的 UI 解决方案，让对前端 UI 开发不熟悉的开发者也能更加方便得加入到插件开发的行列中来。

总而言之，uTools 在朝着更丰富、更自由、更强大的未来前进着。
