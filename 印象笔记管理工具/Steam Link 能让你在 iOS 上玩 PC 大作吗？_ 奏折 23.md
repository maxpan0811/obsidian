#### 应用简介

**@沨沄极客：**Steam Link 是一款由 Valve 公司推出的 Steam 官方串流工具，它能够将电脑上运行的游戏串流到移动设备上。在前段时间介绍的《[最强电视盒子测评：Nvidia Shield TV | 新玩意](https://sspai.com/post/54581)》中，我们就介绍过游戏串流，这是一种 **通过 PC 的性能来运行游戏，将画面通过网络传输到 TV 端或是其他设备上** 的技术。

苹果曾以「应用内有网页内购」为由拒绝 Steam Link 上架 App Store，所以在很长一段时间里这款应用只有 Android 版本而没有 iOS 版本。

![](.evernote-content/6883456F-F282-4F41-94CD-6D93DEC64896/CE6AD102-7738-4328-B22C-CA54E201C752.png)

初次连接的引导

### Steam Link 有什么优势？

其实早在 Steam Link 之前，已经有很多串流和远程的应用了。而「游戏串流」功能更像是一种炫耀的噱头，很多「远程桌面」类的工具并不能达到玩游戏的要求，往往只能进行轻度的游戏，或者用于展示画面，很难进行实际操作。毕竟玩游戏所要求的「低延迟、高画质、便捷操作」等都在远程桌面工具的考虑之外。我曾在 2018 年 2 月就测评过一款第三方的游戏串流工具 [Moonlight](https://sspai.com/post/43189)，它能够实现基本的游戏串流，但还称不上体验优秀。

如今 Steam Link 再次上架 App Store，**我认为 Steam Link 才是一款真正为游戏打造的串流工具**，Steam Link 在三个细节上让我觉得体验很好。

丰富的设置选项

首先是**初次连接的步骤引导**。Moonlight 的启用需要一些操作，比如需要手动启用 Nvidia 的 GameStream 将手机模拟成一个 NVIDIA Shield 设备。而 Steam Link 则有着相对完善的用户引导，包括选择设备、连接设备、网速测试等多个环节，确保你能安全无误地进行游戏。如果网速测试结果不佳，也会出现一些提示告知你可能会出现网络波动。这在 Moonlight 和微软远程桌面等应用中的直接体现就是卡顿。

其次是**游戏手柄自动适配**。得益于 Steam 强大的游戏社区，很多游戏会有玩家分享的手柄配置。触屏状态下的 Steam Link 支持「虚拟手柄、虚拟手柄 + 鼠标、鼠标」三种操作方式，进入不同的游戏会自动适配最适合当前游戏的方案。这一点也是其他工具不具备的，远程桌面应用通常不会提供虚拟按键。

最后是**按键自定义**。当一些游戏没有合适的按键布局，或者虚拟按键遮挡了游戏画面时，Steam Link 允许你自定义按键的位置，比如默认只提供十字键和 A B X Y，你可以拖一个摇杆出来替代十字键。这类功能鬼泣 5 等游戏中体现很明显（鬼泣 5 中的十字键是无法移动的）。

### Steam Link 的实际体验如何？

那么问题来了，在手机上能玩什么串流游戏呢？我这边测试了手柄操作为主的《鬼泣 5》，鼠标操作为主的《太吾绘卷》，还有昨天刚发售的、需要键鼠操作的战略游戏《全面战争：三国》。在虚拟按键方案下，我的感受是这样的 ——

鬼泣 5 游玩实录

1. 纯手柄操作的游戏体验比较良好，能躺在床上用手机上搓出连招，还是挺快乐的。
2. 纯鼠标操作的游戏体验一般，手机游玩文字较小有点费眼，点按精准度不高，建议用平板游玩。
3. 需要键盘的游戏就必须要外接键盘，否则几乎无法正常进行游戏。

手柄游戏可以实现正常游玩，但如果你真的想用串流好好玩游戏，**确实要配一个能够连 iOS 的游戏手柄才能玩得爽快**。

值得一提的是，在**游戏过程中的延迟相当低**，相信 Steam 对自家的桌面平台和 Steam Link 都做了一些优化。而且串流的画面反而比桌面要稍流畅一些，应该是 Steam 让画面优先在串流端显示。

不过串流**对电脑配置也提出了更高的要求**，相同画质下的《全面战争：三国》在进行 Steam Link 时出现了明显的卡顿（非网络原因的画面卡顿）。

### 点错科技树的 Steam Link？

现在回头来看苹果和 Steam 之前的恩怨。Steam 此前一直在桌面领域深耕，尚未染指移动端游戏。而在 Steam Link 下架的这段时间里，苹果针对 Apple TV 推出了多种服务，对游戏内容方面推出了 Apple Arcade，苹果可能从 Steam Link 中看到了来自游戏串流的威胁。Steam Link 的推出不仅不会给苹果带来收益，甚至会影响苹果在移动游戏方面的收入。

当然，尽管我前面说 Steam Link 是真正为游戏打造的串流工具，**但在移动设备上串流并不是玩游戏的最好方案**。因为移动设备不仅本身存在操作不便、网络不稳定的问题，而且玩家群体也不相同。这就造成了一种端游玩家嫌弃「不如直接玩电脑」、手游玩家嚷嚷「功能鸡肋、延迟太高」…… 被两边玩家看不上的尴尬局面。

![](.evernote-content/6883456F-F282-4F41-94CD-6D93DEC64896/33136CB2-9EA1-451E-BFDD-4AF5D693136A.png)

适合串流的小游戏

把原本庞大的端游放到手机上游玩，操作空间缩小，只得到一时的便利。这对很多玩家而言是一种负优化，两头不讨喜。把相同的功能做在 Shield 这种电视盒子上，游戏表现就非常优秀，相当于做了一次免费升级，玩家群体也更乐于接受。我认为**想让玩家接受串流，就必须让它变成游戏体验的加成**。

所以，我仍然坚持一年前的观点：「**手机游戏串流只适合玩更轻量的、操作难度低的游戏**」。如果想在 iPhone、iPad 上玩点桌面端 Steam 小游戏，那 Steam Link 确实是为游戏串流开了个好头。

在玩上真正的云游戏之前，不如先来试试免费的局域网内串流方案吧，说不定 G 胖能让你对「串流质量差」的观念带来一些改变呢。

### Live Transcribe 与讯飞输入法 9.0：在语音转文字上做到极致

**@Fairyex：**说到语音转文字这件事情，国内外做得最好的估计就是谷歌和讯飞了，最近他们俩又在同一时间在「语音转文字」领域迈出了一大步。

![](.evernote-content/6883456F-F282-4F41-94CD-6D93DEC64896/EA62EDFA-F753-42D3-9172-6883865B7152.png)

**Live Transcribe**

[下载链接](https://play.google.com/store/apps/details?id=com.google.audio.hearing.visualization.accessibility.scribe)

Live Transcribe 原本是谷歌专门为听觉障碍人士所设计的辅助功能应用，它的主要功能是实时将周围的声音不断听写成文字，让听觉障碍人士也可以轻松地参与对话，同时也可以在屏幕上输入文本回复。

但实际上这一功能还有其它的用法，比如会议记录。当你想要记录长达几小时的会议，却不想反复收听会议录音，就可以用 Live Transcribe 来记录会议。

Live Transcribe 可以说是极简设计的典范，打开应用就开始记录，没有开始按钮也没有停止按钮，底下切换语言和输入文本。

![](.evernote-content/6883456F-F282-4F41-94CD-6D93DEC64896/30A07851-2BA7-4BC9-96A2-3FB326D82177.png)

**讯飞输入法**

下载链接：[Android](https://play.google.com/store/apps/details?id=com.iflytek.inputmethod.google)、[iOS](https://itunes.apple.com/cn/app/%E8%AE%AF%E9%A3%9E%E8%BE%93%E5%85%A5%E6%B3%95-%E8%AF%AD%E9%9F%B3%E7%BF%BB%E8%AF%91%E6%96%97%E5%9B%BE%E6%89%8B%E5%86%99/id917236063?mt=8)

而讯飞输入法我们就比较熟悉了，作为语音输入的佼佼者，讯飞输入法在 9.0 的更新中加入了两个非常实用的功能：**普通话和英语无缝识别，普通话和粤语无缝识别。**

它解决了语音识别中的一大痛点，以往我们需要语音输入一大段有不同语言的文本时（比如中英文论文），要么在英文出现的地方停下来手动打出英文，要么就直接读过去，整段读完再将识别错误的中文改为英语单词。现在这些文本就可以随心所欲地一遍过。实际效果可以看一下演示视频：

### 时间记录工具 Toggl 又出第三方客户端：Timery

![](.evernote-content/6883456F-F282-4F41-94CD-6D93DEC64896/D4F163F1-C961-49F5-8870-67DB59623D30.png)

**Timery for Toggl**

[下载链接](https://itunes.apple.com/app/id1425368544)

**@Minja：**作为一款主打在线服务的时间记录工具，Toggl 最令人兴奋的就是其开放的 API，总有人能够据此做出有意思的时间记录方式。比起表现平稳的官方客户端，第三方客户端 Timery 主打的是**通过通知中心小部件快速记录**。

![](.evernote-content/6883456F-F282-4F41-94CD-6D93DEC64896/640D4A8E-2965-4348-8CA3-741C38C5B21A.png)

Timery 的小部件很方便

Mac 用户可能觉得这没什么，毕竟在 Mac 上可以通过点击浮窗和使用快捷键来快速开关 Toggl，但是 iOS 上面的限制就比较多，绝大多数操作都要打开官方客户端进行。这带来的问题就是**容易忘记关掉计时器**，手机只要一锁屏（比如开语音会议的时候）丢一边，很可能就会让 Toggl 一直跑下去。相对而言，Timery 的小部件就要醒目很多，可以直接在 widget 里显示当前进行的任务，只要看到计时器在跳，就很难放任它不管。

不过 Timery 需要订阅，而且主打的也只是记录功能，在官方已经支持捷径、手势操作和统计回顾的情况下，除了小部件以外很难说有什么亮点。暂时来看，Timery 还无法取代官方客户端，毕竟我们仍然需要一个地方来回顾历史记录（除非你愿意每次都打开网页去看）。

![](.evernote-content/6883456F-F282-4F41-94CD-6D93DEC64896/4AC5DD53-EBD2-45F2-93AE-F989FEFCD2BF.png)

Toggl 除了记录还能生成表格

Toggl 开放 API 带来了很多有意思的第三方作品甚至捷径动作，我们内部就在用一个 Hum 做的捷径自动打开时间记录（比如先点一个捷径按钮，就会先开计时器，再跳到 RSS 客户端，从而实现自动计时）。

### 免费全能的正则表达式工具：RegExhibit

**@Minja：**诸位或许遇到过这种情况：想删除文章里的一些文本，遇到复杂点的，文本替换无济于事，似乎只得躬亲。比如下面这堆文本，想剔除里面的图片链接就让很多人头疼 —— 这段文本是我用捷径从网页上扒下来的，勉强算工整，诸位做剪藏时可能遇到过更加复杂的情况。

![](.evernote-content/6883456F-F282-4F41-94CD-6D93DEC64896/B3E6D2CA-21D3-4C65-8087-C08B7ADAE1DC.png)

剪藏下来的文本

这种有一点规律但不完全相同的文本就可以用正则表达式来处理，后者可以用一串字符匹配一类文本，比如 Markdown 图片链接，用一段 `!\[.*\]\(.*\)` 就能够悉数揪出。

![](.evernote-content/6883456F-F282-4F41-94CD-6D93DEC64896/0A86A12F-6767-4E5B-A916-9AA201970BFA.png)

找到 Markdown 图片链接

比较奇怪的是，正则表达式这么好用，在 Mac 上似乎一直没有合适的客户端，多数编辑器也不呼应它。Power User 们比较耳熟的大概只有收费的 Expressions，可是它只能显示匹配结果，却不能接着做别的事情，最简单的，想删除指定的文本都不行。

柳暗花明，我淘到一个年代久远的开源项目：[RegExhibit](https://roger-jolly.nl/software/)，支持实时查看匹配结果，可以分组查看，还可以替换、删除文本，圆满。上面搜出并剔除 Markdown 图链的操作，就是通过 RegExhibit 完成的。

![](.evernote-content/6883456F-F282-4F41-94CD-6D93DEC64896/CF83F103-7300-4411-978F-9B78F47340AE.png)

RegExhibit

[> 官方下载](https://roger-jolly.nl/software/)

正则表达式的标准很少发生大改动，在同类工具凤毛麟角的今天，RegExhibit 在最新系统上可说是老当益壮。

其实正则表达式的功能远远不止替换、删除，在捷径、Keyboard Maestro、LaunchBar、Automator…… 几乎所有自动化工具中，我们都会用到正则表达式。

![](.evernote-content/6883456F-F282-4F41-94CD-6D93DEC64896/ACAE220A-CB3D-46EC-A30B-1244345C866D.png)

通过正则提取符合要求的文章（显示效果经过美化）

### iOS 剪贴板链接直达

**@沨沄极客：**今天要介绍的是一个 iOS 的小技巧：剪贴板链接直达。

如今链接被限制的条件越来越多，如今在 QQ、微信中看到一个链接时，直接打开可能要等很久的跳转白屏。哪怕是正常的链接，也可能会出现「网站已被屏蔽」的提示。白屏、屏蔽时点击右上角的「从 Safari 中打开」是无效的。所以我往往更愿意直接复制链接，手动到浏览器中粘贴打开。而这个操作其实有更方便的方法可以实现。

1. 选中并复制链接。
2. 打开 Spotlight，它会自动识别剪贴板中的链接。
3. Siri 建议的下方出现「打开拷贝自某应用的链接」，点击就可以在 Safari 中打开了。

![](.evernote-content/6883456F-F282-4F41-94CD-6D93DEC64896/3E70DE9E-52B0-4E5E-A343-35EE3E981E4E.jpg)

Spotlight 链接直达

值得一提的是，通过 QQ / TIM 的「我的手机」功能发送链接时，就算不解锁手机也会把链接写入剪贴板。所以直接在锁屏界面进入 Spotlight 也能直接在 Safari 中打开（当然，打开时还是需要解锁的）。

### 网站：代码也能玩套壳截图 ——Carbon

**@Minja：**有时候写文章需要提到一些脚本，为了展示代码的结构、组成（实际上就是想要**代码高亮**）1 ，我一般会在代码编辑器界面直接截图。不过编辑器往往带着工具栏、侧边栏、运行结果窗口，这些元素对于不太熟悉代码的读者来说很有「劝退」效果，但每次为了截图而调整半天编辑器布局又特别打断工作（别忘了我还要调回来）。

一番搜寻之下找到一个网站 ——Carbon，**可以给代码高亮显示，并且进行「套壳截图」**，效果比较理想。

![](.evernote-content/6883456F-F282-4F41-94CD-6D93DEC64896/6D9E92FD-A873-4B5B-BF31-C2CC3C49D674.png)

Carbon

Carbon 是一个开源项目，目前支持识别主流语言（很可惜的是 Mac 自动化中的常客 AppleScript 并不被支持），把代码粘进去就能获得一幅渲染好的套壳截图。图片的背景、颜色主题也可以自己设置，如果觉得左上角的窗口按钮有点画蛇添足，也可以把它去掉，制作一张纯色背景的圆角卡片。

网站链接：[Carbon](https://carbon.now.sh/)
