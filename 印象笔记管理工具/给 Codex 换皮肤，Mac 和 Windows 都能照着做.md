# 给 Codex 换皮肤，Mac 和 Windows 都能照着做

给 Codex 换一套皮肤
-------------

Codex Desktop 的默认界面很克制，但每天长时间对着同一套配色，难免会想：能不能换成自己喜欢的壁纸和氛围？

最近有个第三方开源项目 Codex Dream Skin，做的正是这件事。它可以把一张普通横图变成 Codex 的背景，再配合透明面板、侧栏样式和输入框效果，让整个工作区看起来完全不同。

更重要的是，它没有把整张「假界面」盖在窗口上。侧栏、建议卡、项目选择器和输入框仍然是 Codex 原来的控件，照样可以点击和输入。

这篇文章不只告诉你怎么装，也会讲清楚怎么恢复。因为对这种第三方工具来说，「能退回去」和「能装上去」同样重要。

先说结论：它不是官方主题，也不是破解安装包
---------------------

Codex Dream Skin 不是 OpenAI 官方功能。

它的实现方式，是用带本机调试端口的方式启动 Codex Desktop，再通过 Chrome DevTools Protocol，向 Codex 的渲染页面注入 CSS、背景图和少量装饰 DOM。

你可以把它理解成：自动打开 Codex 的 DevTools，然后帮你执行一段换肤代码。

根据项目当前说明，它不会修改 .app、app.asar、WindowsApps 或官方代码签名，也不会自动改写 API Key、Base URL 和模型供应商设置。

但「不改安装包」不等于「完全没有风险」。皮肤运行时会开放一个只监听 127.0.0.1 的本机调试端口。局域网里的其他设备连不上，但同一台电脑上的恶意程序理论上仍可能尝试访问它。

只从项目官方仓库下载；皮肤运行期间不要启动来历不明的软件；不再使用时执行「完全恢复 Codex」，关闭调试会话。

第一步不是安装，而是选对图片
--------------

最容易踩的坑，是把网上展示效果的整张截图直接当背景导入。不要这样做。

效果截图通常已经包含侧栏、卡片、输入框或文字。如果再把它铺到真实 Codex 后面，就会出现两套界面叠在一起：看起来像界面，实际又点不了。

![](.evernote-content/8354DFF2-4ED3-47C6-90EE-B29699497AA1/0EC3EFF1-9C44-4620-9C85-E4255332BF59.png)

主题图应该是纯背景，不要把预览截图当壁纸。

正确的素材建议这样准备：

• 使用 16:9 横图，2560 × 1440 比较合适；

• 优先选择 PNG、JPEG 或 WebP；

• 文件控制在 16 MB 以内最稳妥；

• 画面左侧或文字区域不要太复杂；

• 使用自己拥有权利或明确获得授权的图片。

![](.evernote-content/8354DFF2-4ED3-47C6-90EE-B29699497AA1/25819627-BB1D-45CA-A8A9-779CFEAEFAF9.png)

双平台安装路径：先记住恢复入口，再开始换肤。

macOS：新手直接双击安装
--------------

Mac 版的流程比较直观，而且不需要另外安装 Node.js。项目会验证 Codex 自带的签名 Node 运行时。

1. 安装官方 Codex Desktop；

2. 至少正常启动过一次，让 ~/.codex/config.toml 创建出来；

3. 保存未发送的内容，然后退出 Codex。

下载并解压项目，进入 macos 文件夹，双击：

Install Codex Dream Skin.command

如果 macOS 拦截脚本，可以右键文件，选择「打开」，再确认一次。不要为了省事关闭整个系统的安全检查。

要更换自己的图片，双击桌面的：

Codex Dream Skin - Customize.command

在文件选择器里选图，给主题起一个名字。如果 Codex 已经打开，脚本会询问是否重启并应用。以后主题没有出现，或者 Codex 更新后需要重新应用，可以再运行 Codex Dream Skin.command。

Windows 11：安装后从系统托盘管理
---------------------

Windows 版要求使用 Microsoft Store 安装的官方 Codex Desktop，并提前安装 Node.js 22 或更新版本。

先关闭 Codex，然后进入解压后的项目根目录，在这里打开 PowerShell，执行：

powershell -ExecutionPolicy Bypass -File .\windows\scripts\install-dream-skin.ps1

安装脚本会验证官方 Store 包、备份原来的外观配置，并创建启动、恢复和托盘快捷方式。

安装完成后，可以双击桌面的 Codex Dream Skin，也可以运行：

powershell -ExecutionPolicy Bypass -File .\windows\scripts\start-dream-skin.ps1

皮肤启动后，在任务栏右下角找到 Codex Dream Skin 图标。右键托盘图标，可以应用或重新应用、暂停皮肤、更换背景图、保存当前主题、切换已保存主题，以及完全恢复 Codex。

选择「更换背景图」并导入 PNG、JPEG 或 WebP 后，后台注入器会读取新的主题文件。没有立即变化时，再点一次「应用或重新应用」。

换肤后，至少做一次功能检查
-------------

1. 首页背景是否已经变化；

2. 左侧栏是否仍能切换任务；

3. 项目选择按钮是否可点击；

4. 输入框是否能输入和发送内容；

5. 打开普通任务后，文字和代码是否清晰。

如果背景抢文字，可以换一张更安静的图片，而不是继续叠加更多阴影和装饰。主题的价值是改善工作感受，不是让每个角落都在发光。

![](.evernote-content/8354DFF2-4ED3-47C6-90EE-B29699497AA1/8695B56C-9EE3-4369-9C0C-96AE8A9DC1B1.png)

真正恢复皮肤，需要停止注入并普通重启 Codex。

怎么恢复到官方界面
---------

Mac 用户双击：

Codex Dream Skin - Restore.command

Windows 用户可以在托盘中选择「完全恢复 Codex」，也可以运行桌面的 Codex Dream Skin - Restore。

恢复操作会停止注入器、移除当前皮肤，并以普通方式重新启动 Codex。你的任务、登录信息和工作目录不会因为恢复皮肤而被删除。

特别注意：Windows 托盘里的「退出托盘」只会关闭托盘菜单，不等于关闭皮肤，也不会自动关闭调试端口。真正想退回官方状态，要使用「完全恢复 Codex」。

Codex 更新后失效，怎么办
---------------

Dream Skin 依赖 Codex 页面内部的 DOM 结构和选择器。官方更新后，某个 class、按钮结构或渲染流程发生变化，皮肤就可能错位或暂时失效。

1. 先执行完全恢复；

2. 重新下载 Dream Skin 最新仓库；

3. 关闭 Codex；

4. 重新运行安装和启动脚本；

5. 仍然异常就继续使用官方外观，等待项目适配。

值不值得用
-----

如果你每天都在 Codex Desktop 里工作，又愿意接受第三方运行时注入工具的维护成本，Dream Skin 确实提供了官方外观设置之外的自由度。

它最聪明的地方，不是把 Codex 变成一张漂亮截图，而是在保留原生交互的基础上改变视觉层。你仍然在使用原来的 Codex，只是给它换了一套衣服。

知道怎么安装，也知道怎么恢复，再决定要不要长期使用。这才是对待第三方工具最舒服的方式。
