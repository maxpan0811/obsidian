用快捷指令 + Airtable 记录硬件保修信息
=========================

我有不少电子设备，手机、平板、笔记本、耳机等，每年可能还会新买一些。除此之外，家里还有空调、冰箱、洗衣机等家电。这些硬件的保修期不尽相同，有的是一年，有的是两年，有的还会在购买延保后继续叠加，统统记在脑子里不容易。

去官网查询保修期也麻烦，毕竟使用频率不高，没人记得住那串网址。每次搜完之后，还得从系统设置深处的「关于本机」里、和被磨损得快看不见的设备底部翻找序列号，最后小心翼翼地填入官网验证，整个过程很麻烦。

知道设备的保修日期其实对我们很有帮助，因为你往往很容易错过保修期，丢失了凭保维修、甚至更换整机的机会（尤其是苹果设备）。我曾经用过一台 iPhone 5s，购买不久后就发现了镜头边缘有进灰，万幸的是那颗灰尘不影响拍照，所以我也没有着急着去 Apple Store 维修。直到设备快过保的前一个星期，系统的日历提醒了我，抱着侥幸的心理，我尝试预约了维修，没想到真的可以更换整机。不仅镜头清理干净了，衰退的电池和有着轻微磨损的屏幕也跟着换机满血复活了。

那一次我是手动将过保日期记录到日历，并设置了提醒。所以我在想，是不是可以有一个**统一查看所有设备保修信息的工具，并且把记录和提醒的过程自动化**。刚好前段时间 Power+ 群里的 @Obk 提到想看 [Airtable](https://airtable.com/) 与快捷指令协作的选题，而 Airtable，就很适合用来实现这个需求。

Airtable 是一款在线表格工具，从工具属性上就很适合用来存放多行多列的数据。Airtable 在各个平台都有客户端，也提供了网页版，更重要的是，Airtable 对外提供了开放的 API，让我们可以通过快捷指令等自动化工具，简化填入数据的过程。从输入、到整理、再到浏览，Airtable 都能满足我们的需求。

![](.evernote-content/06109A88-2A97-4B07-A5E6-CF71D012DF46/4EDFDE04-276C-495C-A636-ABF8239B498F.jpg)

将所有设备的保修信息汇总到 Airtable 里

本文会以记录设备保修日期为例，讲解如何用快捷指令配合 Airtable 的 API 简化输入和自动生成提醒。学会这种用法，很容易就能将 Airtable 应用到其它场景上。

![](.evernote-content/06109A88-2A97-4B07-A5E6-CF71D012DF46/D79083E6-8BBC-4CE8-8D1E-A7ECE95713CE.png)

用快捷指令简化输入和自动生成提醒

使用方法
----

首先，**⏬ [点此下载「添加保修信息」快捷指令](https://www.icloud.com/shortcuts/636196a96d3346018c4c4a3b2fb9b0de)**

打开之后，快捷指令会提示你填写 Airtable 的 **API Key** 以及**记录保修信息的表格 ID**。

![](.evernote-content/06109A88-2A97-4B07-A5E6-CF71D012DF46/43E7317B-2649-445D-B558-E2148673CB90.png)

提示填写 API Key 和表格 ID

API Key 可以在 Airtable 的[账户设置界面](https://airtable.com/account)生成：

![](.evernote-content/06109A88-2A97-4B07-A5E6-CF71D012DF46/CA6F775D-1A93-4B58-842C-8E433E5BE444.jpg)

生成 API Key

表格 ID 则需要我们先在 Airtable 中创建一个表格，命名随意（比如「设备保修记录」）。然后访问 Airtable 的 [API 文档](https://airtable.com/api)，选择刚才创建的表格，就能找到表格 ID：

![](.evernote-content/06109A88-2A97-4B07-A5E6-CF71D012DF46/837FD655-5393-4790-AB95-91578291D758.jpg)

表格 ID

API Key 和表格 ID 填写完成后，就能运行快捷指令。运行过程中，快捷指令会提示你填写**设备名称**、**购买日期**、以及**保修期**。最后，快捷指令会根据过保日期自动在日历中生成一个事项，并在过保前一周提醒你。

动作详解
----

这个快捷指令的前半部分很简单，主要是利用了「要求输入」「从菜单中选择」「格式化日期」「调整日期」等操作，提前填写好了设备名称、购买日期、和保修期。大家一看能明白，这里就不多做讲解。

后半部分则是用到了快捷指令的「获取 URL 内容」操作，调用 Airtable API 在表格里创建新条目。

翻看前面提到的 [Airtable 文档](https://airtable.com/api)，在「AUTHENTICATION」这一栏中我们可以了解到 Airtable 的 API 是需要经过验证的。「验证」的意思其实和「登录」差不多，也就是让 Airtable 服务器知道访问的人是你，而不是别人。否则其他人就能在我们的表格里随意添加内容了。另外 Airtable 有多人协作功能，如果表格是共享状态的话，Airtable 会记录添加人的信息。

所以，我们需要在「获取 URL 内容」操作里，添加「头部」内容。参考文档右侧的格式 `Authorization: Bearer 你的_API_KEY` 填进里面：

![](.evernote-content/06109A88-2A97-4B07-A5E6-CF71D012DF46/08B22CEC-8E54-4628-9EE2-07D9C86F78CF.jpg)

验证（图里采取了调用变量的方式，主要是方便大家填写）

接下来，在文档的「Create records」栏中，我们可以了解到创建条目需要采用 `POST` 方法，并且条目里所有的数据需要包含在键 `fields` 之下。所以我们在「获取 URL 内容」操作的「请求体」部分，创建一个名称为 `fields` 的词典。然后在词典内，再创建三个文本值，名称分别对应表格里的名字。

![](.evernote-content/06109A88-2A97-4B07-A5E6-CF71D012DF46/21341C80-8DAD-488D-B05C-89929D80A20E.png)

创建条目

最后，再用「添加新日程」操作创建日历，并设置提醒时间为「1 周前」。

![](.evernote-content/06109A88-2A97-4B07-A5E6-CF71D012DF46/F0BEF52E-BA9F-4896-8A7C-8F50EA59B9A8.jpg)

「添加新日程」操作

至此，整个快捷指令就设置完成了。

优化版本
----

你可能有注意到，这篇文章的第一张配图里还包含了「主要部件保修」信息。这主要是因为部分苹果硬件会提供主要部件保修两年的服务。比如 iPad 的整机保修期是一年，但主板、内存、硬盘、显示屏等主要部件，享有两年的保修期。你可以在[这个页面](https://support.apple.com/zh-cn/warranties)查询到苹果产品在中国大陆的硬件保修政策。

为了节省操作，我将页面里提到的主要部件保修信息都抠了下来，并通过快捷指令的「如果」操作来判断 —— 如果设备名称包含「iPhone」，那么就填入 iPhone 的主要部件保修政策；如果设备名称包含「iPad」，那么就填入 iPad 的主要部件保修政策；以此类推……

![](.evernote-content/06109A88-2A97-4B07-A5E6-CF71D012DF46/E2C2E5FA-529F-475E-9772-02826DAF2D05.png)

通过「如果」操作判断设备名称

小结
--

表格工具的特点，是可以在一个界面内同时浏览大量的数据，同时还能借助排序、筛选等功能，对数据进行对比。本文提到的记录保修日期只是其中一个例子，像书籍、影视等的观看进度以及评分等内容，都很合适记录到 Airtable 中。
