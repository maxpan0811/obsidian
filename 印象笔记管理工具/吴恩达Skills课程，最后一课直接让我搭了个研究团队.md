# 吴恩达Skills课程，最后一课直接让我搭了个研究团队

---

![](.evernote-content/E6B4EEBB-884E-4B1C-8366-5ADD6AEC0B1F/88DF8FDD-2A43-4054-BEB2-2152728404C1.png)

最近我发现一个规律：无论你用什么 AI 工具——Claude、GPT、Cursor、还是各种 Agent——如果你想提高效率，用到最后，本质都是同一件事：

把流程固化

。

你得把自己脑子里那些精华的经验，提炼成一个可以反复执行、减少差错率的东西。这个东西，就是 Skills。

所以我最近花了大量时间研究 Skills。

市面上讲 Skills 的教程太多了，质量参差不齐，为了不浪费时间，我直接找了吴恩达的课程。

听了之后我发现确实很不错——这门课叫「Agent Skills with Anthropic」，由 DeepLearning.AI 和 Anthropic 官方合作推出。

主讲人 Elie Schoppik 是 Anthropic 的技术教育主管，不是外面某个独立开发者自己录的，是官方亲自下场讲。

课程地址：「Agent Skills with Anthropic - DeepLearning.AI 」（网址显示不出来，可以谷歌搜索上面内容，第一个就是这个课程。）

总共 10 个视频课时，约 2 小时 19 分钟。内容并不多，但信息密度很高。

我把每节课的文字版都做了翻译和清洗，有需要的朋友可以在后台回复「skills课程」领取。

![](.evernote-content/E6B4EEBB-884E-4B1C-8366-5ADD6AEC0B1F/E4E34887-E43F-4F61-AAC8-9A2077006914.png)

这门课到底讲了什么

我按自己的理解，把 10 节课压缩成四个层次。

第一层：Skills 是什么，为什么需要它

课程前两节花了不少时间讲清楚一件事：Skills 和 Tools、MCP、Subagents 到底有什么区别。

简单来说，Tools 是让 Agent 执行具体动作的（比如搜索、读文件）；MCP 是标准化的工具接入协议；Subagents 是你派出去干活的「分身」。

而 Skills 不属于以上任何一类——它是一个知识包，装的是「怎么做」的经验。

打个比方：Tools 是锤子、扳手，MCP 是让这些工具能统一插上的接口标准，Subagents 是帮你干活的助手。

而 Skills 是写在操作手册上的「遇到这种情况该怎么做」。

这个定位看起来简单，但很多人用 AI 效率上不去，恰恰是因为搞混了。

每次开新对话都从头交代一遍风格、流程、偏好，本质上就是因为没有 Skills。

第二层：Skills 的结构和设计原则

第三到第四节进入正题。一个 Skill 其实就是一个文件夹，核心是一个 SKILL.md 文件，加上参考资料和脚本。

这里有一个非常关键的设计思想，课程里反复强调：渐进式披露（Progressive Disclosure）。

大多数人给 AI 喂上下文的方式是「全有或全无」——要么一开始把所有东西都塞进去（贵、浪费、撑爆 Token），要么什么都不给，每次从头说起（脆弱、重复、令人抓狂）。

课程里没有详细展开说，我找来 LEJ Guide 的一篇文章，他解释的 “渐进式披露” 大家比较公认。他说 Skills 是把这个问题分成了三层：

第一层——名片。 会话一开始，Agent 只看每个 Skill 的名称和描述，每个大概 30-50 个 Token。它只知道「我有一个 Slack Skill，我有一个写作风格 Skill」。仅此而已。
第二层——目录。 当 Agent 判断某个 Skill 跟当前任务相关，才加载完整的 SKILL.md。这是一个快速入门指南，告诉 Agent 能干什么、怎么干。
第三层——深层资料。 只有真正需要特定知识的时候，Agent 才去读单个参考文件——表结构、API 文档、完整示例。一个 Skill 可以有几千行参考资料，但闲置时的成本是零。

省 Token 是一方面，更重要的是它让 Agent 的注意力始终集中在当前任务上，不被无关信息干扰。

不过，这个设计依旧被人诟病（多是说skills触发不了等问题），但是有研究提到，将 skills 限制在30个以下，识别效率还是可以保证的。

第三层：在不同平台上使用 Skills

第五到第七节分别讲了怎么在 Claude API、Claude Code 和 Claude Agent SDK 三个环境里用 Skills。

对普通用户来说，Claude Code 那一节最实用。你把 Skills 文件夹放到 ~/.claude/skills/ 目录下，Claude Code 就能自动发现和使用。不需要额外配置，开箱即用。

对开发者来说，Agent SDK 那一节才是重头戏——也是让我印象最深的部分。后面单独展开说。

第四层：压轴实战——用 SDK 搭建研究团队

这是整个课程最后的最佳实践，也是让我直接动手复刻的部分。

三个 Agent 并行工作，自动出研究报告

![](.evernote-content/E6B4EEBB-884E-4B1C-8366-5ADD6AEC0B1F/C1ADE9B4-DD79-4907-906D-13D74BE9CA14.jpg)

课程最后一节展示了一个架构：用 Claude Agent SDK 搭建一个「研究团队」。

核心思路很清晰：

一个主 Agent 负责协调
下面带三个子 Agent，各有分工——比如一个负责搜集资料，一个负责整理数据，一个负责撰写内容
你发一条简单的指令，比如「帮我研究 Gemini Ultra 有哪些付费功能」
三个子 Agent 同时并行工作
最后由主 Agent 拼接，形成一份完整的研究报告
报告自动写入你指定的地方——课程示例用的是 Notion MCP

老实说，课程里这个 SDK 的部分理解起来有门槛。

![](.evernote-content/E6B4EEBB-884E-4B1C-8366-5ADD6AEC0B1F/FD6487FF-D5F9-4E82-8378-125E0AACCAFC.png)

我当时看完一遍没完全吃透，后来跟 AI 反复对话，才形成了一个完整的实现路径。

但实现之后的效果确实让我很满意。我把最后的输出改成了写入我的 Obsidian（课程里是写入到notion），然后拿一个真实问题测试：

比如，我最近在纠结要不要开一个 Gemini Ultra ，想知道它有哪些不同于免费版的权限。

于是试水让这个刚做好的 Agent 去研究，过程中三个子 Agent 并行跑，真的就是一句话，最终确实生成了一份质量尚可的报告。

当然还需要根据行业差异、个人经验调整这个skills，但是视频里给的基本框架已经挺不错了。

![](.evernote-content/E6B4EEBB-884E-4B1C-8366-5ADD6AEC0B1F/0F0B5CBF-87DF-442D-9F0D-9706AE2D392B.png)

课程里举的例子看起来比较简单，我做的也只是一个复刻。

但这件事的价值不在于这个 demo 本身，而在于你可以在这个基础上做大量叠加——把它调整成符合你品味的研究框架、符合你想要的输出格式、写成你自己的 Skill，这些都是可以做到的。

适合什么人看这门课

如果你正在研究 skills，并且经常需要重复类似的工作流，这门课值得花两个小时看一遍。

它不需要你有编程基础，前六节对所有人都友好。

最后一节涉及 SDK 开发，有一定门槛，但你也可以借助AI一步步实现。

如果你还不了解 Skills 是什么，建议先看我之前写的那篇《一文看懂Skills：读完50篇文章后，我安装了这些13个skills（附清单）》，那篇是入门概览，这门课是进阶实操。

我觉得现在这个时代真的很好。只要你想学、有那种钻研的劲头，就一定能找到渠道和方法。
