在 Today Widget 中一览本周的大小事务：Eventail
----------------------------------

![](.evernote-content/F4C766F3-D092-4136-96F3-1F7371655730/69A70CDB-6931-444E-AF40-3C791F000D53.png)

**Eventail**

[下载链接](https://itunes.apple.com/cn/app/eventail/id959674103?mt=8&at=10lJSw)

**@Hum：**在写《[Best Of：iOS 上的日历应用](https://sspai.com/article/43127?series_id=9)》时，我窝了一个对比标准没有写，那就是「在 Today Widget 中的显示效果」。

为什么呢？因为当时在榜的应用在这个点上，表现得都让我并不满意。具体地说，是因为它们在用的都是一种月视图和列表视图的集合：

![](.evernote-content/F4C766F3-D092-4136-96F3-1F7371655730/A66523E5-044D-4E7E-B2C5-C67721E6FAAE.jpg)

Fantastical 在 Today Widget 中的样式

这样的集合，可以通过月视图中，每个日期下面不同颜色的「点」，来从一定程度上反应当天的繁忙程度，但如果说我们想知道这一周的情况，就需要每天都点一下？因此哪怕从便捷操作这一点上来说，日历里的周视图，也是非常重要的。

其实理论上，周视图也同样重要。

我们每周都该在周五（今天）晚或者周一早晨，为自己列一下接下来这一周的基本规划。

这个基本规划，就是一个制作周日历的过程：

首先，我们把已经确定了发生时间的事件（不管它来自工作、副业、生活还是娱乐）放到日历中。这时的日历就起到了它的指导效果，通过它，我们可以从视觉上明白自己闲暇时间还剩多少。

接着，我们可以把其它一些有计划，但还没有规划好的事，填到这些空间中。

只有周视图，可以从视觉上，让我们保持对当天和整个星期的事务敏感性。月视图过于分散而列表视图过于集中。

查看周视图最好的地方，就是 Today Widget。因此当时在列的所有大牌日历软件，在 Today Widget 这个方面做得都让我不满意。

所以一直以来我的 Today Widget 在日历展示软件上没有很好的选择，直到我发现了 [Eventail](https://itunes.apple.com/cn/app/eventail/id959674103?mt=8&at=10lJSw)。

![](.evernote-content/F4C766F3-D092-4136-96F3-1F7371655730/583B0BDA-81BB-4D42-8EA8-DA17B064679B.jpg)

Eventail 在 Today Widget 中的样式

它的功能，简而言之，就是**以周视图在 Today Widget 显示日历**。

这款支持 iPhone 与 iPad 的日历工具，可以在 Today Widget 里显示 3–5 天的日期跨度。它可以把没有日历事件的日期给隐藏起来，但是我更偏好把这些日子都显示出来，让日历更能反应这一周繁忙程度。

除此之外，它可以把已经发生过了的事给过滤出去，不让它们显示在日历上。

以上说的功能都是免费的，付费用户可以用的功能包括改成暗色配色、显示 7 天日历情况、显示提醒事项中的事。

换句话说，免费版已经够用，起码没有限制可以显示日历的数量。

如果你需要在 Today Widget 中一览你这周的大小事务，Eventail 就是你寻觅已久的那个工具。

自动判断用什么浏览器打开链接：BrowserOpener
----------------------------

![](.evernote-content/F4C766F3-D092-4136-96F3-1F7371655730/DDDA1BDF-05BB-441D-9B50-115C80162D3A.png)

**BrowserOpener**

[下载链接](https://itunes.apple.com/cn/app/browseropener/id1387480496?mt=12&at=10lJSw)

**@文刀漢三：**iOS 上的默认浏览器没得选，只能是 Safari。为了和 iOS 保持一致的体验，所以我在 Mac 上自然也选择了 Safari。另外从个人偏好来说，我也是更喜欢 Safari，阅读模式、顺滑的手势操作、iCloud 标签页同步、原生的设计风格，这些都是我选择 Safari 的原因。

但我还是在 Mac 上安装了 Chrome 作为备用，有两个原因：

1. 将 Chrome 作为工作专用浏览器，Safari 作为日常使用浏览器，这个有机会我会另写文章介绍；
2. 国内仍有部分视频网站需要 Flash 才能播放，我又不想在 Mac 上安装 Flash，所以只能借用 Chrome 内置的 Flash 插件来播放；另外，部分网站在 Safari 上可能会显示不正常，Chrome 此时也可以作为备用选项。

本文要介绍的小工具，主要解决第二个问题。这个工具的名字叫 [BrowserOpener](https://itunes.apple.com/cn/app/browseropener/id1387480496?mt=12&at=10lJSw)，最近刚上架 Mac App Store 不久。

BrowserOpener 的原理很简单，它首先会把自己设置为系统默认浏览器，然后通过添加规则点开的链接用什么浏览器打开。

![](.evernote-content/F4C766F3-D092-4136-96F3-1F7371655730/55425130-92EB-4F13-B740-59157AA8D79B.jpg)

BrowserOpener 的设置界面

比如在我的场景下，BrowserOpener 设置内的默认浏览器选择 Safari，另外再添加爱奇艺、AcFun、CCTV（最近在热播世界杯）等链接到 BrowserOpener 中，打开方式选择 Chrome。这样当我从外部点开一个链接时，BrowserOpener 会判断是否符合规则，符合的话就会用 Chrome 打开，不符合则选用默认浏览器 Safari 打开。

![](.evernote-content/F4C766F3-D092-4136-96F3-1F7371655730/C88E548E-F73C-4DC9-92A6-A9DE10F2978E.gif)

普通链接（少数派）用 Safari 打开，特殊链接（爱奇艺）用 Chrome 打开

不过 BrowserOpener 目前有一个问题，就是只能处理外部链接，无法处理浏览器内部的链接。比如当我从 Safari 网页中点开一个爱奇艺的页面，它无法自动跳转到 Chrome 打开。我在 Twitter 上跟作者 [@wxt2005](https://twitter.com/wxt2005) 反馈过这个问题，他表示之后可能会通过浏览器插件来解决。

BrowserOpener 属于偏自动化的方案，如果你需要手动的方案，也可以下载我之前做过的一个 Keyboard Maestro 动作，可以将当前 Safari 页用 Chrome 打开，省去了复制粘贴等繁琐步骤。当你没有打开 Safari 时，这组快捷键也可以用来快速启动 Chrome。

**⚙️ Keyboard Maestro 动作下载：** [用 Chrome 打开当前网页](https://cdn.sspai.com/Open%20in%20Google%20Chrome.kmmacros.zip)

iOS 上最纯粹的解压 / 压缩工具：Kpressor
---------------------------

![](.evernote-content/F4C766F3-D092-4136-96F3-1F7371655730/3768D4D8-D6C2-4222-8952-F466ECC0BDAD.png)

**Kpressor**

[下载链接](https://itunes.apple.com/cn/app/kpressor/id1266619677?mt=8&at=10lJSw)

**@Minja：**iOS 上的解压一直是个难题，虽然我们不会三天两头解压文件（除非你有一个总是用微信发 `zip` 文件给你的同事），但遇到时就会非常头疼，简直可以比肩「不是病疼起来却要人命」的牙痛。

当然也有不少文件管理工具自带了解压功能，但是像 Documents 这样的工具都需要我们把压缩包先拷贝到，不能「原地解压」，总觉得在挂靴瘙痒。在 iOS 11 引入 Files 之后，我就更难以接受交互方式不统一的第三方文件管理器了。

而 [Kpressor](https://itunes.apple.com/cn/app/kpressor/id1266619677?mt=8&at=10lJSw) 就是一款非常纯粹的解压工具，开门见山地提供了压缩和解压两大功能，可以解压全部或单个文件，也能往压缩包里添加内容，带密码的也能解压（当然你得输对密码），还能够预览压缩包里的内容。

![](.evernote-content/F4C766F3-D092-4136-96F3-1F7371655730/CA6ED143-6B9C-4AFC-9191-5F065F59D5D6.png)

Kpressor

操作上，Kpressor 支持 Documents Provider、Drag & Drop 和 iPad 的分屏模式。在 iPad 上用 Kpressor 来解压 / 压缩文件尤其顺畅，简单拖来拖去就行了。虽然和电脑上双击解压的操作还是有差距，但 Kpressor 应该是 iOS 上最纯粹而舒服的解压工具。

![](.evernote-content/F4C766F3-D092-4136-96F3-1F7371655730/AAB28C03-B9E7-4D6B-9B64-BDC5B82BA072.gif)

拖拽添加文件到压缩包

对了， `rar` 文件不在 Kpressor 的支持范围之内（[版权限制](https://zh.wikipedia.org/wiki/RAR)），如果你需要解压这种格式的文件，可以下载我的 [Workflow 动作](https://sspai.com/post/44586)。
