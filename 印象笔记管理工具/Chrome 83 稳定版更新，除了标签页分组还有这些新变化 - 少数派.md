Chrome 83 稳定版更新，除了标签页分组还有这些新变化

根据此前的计划，Chrome 稳定版本的更新受疫情影响将直接跳过 v82 来到 Chrome 83，因此很多原本在 Chrome 82 上就要正式发布的功能也悉数积攒到了本周更新的 Chrome 83 中。

在这篇文章中，我们就来对 Chrome 83 的更新亮点进行汇总。

标签页分组正式上线
---------

如果说自 Chrome 浏览器诞生以来改动最小的功能，或许就是标签页 —— 毕竟其可以创新的部分并不多，甚至用户对于标签页的功能形态早已经习以为常，只有当打开多个标签页时才会遇到需要对打开的标签页进行管理的需求。

正因为 Chrome 浏览器基于多标签页浏览，而每一个标签页都是一个浏览器进程，在做到绝对的浏览器安全隔离之外，也带来了如果打开 N 个标签页后内存泄漏，甚至标签页难以查找的弊端。

所以 Google 加入了对标签页分组这项新功能，通过将相同类型的标签页放置在同一标签页组内，通过颜色名称等标记标签页栏，从而将不同的标签页明显地区别出来，对比此前都是同一颜色的标签页，显然在使用上要高效得多。

![](.evernote-content/637D74E3-D6AB-4C24-99F5-1CFB9F33B1CE/6E0EED74-D0E8-4858-B71A-3AFC49AC7839.png)

而在实际使用中，将鼠标移动到标签页栏然后点击右键，选择「添加到新组」就可以将当前打开的标签页中加入到一个新组中，而点击标签页的左侧的这个「组」的标记色，你可以对这个组进行编辑 —— 更改组的标签颜色，或者给组标记上名字，之后你就可以很明显地看出来这个组和其他标签页的与众不同之处：只要是带有相同颜色标记的标签都属于同一个组。

![](.evernote-content/637D74E3-D6AB-4C24-99F5-1CFB9F33B1CE/A4808823-48DC-4376-BA92-4274AB282273.png)

之后在相同的组里的标签页中新打开的网页都会自动加到了这个组中，这样的设计非常适合打开大量网页查找资料的情景，这样同属一个信息源的网页很容易和其他的无关网页区分开。当然传统的新建标签页并不会基于当前的组，你依旧可以用点击​+​的方式来创建无组的新标签页。

至于想要创建属于某一个组的新标签页，你可以在组内的标签页栏点击右键，选择「在右侧新增标签页」。同理你可以将不同组中标签页通过在标签页栏右键选择的形式来加入不同的组中。

![](.evernote-content/637D74E3-D6AB-4C24-99F5-1CFB9F33B1CE/B2AF1AE7-D398-4FA5-B737-204AEC3BD34F.png)

通过组来管理标签页还有一个很大的好处就是它可以区分工作区，比如我的一个组是和「少数派」有关，有一个组和当前的工作有关，双方衍生或者关联打开的标签页互相不影响，甚至我可以实现基于组的多标签页操作 —— 比如直接关闭分组中所有标签页，或者将其整体移动到一个新的浏览器窗口中，再通过窗口分屏的形式进行多任务处理。

![](.evernote-content/637D74E3-D6AB-4C24-99F5-1CFB9F33B1CE/51F8EB18-D655-4EC5-B845-A27D9C169134.png)

对比其他采用扩展的标签页组解决方案，Chrome 原生的标签页组最大的优势在于通过颜色标注直观的展现群组，毕竟他的目的是区分以及整理已经打开的标签页，而很多第三方的组其实更多的是为了控制所谓的标签页系统资源占用，显然在目的上远不如原生的标签页组来的更加实际。

标签页组功能目前已经在最新的 Chrome 83 稳定版中可用，并将逐步发布到新版 Chrome 中。

隐私与安全改进
-------

Chrome 83 本次更新另一个较多的新功能主要集中在隐私和安全方面，针对更容易泄露个人隐私信息的 Cookie，在新版的 Chrome 83 中提供了对 Cookie 管理的四种选项：允许所有、隐身模式下阻止第三方 Cookie、阻止第三方 Cookie 以及阻止所有。对应的在隐身模式下也有切换面板可供选择。

![](.evernote-content/637D74E3-D6AB-4C24-99F5-1CFB9F33B1CE/9C8B3902-27AC-4BC8-B135-BC226F176759.jpg)

另一方面在 Chrome 80 中开始推进的安全检查功能在本次更新中进一步加强，这一次除了会提醒密码是否泄露之外，还会检查扩展是否存在安全问题，扩展部分菜单也进行了单独设计。可以详细地罗列出扩展访问了哪些数据。

![](.evernote-content/637D74E3-D6AB-4C24-99F5-1CFB9F33B1CE/C385FD69-5C2D-4BB2-857D-20BD742E5EF3.png)

另外增强的安全浏览功能还会检查下载内容和网页是否安全，对于混合内容下载时会给出「文件无法安全下载」的提醒。需要说明的是如果开启了增强的安全浏览会要求将浏览数据发送到 Google。

![](.evernote-content/637D74E3-D6AB-4C24-99F5-1CFB9F33B1CE/77E22652-92FB-4924-A58F-B53AFCFFBCBC.png)

图：9to5Google

从 Chrome 79 开始就开始加入了 DNS-over-HTTPS （DoH），本质上说是为了防止用户被跟踪或者被重定向到恶意网站，而在 Chrome 83 上则变成升级到根据当前的 DNS 服务对应的 DoH 服务。而另一项一直执行的计划 —— 逐步结束对 Flash 支持在 Chrome 83 中进一步给予用户提示，比如如果提示激活 Flash 插件时并导致状态更改，将会再次给予警告。

![](.evernote-content/637D74E3-D6AB-4C24-99F5-1CFB9F33B1CE/2FBDF91D-0C0D-4478-8009-8367D9259252.png)

设计与其他功能
-------

除了部分人心心念念的标签页分组和针对隐私安全的改进，一些早先公布的更新计划也随着本次 Chrome 83 稳定版的推送开始逐步落实。

2019 年 10 月，微软 Edge 浏览器正在紧锣密鼓地测试基于 Chromium 内核的新版浏览器，尽管 Chromium 内核有着更为深厚的用户基础，在「换血」过程中微软 Edge 团队还是不忘时不时「调侃」一下 Chromium 存在的各种问题，比如糟糕过时、土里土气的表格控件设计 ——Egde 团队甚至还单独发过一篇 [博文](https://blogs.windows.com/msedgedev/2019/10/15/form-controls-microsoft-edge-chromium/) 来说明这件事。

而今羽翼渐丰的 [新版 Egde 浏览器](https://sspai.com/post/60041) 已经收获了不少好评，要功能有功能、要颜值有颜值，Chrome 这边自然也不得不将今年 3 月公布的 [新设计](https://blog.chromium.org/2020/03/updates-to-form-controls-and-focus.html) 方案一步一步落到实处：更新到 Chrome 83 稳定版后，你会发现一些常见的网页控件，包括勾选框、文本框、下拉选单、滑动工具条等等都已经从原先带有高光、渐变和阴影的「复古」样式演进为扁平、清爽的现代风格。

![](.evernote-content/637D74E3-D6AB-4C24-99F5-1CFB9F33B1CE/35D89D6C-468E-4C96-98F6-2ED38408B035.png)

新、旧设计对比

如果你想亲自看看效果，可以用 Chrome 浏览器打开这个 [页面](https://www.miketaylr.com/pres/html5/forms2.html) 进行测试。

除了设计风格方面的更新，如何解决用户怨声载道的高资源消耗问题、提升便携式设备续航也在 Chrome 团队的日程之上。

![](.evernote-content/637D74E3-D6AB-4C24-99F5-1CFB9F33B1CE/69583DC3-45F7-47A6-9F4C-837B137B1B79.png)

也正是从 Chrome 83 这个版本开始，Google 将向用户陆续推送标签页冻结（freezing tabs）功能，这个功能启用后 Chrome 会自动检测浏览器窗口的焦点状态，如果部分页面中的内容闲置超过 5 分钟，Chrome 就会通过减少页面像素刷新等方式来降低这些标签页面的资源消耗。

不过你也大可不必担心这个功能会对特定体验场景造成影响，从 Chrome Platform Status 的 [介绍文档](https://www.chromestatus.com/feature/5193677469122560) 来看，有网页自刷新、后台音视频媒体播放、WebUSB 连接等活动的标签页面不在标签页冻结的作用范围之内。所以这个功能更适合那些动辄数十个标签页面的重度网页浏览用户，这部分朋友这次不妨试一试上面介绍的标签页分组功能吧。

结语
--

对于完全依赖桌面浏览的现代办公族而言，Chrome 83 终于全面上线的标签页分组功能应该能让网页信息整理变得简单而清晰，可以有效避免打开重复标签页的问题，甚至进一步实现在浏览器「工作区分类」，效率提升不言而喻。另外值得一提的是，另一头基于 Chromium 内核的新版 Edge 浏览器显然已经为「霸主」Chrome 带来了一点点竞争的压力 —— 尤其在最近加入对扩展程序的同步支持后。
