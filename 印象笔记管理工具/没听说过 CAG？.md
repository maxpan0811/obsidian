# 没听说过 CAG？

---

![](.evernote-content/2326CF62-BDE6-4301-B7C6-2AFC25ECA40D/BCD95B38-02E4-4C5C-9A73-8AED0BC8C6D5.jpg)

  

**没听说过 CAG？那你的 RAG 架构可能正面临严重的“效率危机”。**

**简单来说，传统的【RAG 检索增强生成】虽然强大，但它有一个致命的弱点：无论你的背景资料是否发生变化，系统每一次查询都要跑一遍向量数据库。这就像是明明已经背下来的书，每次考试还要翻开课本查一遍，既费钱又慢得要命。**

**现在，【CAG 缓存增强生成】正在改变游戏规则。**

**它的核心逻辑其实非常聪明：既然有些信息几个月都不会变，为什么不让模型直接“记住”它？通过将这些静态信息预先存入模型的键值对（KV）缓存中，模型就能像拥有了长期记忆一样，实现秒级响应。**

**如果将 RAG 和 CAG 结合起来使用，你就能搭建出一个更高效的知识架构：**

**1. 静态数据层（CAG）：将公司政策、技术文档等很少变动的信息，一次性缓存到模型内存中。**

**2. 动态数据层（RAG）：将最近的更新、实时文档等信息，通过检索实时获取。**

**这种“动静分离”的做法，不仅能让推理速度大幅提升，还能显著降低 API 调用成本，同时减少不必要的数据冗余。**

**不过有一个小技巧：千万不要把所有东西都塞进缓存。如果什么都缓存，很快就会触碰到模型的上下文上限。聪明的做法是只缓存那些高价值、低变动的“冷数据”，而让“热数据”保持实时检索。**

**目前主流的 AI 平台，比如 OpenAI 和 Anthropic，都已经在其 API 中支持了 Prompt Caching（提示词缓存）功能。这意味着你从今天起，就可以开始着手优化你的 AI 应用了。**

**那么问题来了：在生产环境中，你已经开始尝试使用这种缓存机制了吗？**
