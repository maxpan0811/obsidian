PowerToys 是 Windows 下一款非常强大的工具箱，内含众多非常强大的功能。其中有不少功能在我派之前的文章中都已经提到过了，这里就不再过多赘述。具体可以查阅下面的这两篇文章：

前段时间，PowerToys 又迎来了更新，删除了 Window Walker，取而代之的是更为强大的 PowerToys Run，同时也新增了一款工具 Keyboard Manager。它们的加入使得 PowerToys 更加强大。

如何安装
----

官方目前给出了两种安装方法：

第一种就是通过 MSI 文件来安装，也是更加推荐的一种方法。只需要在 [这里](https://github.com/microsoft/PowerToys/releases/) 下载最新的安装文件，双击安装即可完成，非常简单。

第二种则是通过之前微软刚刚发布的一款 Windows 下的包管理器 [WinGet](https://github.com/microsoft/winget-cli)，安装只需要一条命令即可

```
 winget install powertoys
```

这种方法虽然看起来更加简单，不过需要首先安装 WinGet，而且这款工具还在测试中，并不是很稳定，因此不是那么推荐使用这个方法。

**关联阅读：**[在 Windows 10 上安装软件，你现在可以用微软的包管理工具了：WinGet](https://sspai.com/post/60592)

PowerToys Run
-------------

> PowerToys Run is a quick launcher for power users that has additional capabilities without sacrificing performance. It is open source and modular for additional plugins.

这是一款快速启动器，类似于第三方应用中的 [Listary](https://sspai.com/post/52725)、[everything](https://sspai.com/post/51961)，能够快速定位软件、文件或是文件夹并迅速打开。不仅如此，它还能搜索已经打开了的应用，并进行切换。

### 操作技巧

通过 `alt + space` 的操作可以轻松唤出 PowerToys Run 的面板，之后就可以通过输入来选择了。测试下来它的搜索功能支持模糊查询，支持正则表达式。搜索得出的结果永远是应用程序或窗口会排在最上方，接下来才是文件或文件夹。

![](.evernote-content/5D09F04A-5FF8-401A-96D2-79E8A092AD95/C41F8824-8937-4D29-83FE-06118738A080.gif)

被选中（蓝色标出）的选项右侧会出现数个选项，对不同类型的选项有不同的操作。

* 应用程序可以选择以管理员方式打开，或是打开该应用程序所在位置，快捷键 Ctrl + Shift + Enter。
* 文件或文件夹则可以复制所在的路径，快捷键 Ctrl + C，文件还可以选择打开文件所在位置，快捷键 Ctrl + Shift + E。

对于已经打开的窗口选项，目前没有任何可选的额外操作。

### 已知的一些问题

目前在 GitHub 上已经列出了部分，这里列举其中影响使用体验的一些问题

1. 多屏幕下，如果切换屏幕，那么 PowerToys Run 会关闭。（我没有多屏所以没办法测试）
2. 第一次启动时（不管是开机启动还是开机后手动启动），无法通过鼠标点击其他区域来关闭它，必须通过 esc 键来关闭
3. 按下 tab 键不会立马选中下一项，而是需要按好几下。这是因为按下 tab 后，会选中右侧的额外选项，但是因为没有高亮显示，导致用户并不知道目前选中的是哪一项。如果使用方向键就可以直接选中下面的一项。
4. 输入 > 之后通过上下键来选择的时候， > 会直接消失不见。
5. 通过鼠标来选择的时候，右侧的额外选项并不会出现。

### 小结

可以看出目前问题还是非常多的，不过目前我使用下来，如果只是简单地打开文件或应用，切换下窗口，不会太过于影响体验，还是能够正常使用的。不过目前就我个人而言并不是很推荐使用，因为除了切换窗口这个功能，其他都被 Listary 替代，而且 Listary 还更加稳定好用。

Keyboard Manager
----------------

这是一款非常简单却又非常实用的小工具，能够将一个按键映射为另一个按键，或者将一组快捷键映射为另一组。之前要通过修改注册表才能完成的操作，现在只需要简单的一个小工具就可以了。

Keyboard Manager 使用时无需重启，只需要保留后台，那么配置后就可以直接生效。目前还不支持将单个按键映射为快捷键。要注意的是，如果需要交换两个按键的内容，**不要忘记要进行两次映射**，不然其中一个将无法用任何按键输入。

![](.evernote-content/5D09F04A-5FF8-401A-96D2-79E8A092AD95/9498FC87-BAA5-474A-8AFA-74C8B0698882.gif)

### 已知的一些问题

* Keyboard Manager 在登录界面或是需要输入密码的界面中是不会生效的。
* 某些游戏所使用的一些 API 会导致 Keyboard Manager 无法生效。
* 由于 FancyZones 和 Shortcut Guide 这两款工具会映射 win 键，会出现问题，需要手动关闭再打开一次 Keyboard Manager。
* 如果操作过 Caps 键，那么大写锁定的指示灯可能会无法正确显示（由于我之前已经把 Caps 键通过注册表映射为了 Esc 键，因此无法测试）。

最后，不建议和其他重新映射按键的应用同时使用，因为不知道会产生什么奇奇怪怪的 bug。

PowerRename
-----------

这是一款早在 14.0 版本就加入了的工具，但是由于之前的两篇文章中都未提及，因此就在这里再简单介绍一下。

这是一款能够批量重命名的软件，选中多个文件或文件夹后，就可以在鼠标右键中选择 `PowerRename` 来使用了。要注意的是，如果选中了文件夹，那么文件夹中的所有内容也会被同时选中，需要勾选 `Exclude Subfolder Items` 来排除这些内容。

![](.evernote-content/5D09F04A-5FF8-401A-96D2-79E8A092AD95/9CD2FCEB-E838-47DD-9A5B-EA1140265228.gif)

其本质就是查找替换，不过支持正则表达式，支持只修改文件名或文件后缀名，或是只修改文件或文件夹，支持大小写敏感。它的功能非常强大，能够在有需求的用户手中发挥出强大的实力。

不过，由于选项繁多，还需要懂得正则表达式，导致学习成本非常高，因此在小白用户或是没有那么大需求的用户手中，就变得不是那么友好了（比如我）。特别是对于那些只需要在非常特殊的，几乎不会再现的场景中使用一两次的用户而言，这款工具并不实用。

总结
--

可以从最近 PowerToys 更新的频率看出，微软对于这款工具箱目前还是非常重视的。虽然目前还有很多功能不是特别的完善，也有不少的 bug，但相信这些都是可以通过维护升级来解决的，也相信微软最终能够给出完美的答卷。
