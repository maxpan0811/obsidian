以下是我用 XBMC 后的一些心得

0. 决定 装载 XBMC 的硬件

这个是最有意思的。 XBMC 是为 HTPC 设计的但是装台电脑再少也要 2000-3000吧 还大 还不一定好看 还噪音. 我有 DELL 这种类似 MAC Mini 的小方盒子主机, 很适合做 HTPC. 但是也要 2000-3000

目前讽刺的是 最适合搭载 XBMC 的主机是 Apple TV. 因为很便宜 99 USD 小巧美观安静设置方便(相对电脑) 还能  Airplay. (很多人对苹果的观念是贵) 但是目前 3 没有越狱无法安装 XBMC， 2 只能支持到 720p.

2. 更改多语言支持

XBMC 默认使用默认皮肤提供的字体 但是默认皮肤字体是无法显示非英语字符的。

System = > Appearance => Skin => Fonts 选成 Arial based

几秒后 XBMC 就能显示非英语字符了 比如 日文 中文 韩文

不要先更改 中文界面！ 在没有多语言支持前更改成中文界面会导致 全部文字 口口口

3. 整理 让你的 XBMC 变得来福.

为啥要用 XBMC? 就是要改变扔掉电脑的操作方式因为它不来福。 这也是为啥说 XBMC 和 iTunes 是类似的东西。 2 者共同的目的是 来福 改变仍掉 电脑文件系统的操作方式.

如果你用的是普通 播放器，你播放歌曲的方式是 我的电脑 => D 盘 => 歌曲目录 => 不知道那么目录 => 下一层 => xxx.mp3 双击播放.

但是如果你使用的是 XBMC 那么你的方式是 坐在沙发上 点击大大的 Music 图标 在众多专辑封面 或者 歌手中 掉选一张专辑 然后开始播放.

同样电影 电视剧也是 一个道理. 抛弃电脑的文件系统概念 让你直接面对

![](.evernote-content/DA00B209-4A0D-460A-8EF2-AD4DEED3EB6D/5C802C60-9016-4EAD-A4E9-2C16499D547B.jpg)

音乐的管理我就不说了 因为 iTunes 是管理音乐的最佳软件 上面图中为啥 这些专辑都有封面其实是 iTunes 的功劳. 这里说下 iTunes 管理思路和 XBMC 的不同

iTunes 的思路是把媒体信息 比如 歌手 曲名作曲家 剧情介绍 导演 插图 等等内容写在 mp3 mp4 文件内部， 也就是说 你把这个 mp3 mp4 copy 被别人 别人里面也就有了 封面 信息 而 XBMC 则不动文件本身只是建立一套对应的数据库。 坏处是 你要把文件 COPY 给别人或者下次再用那么 那些封面 信息 啊 都没有了. 这里音乐的管理我就不说了 因为我用 iTunes 管理音乐。 这里之说视频的管理. 因为 iTunes 限制太多所以播放盗版不是很方便只适合播放正版。 而 XBMC 就是用来放盗版的. (从iTunes下载来的正版电影是不能播放的)

![](.evernote-content/DA00B209-4A0D-460A-8EF2-AD4DEED3EB6D/1F077022-567C-4F75-AE9F-ED2C03FB071C.png)

**视频文件的分类 命名 和目录结构摆放 (非常重要 直接影响到视频能否被认出 是否能下载到对应信息) 下不到对应信息的文件是不会出现在来福的Library(资料库)里的.**

视频文件在 XBMC 中分为2种类型 单独一个文件 比如 Movie 类行. 因为一个电影就是一个文件。

还有就是电视剧 也就是一堆文件是第一集 第二集 第三集 第二季 第一集.....

所以你所有的视频至少要有2个大目录 一个放单个文件的电影的 一个放电视剧/动画片的.

命名 (超级重要)

文件名直接影响到 你能否找到对应的视频信息 和 iTunes 不同 XBMC 只看文件名不看文件内的 Tag.

电影 (单独文件)

比如 你下载了 <The Matrix> 这个电影。 而你将它改名叫做 黑客帝国.mkv 或者 骇客帝国.mkv 或者 [x264][ac3]黑客帝国 xxxx .mkv 这些都是不靠谱的。 因为翻译名是找不到的, 也不靠谱.

推荐的命名方法是 电影年份\_电影原名 中间不要有空格. (不要有空格只是在跨平台上的命名好习惯不是强制) 比如 上面提到的这个电影 最好的命名方法是 1997\_The\_Matrix.mkv

如果不是英语电影 比如日文 或者 中文 你可以有2个选择

1. 去 the movie db 找对应的英文名称 (因为文件名最好是英文的)

2. 使用官方名称 (运气好可以找到)

电视剧/动画片 (多个文件)

比如你下载了最近热播的 Fate/Zero 系列 如果你把它命名成 圣杯战争0-1.mkv 这样的名字肯定是认不出的

我再三强调 不要使用翻译名称！ 要使用官方名字!

正确的方法是 在你放电视剧的目录下再建立一个目录名字叫 Fate\_Zero 这个剧集的官方名称 (特殊字符用 \_ 代替)

然后里面放 第一集的名称应该是 Fate\_Zero\_S01E01.mkv 第二集 Fate\_Zero\_S01E02.mkv

前面是目录名字 后面是 集数 S01 代表第一季 E01 代表第一集.

如果遇到特别视频 比如 介绍 讲解 则使用 第0集 Fate\_Zero\_S00E01.mkv

在理解命名原则后就是找一个好的 Scraper

系统默认的 Movie Scraper 是 The MovieDB 默认的 电视剧 Scraper 是 The TVDB.

你也可以添加其他的 比如 电影可以添加 IMDB(需要爬墙) 或者 豆瓣

动画片推荐 AniDB 或者 豆瓣

以上的 Scraper 都是要另外下载的.

![](.evernote-content/DA00B209-4A0D-460A-8EF2-AD4DEED3EB6D/83C7E6DC-CEE9-4E41-AE66-B64206D5543D.jpg)

![](.evernote-content/DA00B209-4A0D-460A-8EF2-AD4DEED3EB6D/B44A0F0C-B60F-4ADD-8B33-6415AD786E55.jpg)

(找动画片巨牛的 AniDb)

建立了好的命名系统 和 Scraper 后你就可以去 XBMC 里创建视频源 电影类的指向单一文件的那个目录

电视剧指向多文件的那个目录 推荐动画片 和其他美剧 分开 因为需要用不同的 Scraper 找信息. (但是动画电影无需分开 The Movie DB 基本都有)

如果我不知道原名或者英文名怎么办？ 去 Scraper 的网站查看 原名是什么 或者 Google.

3. AirPlay

XBMC 从 11 开始支持和 iOS 连动并能 AirPlay (模仿Apple TV) 的功能

但是做得并不是很好，音频很难 AirPlay 视频Buffer 做的有问题，老卡。 而且貌似不支持 Mirroring.

开启方法是 去 Settings => Network => 开启 Airplay

如果你的 XBMC 装在 Apple TV 里 记得关掉 XBMC 的 Airplay! 使用 Apple TV 自己的原生 Airplay.

注意 Windows 电脑要开启 Airplay 必须安装 iTunes.

4. 用 iPhone / Android 作为遥控器

1. 在 Settings => Network => Allow Control of XBMC via HTTP 打勾

2. 去 APP Store 下载XBMote 或者在 Google Play 商店里搜索 XBMC Remote 下载遥控器客户端

3. 在客户端上设置 你要遥控的 XBMC 电脑的 IP 地址端口号就能遥控了.

接下来你就能在沙发上享受来福的 盗版体验了.

问题:

1. XBMC 使用内置播放核心对硬解支持好 但是软解不佳 遇到 H.264 10bit 的片子效果会有问题.
