剪贴板增强：Paste
-----------

剪贴板是系统提供的一个暂存数据的地方，无论是平时剪切还是拷贝的内容，都会存到剪贴板之中。但是它有个不足之处，就是无法保存剪贴板历史。复制了新的文本段之后，上一次保存的内容就会被覆盖了。

为了解决这个问题，来自俄罗斯的 Dmitry Obukhov 和 Marat Saytakov，开发了这款名为 Paste 的剪贴板增强工具，它可以将每次复制的内容保存起来，方便再次使用。

### 初次使用：历史记录

安装完 Paste 初次运行会出现如下的配置界面，其中激活软件的快捷键默认设置是三键激活。此处建议可以配置为双键激活，例如图中所示的 `⌥+E` 激活，可以让 Paste 用起来更加方便。

默认开机自启是关闭的，但是考虑到这是一个高频使用软件，此处可以将开机自启一并开启。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/B4F023D0-FD7E-42CF-ADB8-CF62F1C733D7)

在完成欢迎界面的简单配置之后，按下快捷键激活，即可从底部滑出 Paste 的界面。

笔者近期正好在写一篇关于噪音消除的物理小论文，需要在不同的网页中查找相关资料进行整合。在传统的文本资料收集过程中，如果想要保存每一条复制的文本，就免不了在浏览器和笔记软件之间来回切换，复制粘贴文本。

而在有了 Paste 之后的复制历史记录功能之后，就可以让我专注于查找资料，所有复制的文本和图片都会被 Paste 自动保存到它的历史记录中。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/30DD0D8E-F294-43C6-9E00-9C029485A2F5)

当资料查找完成之后，按下快捷键弹出 Paste 的主界面。任意选中一个文本块，右键可以调出功能菜单。其中最值得一提就是第一个功能，**回车将文本粘贴到当前所使用的 App 中。**如果文本包含超链接或者加粗等文本格式，还可以使用第二个选项将其粘贴为纯文本。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/66534FCB-3F65-470C-A15A-C7351E3763BC)

接下来看到**编辑功能**，按下 `⌘+E` 就会弹出一个新的窗口编辑当前选中的文本片段，文本修改好之后只需要按下 `⌘+W` 关闭窗口，修改后的文本就会自动替换原始文本。例如，在下图我将最后的文本来源声明剪切到剪贴板，稍后它将作为参考资料插入文章末尾。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/4DE56424-64A5-4017-9C7A-7175302549FE)

如果你认为使用方向键选择文本稍微有点麻烦，可以在此处按下 ⌘ 键，可以看到每个文本块的左下角都出现「⌘数字」这样的文本，使用 `⌘ + 数字` 即可**快速复制指定的文本块**。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/F72F08BF-3D57-4D07-AB50-D768F7C4A4FB)

### 文本管理：收藏 & 查找

笔者会收藏一些终端命令，可以在需要的时候快速找到使用。这时候就可以借助 Paste 的 Pinboard 来实现分组管理。

点击功能栏右边的 + 号，可以新增一个 Pinboard，用户可以使用 Pinboard 将文本片段保存到不同 Pinboard 上用以归类。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/59ABAA94-72C1-40D5-B302-EF24F42FC35B)

有两种方式可以将文本片段保存到某个 Pinboard，一是将文本片段拖动到该 Pinboard 标签处；二是通过右键的固定功能，将选中的文本归入指定的 Pinboard。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/FFD85513-0DD1-4516-9B04-7D3735991D04)

如果分组数量多了之后，查找起来依旧有点麻烦。这时候就可以使用**搜索功能**进行搜索，点击搜索图标，可以根据关键词搜索相关的片段或者文件，例如此处的「youtube-dl」，默认使用全局搜索且无法改变。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/5656ADB5-3427-4D8A-BDBB-35DC5F86F57D)

另外，除了可以使用片段的关键词搜索之外，Paste 还可以**通过文本块来源进行搜索**。如果记得自己需要的某段文本是在某个软件上复制的，就可以在搜索栏中输入软件名，这样可以进一步缩小搜索结果，Paste 就会显示所有从该软件复制来的文本。

如下图所示，在搜索栏输入：chrome，就会过滤出所有从 Chrome 上复制的文本。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/731A56D3-0F5A-447C-B949-5D76F3A479D9)

如果觉得即使是根据软件名搜索之后，得到的结果依旧太多，可以再加上文本内容可能包含的词再次过滤搜索结果。例如，想搜索从 Chrome 上复制的少数派网页链接，那么就可以输入：「chrome sspai」，就可以得到如下的搜索结果。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/B2D72CC5-25BA-4323-9FCA-6C858DF098EF)

如下图所示，在偏好设置中可以开启链接的预览功能，预览链接功能会为链接添加上一个网站的 logo 或者预览图在文本片段上，这样对于从大量的文本片段中找到某个网站的链接，还是比较有帮助的。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/F4A2F652-AB71-499B-A471-6D9EF08B3F33)

使用快捷键 `⌘ + [` 和 `⌘ + ]` 可以选择上一个或者下一个 Pinboard，方便在多个 Pinboard 之间进行切换。在切换到对应的 Pinboard 之后，就可以直接使用 `⌘ + 数字` 选中文本或者方向键移动，然后回车选中。

### 进阶功能：Paste Stack

偏好设置的快捷键菜单里提供了比初始配置更多的快捷键设置，除了刚刚配置的激活 Paste 之外，还有一个陌生的功能：激活 Paste Stack。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/DF1216EF-88D3-47A1-ACB4-79098185E37A)

正如其名，该功能的作用就是**将复制的文本都保存到一个堆栈中**，等待后续处理。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/2C2D78E7-38D4-4C15-A85B-CE2739FC8B41)

具体使用场景：需要从多个网页中摘抄不同的文本，并将它们按照顺序梳理之后，粘贴到某个地方。先使用默认的快捷键 `⌘+⇧+C` 呼出 Paste Stack 的界面，然后就可以在不同的网页之间选中文本并复制，这些文本就会进入 Paste Stack 列表。但是需要注意的是 Paste Stack 不支持文件内容，所有的文件内容都会显示为 Unknown，所以它的使用范围就被局限在了文本。

点击 Paste Stack 列表的右上角的排序符号可以选择正序或者逆序排序，如果某条文本内容觉得是无用的，也可以使用双指右滑删除。

当需要的文本都摘抄完了之后，就可以进入任意一个文本编辑软件，使用 `⌘+V` 粘贴堆栈里的文本。

### 其他特色功能

#### 格式处理：纯文本模式

在通用设置菜单里值得关注的是**粘贴为纯文本功能。**例如从网页上复制下来的网页链接，可能是富文本的格式，勾选该功能后，文本在粘贴出来之前就会被处理为纯文本了。这对一些文本工作者来说是比较实用的功能，可以省去手动再处理格式的步骤。或者使用文本的时候同时按下 `⇧` 快捷键，可以把文本以纯文本的形式粘贴出来。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/7A699A58-B631-48B9-B0EB-5A853CAA2DE1)

#### 排除不记录的软件

规则菜单是用于**排除某些文本的保存**，或者**在某些软件中禁用 Paste**。建议可以将两个选项都勾选，避免保存到密码管理软件的复制或者随机生成的密码。对于「不保存以下应用的历史」功能，普通用户可能无需另外再添加其他软件，对于经常使用终端的用户来说，或许可以考虑将使用的终端加入禁用名单，以免 Paste 保存了过多无意义的文本之后所带来的查找不便。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/BC6A93CB-175A-4D30-9D6B-CBF006496EAC)

#### iOS 同步查看

Paste 订阅之后多端都可以使用，所以建议在 iPhone 或者 iPad 上下载 Paste 来同步剪贴板数据。使用场景像是在 macOS 上处理完文本之后保存到 Paste，然后带着 iPhone 或者 iPad 在外使用，如果这时候正好需要使用到剪贴板上记录的某条内容，就可以通过 Paste 很方便地找到需要的文本了。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/F5DF6A55-1C91-4844-A7C0-56D54D52C6B0)

### 小结

Paste 像是为剪贴板文本找到了一个家，让他们有了归属，作为用户可以对这些文本块进行筛选归类整理，留下有价值的文本块。分类整理之后方便以后快速使用。

该软件比较有创新的就是 Paste Stake 功能，如果是使用传统的复制粘贴方式，需要在网页和文本编辑器之间来回切换复制粘贴。而 Paste 利用堆栈的概念，让用户只需要专注于文本的收集即可。

解压缩工具：BetterZip
---------------

解压或者压缩文件，是一个很常见的文件操作。但是对于 macOS 来说，自带的「归档实用工具」只能解压和压缩 zip 文件。对于其他类型的压缩格式，它就束手无策了。为了解决这一问题，我们就需要使用第三方的解压缩软件，用以完善对其他格式的压缩文件的支持。

而 BetterZip 就是这样一款全能的解压缩软件，接下来就来看看 BetterZip 究竟有多好用吧。

### 软件安装 & 设置

下载 BetterZip 初次运行之后，会显示如下的欢迎界面。可以看到下面有四个勾选框，建议就按照默认配置直接下一步即可，但是如果你没有使用 Alfred 可以取消勾选最后一项。最后一项的 Alfred workflows 功能将在后文讲解。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/2CA198C8-8979-46BD-9EAC-2F6E3B3135A5)

### 解压文件

简简单单地设置之后，BetterZip 就会全面接管系统的解压缩工作。但如果双击压缩包尝试解压之后，就会发现它会跳出主界面，需要手动选择解压路径之后，才会进行解压的，并不像归档实用工具一样很自然地解压。

对于这种情况，我们只需要打开偏好设置，在「通用设置」中勾选「从 Finder 中打开压缩文件时直接解压缩」，即可做到「一点就解压」的效果。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/F51E4FC5-F04E-4390-9C9C-29AB26A33630)

### rar 文件解压缩

由于 rar 是一种专利文件格式，rar 解码器是开源的，但是编码器却是要收取版权费的。所以目前多数的解压缩软件都是可以解压 rar，但是却无法创建 rar 文件。

为了解决这一个问题，BetterZip 使用了挂载外部 rar 命令的方法，实现了 rar 压缩文件的创建。为了使用这个功能，需要先前往偏好设置的程序助手菜单下，点击「下载并安装 Rar」。这样 rar 就可以外挂到 BetterZip 使用了。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/C9EF2F1C-B577-411B-908F-CE8A1ED1F898)

### 压缩文件

BetterZip 在安装完成后，会在右键菜单添加功能选项，可以让我们很方便地解压缩文件。BetterZip 在这里提供了个性化的设置，请注意看灰色字：「从预设解压」和「使用预设压缩」。而这个预设是可以在偏好设置的「预置」菜单下进行更改的，我们可以在这里修改一些设置，让它使用起来更加适应我们每个人的实际需求。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/473B4A35-A538-4B3C-A1CA-4FF187E632F9)

打开解压缩的预置菜单，在这里我个人对解压成功后「移动压缩文件到」的位置进行了修改。原本的移动到废纸篓，我修改为了不移动。由于此处的偏好设置较为个性化，所以在这里的设置需要根据个人实际需求进行调整。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/047EC6F7-8BC7-4900-BC5C-6E2C8E9E7B05)

### 为 PC 保存

使用 macOS 自带的归档实用工具，打包出来的 zip 文件，如果发给 Windows 用户看的话。就有可能出现两个问题：根目录多出来一个 \_MACOSX，文件目录下多出一个 .DS\_Store 文件。以前我将文件使用归档实用工具打包发给老师，在安卓上解压一下都是乱码出来。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/F3FE9CDE-6CA9-4FE2-B05B-ED56056DA000)

为了解决这个问题，避免在实际工作上带来麻烦。可以使用 BetterZip 的「为 PC 保存」功能，确保压缩文件能在各个系统上正确解压。使用「为 PC 保存」功能压缩后，在 Windows 下解压出来的文件目录如下，与 macOS 目录结构完全一致，且没有乱码问题。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/D5709B53-694A-4D8C-993F-89919B54E736)

### 小结

BetterZip 提供了全面的压缩文件格式支持，且通过挂载 rar 外部命令的方式，实现了 rar 文件的创建。还可以使用预设功能，调整软件的解压缩设置，让它变地更加适合我们的实际使用需求。除此以外，空格预览功能也能让给我们快速查看压缩包。赋予它一个解压缩全能选手的称号，是实至名归的。

NTFS 磁盘写入支持：iBoysoft NTFS
-------------------------

对于 macOS 用户来说，NTFS 文件的写入有时会带来一点小麻烦。

比如作者以前就碰到过这样的情况，因为平时 U 盘都是我自己使用，自然就格式化成了 macOS 扩展格式。有一次需要跟老师拷贝资料，老师问有 U 盘吗？我拿出 U 盘给老师拷贝资料，可是 Windows 是无法读取 macOS 扩展格式，就给我带来了一点麻烦。

为了能够让 U 盘同时支持 macOS 和 Windows 的读写，我们就需要将 U 盘格式修改为 NTFS 格式。但是要注意的是，macOS 只支持 NTFS 的读取，而不支持 NTFS 格式的写入。

为了解决这一问题，就需要可以使用 SetApp 里面的 iBoysoft NTFS for Mac 软件，来实现 macOS 对 NTFS 格式磁盘的写入。

### 基本使用

安装好 iBoysoft NTFS 之后，运行程序进入欢迎界面。在这里你可以根据需要，设置是否让软件开机自启。如果只是偶尔读写 NTFS 磁盘，可以考虑关闭开机自启。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/B07AD85C-1760-49CF-871C-C32710791BDA)

开机自启配置好了之后，iBoysoft NTFS 就会在状态栏上保留一个图标供使用。

以我手上的 macOS 扩展格式 U 盘为例，需要先将它格式化，更改为 NTFS 格式。使用 macOS 自带的磁盘工具，选中 U 盘，点击上方工具栏中的「抹掉」按钮，将格式修改为 Windows NTFS 格式即可。虽然 iBoysoft NTFS 自带 NTFS 格式化功能，但是不建议使用，因为它无法在格式化时自定义磁盘名称。而且格式化磁盘这么重要的工作，还是交给系统自带的软件会安心一点。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/3E0E9F43-9401-4D5E-BB86-9D6D8F9FFD6E)

格式化完成之后，推出 U 盘，再重新连接 MacBook，即可看得到 iBoysoft NTFS 弹出提示框。提示检测到了 NTFS 格式的磁盘，是否需要以读写的方式打开。点击「确认」即可在 Finder 里面打开磁盘的目录。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/08F686D7-01ED-4ED7-9F0D-BF9AAB6CE703)

如果点击确认之后，Finder 没有反应，也可以通过状态栏上点击图标打开目录。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/DD39A3D9-0537-4974-97A6-1EF45C25D0F1)

### 推出磁盘

很多在 macOS 上使用 U 盘或者其他移动设备的用户，都可能遇到过下面这个提示。

![](.evernote-content/C63DF629-05D4-4D6E-B7C6-095670C3F0D9/CDAD8BD8-0386-4620-82D7-D6F6E9F58A71)

U 盘使用完了之后，点击 Finder 磁盘旁边的推出按钮，却提示磁盘未能推出。有时即使按照它的提示关闭相应的程序，也依旧是这个提示。（因为有一些后台并不能直接从 Dock 关闭）

为了解决这个问题，iBoysoft NTFS 给磁盘增加了推出功能。当使用完 U 盘之后，只要从状态栏选中 U 盘，再点击推出即可安全推出磁盘。

### 小结
