2. **v2（品牌过滤）**：标记非携程系跳过 → 漏配严重，匹配率从 36→10%

```
提取订单系统门店名和联盟条目的独有地点关键词（3-6字，剔除通用词）

```

## 六、v2 修正：品牌门禁匹配（2026-07-12）

> 上一版本数据**全部作废**。原因：匹配逻辑有根本性缺陷——仅做关键词交集，忽略品牌体系。52% 的匹配是品牌错配。

### 问题发现

Hy 指出核心问题：**联盟条目中的品牌体系完全被忽略。** 渝之旅门店被匹配到百事通/去哪儿门店，仅靠地点关键词碰巧撞上。

| 联盟条目 | 品牌 | v1 错配到 | 正确结果 |
|---------|------|----------|---------|
| `渝之旅九龙坡区步行街二门市部/海航乐游...` | 渝之旅/海航乐游 | ❌ 去哪儿/百事通 | ✅ 驳回（非携程系） |
| `海外五小区门市` | 海外 | ❌ 百事通 | ✅ 驳回（海外≠百事通） |
| `海外百事通南方花园` | 百事通 | ✅ 百事通 | ✅ 含「百事通」，正确匹配 |
| `渝之旅南坪万达广场/重庆携程江南大道` | 携程 | ✅ 携程（靠地点撞） | ✅ 含「携程」，正确匹配 |

### 修正方案

```
阶段1 — 品牌体系门禁（新增）
  携程系品牌 = [携程, 百事通, 旅游百事通, 去哪儿]
  ✅ 含任一 → 放行
  ❌ 不含 → 标记「未匹配（非携程系: 渝之旅/海外/...）」

阶段2 — 地点关键词交集（仅放行条目）
  独有地点关键词交集 → 加权评分 → 阈值筛选 → 去重输出
```

层级把关：海外/海外旅业 NOT mapped to 百事通（Hy 确认严格匹配）。
