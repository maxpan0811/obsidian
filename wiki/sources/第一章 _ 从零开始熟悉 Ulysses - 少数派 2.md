# 第一章 | 从零开始熟悉 Ulysses - 少数派

---

从零开始熟悉 Ulysses
==============

> 小说《挪威的森林》的开头部分， 是在希腊各地咖啡馆的小桌上、轮渡的座椅上、机场的候机室里、公园的树荫下、廉价旅馆的写字台上写的……因为总也找不到固定的居所，到后来仍旧在欧洲 各地游荡， 在各种场所继续写这部小说。

村上春树在《我的职业是小说家》一书中，绘声绘色地描述了自己这段写作经历。这让我意识到，原来作家的并不只是整日在书房正襟危坐，而是在能写字的地方，就能开始创作。

而 Ulysses，正是每个写作者用来不分时间、地点地，捕捉转瞬即逝的灵感、施展才华的帮手。

然器具虽利，但也要使用者对其捻熟于心。打开 Ulysses，也许你会被其中有趣又实用的功能所吸引；也可能会因为它功能繁多，不知如何开始而一筹莫展。此时就要警惕，不要被细节一叶障目，看不见 Ulysses 整体全貌。

Ulysses 系列教程的第一篇，就**以鸟瞰的视角，看看 Ulysses 如何嵌入到写作的每一个环节**。

一切从 Markdown 开始
---------------

如果你之前只用过 Word 来作为自己的写作工具，打开 Ulysses 后想必十分「惊慌」。在 Ulysses 的界面中，完全找不到类似 Word 中的「开始」、「插入」等一系列编辑文字样式的工具。

难道 Ulysses 只能像记事本一样光秃秃的书写吗？

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/5F95DB3D-D74C-4394-982C-2A977DC8217C)

Word 菜单栏界面

正如序言中所言，Ulysses 是一款写作工具，主要侧重优化我们在文章写作时的体验。和 Word 不同，当我们调整文章的「外貌」（样式）时，在 Ulysses 中用的是一种被叫做「Markdown」方式1。

让我们从 Markdown 这个新概念开始，体会在 Ulysses 中写作是一种什么样的体验。

### 发现 Markdown 的本质

Ulysses 和你早已熟悉的 Word 相比，关键的不同**在于如何给文章加上样式**。

使用 Word 工作时：

1. 使用者先输入文稿；
2. 再**在工具栏选中**想要的功能，为文字设置各式样式。例如字体字号，加粗排序列表等；
3. 有必要的话，还会为文章整个页面加上页眉页脚、页码符号等。

这些步骤中所选择的样式效果，会**直接显示出来**。这些带有格式的文字被称为**富文本**，相应地 ，Word 就被称为**富文本编辑器**。

而在 Ulysses 中，同样是输入文稿，但不需要使用者手动的去点击菜单栏工具，而是给想被赋予样式的文字，**用特殊的符号包裹起来**。例如下面这句话中，用 `*` 包裹起来的文字，就会显示出粗体的效果。而句子前的 `>` 符号，会将整句设定为引用的样式。

在 Ulysses 中输入为：

```
> 除了我的**天才**，其他没什么可申报的。——王尔德
```

在 Ulysses 中按快捷键 `⌘Command + ⇧Shift + P` 预览，显示效果为：

预览标记显示效果

这类能控制文稿样式的符号，被称为 **Markdown 语法标记** 2；所有语法标记集合起来，就是一套 **Markdown 语法**；保存的文件相应地被称为 **Markdown 文件**。Ulysses 能编辑和显示这类文本，就被称为 **Markdown 编辑器**。

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/6DF160FD-549C-4D11-B118-90E6CF1C47AE)

Word 与 Ulysses 的对比

再回过头来将两者的操作流程平行地对比来看，就能发现 Markdown 的本质：**用语法标记来注明文字的样式**，之后由软件（例如 Ulysses），将夹带着语法标记的文稿转换成富文本。而不是像 Word 一样，直接通过操作功能键来得到富文本。

上文是从技术实现的角度来区别 Markdown 和富文本。从使用者的视角，更能发现两者的差异。在 Markdown 编辑器中，使用者**只有「输入字符」**（文稿和语法标记）这一个动作，就完成书写和排版两件事。编辑界面中只会显示字符，想要查看效果，需要按**预览**功能键。而在 Word 中，输入文稿和调整样式，被泾渭分明地**区分成了两个动作**。而且调整好样式后能直接查看效果。

预览 Markdown

### Ulysses 为什么选择 Markdown ？

Markdown 工具和 Word 为代表的富文本工具，在最终目的——得到一份带样式的文稿——上其实是一致地，只是两者在赋予文本样式时选择了不同的方式。那 Ulysses 为何不借着 Word 的普及，采用对多数人来说，学习成本低的富文本编辑方式呢？

作为一款写作工具，**使用时能保持连贯流畅的书写**，是最基础也最核心的体验。而 Markdown 工具能更好的做到这一点。

在 Ulysses 中，**我们的手指和思路保持连贯，集中在输入字符这一个动作上**。而在使用 Word 时，需要不停在键盘输入和鼠标选中按键中来回切换。回想下，有多少次使用 Word 时，你是能一口气先写完全文，再去一次性完成排版？

不管是使用 Word 还是 Ulysses，将文稿发送给他人进行编辑、修改都是常见的需求。富文本编辑器另一大短板，是不利于分享给他人。且不说不同编辑器的文件格式不能互相打开，就是同为 Word 的 `.DOCX` 格式文件，你肯定也遇到过不同版本、不同平台之间的文件，打开后排版面目全非的状况。而 Markdown 的 `.md` 3文件，是真正做到「**一处书写，随处打开**」。

其实 Word 最主要的价值，在于它有行业标杆级的字处理和排版能力。选择富文本编辑器，利于使用者用直观的方式进行排版。而 Ulysses 作为一款侧重写作的工具，则选择了**能让用户从书写到分享更省心**的 Markdown。

### Markdown XL 标记简单分类

讨论过 Markdown 的概念和优势，希望你已经放下对它的恐惧和抵触。它易用，且易学。接下来花几分钟，你就能掌握 Markdown 的基本用法。

多个 Markdown 语法标记集合起来被称为 Markdown 语法。在 Ulysses 中，应用默认选定的是被称为 **Markdown XL** 的语法。限于篇幅，这里不展开介绍它与原生的 Markdown 语法的区别。对于初学者来说，只需要知道，Markdown XL 与 Markdown 的关系，**就像方言与普通话**，能表达相同的意思，但在具体的标记上有细微差别。

Ulysses 一共支持 25 种语法标记，从使用的阶段来分，我将它们大致分为「书写时使用的」和「编辑时使用的」两类。

1. 书写时使用的：标题、分隔、粗体、重点、列表、引用、链接、图片、视频、脚注、代码、原始源；
2. 编辑时使用的：标记为、删除、注解、评论、注释块。

在这一章，我们**先只关注第一类标记**，掌握了它们就能完成写作的任务。在下一章再对第二类标记进行介绍。

### 在 Ulysses 中输入语法标记

点击「窗口 - 标记」（快捷键 `⌘Command + 9`），便能看到这些语法标记对应的字符。

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/A288CA20-A340-4B23-A9A3-108AD38E74E4)

Ulysses 的语法标记窗口

我们要做的，**就是把想要修改样式的文字，用语法标记注明**。有两种方式你可以做到这一步：

* 像使用 Word 一样，先**选中文字**，再**点击菜单栏「标记」**下的语法符号。

使用语法栏修改样式

* 更快速地操作是，在输入文稿时，**直接键入语法标记**。

通过「标记」添加标记

有的标记只需要在文字前加上即可（例如引用的 `>`），而有的则需要在文字前后都加上字符（例如粗体的 `**`）。可能你会抱怨，需要在文字前后都要敲一遍标记太麻烦了，尤其是标记的字符都不方便输入的时候。

这时有一个操作小技巧，输入完文字光标会停在文字后，而不是标记后（因为 Ulysses 没有智能到知道你是否输入完了），直接按回车键，Ulysses 就会**自动补全**文字后面的字符。

回车键入标记

### 了解 Ulysses 中的语法标记

第一类中的大部分标记，都能直观地从字面理解它们的作用。不过仍有几个用法特殊一些的标记，需要特别说明。

#### 脚注、链接、图片和视频

在文档中选择插入脚注标记的位置后，Ulysses 会弹出窗口，在窗口中进一步附上相应的内容。想要删除或者修改，双击文字，窗口就会再次显示出来。

插入脚注

而插入链接稍微特别一点，需要**先选中文字**，**再给文字添加链接**。添加完成后文字的样式也会相应地有所变化。

插入链接

插入图片和视频在 Ulysses 中也非常方便。因为这两种语法标记的输入稍微麻烦，我更喜欢用快捷键的方式输入。**直接将图片或视频拖到弹出的窗口中即完成插入**。视频还可以在窗口中直接播放，不过预览时不会显示视频。另外值得注意的是，只有在 iCloud 文库中的文稿才能插入视频。

插入图片和视频

#### 代码与代码块

至于代码和代码块，只是**同一元素的两种不同呈现**。代码标记顾名思义，是**为了显示一段字符是程序代码**，其实就是行内代码，不过也可以用于标出一些不属于文稿的内容，比如前文中我用来标出快捷键的方式，实际上就是使用了代码这种语法标记 。例如：

```
点击「窗口 - 标记」（快捷键 `⌘Command + 9`），便能看到这些语法标记对应的字符。
```

如果要标注的范围**不是句子中的某个单词，而是一整段，则使用代码块。**

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/CFBE1A08-74AB-491C-BC06-987D883DA869)

代码块演示

#### 原始源与原始源块

对于平时接触 Word 的用户来说，原始源和原始源块这两种语法标记可能有些陌生。Markdown 语法覆盖了大部分对字处理的需求，不过总有一些想要的样式没有内置。得益于 HTML 语言与 Markdown 的血缘关系（在第四章会详细展开讲），在 Ulysses 中可以直接写 HTML 文稿，就能**正常显示 HTML 所表达的内容**。

最常用的例子是表格。Ulysses 没有内置表格的语法标记，但当你急需在文中插入一个简单的表格，就可以使用 HTML 的语法来写：

```
~~<table border="1">
~~<tr><th>姓名</th><th>职业</th><th>格言</th></tr>
~~<tr><td>奥斯卡·王尔德</td><td>作家</td><td>除了我的天才，其他没什么可申报的</td></tr>
~~</table>
```

而实际效果将是：

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/04FF3DD5-9B0B-41F2-A483-E2BD3B958425)

利用原始源插入表格

值得注意的是，这些 HTML 元素**只能在 HTML 预览时显示**，其它格式的预览中将不会显示出表格和原始源块中的字符。

使用原始源不仅能显示 HTML 支持、但 Ulysses 不支持的元素，更强大的是它能借助 HTML 的特性实现更极客的功能。例如在 [Ulyssses 的博客](https://ulyssesapp.com/blog/2016/11/eline-explains-5/)中，就介绍了通过 HTML 代码，实现在 Ulysses 中预览 LaTeX 公式的效果。这里限于篇幅不做展开，在第七章中再来详细介绍。

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/081A2A22-7749-4CF7-AC54-50BA975BF316)

#### 转义符

最后我们来了解一种在语法标记窗口中没有告诉你的标记——转义符。

Markdown 中的语法标记都是平时书写时可能会用到的符号，虽然都是在英文状态下的半角符号，书写中文时一般不会有问题，但是书写英文时就面临一个难题：如果我就是**想输入这个符号本身**，例如 `#`，在 Ulysses 中有可能错误的认为你想输入一个一级标题，这该怎么办呢？

其实解决方法很简单，**在符号前面加上英文的反斜线  符号**，注意和你想输入的符号之间不要留有空格。Ulysses 就会明白：哦，你是想直接显示这个符号。 使得符号失去了语法标记的含义，所以被叫做转义符。

#### 小心区分代码、原始源和转义符

代码、原始源和转义符虽然从定义上来说各司其职，不过在直观显示上，都有「直接显示某些内容」的效果。所以使用时，必须小心区分它们的用法，以免导出文稿时发生错误。

原始源块与代码块这两种标记首先可以区分出来。它们都是**针对一整段文稿的语法标记**，不用于一句中的某个词或字。

而原始源中的内容，**只会在 HTML 和 ePub 文档中显示出来**。在 PDF 和 DOCX 文档中，既不会显示原始源中想表达的内容，也不会显示原始源文本本身。

原始源的效果

行内代码在一些 HTML 预览中会有特别的效果，而转义符则不会有。除此之外，转义符只对符号有效，而代码可以包裹类似函数名的内容。

虽然这几类标记在使用上并没有严格的限制，不过我这里还是归纳一下自己的使用方式，以供大家参考：

* 转义符：想直接输入符号；
* 代码：在行内标出想突出的符号或者代码中的函数名、变量名等；
* 代码块：一段或多段计算机代码；
* 原始源：在行内想添加特殊的 HTML 元素，例如上标；
* 原始源块：一段或多段有特殊功能的 HTML 代码。

看到这里，「怎么在 Ulysses 中输入文稿」对你来说已经不成问题。 Markdown XL 的第二类标记，会在接下里的章节中穿插演示。

如果你想对 Markdown 理解的更透彻一些，可以阅读《[Markdown 完全入门（上）](https://sspai.com/post/36610)》和《[Markdown 完全入门（下）](https://sspai.com/post/36682)》这两篇文章。

理解了 Markdown XL 只是熟悉 Ulysses 的第一步。如果只是想要一个能写 Markdown 文本的工具，市面上有很多优秀且便宜的替代品。 **Ulysses 的价值，在于你能用它完成整个写作的流程**。

与其夸夸其谈，不如实际操练一番。从一个使用者的角度，体验使用 Ulysses 完成一篇文章的全过程。

从开始的地方就做好准备
-----------

微信小程序号称「开箱即用」、「用完就走」，但这些工具大多功能简单，使用频率不高。Ulysses 兼顾了你日后写作时的纸、笔和书桌，甚至资料夹的作用，若是不在开始使用时好好规划，必然大大影响使用的效率（想想你电脑的桌面和办公室的工位）。

### 什么是文库

使用 Word 的用户，习惯将写好的文稿直接保存在项目的文件夹中。这看起来顺理成章，但是积累若干时日，就得在不同的文件夹中奔波，找到想打开的文稿。为了更好的管理所有的文稿，Ulysses 采用的是**文库**的方式。

文库与文稿的关系可以类比为 iTunes 与音乐文件。我们**在 Ulysses 中可以查看编辑文稿，但是看不到文稿的具体文件和文件在硬盘中的位置**。在 Ulysses 中，文库根据保持文稿的方式分为三种：

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/8A473109-A28C-4746-B5DD-262AAEF6B9E9)

文库

在「偏好设置 - 文库」中，可以控制哪些文库显示在 Ulysses 的边栏中。

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/8F422917-2736-4B2C-9830-46BB7B756A69)

偏好设置 - 文库

### 在 Ulysses 中使用 iCloud 文库

Ulysses 中的 **iCloud 文库**是最能保证使用体验的地方。

iCloud 文库实现了**和 Ulysses for iOS 无缝同步**的效果。在 Mac 上编辑过的内容，我们**不需要**做任何额外的设置，Ulysses 会即时地同步到 iOS 上。如果两台设备在不联网的情况下修改了同一篇文章，产生了冲突，Ulysses 也会提醒你选择其中一份保存。整个过程悄然无声，又可靠稳定。

Ulysses 采用的是 CloudKit 而非 iCloud Drive 技术，将文稿保存在 iCloud 上，在「Finder - iCloud Drive」中是看不到 Ulysses 文件夹的。

而在 iCloud 文库的下一层级，则被 Ulysses 称为「**组**」，文稿就保存在不同的组中。**组就是我们习惯的文件夹**，组之间可以嵌套，可以为它选择特定的图标。

新建组

和常常困扰我们的「该如何高效的组织文件夹」一样，**在 Ulysses 中我们该如何组织「组」**，以提高使用 Ulysses 的效率呢？

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/86A17D2A-8256-47C9-912C-B0B5204A82CC)

Ulysses 教程组

以在少数派写一份教程为例，总共会有不下十篇文稿，于是值得**为教程创建一个组**，而不是与少数派首页的文章混在一起。而这些文章会有教程正文，答疑汇总，为了写作还需要收集素材和灵感。于是我都**单独在组下创建了不同的组来分开放置这些文稿**。

总结我自己的使用经历，在 Ulysses 中规划组的经验有三条：

1. 一个需要**多篇文稿** （起码不低于三篇）的大型项目，可以创建一个组；
2. 根据**文稿使用的场合**，可以分类文稿，创建子组嵌套在上一层级中；
3. 素材等**非正式写作的内容**，可以单独但在一个组中，利于写作时及时查阅。

### 在 Ulysses 中使用外部文件夹

使用 iCloud 文库能让文稿顺畅地在 Mac 和 iOS 间同步，我们也不担心「文件保存在哪」、「iOS 能不能方便的打开」这种问题。绝大多数时候，iCloud 文库都是一种更省心的方案。

不过有时候要打开本地文件夹中的 Markdown 文稿，或者为了把文稿放在 Dropbox 上与同事共享，就需要使用外部文件夹功能。

首先要在「偏好设置 - 文库」中**勾选上「外部文件夹」**。然后在文库界面中，**通过「+」按钮添加文件夹**。

引入外部文件夹

例如这份 Ulysses 的教程文稿，我需要和编辑共享，方便他们直接在文稿上进行修改。于是乎便把文件夹作为外部文件夹添加进 Ulysses。外部文件夹可以和 iCloud 文库中的组一样，进行新建组等操作。

在 Ulysses 中编辑外部文件夹或者 iOS 上的 Dropbox 文件夹，大体上和编辑 iCloud 文库中的文稿无异。不过由于文件是单独保存在 Ulysses 之外的地方，两者还是有些细微的差别的。

最明显的差异就是**能用的标记变少**了。在外部文件夹中查看标记窗口，就会发现少了「标记为」、「删除」、「注解」、「评论」、「注释块」和「视频」，这六个标记都是 Markdown XL 独有的语法。在外部文件夹文稿中使用了这些标记的话，会无法正常预览它们的效果。

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/7B776857-B380-4AF2-B4A3-0B74330A00FB)

外部文件夹中可用的语法标记

即使是在外部文件夹中可用的图片标记，用法都和 iCloud 文库中有所不同。在 iCloud 中的文稿，可以直接拖拽图片进文章中，不同平台间 Ulysses 会自动同步。

而在外部文件夹中的文稿中，如果你想让 Ulysses 自动同步图片，就需要在组的设置中将文稿的格式设为 `TextBundle` 或 `TextPack`。如果继续使用 `md` 格式文本，只能用插入文件链接的方式插入图片，而图片是不会在 Mac 和 iOS 间同步的。

设置 TextBundle

在 Ulysses 中文稿不需要考虑文件名的事，就像 iTunes 中不需要考虑 mp3 文件的文件名。但在外部文件夹中，就必须要给文稿取文件名。默认情况下 Ulysses 会将一级标题作为文件名，也可以用 `@:` 标记来设定文件名。

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/80494384-7679-4EFD-9975-8490CD880AAB)

在文稿中设定文件名

Ulysses 的各项功能，都是基于 iCloud 文库而设计。使用外部文件夹，也会使写作目标等功能不可用。如果「将文稿保存到外部文件夹」不是刚需，我建议**最好是使用 Ulysses 的 iCloud 文库**，可以省去后期不少麻烦。

### 备份，保证万无一失

备份文稿的重要性毋需多言，无论是担心论文写到一半硬盘坏了，还是想找回几天前写好的上一版，备份文稿都是最好的解决方案。和同步一样，Ulysses 也将备份这个动作做到悄无声息，不需要我们使用额外的工具。

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/E43C6AC7-35A2-4355-9ED2-5DF9A2F70618)

偏好设置 - 备份

Ulysses 中的备份是完全自动后台进行的，不需要我们手放在「⌘ + S」键上随时准备保存。我们所要做就是在 Mac 上的「偏好设置 - 备份」中**打开备份**。一旦需要找回之前某个时间点的文稿，点击**浏览备份**，就可以在备份浏览器中**恢复文稿到收件箱中**。

在 Mac 上恢复文稿

值得注意的是，只有 iCloud 文库和「我的 Mac 上」文库会有备份功能，外部文件夹的备份就只能依赖「历史版本」功能。依靠 macOS 系统的「历史版本」，相比 Ulysses 自带的备份，为了节约存储空间，在备份次数上会少很多。

在 Mac 上恢复历史版本

调整到舒适的码字姿势
----------

对于一把厨师刀而言，坚硬的刀身和锋利的刀刃并不是一把好刀的全部，一把握持感极佳的刀柄也是好刀的一部分。而对于写作工具来说，写作时用到的功能固然重要，能让写作这个过程变得更加享受的特性也不可小视。

在正式写作前**将 Ulysses 调整到最让自己舒服的状态**，无疑是高效写作的重要一步。

### 如何在 Ulysses 中沉浸式的写作

古今中外的文豪都是从一张稿纸开始写作。纸张谈不上功能性，但确实能让作家不为他物所打扰。而相比 Word 这类富文本工具，Ulysses 没有一排排复杂的工具栏，已经非常接近纸笔的效果。再花时间调教一下，就能在 Ulysses 中达到**沉浸式写作**的效果。

#### 去掉一切干扰写作的元素

想要沉浸的体验，就要**让桌面上显示的元素尽可能的少**。

在菜单栏「视图 」中，能控制文库、文稿是否显示。也可以直接按快捷键 `⌘Command + 3`，选择「仅限编辑器」。如果嫌界面上方一条工具栏碍眼，在「视图」中取消勾选「总是显示工具栏」即可。工具栏就会像视频播放器中的进度条一样，只有光标移动到上方才会显示出来。

此时的 Ulysses 界面应该去掉了一切干扰写作的元素，但是 macOS 的菜单栏还明晃晃地显示在上方。在「视图」中选择「进入全屏幕」（快捷键 `⌃Control + ⌘Command + F`），即可进入全屏状态。此时的 Ulysses，已经和白纸无异。

全屏状态

#### 调整到合适的版面

用电脑输入文字时，编辑界面版面的重要性往往容易被人忽视。行与行，段落与段落之间隔的太密则不易阅读，太散则让眼睛焦点跳跃太多；行宽太宽需要头部来回摆动，太窄又使得换行过与频繁。所以调节编辑界面这一步必不可少。

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/2F5A86A9-A56A-4459-896B-B0BA59C960B4)

偏好设置 - 一般

在「偏好设置 - 一般」中就能调节编辑界面版式。在这里我很难告诉你，到底多少的行高和段落间距是正确的，需要各位根据自己的习惯调节。不过对于字号，根据我的经验建议，无论是在 Macbook 上还是外接显示器，**字号最好不要小于 14 号**，否则字会太小不易阅读。

而行宽我设定为 80，因为这基本与少数派网页上的行宽一致，可以帮助我判断文章放在网页上的显示效果，免得写出一段占了整整半个页面的豆腐块。

#### 使用打字机模式

不过版式调整的再合适，光标还是会随着输入逐渐滚动到屏幕下方，头部也跟着不断移动。长期写作后不仅容易让注意力分散，也会不利于颈部健康。

在张扬的教程《[久坐的你，要保证正确的坐姿](https://sspai.com/series/2)》中，对于视线和显示器的角度是这样建议的：

> 为了不让头部前引，显示区域的中心应该尽量和视线在一条直线上，让视线保持水平为最佳。

而使用 Ulysses 的**打字机模式**，就能让视线始终保持水平。

在「视图 - 打字机模式」中启用（快捷键 `⌥ Option + ⌘Command + T`），固定滚动的位置根据自己的屏幕位置来设置。因为我经常使用的是外接显示器，故而将固定滚动的位置设定为高，就可以一直让视线保持水平。同时还打开了标记当前行，让眼睛更容易注意到输入的位置。

打字机模式

经过一番调整，你就能在一块白纸一般的画面上，视线始终保持在合适的位置，把全部的注意力投入到写作中。

### 让写作充满色彩

虽然我们希望能在 Ulysses 中达到沉浸的写作体验，但若屏幕上只有白纸黑字，未免太索然无味。而且 Markdown 文本中标记字符和文字杂糅在一起，如果不加区分，势必让修改文稿变成一件费神的事情。

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/331EEE03-A458-4C4D-BB2E-028DDC42221F)

使用「文本编辑」打开 Markdown 文稿

不过你肯定已经发现，在 Ulysses 中，例如标题，带链接的文本，这些被标记的文字和相应的标记字符，都已经被不同颜色显示出来。**这样一套颜色的组合，被称为主题**。除了默认设置好的主题，打开「偏好设置 - 标记」，下半部分显示了更多内置的主题。

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/329F1BF0-74BD-4303-B29C-6A0C34259D36)

偏好设置 - 标记

对颜色的喜好可以说毫无标准可言，是完全主观的选择。值得庆幸的是，Ulysses 作为一款热门的写作工具，有众多第三方主题。在「标记」界面下方点击「获得更多主题」，能在官网专门的主题页面下载其它人上传的主题。

下载主题

这些主题中即有 [Minimal](https://styles.ulyssesapp.com/bundle/Minimal/5389933f081b0faa4d320f30) 这类极简风格的主题，也有 [Blue Screen](https://styles.ulyssesapp.com/bundle/Blue+Screen/5394488b6162874f02615e31) 这类模拟特殊效果的主题。我在使用的是 [Dracula](https://styles.ulyssesapp.com/bundle/Dracula/56319778b634a4410207d126) 这款主题。除了本身它的配色优雅，对标记字符的显示明显这些特点以外，它还是一套热门的代码编辑器配色。这意味着我在使用 Pycharm 或者 Clion 等代码编辑器时，也可以使用同样一款配色，省的打开不同软件时配色的各不相同。

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/22527ECF-C289-41C4-BADE-69DA291A240C)

Dracula 主题

一旦使用了配色主题，白色为底的屏幕中，各色符号难免显得有些出挑，从而使得注意力被五颜六成吸引走，此时就可以使用主题的「黑暗主题」（下载的主题会默认包含）。

点击「视图 - 黑暗主题」（快捷键 `⌥Option + ⌘Command + X`）打开，白色的文字在黑色背景下更易辨认，而且不会被彩色的符号夺走注意力。如果还嫌文库和文稿的界面的亮色，点击「视图 - 黑暗模式」快捷键 `⌥Option + ⌘Command + L`）就能让整个 Ulysses 浑然一体。

打开黑暗主题和黑暗模式

不过值得注意的是，有的主题（例如 Dracula）选择黑暗主题是没有变化的，因为它们本身就是暗色系的主题。

### 先给自己定下一个小目标

如果是要准备写报告论文类的文章，字数往往是刚性要求。即使是在少数派等网络媒体，也要注意控制篇幅。正式写作前的最后一小步，就来给自己的文章长度定下一个写作目标。

点击「窗口 - 附近」（快捷键 `⌘Command + 4`），就能找到写作目标工具（使用外部文件夹时这个功能不可用）。Ulysses 的写作目标工具可以通过不同的统计方式，设定期望的目标。

设置写作目标

对于中文文稿来说，最合适还是**通过「不计空格的字符数」**来统计。通过「字」来统计，实际上统计的是单词的数量。在写作过程中，除了通过写作目标环的颜色，直接打开统计窗口（快捷键 `⌘Command + 7`）查看写作进度来的更为方便。

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/71FC7C67-EFE8-424C-8C98-E22BEC223F45)

统计窗口

最后一步，导出并分享你的文章
--------------

历经九九八十一难，你只剩最后一步导出文章这一步了。

在写作时的编辑窗口中，文稿一直以 Markdown XL 的样貌示人，而在保存和导出时，可以**灵活地选择要保存的格式，以及文章的样式**。

### 选择导出时的格式

在「快速导出」窗口中，可以选择导出文稿的格式。在 Ulysses 中，可以将文稿导出为五种格式：

1. 文本
2. HTML
3. PDF
4. ePub
5. DOCX

其中文本又可以细分为纯文本（txt 格式）、富文本（rtfd 格式）、Markdown 和 TextBundle 这四种格式。其中导出为纯文本时，文章内的图片、视频内容都不会被导出。而富文本格式能直接在文稿中显示图片。选择为 Markdown 格式时，如果有图片，就会将 Markdown 文稿和图片放在文件夹中一齐导出。

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/4EA7E2CC-26D9-43C4-BAEB-8F8DAF6DCC34)

支持 TextBundle 的应用

TextBundle 本质上也是图片和文稿合成的文件夹，不过它能被支持的编辑器解析，使得它既能保持文本文件的通用性，又能方便地插入图片等内容，导出时也能以单个文件的形式去分享。

而选中导出 HTML 时的选项「整页」和「片断」，其实 HTML 中的概念（对 HTML 不熟悉的读者可以先跳过这一段）。选择为「整页」时，导出的 HTML 文件中有完整的 `head` 和 `body` 部分，CSS 文件也会一并导出；而选择为「片断」时，导出的 HTML 文件中只会包含文稿本身的内容，也没有相应的 CSS 文件。

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/7EB31F0E-5274-4255-9548-303795DDAB80)

导出为 ePub 时的选项

选择 ePub 格式时，就可以直接把文稿变成一本电子书。在窗口中还可以设置电子书的封面、标题和作者。

至于 PDF 想必大家都很熟悉，除了可以设定样式外，还可以设定纸张的大小，方便打印。而如果你之后想在 Word 中继续编辑， Ulysses 可以直接将文稿保存为 Word 的 DOCX 格式。

### 选择文稿的样式

在 Word 中当我们编辑完一篇文稿时，文稿的样式就固定了下来。也就是说，经过了编辑的文章，在保存导出时就不能再随意修改了。

在 Ulysses 中完成写作后，在「快速导出」窗口中，除了能选择要保存的格式。更关键的是，你可以在此时再决定你的文章样式，还可以**不断地再几种样式间来回切换**，看看哪种样式最合适。值得注意的是，当你选择导出格式是文本时，就不能带有样式的预览了。

切换样式

和标记主题一样，导出时的文章样式也是越多越好，才能让众多用户都找到自己最中意的。而想实现这个目标，光靠几套内置样式是满足不了的。点击「样式 - 获得更多样式」，就能像主题一样去 Ulysses 官网下载第三方样式。

下载使用第三方样式

值得一提的是，不同文本格式间下载的样式并不通用。先在页面右上方选择文本格式，再选择样式。PDF 和 DOCX 的样式可以通用。

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/5C28B6E0-559A-49C2-8AA9-32E963880E71)

根据格式选择样式

### 选择文稿的去处

在 Word 中，文档导出后保存为文件，它的使命就结束了。但在 Ulysses 中，文稿的去处不止一处。

![](.evernote-content/2A983729-48BF-499E-9737-2F5B327DF39B/FB7BDFAE-C65B-475E-9786-620803726C04)

选择去处

在「快速导出」窗口中上方右侧有五个图标，意味着有五种操作文稿的方式：

* 预览
* 拷贝到剪贴板
* 存储到
* 打开方式
* 共享

「预览」和「存储到」非常好理解。其实在快速导出窗口的下侧，即可以选择预览，以及直接拖动文件图标来保存文稿。

直接拖动保存

而「拷贝到」则有些复杂。它不是拷贝编辑器窗口中的 Markdown 文稿，而是**根据选定的文稿格式**，**拷贝相应的格式内容**。例如选择 HTML 格式，拷贝的就是 HTML 文本；选择 DOCX 格式，拷贝的就是带样式的富文本。而 ePub 格式因为是电子书格式，所以没有「拷贝到」选项。

「打开方式」同样是根据选定的文稿格式，选择其它软件来打开文稿。例如 Markdown 格式可以用 Marked 打开，而 DOCX 格式可以用 Word 来打开。最后，「共享」就是 macOS 内置的系统级分享功能。

小结
--

Ulysses 复杂吗？

单个功能来看不复杂。例如同步、备份等在其它工具中令人头疼的事情，Ulysses 都帮你完成了。但它也绝不简单，只不过它的不简单，是因为写作这件事本身就不是你想的那样，提起笔写字就行。

我们花了一整章的篇幅，来剖析使用 Ulysses 的基本流程。先来了解 Ulysses 的全貌，搞清使用它的脉络，之后的章节深入到进阶技巧时，我们才不至于迷路。

等等，说是「使用 Ulysses 的基本流程」，但却偏偏对写作这一步一闪而过？当然不是因为在这一步中 Ulysses 无能为力，而是有太多精彩需要细细品味。接下来的一章，就看如何用 Ulysses 高效地写作。
