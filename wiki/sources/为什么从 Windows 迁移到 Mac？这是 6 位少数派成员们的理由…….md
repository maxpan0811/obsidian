选 Windows 还是 Mac？这个问题从来没有标准答案，却一直被许多人乐此不彼地提起、讨论。随着节日季来临及新款 MacBook Pro 的发布，「该不该选 Mac」「选哪款 Mac」等问题又渐渐有了一些声量。

这不是一篇关于 Mac 的选购建议与指导内容，而是来自少数派编辑部的大家从 Windows 迁移至 Mac 的真实经历，和你分享我们最后选择了 Mac 的原因乃至思考，希望能为你提供一些参考和帮助。当然，选 Windows 还是 Mac 本就不是一个「非此即彼」的绝对选项，只要适合自己，就是对的选择；如果你有不同的经历、不同的观点，也欢迎在评论区与我们一起讨论。

▍鲸鱼鱼：从「被迫选用」到「习惯依赖」
-------------------

**在用设备：MacBook Air（M3）**

我换用 Mac 其实是个意外。此前因为工作中 ERP 软件（公司用的进销存系统）的平台支持限制，我一直需要主力使用 Windows 工作，但去年初 Windows 笔记本突然持续重启，没办法进入使用状态，在我犹豫维修还是另购电脑的期间，发现 ERP 软件提供了网页端，于是我干脆绕过维修和另购，直接领用公司的 MacBook Air M3，节省一笔支出。

刚迁移时因为习惯的工作流被打破，我确实不适应了一段时间。除开最重要的 ERP 软件，工作中原本还有几个仅支持 Windows 的软件如 Quicker 这类效率工具也得寻找、熟悉替代品，所以期间我好几次向同事表达了对 Windows 的怀念之情。不过这种不适其实只持续了几周，重新建立好工作流之后，我对 Mac 的感受主要可以落在两个词：舒适、稳定。

舒适主要是因为 Mac 较低的使用门槛以及良好的生态协同。合盖休眠以及开盖即用这件事上，Mac 的整体体验要更优，而 Windows 电脑因品牌、型号繁多，虽然也有能追赶甚至超越 Mac 开合体验的设备，但对于用户来说还存在一道挑选、设置的门槛，相比之下 Mac 的到手即用就便利得多，开会时需要带着电脑离开工位，合盖、拔线就能带走，开盖几乎是秒醒，后台软件基本不受影响，而之前我的中端 Windows 笔记本在这种场景下就需要保持开盖状态端着走，否则开盖后需要一点时间恢复设备状态。

生态协同上，对我而言最有用的是通知同步和 iPhone 镜像。iPhone 上的网店软件如果收到店铺通知，在 Mac 上也会迅速弹出通知，工作状态下无需分出太多心思解锁手机查看；iPhone 镜像则很适合用于偶尔摸鱼，电脑屏幕上镜像 iPhone 看看小说、挂个游戏，老板还以为你在盯着电脑认真工作呢，当然，实际用得更多的还是点开 iPhone 同步到 Mac 的通知，查看店铺信息、邮件等内容，节省拿起手机或是在电脑上单独登录这些软件的时间。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/C7444F89-0160-45C9-BA0E-C5D9D5C2580E.jpg)![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/047D05EF-0EFD-4F80-9538-7C2F5024B96D.jpg)

稳定感则主要源于 M 芯片、统一内存等硬件与软件的结合。M 系列芯片在性能和能效上的表现足够优秀，在 Mac 上使用 Photoshop 等中大型软件很少出现卡顿的问题，工作中开多标签浏览器、ERP 软件、即时通讯、效率工具这类典型场景，也能保持优秀的性能；较长时间外出例如到仓库盘点库存，Mac 的长续航表现也足以让我不再担忧中途需要为电脑补充电源，能够轻装上阵。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/04DA91F7-D61A-4547-A9F6-B122A4E72BF1.jpg)![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/04DA91F7-D61A-4547-A9F6-B122A4E72BF1.jpg)

唯一的遗憾是，Mac 在游戏上的兼容性对比 Windows 还是弱了些，哪怕有一些移植和云游戏的方案，目前的体验仍然难与 Windows 同级，需要一段发展时间。不过，如果看中电脑续航以及系统体验，没有专业软件限制，我认为 Mac 是一个足够舒服的选择，现在回头看，Windows 电脑故障只是个小意外，却把我从「被迫选用」带到了「习惯依赖」，以后再让我选工作电脑，至少我不会再本能地只想起 Windows。

▍PlatyHsu：对于细节和品味的关注，本身就更值得追求

**在用设备：MacBook Air (M2, 2022)，MacBook Pro (M4 Pro, 2024)**

当我还在用 Windows 的时候，每次新装系统，最先安装的软件之一就是 MacType。我最早是从 Windows 美化论坛上知道这个软件的，当时作为一个小学生（字面意义上的那种），还不知道背后的原理，只知道别人晒出来的截图上，汉字一个个都显得饱满优美，跟我自己电脑上那种犬牙差互、瘦骨嶙峋的样子完全不同。后来才知道，MacType 的原理就是劫持 Windows 的字体渲染引擎，更换为类似于 macOS（当时还叫 Mac OS X）的渲染机制。这奠定了我对 macOS 最初的好印象，也是后来吸引我更换到并一直使用 Mac 的主要原因之一。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/FA7FD8AC-BA0F-4217-8FA6-5A962B8C91BA.jpg)

相同字体在低分辨率环境 Windows（上）和 Mac（下）的显示效果差异，图片改编自腾讯 IXUS

在当时那个屏幕分辨率普遍不高，WSXGA+ (1680×1050) 和 WXGA+ (1440×900) 已经算高分屏的年代，Windows 和 macOS 字体渲染的质量差异特别明显。原因在于，两者采用了完全不同的思路来处理「如何用有限的像素最大程度地表现出文字」这个问题。Windows 的字体渲染大量使用渲染提示（hinting），简化、变形文字的笔画，从而降低对像素数量的要求，只求「神似」。而 macOS 采用了亚像素渲染（sub-pixel rendering），在笔画的周围加上大量浓淡不齐的辅助像素。借助人眼的错觉，这些像素在远看时相互重叠，文字的轮廓就像化学反应一样显现出来了。

应当承认，字体渲染是否美观是一个主观问题，虽然包括我在内的许多人非常喜欢 macOS 那种饱满浑厚的效果，我也知道 Windows 那种「一板一眼」的渲染方式其实不失拥趸；macOS 的渲染效果在这个群体看来反而太「糊」。同时，随着「视网膜」级别的高分屏不断普及，操作系统有了更充裕的像素数来渲染字体，亚像素渲染已经不再必要，Windows 和 macOS 的效果差异也随之变小了。

但即使这样，对字体有要求的人仍然有理由更偏好 macOS。首先，即使在现代的高分屏上，macOS 的 Quartz 仍然比 Windows 的 DirectWrite 更注重抗锯齿和保留文字轮廓的原始粗细，因此相同的字体在相同大小和分辨率下，仍然看起来更饱满。

更重要的是，由于微软至今没有更新传闻多年的微软雅黑新版（Noble Scarlet），并且旧版 Windows API 构建的界面仍然使用老旧的中易宋体作为默认回退字体，中文 Windows 用户至今还在继续忍受这些为低分屏设计字体的拙劣效果，以及中西文混排时无衬线和衬线体打架、中文字体显示西文字符等等丑态。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/51E7F5D8-7F7C-414B-890F-05448BBDDFFA.jpg)

Windows（上）在使用英文软件时，经常遇到字体回退问题，不能调用正确的中文字体显示

相比之下，macOS 的中文系统字体苹方在设计上虽然不能说尽善尽美，经过持续多年的更新也成为了一个字重、字形支持全面的成熟字体家族，并且与西文部分的 San Francisco 能和谐搭配。对于中文用户来说，观感上优于 Windows 是没有疑问的。

最后，在可以选用的字体方面，你或许没有注意到，macOS 中内置的字体也是在不断增加的，其中不乏质量很高的选项；这些字体根据 macOS 的许可协议，通常都可以用于平面设计等商业内容创作。又因为 macOS 传统上与设计行业的紧密联系，它在字体管理和字体功能的支持上也具有优势。如果你有自己安装字体的需求，选用 macOS 可以获得更好的兼容性和更顺畅的体验。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/0031CE0E-77ED-43C5-83AD-23C51AAD584E.jpg)

macOS 在字体管理工具和内置字体质量方面具有优势

诚然，字体的显示和管理只是操作系统中一个非常小的方面，作为更换操作系统的理由或许显得有些琐碎。然而，屏幕上的文字几乎构成了我们数字生活的全部肌理。无论是阅读文档、编写代码，还是浏览网页，本质上都是在与文字打交道；积少成多，体验上的差异就很可观了。从一些更形而上的角度说，对于细节和品味的关注，本身就是更值得追求的。

▍张奕源 Nick：为应用生态而转向
------------------

**在用设备：MacBook Pro 16 英寸（M3 Max），Windows PC（AMD Ryzen 7 9700X + NVIDIA RTX5080）**

辱骂 Windows 不行已成陈词滥调，但情况就是这么个情况——只要你是个正经用过 Mac 和 Windows 的正经人，肯定很难得出「Windows 是更好用的那一个」的结论。即便长期单用 Windows，你都不难发现这个操作系统里无处不在的魔幻和吊诡。

我一直同时在使用这两个生态的设备，而且硬体配置都保持着交替更新，并不老旧，所以对于它们分别的优劣之处都算得上熟悉，也早就过了对 Windows 满腹怨言的阶段。更何况，我偏向用 PC 打游戏而非主机，既然「有求」于人家，那人家隔三差五闹点脾气、出个小 bug 之类也属理所当然。

——心静自然凉嘛，善哉善哉。

严格来讲，我并未完成严格意义上的「从 Mac 迁移到 Windows」，但差不多就在过去这一年半载里，我已经在办公层面弃用了 Windows，完全转向 Mac。弃用的理由很简单，其实就是 Windows 上没什么好 app 可以用——在我的使用经验里，只要是 Windows 和 Mac 上都有的工具类型，就没有哪款是 Windows 版本更好用的。

哦对，IDM 算是个例外。

这里有个背景：在我漫长的「温麦双修」时期，我一直尽可能坚持一些产品原则，譬如尽量使用跨平台工具，尽量使用开源工具，尽量保持重要文件可以在双系统之间同步等。但随着 Windows 的逐渐拉垮和 Mac 的愈发完善，这种坚持也显得越来越没有意义。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/83987698-1B26-4298-AF3D-5FD9F17C4D88.jpg)![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/59BB36BB-A91D-4C67-BF6B-F7600BF5F85D.jpg)

因为要同时兼顾 Mac 和 Windows，所以我的桌面一直是个有多套键鼠的混乱空间

我心有不甘，去问大师。大师抱出一只鸡，在它脚上缠了条绳子。鸡每走几步，大师就拉住绳子，鸡就摔个跟头。鸡又站起来向前走，大师又拉绳，鸡又摔跟头。

如是往复了八次。

我大悟：师父，我懂了，你是让我像这只鸡一样百折不挠，就算 Windows 不行了也要保持兼顾对不对？

大师冷笑：我是让你拉鸡八倒吧。

所以，现在的我在工具层面已经走在了「彻底果化」的路上，譬如：

* 主要文档工具从《Notion》逐渐搬迁至《Craft》
* 草稿工具使用 Apple 独占的《Drafts》
* 主要邮件客户端使用 Apple 系统自带的《Mail》
* 使用 Apple 第一方的《Journal》记录工作日志
* RSS 订阅工具从《Inoreader》迁移至 Apple 独占的《Reeder》，顺带还用来订阅我新关注的 YouTuber
* 使用 Apple 生态友好的《Letterboxd》记录影剧片单
* 主力移动硬盘直接格式化为 APFS
* 开始使用 Sign in with Apple

另外，还有一些是 Apple 一直做得很好、我也一直在用的服务，如：

* 有很多好剧的《Apple TV》
* 我不依赖任何推荐算法、主要靠自己找歌听，所以觉得还不错的《Apple Music》
* 聆听经典乐体验舒畅的《Classical》
* 有了它就根本不需要再找第三方健身教程的《Fitness+》
* 我之前已经讲过使用方法的天下第一白板工具《Freeform》
* 定位和找路都蛮准，且没有傻缺开屏广告的《Maps》

除此之外，还有一些工具（譬如《Todoist》《1Password》等）并非 Apple 生态独占，但我也在用。使用它们的原因纯粹是因为它们好用且刚好做了 Windows 版本，并不是为了专门照顾跨平台的需求。

目前在干活方面让我唯一不能摆脱 Windows 的就是我那张 RTX5080 显卡。因为我剪影片时喜欢外挂胶片模拟插件来调色，这些东西很吃显存和 GPU 性能，我手里的设备中只有 RTX5080 能扛得住渲染量且不出错，我也因此还得把它当成重要的剪辑机来用。

从工作流的角度来说，用 Windows PC 其实并不方便，因为我有大量的素材都是用 iPhone 拍的，互传文件和编解码都是 Mac 更简单。但 5080 在手，不用也是浪费，唯有继续好好剪片，希望一两年后我的水平能配得上一部 128GB 内存的 Mac 吧。

总体来说，我对 Windows 这个平台最根本的不满意不在于某个具体的介面或者功能做得不如 Mac 好，而是我在微软身上看不到这个系统有洗心革面、大幅改良的希望和迹象。作为一个年过三十旬的老系统，陈年老代码堆积如山、尾大不掉可以理解，但隔壁 macOS 的岁数也没小到哪去，功能更新却一个接一个，市占率也越来越高，足见无论历史包袱有多重，只要决策者愿意变革，全都不是问题。

如果倒退个十年八载，我肯定还有各种打补丁、改注册表、到处找工具堵窟窿的折腾劲。但如今的我只求这部 PC 可以少生事端，千万别过年回了趟家就又开不了机了。

阿弥陀佛。

▍Tp：为了「生态融合」体验的坚定选择

**在用设备：MacBook Air（M1）**

与其说我是因为受不了 Windows 的什么缺点才迁移到 Mac，倒不如说我其实从最开始就觉得 Mac 更适合自己。在上大学选购电脑时，我就考虑过是否要直接一步到位选择 MacBook，但综合考虑到防盗、交作业以及游戏需求，最终选择了 Windows 作为大学期间的电脑。但大一时我还是折腾着给电脑装上了黑苹果，在双系统的情况下用了下去。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/74B763D5-221F-4B19-B481-B446DD668529.jpg)

大学时期的照片

我之所以觉得 Mac 更适合自己，主要还是因为 macOS 的软件生态是我喜欢的那种「优雅且省事」的类型。于是在工作后，我便立刻入手了这台 MacBook Air（M1），过了将近五年的时间，它仍然能在最新的 macOS 26 下流畅使用。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/3EAD828C-5177-4522-87B5-CE6A4966FEE2.jpg)

我大学读的是汉语言文学专业，不像理工科专业那样对特定软件有需求，但 Windows 上始终没有一个能让我满意的 Markdown 写作软件。就算是许多人赞不绝口的 Typora，在当时给我的感受也并不算好用。而那时在 macOS 上就已经有 iA Writer、Ulysses 这样设计简洁且带有「聚焦模式」的写作软件，真的非常吸引我。以至于大学期间我绝大部分的写作都是通过 iPad 上的 Ulysses 完成的，Windows 电脑则更多承担的是 Steam 和战网的启动器。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/E5642E15-BDE6-4797-8EEB-D91703397471.jpg)

摄于 2019 年

即便抛开写作软件的需求，Windows 和 macOS 给我的感觉就是一个需要自己装修的毛坯房和一套精装修。你可以把 Windows 用得十分高效优雅，但前提是你要花时间折腾。

比方说，我想将 iPad 上用 GoodNotes 写的笔记放到电脑上看，在 Mac 上我可以直接通过 AirDrop 投送给电脑，甚至可以直接打开电脑上装的 GoodNotes 应用同步过来。但在 Windows 上，如果想打通电脑和手机、iPad 的文件壁垒，要么就得借助第三方软件（比如微信的文件传输助手），要么就得像我一样 [折腾局域网共享](https://mp.weixin.qq.com/s?__biz=MzU4Mjg3MDAyMQ==&mid=2247493508&idx=2&sn=43f8054c87606f4e433c0a20d31c0ba6&scene=21#wechat_redirect)。

这种生态间的完美融合，在近几年的 macOS 中更是得到了进一步加强，这里我想通过 2 个自己常用的功能来分享这种惊喜感。

首先是「iPhone 镜像」。

如果你开启了 Mac 上的「iPhone 镜像」，那么你手机上的推送通知都会被同步到电脑上，点击通知可以直接打开「iPhone 镜像」操作手机。但真正惊艳到我的，是新系统中它甚至还能同步 iPhone 的实时活动。尤其是在点了外卖后，Mac 的状态栏便会有出现实时活动的小黑条，像灵动岛一样展示预计送达时间，点击便会展开为实时活动的卡片。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/A862FCCD-5C66-4CA4-B098-85D213F5309E.jpg)

不仅如此，就连桌面小组件也能直接调用 iPhone 上安装的 App。「iPhone 镜像」给我的感受并不只是简单的远程控制，而更像是为 Mac 与 iPhone 消弭壁垒的粘合剂，让人真正体会到「生态融合」的畅快感。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/98D2E8CA-B4AC-4BE9-87DC-1279DD7CED42.jpg)

比如在电脑桌面摆上 GR 拍的照片 🌝

其次是 Apple Music。

这里说的倒不是 Apple Music 这个订阅服务，而是 Apple Music 的 iCloud 资料库。我平时除了会用 Apple Music 听歌以外，还会创作一些自娱自乐性质的音乐作品，其中既有通过 AI 生成的「快餐」，也有用 Logic Pro 花时间雕琢的 Demo 与成品。而我会将这些作品统统导入 Apple Music 的资料库，不仅可以在 iPhone 上随时收听，甚至还能直接通过「音乐」应用导入到 iPod。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/A3E97EB2-5C94-48C9-B031-832902A0F4B5.jpg)

尤其是在 Logic Pro 里完成一首歌后，还能直接选择共享到「音乐」，后续只需要简单编辑一下歌曲信息，就能做出一张像模像样的专辑了。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/0D2FD577-4612-4757-BFC2-75AD1FDC0350.jpg)

我从入手 Mac 那天开始就一直不吝对 Apple silicon 的赞美。即便用了将近 5 年的时间，在 macOS 26 下它也依然能提供足够流畅的体验，即便是剪一些简单的视频，或是用 Logic Pro 做音乐也不在话下。说实话，在迁移到 Mac 后我并没有如预期般思念过 Windows，尽管 Windows 11 在初登场时凭借 UI 设计让我感到有些惊艳，但在短暂使用过后，我还是坚定地认为自己会一直选择 Mac。

尽管如此，我也并不会向所有人推荐 Mac，尤其是在 Windows 占主导的环境下，macOS 很可能在某些特定场景下无法满足你的需求。对于适合使用 Mac 的人来说，大家一定会在某个时刻突然意识到「啊！我需要一台 Mac」，这时候你所需要烦恼的，就只剩下加到多高的配置了。

▍Lincoln：未必适合人人，但一定值得考虑
-----------------------

**在用设备：MacBook Pro 16 英寸（M3 Max）**

我在用电脑这方面其实很「骑墙」，哪一个阵营好就选哪一个阵营的电脑。所以在 2017 年见识到 MacBook Pro 系列单热管压 Intel i7 的操作以后，毫不犹豫跳到了隔壁的 Windows 笔记本阵营。在当时看来，比起时不时撞温度墙的 Mac，同样是风扇呼呼猛转，Windows 笔记本至少能让我获得相应的价格匹配的性能。

再加上当年全新 USB-C 模具带来的不少兼容性问题，以及 Touch Bar 的市场反馈不佳，因此即使放到今天回看，我认为当时选择避开 MacBook 系列依然是一个理性的决定。

但在 M 系列的 MacBook 发布以后，才发现原来笔记本在正常浏览网页中风扇并不会发出恼人的噪声，出门之前也不需要反复检查笔记本到底是不是进入睡眠状态，加上 Windows 各种令我抓狂的问题。所以，在反复对比、更多的接口、以及未来的使用需求以后，最终决定换新到了 MacBook Pro 16 英寸（M3 Max）。

在正式上手以后，我对 M 系列的 MacBook 感受就是「可以灵活伸缩的性能」。具体而言，日常使用时设备安静且响应迅速；当需要高性能输出时，也能及时释放算力，表现干脆利落。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/F9228998-824B-4004-B28B-CC5925F305A1.jpg)

日常使用性能占用和整机功耗

例如，在日常浏览网页、准备选题或修改稿件等看似轻度的工作中，实际上对处理器的单线程性能要求并不低。自发布以来，M 系列芯片在单线程性能上的表现一直较为稳定，这使得这类单线程操作能够迅速完成；而在任务结束后，处理器又可以快速回落至较低的运行频率，在保证响应速度的同时降低功耗，显著延长了续航也减少了发热。

当然大小核的架构和 macOS 的系统优化在这方面也出力不少，大家可以 点击这里 了解 M 芯片的调度原理。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/01BDF425-8126-44F3-919F-F24822EE24D5.jpg)

Lightroom 批量导出性能情况和整机功耗

至于到了需要性能的地方，比如在 Lightroom 里处理导出照片，或者是在本地运行 Whisper 或者 LLM 模型的时候，M3 Max 确实又能榨出足够的性能，凭借着统一内存的设计和独立的 NPU 加速单元，在 Whisper V3 模型转译文字的速度，甚至稍快于在用的 7900X 台式机，但是台式机耗电量可比 MacBook 大得多。

所以硬件的角度来看，买一台「可以灵活伸缩的性能」的 M 系列 MacBook 几乎覆盖了大多数使用电脑的场景。

正如上面 Nick 说的那样「只要是 Windows 和 Mac 上都有的工具类型，就没有哪款是 Windows 版本更好用的」。如果只用 Windows，那么捏着鼻子找点替代方案也就将就着用了。但是随着回到 Mac，Windows 上的软件生态让我觉得哪哪都不顺手。

比如备份，macOS 上非常成熟的 Time Machine 已经存在 N 年了，付费软件上也有很不错 Carbon Copy Cloner 作为 Time Machine 的升级方案。而 Windows 内置的文件历史管理器很久没更新，更别说功能也不全；而为数不多的能克隆 NTFS 格式硬盘的 Windows 7 备份与还原，微软也表示过即将退役的情况。至于第三方的 Windows 备份工具 Acronis 感觉也志不在个人用户上，更别说 Essentials 纯本地备份服务也是按年收费的了。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/51F5E8C8-BA44-45CA-8C02-7592B556E96E.jpg)

Acronis 目前定价策略一览

至于迁移到 Mac 以后，我承认那么一点点的地方还是很怀念 Windows，特别是 Windows 的托盘机制。记得以前用 13 寸 Mac 的时候菜单栏上右边常驻的图标一多，就有很大的概率会和左边的菜单打架；更别说在 MacBook 系列有刘海以后，Mac 上的菜单实际的空间就更少的了。Apple 至今都没给菜单栏那样的托盘的机制，而要等第三方用 Hack 的手段实现类似的设计。如果你说 Mac 已经可以关闭菜单栏上的图标了，那偶尔要用到的时候怎么办呢？

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/E77AA17F-5C4A-45CE-B139-637438870F46.jpg)

我目前的完整的菜单栏（上）和 16 寸 MacBook Pro 可以显示的菜单栏区域（下）

最后，我知道不是每个人都适合 Mac。不过，随着 Apple Silicon 迁移计划完美收官，原生适配 ARM 架构的应用数量已经明显增加，整体软件生态也趋于成熟。在这种背景下，如果你正在考虑选购下一台电脑，不妨先拿着自己的工作流，在各类社区中了解一下当前 Apple silicon 的实际使用体验，再做考虑。

▍克莱德：缘于预算，落在体验

**在用设备：M4 Mac mini**

没错，我就是那个吃到国补红利、2400 元不到买到 M4 Mac mini 的人。所以「为什么换到 Mac」这个问题的答案，首先自然是因为价格——拿着这个预算我真的很难选到一款性能相近的 Windows 小主机了。

对一个以往办公、游戏场景二合一，什么事儿都在一台联想拯救者 2021 台式机上完成的 Windows 用户来说，Mac mini 可以说是一台有点颠覆性的设备，尤其当我渐渐意识到除了游戏场景之外的大部分使用场景都能交给它完成时，启动一台体积 34L、重负载下显卡高频啸叫的主机也就有了一点心理上的阻力：毕竟 Mac mini 体积小巧，放在老旧的升降桌上可以完全不用考虑承重问题；即便放在桌面，运行过程中也从来不会发出任何让人不悦的噪声和热量…… 过去这半年多时间里，我的 Windows 设备除了偶尔内容需求得开机检查下更新、对比更新内容，开机次数应该还停留在个位数（必须要说这里也有 PS5 Pro 的功劳）。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/951AB967-1803-497C-9578-2625F8FE95C9.jpg)

也就比我的 Pixel 9 Pro 大不了多少吧

具体到用法，对我来说这台 Mac mini 又算不上什么「造梦引擎」。

论工作我大部分时间都在和浏览器、飞书、Figma 打交道，这些我自认为算是轻度需求的任务对搭载 M4 处理器的设备来说肯定是不痛不痒。而考虑到它极低地待机功耗，我很快也开始将此前运行在 Windows 上的 Jellyfin 服务器、Cloudflared Tunnel、Home Assistant 等自托管服务陆续迁移过来，这一部分就像今年年初同事在 这篇文章 里提到的，macOS 对这类自托管服务的支持格外友好，一方面有 Homebrew、Orbstack 这类优秀的工具可以减少很多配置门槛，另一方面 macOS 本身也有「唤醒以供网络访问」「自动硬盘休眠」等适用于托管服务的系统设置，似乎从设计时就考虑到了类似的使用场景。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/124C82C0-E8AA-42FC-9765-1B96326C3DFF.jpg)

另外，切换到 Mac mini 之后我对「Apple 生态」也算是有了新的理解，比起 Android、Windows 那种数量和安装来源上的自由，macOS 和 iOS 这边虽然随便找个小工具都是 0.99 美元 / 月的订阅起步，开源软件世界里那种兼顾免费、好用甚至好看的选项少之又少，但胜在有头有脸的选项基本上也都能提供及格线以上的质量。

如果我想找一个方案解决一个问题，大多数时候我需要担心的并不是找不到、而是得花多少钱——所以像 OrbStack 这种只有 Apple 生态才有、并且个人使用完全免费的东西更是宛如珍宝。

Apple 生态和应用开发者生态是相互吸引、而不是 Google 那种单向奔赴的，这在我看来才是 Apple 最厉害的地方。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/586EEAA4-E1A6-4805-BDC2-AD99E2C57EF7.jpg)

当然了，后续主力机从 Pixel 9 Pro 切换到 iPhone 17 标准版之后，实时通知同步、iPhone 镜像、Apple Watch 解锁这些生态内硬件协同效果也让我非常满意，其他同事有介绍我就不过多展开了。

![](.evernote-content/33314CCC-74FF-4458-B170-16B2BDAFE3AE/6580D112-F9A3-4568-9D0A-B1F86D57ADAE.jpg)

Windows 都比你规矩

唯一需要吐槽的是 M4 Mac mini 在接口上过于激进，没有 Type A 接口意味着我没办法在不借助转接头的情况下直接使用现有的无线鼠标，另外 macOS 26 相比 iOS 26 在很多细节上都更显粗糙——比如最「臭名昭著」的圆角，把 Windows 跨越几个时代的窗口放在一起圆角花样都没 macOS 26 多，希望 Apple 后续的系统更新能勿忘初心好好打磨下细节吧哈哈。

最后，关于选 Windows 还是 Mac 的问题，如果你有不同的经历、不同的观点、不同的想法和思考，也欢迎在评论区与我们一起讨论。
