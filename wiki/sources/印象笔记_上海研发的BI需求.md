# 上海研发的BI需求

---

ML PUBLIC "-//W3C//DTD HTML 4.0 Transitional//EN">

**发件人:** shizhurong [<mailto:shizhurong@hytours.com>]  
 **发送时间:** 2015年7月8日 17:53  
**收件人:** 邹雨 (上海总经办); 林思辰 (上海研发)  
**抄送:** 黄俊(技术开发); 陈翔宇(上海研发)  
**主题:** BI需求收集-研发

 

Hi 邹&林，

非常感谢协助，根据我们沟通的内容，简单整理了下需求。

明确下来的需求会再实施阶段通知你，大概的上线时间是8月3号前。

如有新的需求或下列需求的更新，请及时联系我，非常感谢！

需求提出：研发部

需求方对接人：林思辰

需求概要：

=收客表（订单）相关=

1. 根据钻级，显示订单的分布情况。筛选条件为：分公司名称、出行月份、目的地区域。

2. 根据线路别名，显示订单的分布情况。筛选条件为：分公司名称、出行月份、目的地区域。

3. 根据客户，显示订单的分布情况。筛选条件为：分公司名称、出行月份、目的地区域。

4. 根据销售，显示订单的分布情况。筛选条件为：分公司名称、出行月份、目的地区域。

5. 根据价格区间（区间未定），显示订单的分布情况。筛选条件为：分公司名称、出行月份、目的地区域。

=航空公司相关=

1. 根据航空公司，显示产品的上客率

=预警功能=

1. 成团预警（根据送签时间）：送签截止时间前N天内，列出还未成行团队，每天凌晨以邮件方式推送（邮件接收人未定）。

2. 成团预警（根据库存）：在产品上线后到送签截止时间前20天的时间段内，列出库存用满的团队，每天凌晨以邮件方式推送（邮件接收人未定）。

3. 收客表同名预警（判断销售是否虚报）：列出不同的团存在同名客户的名单，列出客户名称、销售、团信息，每天凌晨以邮件方式推送（邮件接收人未定）。

=销售相关=

1. 根据线路名称，分析每个销售的贡献度（单个销售的销售总额/线路销售总额）

2. 根据订单，显示销售总实报、总切位、总虚报（取消率）

=线路相关=

1. 显示每条线路的成团率和总占比。

2. 显示每条线路的使用率（实报总数/（总机位+已取消机位））

=客户相关=

1. 客户感兴趣的线路：根据客户，显示每条路线的订单量占比。

2. 恶意客户判断（根据换手率-订单人均人数判断）：根据订单人均数，显示客户排名。

3. 客户贡献度排名：根据订单总人数/总价，显示客户排名。

展现形式：日报/周报/月报

查看方式：邮件推送

林仪

1.   客户切位的转化率应该有，显示（每个团、总共）切了多少，在没卖掉之前收回去多少，最终切位使用多少，在有切位团队中（每个团、总共）实际使用多少；

2.  对于各种分类产品应该增加报名时间监控；

3. 团队让价不应该光是文本输入（现在线路属性中录入、显示），然后更改价格，应该有完整的一套记录让价时间、让价幅度、让价后收客情况的东西，然后可以监控让价的效果；

4. 对于销售的改名率应该有监控；

5. 对于销售在出发前10天之内取消的（不管是销售自己取消还是操作踢的）应该有监控；
