从场景自动化到自然语言支持：更易使用的 Shortcuts | WWDC 特刊
=======================================

| 本文为付费栏目文章，您已订阅，可阅读全文 |

今年 WWDC 上好戏迭出，Shortcuts（中文名「快捷指令」，原名「捷径」）并不是主角，但是它的改进仍然让一批自动化玩家兴奋不已：

* **期待已久自动化运行**，可以根据甚至时间、位置甚至 Wi-Fi 状况来触发不同动作。
* **新设计的分享菜单**，让 Shortcuts 动作和系统操作排在了一起，点击起来更加方便。
* **第三方应用参数支持**，可以直接在 Siri 界面和 Shortcuts 动作「聊天」，完成一次细节完整的在线订餐。
* ……

上述都是肉眼可见的改变，此外在动作制作方式上 Shortcuts 也引入了一些更加 powerful 的特性。在迎接这些增强的同时，我们也要面对在操作方式和兼容性上可能带来的问题。总之，现在就开始探索这款 iOS 顶级自动化工具的新变化。

更简单丰富的使用方式
----------

对于普通用户而言，Shortcuts 此次最可见的变化就是增加和优化了各类触发方式 —— 包括对分享菜单和 Siri 增强以及千呼万唤使出来的「自动化运行」。在深入动作的制作细节之前，我们先从触发方式的变动来剖析 iOS 13 的新 Shortcuts。

### 期待已久的自动化

**自动化运行（Automation）**是此次 Shortcuts 应用升级的重头戏，它可以根据时间、地理位置和网络环境等等**场景**条件来触发动作。Shortcuts 应用内甚至新增了一个单独的自动化 Tab 来容纳用户自己创建的触发场景。

![](.evernote-content/3E81803F-6EB8-4B77-8F2C-68E4D28DF14A/4967E4C9-1A30-4BB6-A7D0-DE3781C05853.png)

基于各种场景的自动化

首先要指出，Shortcut 的自动运行和我们认识中的自动化还是有很本质的区别：它只是弹出一个**运行提示**，比如每天到办公室后提醒你打开静音模式，但是并没有权限自顾自执行一些动作。尽管如此，自动运行特性还是明显拓宽了 Shortcut 场景。

最显而易见的，自动运行让 Shortcut 摆脱了一批第三方应用的辅助。过去，不少玩家知道通过 Launch Center Pro 来定时、定位运行 Shortcut 动作，也有一些聪明的玩家会利用系统提醒事项来实现类似的触发（好处是不用花钱），比如：

* **每天睡前提醒开启飞行模式，早起后提醒关闭，避免手机影响睡眠。**这类动作虽说技术含量不高，但很多 Shortcut 玩家都在交流切磋，可见受欢迎程度之高。配合定时的开启提示后，实用性的确上升了很多。
* **到电影院后，提醒运行动作，一键开勿扰、打静音和设低亮度。**手动操作中前两项可以通过实体按键完成，所以我自己看电影时已经养成了习惯开静音、降亮度，但是勿扰模式总忘记开，这批简单但容易丢三落四的操作还是用一个动作一次搞定比较让人放心。
* **预先设置好时间，打开几个常用购物应用领取优惠。**遇到 618、双 11 等购物热点时期，这类动作就会流行起来。
* ……

这些用法延伸了 Shortcut 的用法，但留下了一些瑕疵。一方面，同一条工作流要涉及好几个分散的应用，管理起来不方便；另一方面，这些兜兜转转才实现的场景触发都仰赖 URL Scheme，哪天随手改了 Shortcut 动作的名字，所有带相关链接的地方都要跟着改。所以，相比原先的方案，**统一管理**就是新 Shortcut 自动化的第一个优势。

更进一步，由于 Shortcut 拥有系统级权限，它还获得了网络条件等等**独有触发方式**。在 Mac 上，我们早就可以根据当前所连接的 Wi-Fi 来运行自动化动作，比如通过 Keyboard Maestro 来自动切换「生活模式」和「工作模式」，连上不同 Wi-Fi 后开启不同设置、预备对应软件（订阅过 Power+ 1.0 的读者可参考 [这篇](https://sspai.com/article/42322?series_id=9)，只是拓展阅读，没订的朋友不用破费）。这种**模式切换**类功能在 iOS 设备上向来很孱弱，最后还是要 Shortcut 自己出手来解决问题。

![](.evernote-content/3E81803F-6EB8-4B77-8F2C-68E4D28DF14A/0D10D522-758D-4A80-AE24-A46C577FE387.png)

系统级触发条件

借助新的触发方式，我们可以实现一些过去想都不敢想的用法：

* **利用「Airplane Mode」**：开启飞行模式后提示顺手打个静音。现在乘飞机已经不需要关机，但闹铃或提醒事项响起来时会被空乘提醒，还是很让人尴尬，下次可以试试开飞行后顺带静音。
* **利用「Wi-Fi」**：到家连上自家无线网络后，自动提醒下载新的电视剧集。
* **利用「Open App」**：打开 Ulysses 时，自动提醒开启 Toggl 计时，避免忘记时间记录。
* ……

的确，Shortcut 现在能做的事情很多第三方应用以前就能做，独一无二的功能也受限于安全机制无法真正自动运行，但是 Shortcut 能够内置这些新触发方式，无疑降低了使用门槛。可以确信的是，如果一个新手发现要购买 Launch Center Pro 才能定时运行（其实只是弹出提醒通知）Shortcut 动作，他多半就会放弃继续尝试。对老玩家来说也许觉得 Shortcut 还是差一口气，但新 Shortcut 确实帮大家扫清了很多障碍。

### 更易点击的分享菜单

iOS 13 重新设计了分享菜单（Share Sheet），最大的受益者估计就是 Shortcut。在新系统的分享菜单中，Shortcut 动作的优先级被大大提高，可以和「拷贝」「打印」等系统自带操作摆在一起，常用操作已经不需要经过「Run Shortcut」这一关。

![](.evernote-content/3E81803F-6EB8-4B77-8F2C-68E4D28DF14A/9B4A0273-B6BD-4C3B-8BAC-79D86612D774.png)

优先级更高的 Shortcut 动作

但是和多数人的交口称赞不同，我认为现阶段的分享菜单设计**给 Shortcut 动作带来的麻烦反倒变多**，仍然需要更多的时间去改进。这回分享菜单的改动显然在向 macOS 学习，但是 Shortcut 没有把优点吸收到位。

首先是动作的**排布方式**，Shortcut 在动作变多后会把分享菜单变成灾难现场。用过 Automator 的读者应该还记得，新安装的 Automator 服务（Services）都会出现在右键菜单中，方便直接点击，如果动作装得多了，它们会被统统收纳到一个「Services」菜单选项中，避免右键菜单溢出。相比之下，Shortcut 做得还不够好的就是**无法折叠**，一些玩家可能有几十上百个 Shortcut 动作，很不幸，它们会全部出现在分享菜单中，找起来其实比原先还麻烦，而旧版 Shortcut 分享菜单和应用内布局基本一致，可以凭印象去点。

![](.evernote-content/3E81803F-6EB8-4B77-8F2C-68E4D28DF14A/CD8AF191-FDEF-4BA8-8E3D-792205BBE9AB.jpg)

iOS 12 分享菜单中的动作布局和 Shortcuts 应用内部一致

其次是**点选方式**上，Shortcut 还**不能通过键盘快速定位动作**。macOS 上的动作（以及几乎所有的菜单栏选项）都可以通过键盘来迅速定位，例如按一下 `i` 键，就会在一堆动作里选中第一个 `i` 开头的动作（出现首字母同名时就继续按下一位字母，以此类推），如果我想运行 `image 转 PDF` 动作，只需不假思索地按下前几个字母。不过这好事儿暂时没轮到 iOS。

![](.evernote-content/3E81803F-6EB8-4B77-8F2C-68E4D28DF14A/705FFFC4-318C-465B-B96A-C095AE8009E9.gif)

按首字母快速定位动作

Shortcut 动作一旦数量过多 —— 比如说超过两屏幕 —— 就是纯粹考验眼力的时候了。抛开很少接键盘使用的 iPhone 且不谈，iPad 也不能定位 Shortcut 动作就有点可惜，也许未来可以通过辅助功能的全键盘操作给动作绑定快捷键，但还是不如 macOS 那套定位方式来得泛用。

### 在 Spotlight 中使用「Type to Siri」

最后一个比较隐蔽的新触发方式是 **Spotlight 中的 Siri** 。不少人对于 Siri 应该是又爱又恨，一部分原因大概是不想动嘴（不管是因为觉得尴尬还是发音不准）—— 我自己也是「君子动手不动口」，还写过一篇关于 [打字和 Siri 交互](https://sspai.com/post/44790) 的文章。

到了 iOS 13，我们已经不用为了打字而砍掉原有的语音交互 —— 现在，直接在 Spotlight 中就能运行 Siri 指令。Siri Shortcut 当然也一起沾了光，得以**在 Spotlight 中打字运行**。

![](.evernote-content/3E81803F-6EB8-4B77-8F2C-68E4D28DF14A/0EA62594-4589-47B3-A53F-B6048B7B15C6.png)

集成在 Spotlight 的 Siri

比如你想 [看一下最近几天的油价](https://www.icloud.com/shortcuts/9387b5155caa4bfe8353c339ff5082f2)，不需要张嘴，拽出 Spotlight 或按下 `⌘Command-空格`，输入 Shortcut 动作名后跑一下就完成了（查油价的动作来自 @Zuchen Liu）。

![](.evernote-content/3E81803F-6EB8-4B77-8F2C-68E4D28DF14A/250FBA7B-658F-45C0-ADA5-7DA1A1F8BD7C.jpg)

查询油价

有一类 Shortcut 动作天生带着「瞥一眼」的特性，它们不需要复杂的交互，只是临时展示一些精要信息。新的 Spotlight&Siri 触发方式方法天然就是为这些动作准备的，除了刚才提到的油价查询，还有很多：

* 一览近期热门电影评分（[获取](https://www.icloud.com/shortcuts/6442bc60b8204604b9fbcdf59053f6ff)，作者 @Dadong）
* 查看最近 NBA 赛况（[获取](https://www.icloud.com/shortcuts/67deb8faf7284ddbb49bdda7355dd66c)，作者 @Jack）
* 检查心率报告（[获取](https://www.icloud.com/shortcuts/fc59a35a88e342968d26f6155f902ec1)，作者 @RogerLin）
* ……

当然，Spotlight 搜索框本身就可以搜到 Shortcut 动作，但是每次都需要你手动确认运行，并且运行结果要跳到 Shortcuts 主应用中去呈现。如果你只想看点轻量的信息 —— 比如扫一眼当天的工作安排（参考文刀的 [这篇文章](https://sspai.com/post/53288)），或者搜一下汇率 —— 这种蹦来跳去使用方式就不怎么讨人喜欢。

![](.evernote-content/3E81803F-6EB8-4B77-8F2C-68E4D28DF14A/47C265DC-BFAE-4C78-8072-7BF6C1171016.jpg)

Spotlight 搜出来的动作需要手动确认才能运行

考虑到 Siri Shortcut 未来还可以接收第三方应用的参数传入（下半部分马上就会展开讲），在 Spotlight 中集成 Siri Shortcut 是非常自然的设计，甚至可以全靠打字就完成和应用的复杂「交流」。

更强大的动作制作
--------

从基础常见的自动运行到新设计的分享菜单，这些都是每个用户可以看到的变动；而对于有能力或有兴趣自己制作动作的读者来说，Shortcut 在具体功能上的改进同样令人兴奋。

### 第三方应用参数

Shortcut 在去年 WWDC 后就具备了连接第三方应用模块的能力，国内的 JSBox、国外的 Scriptable 等一批自动化工具最先提供了支持。美中不足的是，彼时还无法直接把数据传递给这些第三方模块，比如需要用 Jayson 处理 JSON 格式数据时，就得借助剪贴板来当中转站。

![](.evernote-content/3E81803F-6EB8-4B77-8F2C-68E4D28DF14A/25CC8580-BB80-4A98-B9C0-6AD7AA5A20C4.jpg)

原先的第三方模块非常依赖剪贴板

在过去这种状况下，第三方模块非常像一只黑箱，我们最多只能经由剪贴板这一通用渠道和它们沟通（几乎任何应用和 Shortcut 模块都拥有权限访问），除了让动作变动冗长，每次也只能传递一段数据。

**新 Shortcut 也赋予了第三方应用接受参数的能力**，喜欢自己制作多步骤动作的读者终于不用插入一大堆「拷贝到剪贴板」和「获取剪贴板」操作。

除了在动作制作中的表现，第三方应用参数支持还带来一个明显的变化：Siri Shortcut 也可以接收多段参数，在 Siri 界面实现更丰富的功能。Siri 和 Shortcut 前身 Workflow 刚刚融合的时候，前者扮演的角色只是一个触发开关，并发掘好语音交互的潜力。这一情况在 iOS 13 中有了很大改变。

第三方应用提供的 Siri Shortcut 也可以在 Siri 界面接收参数。Apple 展示了一个 [用 Siri 向第三方服务平台点餐](https://developer.apple.com/videos/play/wwdc2019/213/) 的例子，和 Siri 说点菜后，会被问及具体需要那种口味，同时 Siri 也会给出几个推荐选项（需要第三方应用本身支持），突破了以往用户单方面下达命令的使用方式。

![](.evernote-content/3E81803F-6EB8-4B77-8F2C-68E4D28DF14A/075566A0-A4C5-4E0D-B5B8-A2E54CC2EE40.png)

Siri 会提醒你说出参数 图 / Apple

反观过去，如果通过 Siri 来跑 Shortcut 动作，所有参数都要预先填好，往往造成同一个动作要创建好几个副本、才能容纳几组不同参数，比如我在用的计时动作系列，就包含了「开始计时 - 写稿 - 奏折」「开始计时 - 写稿 - 单篇文章」「开始计时 - 写稿 - 读者回复」…… 等等一串动作，很不灵活。

参数支持已经是一次不小的进步，不过和理想的语言交互还有很大距离。现阶段的 Siri Shortcut 具备了「一问一答」的能力，但这还不是最自然的交互方式，系统自带的语音交互可能更加理想 —— **用一句话同时触发动作和传入参数**，典型的例子是 `提醒我晚上买牛奶`，首先 Siri 知道要往提醒事项里加任务，同时它也获得了具体的任务 `买牛奶` 和时间 `18:00`。此外，**一句话没把参数说完时，Siri 也可以主动提醒你说出内容**，就像说出 `帮我记一下……` 后 Siri 会等待你继续说要记录的具体内容（默认保存到系统备忘录）。

![](.evernote-content/3E81803F-6EB8-4B77-8F2C-68E4D28DF14A/3C8B38A9-FA87-4398-B88D-0A8735F87DE3.jpg)

系统自己的语音交互可以作为榜样

和系统自带语音交互比起来，官方 session 上那个一问一答才能订只批萨的 Siri Shortcut 就有点「蠢」了。当然一口吃不成胖子，能够让第三方应用也支持传入参数，足以证明 Apple is listening。

### 自然语言变量

对于需要自己动手制作动作的人而言，另一个较大的变动就是提供了**自然语言变量**。不过还是那句话， 别指望 Shortcut 真能「听懂」你说的话，所谓自然语言还是需要遵从一定的格式来写。

简单说，Shortcut 此次加入的自然语言特性就是把原先复杂的操作缩成**一句话**，主、谓、宾都是变量。这么讲有点抽象，来看一个解压文件的例子就能明白。解压前需要判断文件格式是不是压缩包，以前我们可能要获取文件拓展名后再进行 IF 判断，现在只需要一句话：

```
If File Extension is zip
```

其中 `is` 作为谓语居然也是变量（不过和主流编程语言中的变量还不能划上等号），这就很有 AppleScript 的语言风格，用了很多近似日常对话的形式来编写自动化动作。

![](.evernote-content/3E81803F-6EB8-4B77-8F2C-68E4D28DF14A/1C75EB01-E4E4-4D81-A702-58780CF2A3CB.png)

用自然语言变量判断文件类型

因为平时经常制作 Shortcut 动作，我对于自然语言变量的出现还是保持积极态度，但**跨语言支持的问题不容忽视**。iOS 12 中的 Shortcut 已经迭代了多个正式版本，但对于非系统语言的变量支持仍然令人不满意，最近还在困扰我的就是 `网页链接` 变量，英文系统下的 Shortcut 只能识别其「洋名」`Page URL`，这就导致我制作的动作在不同语言的设备上无法直接使用，分享给朋友时还要让对方自己改一下。等自然语言正式出来后，这些跨语言变量名的处理非常让人担忧。

![](.evernote-content/3E81803F-6EB8-4B77-8F2C-68E4D28DF14A/8DA079B1-E507-4320-B27E-D54F4562742C.png)

不同系统下需要修改变量名称

新手可能不能立即体会自然语言的好处，一定程度上还要承担更多的语言切换成本，但是动作本身较长、或者平时制作动作较频繁时，积少成多，自然语言变量的优势就体现出来了。MacStories 秀了一把改进后的超长套壳截图动作，利用新的自然语言变量， 足足精简了几十个步骤，这不得不让人想起当年魔法变量的引入，你可以选择不用，但是当你开始追求简洁精巧时，这些优化工具就在手边。

bug 汇总
------

当前的测试版主要面向开发人员，但仍然挡不住热情的玩家升级测试系统。与此同时，升了级的用户自然会在 Shortcut 的使用中遇到一些问题，所以文章最后也记录一些当前较为明显的 bug，希望给上了早班车的读者们一个提醒（尽管我个人不推荐升级尝试）。

* **不兼容低版本系统**：新 Shortcuts 应用最大的问题是不兼容旧版系统，如果你在新系统中编辑甚至仅仅是运行过一个动作，它在 iOS 13 以下的设备中就无法再被打开。建议所有已经升级或有此打算的读者都将 Shortcut 动作备份一遍，以免同步到正常设备上就无法使用。  

  ![](.evernote-content/3E81803F-6EB8-4B77-8F2C-68E4D28DF14A/48193A1B-064C-43DA-B6BE-D60C38C0AAEA.jpg)

  新动作无法在旧系统中运行
* **分享菜单选项不全**：重新设计过的分享菜单还不够完善，可能会不显示部分 Shortcut 动作。同时文刀和沨沄极客都遇到了这一情况，如果你比较依赖各种通过分享菜单运行的动作（常见如套壳截图、翻译文本），可能需要暂时委屈一下在 Shortcut 内部使用，或者在 iPad 上通过 Drag & Drop 运行。
* **Widget 不显示 Shortcut 动作**：升级后有一定概率在 Widget 中看不到任何动作，虽说可以手动添加，但会触发第一个 bug（导致低版本系统的设备用不了）。如果不幸遇到，在 Apple 提供进一步更新之前建议先不要自行修改。
