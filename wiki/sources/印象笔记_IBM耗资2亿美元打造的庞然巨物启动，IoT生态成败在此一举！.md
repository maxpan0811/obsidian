> 在昨晚举行的IBM Genius of Things 峰会上，IBM携手数十家客户和合作伙伴宣布共建物联网生态系统，同时还正式启动了耗资2亿美元打造的位于慕尼黑的全新Watson 物联网总部，这无疑是IBM在物联网领域挥出的一记重拳！这一举动也显露出了2017物联网的一些发展趋势：生态将变得越来越重要！

在昨天晚上举行的（2017年2月16日）IBM Genius of Things 峰会上，IBM 携手数十家客户和合作伙伴**宣布共建物联网生态系统**，以更好地推动物联网合作创新。同时，**IBM还正式启动了耗资2亿美元打造的位于慕尼黑的全新Watson 物联网总部。**这一系列重磅成果的落地，预示着IBM物联网生态系统建设的进一步升级。

![](.evernote-content/1B8AAD85-B707-4CAC-BD8C-F9E68D8D953E/12EFB8BE-48C0-4C6D-997A-D5EA3D42F4AB.jpg)

**生态系统开始决定平台商的成败**

从IBM的这场峰会来看，**“生态”**毫无疑问是关键词！

作为物联网生态建设推进的重要措施，IBM在启动全新Watson物联网总部之际，联合Avnet、BNP Parabis、CapGemini 和 Tech Mahindra等公司，**成立业内首个认知联合实验室**，所有合作伙伴将在IBM的全球中心共同配置开发团队。

这些来自于IT分销、咨询、金融等各行各业合作伙伴的专家，将和IBM专家一起，借助Watson物联网和Bluemix平台加速各行业解决方案的开发、验证和应用。

IBM Watson 物联网、认知互动和教育部门总经理Harriet Green表示：“很多客户都希望加入我们新的全球Watson物联网中心，和我们共同创新，我们将共同建立一个全新的全球物联网创新生态系统。”

以前，iot101君总觉得生态对于物联网来说是个遥遥无期的事儿，现在，都已经能决定物联网从业者的生死成败了（甚至有业内人士将2016年定为“物联网生态元年”）。在一些B2B和企业软件行业，通过构建强大的合作伙伴生态来凸显产品价值的成功案例已经数不胜数。而在物联网领域，生态的价值将于2017年被进一步放大。

于是问题来了，**为什么“生态”如此重要？或者换个问法，为什么“生态”对IBM物联网来说如此重要？**

为了回答这个问题，我们首先需要明确IBM自身在物联网领域的定位——**I****BM不会跨入物联网设备市场，只专注发展物联网云端平台和提供认知计算能力。**

**而物联网平台正是物联网网络架构和产业链条中的关键环节，通过它不仅能够实现对终端设备和资产的“管、控、营”一体化，向下连接传输和感知层，向上面向应用服务提供商提供应用开发能力和统一接口；还能为各行各业提供通用的服务能力，如数据处理与挖掘、业务流程和应用整合、应用开发、设备维护等等。**

![](.evernote-content/1B8AAD85-B707-4CAC-BD8C-F9E68D8D953E/1F8A1859-3BE9-49C5-B755-C1974C4BD475.jpg)

图：物联网产业全景图谱（物联网智库）

平台层处于物联网产业架构的中间，啥叫中间？就是必须有上也有下才行。而这所谓的“上”和“下”就必须由合作伙伴来构建。

![](.evernote-content/1B8AAD85-B707-4CAC-BD8C-F9E68D8D953E/8A7FA1C5-FD33-457F-925E-1D8185C4AECF.png)

我们可以类比移动互联网时代苹果公司的iOS平台，一方面，iOS平台向下面对大量的开发者，平台为开发者提供工具包等各种便利，助力他们开发各种面向消费者的APP。在这个过程中，开发者只需关注如何解决问题，而无须关心如何获取以及连接消费者数据。另一方面，平台向上面对用户，用户实际上也是数据的托管方，可以直接使用这些APP，便捷自己的生活，而无需关心开发过程。

开发者、用户和平台方（苹果公司本身）三者共同构建了苹果公司强大的生态系统，在这个生态中，开发者和用户相互吸引——用户知道你的平台上有更多好玩易用的APP，就愿意用你的产品；开发者知道你这里有很多的用户，就愿意来你的平台上开发APP。两者互相反哺，正向激励，平台方坐收渔翁之利，这岂不就是生态系统中的“循环”模式？

所以，IBM要想把沃森物联网平台做好，就必须让这个平台置身于一个上行和下行“环境良好”的物联网生态系统之中，即需要着手构建强大、可靠的合作伙伴生态，携手开发各类应用及工具开发者，与合作伙伴共同实现项目落地，开启万亿市场规模。**如果没有这些准备，那所谓的“平台”就是空壳！**

除了联合实验室外，在此次峰会中，另一重大看点是IBM也宣布了和不少各行业顶级的厂商达成了战略合作，构建基于平台的生态系统，包括：

* **牵手博世：**IBM和博世宣布建立面向工业物联网的全球合作伙伴关系，通过IBM云端的Watson物联网平台，为全球各行各业的客户提供博世物联网套件服务，以便大规模安全、高效地实现物联网设备更新。例如，家用电器制造商能够为成千上万的洗衣机与烘干机远程推送更新，确保消费者能够获得最新的更新、服务与产品，比如洗涤剂自动再次购买。
* **与SNCF和理光建立合作伙伴关系：**年客流量高达50亿人次的法国第一大铁路公司SNCF正通过Watson物联网让数百万个传感器发挥功效——消除冗余，持续为乘客提供优质服务。此外，理光（Ricoh）正在改变人们的工作方式，通过智能认知白板提升办公和会议效率。
* **与VISA建立独一无二的合作伙伴关系：**与VISA建立业界独一无二的合作伙伴关系（VISA处理的交易占全球总交易量60%），在Watson物联网平台上嵌入Visa令牌技术，让任意一台互联设备变身真正的销售点。IBM是首家与Visa合作，助力其通过物联网实现创收的公司。

其中IBM与Indiegogo公司及Arrow Electronics公司的合作成功引起了iot101君的注意。

![](.evernote-content/1B8AAD85-B707-4CAC-BD8C-F9E68D8D953E/9CAC83CF-90B6-48F1-B8CB-E1DF00DB560B.jpg)

Indiegogo公司是世界最大的众筹平台，目前众筹金额已达11亿美金。

凭借这一合作，Indiegogo平台上的合格创业者将能够通过IBM Bluemix以及其他业界领先的云服务，免费访问IBM的Watson IoT平台，并获得人工智能、区块链、高级数据分析和网络安全等即用型功能。

这场合作有趣在哪里呢？**相****当于IBM这个平台商牵手了Indiegogo这个平台商，所以抓住的就不只是这一个客户，而是这个平台上可能产生的千千万万的物联网创业者、开发者和应用商。**

嗯，好买卖，这才符合生态思维嘛~

> 博世软件创新部首席执行官Rainer Kallenbach也道出了物联网生态的核心：“物联网技术正在潜移默化地改变商业世界和日常生活的面貌，在物联网的世界中，只有那些能够在生态系统中开展协作的公司才可能取得成功，因为其能够为开放平台和互操作性提供战略性的基础。”

****要用认知计算在一众平台中开出花儿来****

平台商构建物联网生态的愿景是无限美好的，但是不好意思，你能想到的事情，你的竞争对手同样也能想到。

毕竟互联网时代成就了腾讯、阿里这样做平台的巨头，所以很多企业都认为，**在物联网时代，依然会“平台为王”，**所以巨头们纷纷撸起袖子，争先恐后的开始做平台、建生态！

近几年，各大厂商都在加大物联网平台的投入。思科在2016年花费14亿美元收购物联网连接性管理平台排名第一的Jasper；PTC在2013年至2015年先后收购Thingworx、Axeda和ColdLight，分别在连接性、应用生成、大数据和机器智能等物联网各方面的话语权有显著提升；微软收购 Solair，加强Azure物联网和企业云服务；三星收购SmartThing，形成其统一云平台；亚马逊收购21emetry，整合至亚马逊云计算服务AWS。

平台无疑也是物联网领域增长最快的细分市场之一。根据知名市场研究机构IoT Analytics的数据，物联网平台全球市场规模在2015年达到3亿美元，预计未来6年，年复合增长率为33%。到2020年，IoT平台的市场规模将达到16亿美元。

**对于平台商，构建生态说白了就是吸引更多的合作伙伴投入自家平台的怀抱**，IBM面对的平台对手一抓一把：**左有亚马逊的AWS，右有微软Azure，后有科技巨头谷歌自家的云平台穷追猛赶，前有GE的Predix已经在工业领域锋芒毕露。你IBM凭啥能在一众平台中开出花来，说服客户转向你家的云平台，与竞争对手构建起真正的差异化呢？**

iot101君相信IBM这时一定会跳出来回答说：靠的是**认知计算！**

认知计算，是一种全新的计算模式，通过信息分析，自然语言处理和机器学习领域的大量技术创新，能够“理解”非结构化数据，就包括语言、图像、视频等，让计算机系统能够像人的大脑一样学习、思考，进行数据分析并做出正确的决策。

认知计算与我们熟知的人工智能有什么区别呢？以下是援引《哈佛商业评论》的解答。

1. **规模性：**传统人工智能不强调规模，而认知计算必须进行大规模的学习。
2. **交互性：**传统人工智能梦想建造具有人类智能的自助机器，而认知计算侧重于与人类交互和协作。
3. **概率性：**传统人工智能梦想建立一种精确的智能机器，而认知计算则致力于应对各种非结构化、非确定性数据，因此表现出一定的概率性。

|  |  |  |
| --- | --- | --- |
| **差异点** | **认知计算** | **人工智能** |
| **特点** | 强调认知、理解 | 以人控制为主，由人告知机器如何行动 |
| **要求能力** | 有学习推理能力，能通过分析做出适当决策，为人们提供参考 | 接收人的训练、培养 |
| **与人关系** | 与人和环境之间有互动，增加人类智慧 | 没有相互的反馈，主要由人控制，根据人类的需求而工作 |
| **衡量标准** | 没有唯一标准，遇到具体问题具体分析 | 有像图灵测试这样的衡量标准 |
| **关注领域** | 大数据，尤其是海量非结构化的数据 | 模拟人、人类世界 |

表：认知计算和人工智能的区别

上面说的可能很学术，通俗点儿来说就是：IBM 的Watson IoT 平台有别于一些物联网厂商所推出的IoT平台，一般的物联网平台多是透过管理和分析从物联网设备端收集数据，以帮助企业改善组织生产效率或产能，但未来的物联网平台更需具备认知运算的能力，能让物联网装置变聪明，不只能与人沟通交流，甚至可以透过学习来理解使用者的意思。

在这些案例中，IBM的认知领域和物联网是如何联系起来的？原来，物联网通过终端搜集的数据很多都是非结构化的（2020年左右，物联网中非结构化的数据占的比例要达到80%）。非结构化的图像、视频、声音数据怎么处理？**IBM认知技术用人感知世界的方式，更好更深入的去处理这些物联网世界当中搜集来的数据，并做出整合、分类，最终得出一个分析结果。**这就是二者的深度融合！
