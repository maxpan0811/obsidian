### Gear Browser：支持油猴脚本的移动浏览器

**@Tp：**随着我们越发频繁地用手机浏览网页，我们对于移动浏览器的需求也早已不仅限于「能加载网页」这么基础的功能。Gear Browser 就是一款全能型移动浏览器，它不但自带了广告屏蔽等常见的实用功能，甚至还可以安装 Tempormonkey 油猴脚本，为你发挥出它的更多潜力。

你只需要在应用的网页菜单里找到「插件」，即可管理已安装的插件或是安装新的插件。你可以在 GreasyFork、OpenUserJS 等网站上找到优秀的油猴脚本，并直接安装到 Gear Browser 里。由于部分脚本并非针对移动设备进行设计的，因此这些脚本有可能出现无法使用的情况。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/C621E7CF-5788-49A3-99EA-162AB54CEB5D.jpg)

除了支持安装脚本以外，Gear Browser 还支持丰富的调试功能。比如检查网页元素、使用控制台执行 JavaScript 代码，你甚至还可以利用 Gear Browser 内置的「实时样式编辑器」来更改网页的 CSS 样式。如果你觉得这些功能太过「硬核」，Gear Browser 也有更加简单易用的「设计模式」，你可以直接编辑整个页面里的内容和图片。当然，这些编辑都是基于本地的，并不会对网页造成任何影响。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/33C5CCBC-AE46-4C10-B638-7EC82DFEDE1E.jpg)

总的来说，Gear Browser 作为一款移动浏览器，是足够优秀的，应用不但做到了浏览器常备的阅读模式、深色模式等基础功能，还将桌面浏览器的调试功能和油猴脚本带到了移动端，大大提升了它的实用性。

你可以在 [App Store](https://apps.apple.com/cn/app/gear-browser/id1458962238) 免费下载 Gear Browser，应用采取订阅制收费，价格为 9 元 / 月、48 元 / 半年，70 元 / 年。

**关联阅读：**[iOS 浏览器也能装扩展，设计美观、功能强大的浏览器应用：Gear Browser](https://sspai.com/post/60655)

### Backdrops：多样化壁纸「供给」

**@Snow：**之前少数派推荐的许多壁纸应用，都是基于 [Unsplash](https://unsplash.com/) 这个巨大的公版图片库。Unsplash 的图片可以很好地满足精美且高清的壁纸需求，也不用担心版权问题，但其中大多数素材都是具象化的摄影照片，可能并不一定适用于扁平化的手机桌面。而且同类应用太多，库却是同一套，说不定会出现「撞壁纸」的情况，这就有一些尴尬了。

少数派很久以前推荐过 Android 平台上的 Backdrops 这款壁纸 App，不久前正式推出了 iOS 版本。Backdrops 有着自己独立的社群和壁纸库。在 Explore 一栏中，除了摄影作品外，还有许多设计或绘画素材可供使用，相比于实景照片，这些素材或许更适合作为壁纸。除了开发者提供的素材外，注册用户还可以上传自己的摄影或设计作品与他人分享，这也为 Backdrops 的更新量提供了一定的保障。

不过，由于内容来源多样化，Backdrops 可能缺乏像 Unsplash 那样对版权的保障，如果你是想通过这一途径获取图片作为它用，会有一定风险。

除单张壁纸外，应用也在 Collections 一栏中提供了同一主题的「壁纸套装」。套装中的壁纸相关度更高，可以更好地满足像我这样的「强迫症用户」那种锁屏和主屏的「配套化」需求。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/F98BDF79-97DD-4BBF-82B8-2BD55227ADEB.jpg)

应用包含了一个 25 元的 Pro 服务，付费后可以去除广告并解锁壁纸套装以及部分 Pro 专属壁纸。你可以在 [App Store](https://apps.apple.com/cn/app/backdrops-wallpapers/id1500143735) 和 [Play 应用商店](https://play.google.com/store/apps/details?id=com.backdrops.wallpapers) 下载 Backdrops。

### Usage：时刻监视手机状态

**@Snow：**Usage 是一款颜值颇高的手机状态监控应用，它内置 8 种主题配色，并且支持深色模式。Usage 可以实时反馈设备的 CPU、内存、存储、网络通讯、电池等运作情况。应用提供了通知中心小部件，即使不在应用内，你也可以随时调用小部件了解设备运作情况。如果你是一位开发者，这款应用的数据可以为你的测试提供参考。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/7BBBE20A-BB32-4F0E-B1D8-2B99197B502D.jpg)

在 Compare 一栏中，你还可以获取使用设备详细的配置信息，并与其它 iPhone 或 iPad 产品进行对比，了解二者间硬件的差距。

你可以在 [App Store](https://apps.apple.com/cn/app/usage-system-activity-info/id970353453) 上免费下载 Usage。

### Figurative：第三方 Figma 移动客户端

**@化学心情下 2：**如果我们将 iPad 认为是另一种形式的「桌面设备」，那么其浏览器或者是其最大的生产力工具之一，它让在 iPad 上完成互联网产品设计变成一种可能；早些时候 iPadOS 发布时已经有很多设计师尝试在 iPad 使用一些网页端原型设计工具进行产品设计，而 Figurative 则更进一步，将知名的原型设计工具 Figma 带到了 iPadOS 平台。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/62AD4036-F181-4F2E-8BF7-3FAA0FF571AD.png)

简单来看 Figurative 就是将 Figma 打包成了一个 iPadOS 应用，因此它几乎完整保留了 Figma 的所有使用体验，甚至是右键菜单也可以完全正常使用，可以说和桌面浏览器上的体验几乎一模一样。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/F9460E52-E315-4C9D-8E7D-CF268FCE73D0.png)

另一比较让我关心的功能是鼠标指针部分，在精度方面，经过测试，像在进行创建多边形组件然后拖动边缘调整大小等操作时，应用总体来说精度还是相当不错，当然如果觉得可能存在误差，那么在右边直接使用数字调整就可以了。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/E27FE093-17CE-4581-A5C6-217AA85F85AB.png)

在 iOS /iPadOS 上使用 Figma 最让人关心的可能是字体问题 —— 如何导入合适的字体。好在 Figurative 也提供了一整套字体安装方案，总的来说就是为你的 iOS 安装自定义字体、从而在 Figma 中显示出来，当然如果可以在传统桌面上使用 Figma 的话不妨再做二次调整。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/B76AA93D-933E-4133-A7F8-C5CBA84B7A9C.png)

你可以在 [App Store](https://apps.apple.com/cn/app/id1510607687) 免费下载 Figurative。

### Action Blocks：Google Assistant 的「捷径」

**@Clyde：**如果克服使用的网络门槛，Google Assistant 在日常生活中可以发挥出不小的作用：大至与米家智能家居联动、小至烹饪时设置定时器，即便在语音还欠缺自然的 [简体中文](https://sspai.com/post/53796) 语言环境下，Google Assistant 灵敏、多样的唤醒方式和准确的语音识别也能派上不小的用场。

为了让这些 Google Assistant 日常指令更加方便易用，Google 上周在 Play 商店中上架了一款名为 Action Blocks 的官方应用，这款应用与 Siri 快捷指令的用法类似，初版完成度又有点类似小米的小爱捷径。借助这款应用，我们可以将大部分需要语音、文字输入才能执行的 Google Assistant 指令「图形化」，变成一个个图形开关放在桌面，最终实现一触直达的自动化效果。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/AFB6B7B2-11BF-4F02-B4CE-ED9A0032C315.png)

确认指令工作后接下来便是添加方块图片和方块名称，这里我们可以为方块选择自定义图片或应用内置的插画风格图片，添加完成后根据应用提示添加到桌面便大功告成啦。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/FEB157CD-291A-4D4F-85A4-3DDC38B4BC2B.png)

Action Blocks 对于日常生活中常常需要用到 Google Assistant 的人而言无疑是一个「必备」应用，它把常用的语音指令都变成了可以点击触发的图形方块，不管是放在主屏还是单独用一个页面来进行收纳，用起来都会非常方便。

你可以在 [Play 商店](https://play.google.com/store/apps/details?id=com.google.android.apps.accessibility.maui.actionblocks) 免费下载 Action Blocks。

### 睡了吗：全局睡眠定时器

**@ln - 勇字当头：**大学时代我们没少经历过挂视频课的痛苦，手机经常一挂半天自动关机而不知；工作后我们也时长倔强地熬夜，手机屏幕在我们疲倦入睡后不知不觉地彻夜常亮。为了解决由于遗忘等原因造成的手机常亮、电量耗竭等情况，开发者根据自身的需求开发出了「睡了吗」这一应用。

睡了吗可以根据我们的需求为应用设置「睡眠时间」，到达设定时间后手机自动熄屏睡眠。我们可以将睡了吗理解为一个全局的睡眠定时器，所有应用都可以通过它来进行睡眠时间设定。

首次打开睡了吗，我们需要授予应用所必须的权限，使其有权限监测手机当前正在进行播放或是使手机保持长时间亮屏的应用。之后，我们可以在睡了吗当中添加需要监测的应用，如哔哩哔哩、抖音等视频类应用或是其它会使屏幕常亮的应用，并为相应的应用分别设置锁屏时间。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/E347A813-A6B4-4705-8139-82098D27D5FA.jpg)

总的来说，这一应用比较适合经常在夜间看视频、入睡前听音乐或是有挂视频时长需要的用户，由于其索取了设备管理员权限，每次亮屏后都需要用户再次手动输入密码，且应用范围较窄，并不推荐日常使用。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/5328B34E-D629-4C4A-A3B3-4B814CB5E172.jpg)

目前你可以通过 [酷安](https://www.coolapk.com/apk/io.alcatraz.afkprotect) 下载使用睡了吗。

### Wormhole：在电脑上操作 iOS 和 Android 设备

**@huhuhuang：**Wormhole 是一款支持 iOS 或 Android 设备投屏至 Mac 或 PC、并在电脑端直接操作手机的应用。开发者表示，Wormhole 是全球第一款支持在 Mac 或 PC 上操作 iOS 设备的应用。

以通过 Mac 操作 iPhone 为例，你需要先 下载 Wormhole 的 macOS 版本应用。在我体验 Wormhole 时，由于该应用的签名问题，第一次打开时会提示你移动到废纸篓。此时可以找到应用安装目录，点击鼠标右键进行打开。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/CCC83452-27DE-4889-9A77-6060C268758A.png)

打开 Wormhole 之后，你需要同时通过数据线和蓝牙将 iPhone 和 Mac 进行连接，此过程 Wormhole 会提供引导。如果没有问题，几秒钟后你就可以在电脑上看到 iPhone 界面了。Wormhole 应该是利用了 iOS 提供的屏幕镜像功能实现投屏，设备连接之后，下拉菜单中的基座接口和系统捕捉选项会被激活。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/A22670D8-A789-4898-B377-DF1AC11ED184.png)

Wormhole 的使用体验还算不错，你可以通过虚拟鼠标触点实现滑动、点击等操作。理论上来说，你在 iPhone 上的所有操作应该都可以通过 Wormhole 完成，甚至包括触感触控。Wormhole 还在左侧功能区提供了几个快捷键，包括：快速切换到主屏幕、打开多任务管理、截取屏幕图像和调节音量大小。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/4FE13810-77C3-43B9-B95A-B98B6CF18B1C.png)

不过，由于鼠标并不适合滑动操作，在下拉菜单、滑动屏幕时会显得比较吃力。 操作的延迟能明显感受到的，尤其是连接 iPad 时操作会异常卡顿，我觉得官方宣传图中打游戏的场景几乎不可能完成。

Wormhole 是一款挺不错的应用，你真的可以在 Mac 上打开 iOS 应用了，而不必频繁拿起手机。此外，就算是将 Wormhole 单纯作为一款投屏应用使用也是不错的。如果你有兴趣，现在就可以到 [官网](https://er.run/) 购买和下载 Wormhole，49 元可以激活 3 台设备，支持支付宝付款。

### 白描：在线批量 OCR，白描网页版上线

**@Tp：**在移动端，白描可以说是综合体验最好的 OCR 应用之一。但如今我们绝大部分工作还是得在电脑上完成，而白描没有桌面客户端是长久以来的一大遗憾。最近，白描上线了网页版，很好地解决了它没有桌面客户端的问题，让我们在全平台都能用上白描。

白描网页端与移动端一样，支持文字和表格的扫描识别，你只需要将想要扫描的图片文件上传至网页端即可。白描网页端支持批量扫描最多 50 张照片，在完成扫描后，你还能一次性合并所有结果，导出为 DOCX 文件或 TXT 文件。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/A98C9AEC-BDCB-4288-A856-2138CE051F13.jpg)

不仅如此，网页端还支持直接上传 PDF 文件进行扫描。无论是哪种 OCR 方式，在扫描完成后，你都可以对照着原图来修改扫描结果。即使你扫描的是表格，也能在网页上对其进行编辑校对，并导出为 Excel 文件。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/14480D95-917E-4685-AA02-3B198E44AD56.jpg)

实际体验下来，白描网页端的扫描效果与移动端几乎没有差别，都具有相当高的准确性和较快的扫描速度。不论是书本的照片、扫描件，还是屏幕截图、中英混排，白描网页端足以应对生活、工作中的绝大部分 OCR 场景。

不过白描网页版在应对一些不平整的书页、文件时，会出现大段文字识别出错的现象。因此在拍摄照片时，最好还是保证文字能够平整地显示，从而确保识别准确率。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/8A92EF3D-AAB2-4935-9C23-6C90D67D3CA1.jpg)

白描网页版的账户与移动端共通，你只要将账户绑定手机或邮箱即可在网页版登录。在网页版上，白描会员权益也与移动端共享。白描网页版暂不支持移动浏览器，你可以在电脑或 iPad 上点击 [这里](https://web.baimiaoapp.com/) 使用。

**关联阅读：**[在网上批量高效 OCR，白描网页版是个不错的选择](https://sspai.com/post/60646)

### Framer for Web：让原型交互摆脱平台束缚

**@化学心情下 2：**从 Axure 到 Sketch，互联网的产品原型设计工具在很长一段时间都对于平台有比较强的依赖属性：比如只能在 macOS 平台上使用的 Sketch，显然这种平台独占工具并没有真正地促进平台的进步，也就有了之后类似 Figma 这样的等基于 Web 进行设计和开发的产品设计工具，让产品设计真正意义上摆脱了平台的约束。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/21D12A48-BB83-457A-90BC-A1C983108CCF.png)

这种风潮几乎影响了「后 Sketch 时代」 的一批产品设计软件，比如曾经被认为是补足 Sketch 最大缺失的交互设计工具 Framer 也在近期正式推出了真正的纯 Web 版本，换言之我们在设计产品交互的时候可以仅仅依靠一个网络浏览器，至于平台都不再重要了。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/62848C5E-B05B-42E2-9F0E-F92DE97C52A7.png)

对于初学者而言，登录 Framer for Web 之后将会首先通过一系列在线入门视频进行一系列讲解，如果你使用过类似的软件那么会很容易上手；对于 PM 而言，这款工具最需要关注的点是能否将各个页面链接起来，成为一个可以展示的产品交互原型。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/0E1071C3-202D-4A35-87BE-E3663FE0AAA1.png)

虽说 Framer 算是比较新的平台，但是你依旧可以非常轻松地找到各种平台上的样式规范和组件，只需要动动手搜索即可，添加好对应的样式库只需要新建设备画板就可以将需要的组件拖入到画板中进行设计了，不过 Framer 定位在原型设计上，所以要想做一系列的组件和元素的效果，可以选择将 Figma 的设计文件导入到 Framer 的项目中。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/7DB20A89-EB73-4D73-87CF-2FB59B2FA0B0.png)

在我看来 Framer 的交互设计中可使用的动作还是相当丰富的，虽然比不上 Axure ，但对于一般设计师而言其灵活性要远超 Adobe XD 或者 Sketch。其实 Framer 推出 Web 端在我看来更像是让习惯在 Figma 中进行创作的设计师不用单独再打开客户端，从设计到交互呈现都可以在浏览器中呈现。

Framer for Web 也提供了团队协作等一些可能在小团队中用到的功能，如果你希望增强你的 App  展示效果，方便开发团队跟进，不妨试试免费的 [Framer for Web](https://www.framer.com/)。

### Chrome：标签页分组功能正式上线

**@化学心情下 2：**因为疫情的原因，Google 直接选择跳过 Chrome 82 而直接推出了 Chrome 83，因此原本在 Chrome 82 上发布的新功能也被合并在 Chrome 83 中一并推出，比如这个测试了相当长时间的标签页分组功能。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/C450D172-7BCD-4E8C-898D-8DBA131EF0D3.png)

标签页分组主要是针对多标签页浏览时会遇到的标签页管理需求，比如说打开大量的标签页时往往会出现不易查找、容易重复打开的情况，而标签页分组可以有效地避免这种情况，建立采用不同颜色标识的标签页组可以说是一目了然，通过简单地加入组以及将组标签在另外的浏览器窗口打开，你可以建立起一个行之有效的标签页工作区，这样再也不会遇到标签页找不到的情况了。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/1479B1A7-E0B4-4D25-B2BC-F298462A975E.png)

其它更新则主要集中在隐私与安全方面，比如说安全检查功能进一步加强，安装扩展使用了哪些权限也会被展示出来，还有 DoH 支持、混合内容中的下载安全提示等等；新版 Chrome 在设计上也有所调整，比如网页控件的显示更加现代、标签页冻结功能也在本次更新中正式可用，显然在微软 Edge 羽翼丰满之后，Chrome 需要加速创新来进一步维持其霸主地位。

![](.evernote-content/136585FB-F890-4071-9A61-4AC070FE1255/DBBF8435-5CCD-441E-AD0A-73E66A5DBA65.png)
