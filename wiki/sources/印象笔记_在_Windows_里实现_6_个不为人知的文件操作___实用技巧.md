一键创建模板文件夹
---------

每当我购买一块新 U 盘时，我都会习惯性把一套**已经搭建好的文件夹模板**直接复制进 U 盘，免去了我为他们一一创建文件夹的麻烦，也能避免 U 盘用了没几天就乱的一塌糊涂的情况出现。顺便把一些 U 盘里常用的小工具一并复制进去。

这种做法其实就是**模板文件夹**，把一套可复用的文件夹作为模板，二次使用时可以直接套用。这样的做法还可以用在**设计素材模板、文档编辑模板、新建文件分类**上。

不过在创建文件夹时往往操作繁复，需要在生成位置和模板文件夹之间来回跳转。而使用模板本身是节约时间的操作，应当使用更方便的方式触发。

我做了一个 Quicker 动作来解决这个问题。使用方法也很简单，直接在目标文件夹运行这个动作即可。

一键创建模板文件夹

支持用「列表内容」和「模板文件夹」两种形式来创建模板文件夹。

1. 根据列表内容创建，不必在本地创建文件夹。
2. 根据已有文件夹创建，可以复制一份模板到当前文件夹，通用性更高。

**这个动作本身可以直接使用**，调整不是必须的。但由于每个人对文件夹模板的需求不同，如果需要自行做一些调整，这里也给出修改模板的方法。

如果想修改模板，可以在编辑界面手动修改 `default_list_0X` 变量，将会根据列表自动创建所需要的文件夹。默认提供了 5 种模板，不够还可以手动添加。

批量获取文件列表
--------

曾有人在 Power＋ 的群里提问「如何批量获取文件夹名称」。其实这个操作本身很简单，应用范围比较窄，但我后来发现这一步往往是很多文件批量操作的开端，和其他动作组合起来使用时，用途就变多了。比如把文件夹作为选择列表、筛选特定名称的文件等。即便单独使用，也能节约一个一个选中、复制文件名的时间。所以有必要在这里讲一讲。

这个操作本身可以用两种 cmd 命令实现。

第一个方法是用 Tree 命令。按下 `Win ＋ R` 呼出「运行」窗口，输入 cmd 进入命令行。

输入 `tree 文件夹路径 /f > 输出文件的位置/文件名` ，例如：`tree D:/Folder /f > D:/Folder/List.txt`。

运行后，D:/Folder 目录下的文件列表就会被写入 List.txt 文件中。

第二个方法是用 Dir 命令。同样是按下 `Win ＋ R` 呼出「运行」窗口，输入 cmd 进入命令行。

输入 `dir 文件夹路径 *.* /B > 输出文件的位置/文件名`，例如：`dir D:/Folder *.* /B > D:/Folder/List.txt`。

你也可以输入 `cd D:/Folder`，在目录下直接输入 `dir >> List.txt /b` 来获取文件列表，最后实现的效果是一样的。

给文件夹添加备注
--------

Windows 中还有一个不为人知的属性：「备注」，你可以在文件夹的空白位置「右键 - 排序方式 - 更多」里开启备注的显示。

![](.evernote-content/1FD1BBCC-5392-4553-AE6A-0785CA7AF023/638DDF09-A920-4308-849E-2032AAFBD8AE.png)

显示文件夹备注

这个属性原本是给一些文本格式的文件添加的，你可以在「查看 - 详细信息窗格」中直接对 docx、xlsx 等格式的文件添加备注。但并不是每个文件都能通过这种方法添加备注，而且多数时候不必这么做。相反，这个属性给文件夹使用更为合适。

由于根目录下的文件夹名称有较多的文件牵连，往往不方便改名。那么当我们想对文件夹进行分组时，就可以用「以备注分组」的方法来分类文件夹。这里要讲的是给文件夹添加备注的小技巧。

### 通过 desktop.ini 文件手动添加备注

这个方法是 Windows 内置的修改方法。

每个文件夹的信息是由文件夹中的隐藏文件 desktop.ini 决定的，这个文件中包含了文件夹的图标、信息、显示风格等。我们可以手动在文件夹中创建这个文件。

新建一个 txt 文档，输入下面两行代码。

```
[.ShellClassInfo]
InfoTip=备注内容
```

Windows 读取 ini 时只能读取 ANSI 或 GB2312 编码。所以如果备注中出现了中文，记得保存为这两种类型的编码，否则可能会出现乱码。然后保存为 desktop.ini。

最后是 Windows 的一个坑，它要求**只有当文件夹本身具有「只读」属性时，desktop.ini 文件才会生效**，网上有许多资料写的是「系统属性」，显然是错误的。

所以我们需要在 cmd 中跳转到文件夹的上层目录，输入 `attrib +r /d` 为文件夹添加 r（只读属性），使得 desktop.ini 生效。

还有一款简单的小工具 [WindowsFolderRemark](https://github.com/piratf/windows-folder-remark)，也可以直接为当前文件添加备注。

总结
--

除了重命名之外，这里介绍的其余五个技巧在别的地方很少看到。这是因为这些动作操作起来相对复杂，很难通过简单介绍实现。但由于这些动作往往有着相同的操作步骤，所以本质上是一些机械的重复劳动，在一些效率工具的加成下很容易实现。

其他的重复动作也能通过「先懂原理  - 再懂步骤 - 总结规律」这几步来实现自动化，把繁复的动作简化成「一键完成」。这样，每个人都能做出属于自己的自动化动作，提高整体的工作效率。
