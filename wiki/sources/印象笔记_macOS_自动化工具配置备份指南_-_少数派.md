macOS 自动化工具配置备份指南
=================

用了很久的自动化工具后，偶尔摸到朋友那台「原版出厂设置电脑」1，我不禁脱口而出：

> 这什么玩意儿？

相信每个自动化玩家使用别人的设备时，都有过和我类似的经历。

的确，对于每个自动化玩家来说，这些**工具的配置**都是长久积攒的心血。「配置」指一类和用户数据相关的文件，包括**自动化动作的源文件**，**使用频率的记录**，以及**使用习惯的记录**…… 无论更换电脑还是升级系统，配置文件都能让用户延续以前的操作习惯 —— 反过来讲，如果没了这些配置文件，一时间我们可能都不知道该怎么用电脑。

平时在 Slack 群内就经常有读者询问各个应用的配置备份方式，我以前也写过一篇《[备份 LaunchBar Actions](https://minja.bitcron.com/post/titian-tool-bei-fen-launchbar-actions)》。这次文章，希望能解决更多自动化工具 —— 乃至其他工具 —— 的配置备份问题。

找到应用配置文件
--------

备份操作需要有的放矢。找到要备份的配置文件就是整个备份操作的第一步。下图是启动器 Alfred 的配置文件夹，其中就包括了注册信息、偏好记录和自动化动作文件。

![](.evernote-content/8250306C-91D0-4E9B-8B52-0B7B224E2591/81004819-1D92-4D29-BB61-29F576062F7C.png)

藏在系统资源库深处的 Alfred 用户配置数据

但是，并非所有应用的配置文件夹都有和 Alfred 一样的结构，例如另一款启动器 LaunchBar 就把每个自动化动作分开保存，而不像上图的 Alfred 那样全放在一个配置文件里。这一节中我们将列举几款主流自动化工具，通过这些案例了解配置文件的常见保存位置和储存方式，并在文章后半部分结合实现**批量备份**乃至**自动备份**。

**知识点：区分备份和同步**

关于应用数据的备份，可能有读者听说过 [Mackup](https://github.com/lra/mackup)，一个开源备份工具，支持的应用范围很广。但在这个一站式解决方案其实不是多数人理解的「备份」，而是 **同步** —— 把配置源文件移到独立文件夹里，本来的地方则用一个快捷方式代替，让两者强制同步起来（Mackup 基于可用于文件同步的 `ln` 命令打造）。

同步的风险很明显，**如果误删了「备份文件」，或者网盘宕机，那本机用 Mackup 备份过的应用可能都会被清空用户数据** —— 我就遇到过需要重新注册、记录全丢等情况。如果备份文件存在反过来破坏本机文件的可能，就违背了「备份」的本意。在应用备份用户数据这关上，我们还是需要自己动手。

我们以几款知名自动化工具为例，可以把用户数据备份分为几种常见情况：

1. **自带备份**：应用自己可以定期备份，比如 Keyboard Maestro、Hazel，这种最轻松，把配置文件统一移到一个地方（比如 iCloud）就行。
2. **独立文件**：数据主要以独立文件的形式存在，比如 LaunchBar 的动作（action）、Automator 的工作流（workflow），一个自动化动作就是一个文件，找到源文件备份起来就能解决问题。
3. **配置文件**：用户数据合在一份或几份文档里，比如 Alfred 的用户习惯、TextExpander 的自定义缩写词（用于快速输入文本），这些文件也可以手动备份，唯一的遗憾是要恢复的时候一般只能扎堆儿恢复。

上述几类文件，一般在系统的应用支持文件夹中，往往文件夹名就是对应的应用名，例如下面的 LaunchBar 配置文件夹。

![](.evernote-content/8250306C-91D0-4E9B-8B52-0B7B224E2591/9201701E-CA0C-4DA2-889F-4A9068A8B5A5.png)

LaunchBar 的用户数据文件

不过系统自带应用偶若会有特例，比如 Automator 在资源库里就一人占了三四个位置，这种特殊情况不多见，稍加注意即可。

```
~/Library/Application Support/
```

总的来说，最省力的方法是把想要备份的应用数据汇集到同一个文件夹中。由于我个人有和人分享动作的需求，所以还会把配置文件统一保存到云端，便于随时取得 2。

![](.evernote-content/8250306C-91D0-4E9B-8B52-0B7B224E2591/C68BBAF6-BA53-4003-B79F-A43F81A32A71.png)

我的应用数据备份在 iCloud

下面每一类备份方式都会给一些典型例子，看看具体怎样操作。

### 自带备份：Keyboard Maestro 和 Hazel

Keyboard Maestro 是一个通用型自动化工具，给键盘、鼠标都带来了很高的自定义空间。它的应用设置中有个「Sync Macro」的选项，有些人可能会径直把备份放到 iCloud，免去自己编写命令这一步。

![](.evernote-content/8250306C-91D0-4E9B-8B52-0B7B224E2591/B3D7B006-7930-4C4C-82D9-A76DC65C2DD3.png)

Keyboard Maestro 的同步选项

方便归方便，这些自带备份功能的应用，都要留个心眼。Keyboard Maestro 设置里写的是「Sync（同步）」，它的本意其实是同步自动化动作，这样在多台电脑之间也可以保持同样的使用习惯。换个角度看，如果动到了 iCloud 里的文件，就会直接影响 Keyboard Maestro 的正常使用；iCloud 还没有历史版本回溯功能，**网路不畅的话很可能发生同步冲突，丢失数据。**

这是同步和备份区别的又一次表现（上次是前文的 Mackup）。相比之下，文件自动化管理工具 Hazel 的同步提示就写得比较明白，但还是有人直接丢到网盘，从而中招。

![](.evernote-content/8250306C-91D0-4E9B-8B52-0B7B224E2591/AF9A3B78-F034-4379-9473-936EF3408BD4.png)

Hazel 的同步提示

保险起见，并不建议直接同步到网盘，更安全的操作是：

1. 把用户数据同步到本地文件夹
2. 通过其他方式同步本地数据到网盘

关于备份方式的选择，后续会介绍一条简单实用的**备份命令 rsync**，它是单向的备份操作，在本文的使用场景中只负责把本地的文件备份到云端，即便网盘服务挂了也丝毫不影响本地的用户数据。

### 独立文件：LaunchBar

不少自动化工具通过独立的文件来保存数据，典型如 LaunchBar，用户自制的 Action 就放在这里：

```
~/Library/Application Support/LaunchBar/Actions
```

此外还有 Alfred 和 Automator 的工作流文件，它们在本地都能找到源文件，可以安心地直接备份一份到网盘。

对于要求较高的用户来说，手动或定期运行 rsync（或别的同步方式）还不够，还需要想办法让网盘里的数据保持最新。**更理想的是，每次做过新的自动化动作，就把最新版本备份起来。**

自带的 Automator 可以一定程度上满足自动备份。创建一个文件夹动作（Folder Action），监控要备份的文件夹，加上一个运行 Shell 脚本的模块并填上备份命令，简单的自动备份就做好了。每次创建一个新自动化动作时，就会触发 Automator 自动备份。

![](.evernote-content/8250306C-91D0-4E9B-8B52-0B7B224E2591/DE6584DF-3300-4B87-9A53-B07F7204CC39.png)

用 Automator 来自动备份

如果有 Hazel 的话，也可以不写脚本，直接使用自带的同步模块把数据改动更新过去。比较遗憾的是 Automator 和 Hazel 都只能用于通过独立文件保存动作数据的应用，下一节要介绍的 TextExpander 把所有数据塞进同一份文件，那就不能监控文件变动了。

![](.evernote-content/8250306C-91D0-4E9B-8B52-0B7B224E2591/4665D7A9-AEE0-466A-A9CE-30E2C0D30095.png)

### 配置文件：TextExpander

还有一类应用，用一份或很少几份文件保存几乎所有的用户数据，其中代表就是 TextExpander。

![](.evernote-content/8250306C-91D0-4E9B-8B52-0B7B224E2591/4501AB7E-DB35-44E6-B893-BCB3E39E4FDF.png)

TextExpander 的用户数据

老实讲，我不是很喜欢 TextExpander 的数据保存方式，因为一般人在 TextExpander 里面的缩写都会分组整理妥当，但是 TextExpander 的数据文件只有一份，把所有缩写全塞里面，恢复的时候只能一股脑全恢复，分享的时候还得手动导出分享。不过 TextExpander 主打的就是在线服务，它提供了 TextExpander 账户来同步数据，体验也比较稳定，无可厚非。

TextExpander 的用户数据也在 Application Support 文件夹里：

```
~/Library/Application\ Support/TextExpander/
```

和独立文件的操作一样，这份文件直接同步到网盘即可。

每个应用的数据位置并不相同，这里也只能介绍最常见的一些情况。挨个备份所有应用无疑是低效的，还不如只做全盘备份。用户各种命令、自动化工具，无非是为了在常规全盘备份以外，再把或使用高频、或重要度高的文件再拿出来备份一下，便于日常取用。

通用命令：rsync
----------

对于常用文件，我们要的就是指哪打哪，**速度快、不重复、不遗漏。** Mac 自带了 **rsync 文件同步命令**，就可以实现轻量级的备份。

rsync 是一种增量的、采用差异算法的备份（同步）方式，效率比较高。举个例子，你在 Sketch 文件夹里修改过一个文件，rsync 就会认准了这次备份只需要改动这个文件，精确打击。

rsync 的各种参数较多，不过为了备份的话，这几条就足够：

1. **备份：**

   ```
   rsync -av 源文件或文件夹 备份文件或文件夹 
   ```

   最基础的备份命令，就是把文件从一处备份到另一处。**特别小心，两处如果是文件夹，路径最后都要用 `/` 结尾，稳妥起见可以用 `""` 包起来，避免很多怪异名称带来的问题。**
2. **备份时删除旧文件：**

   ```
   rsync -av 源文文件或文件夹 备份文件或文件夹 --delete
   ```

   rsync 采用的是增量备份，也就是说，默认只进不出，源文件夹里删掉的文件仍然留在备份文件夹中，颇有貔貅风范。正常情况下我们是不需要这些文件的，为了让备份文件的源头保持一致可以加上 `--delete` 参数删掉多余的文件。
3. **排除特定文件或文件夹：**

   ```
   rsync -av 源文件或文件夹 备份文件或文件夹 --exclude "想要排除的文件或文件夹"
   ```

   最后这个参数可以排除掉不想备份的文件或文件夹，比方说备份本地曲库时，并不想要把歌词文件夹和流媒体音乐缓存备份进去，就可以把它们排除掉。

注：上面几条命令中的各种路径，都可以通过直接把文件或文件夹拖进脚本编辑器来获得。

![](.evernote-content/8250306C-91D0-4E9B-8B52-0B7B224E2591/2F8294D4-BBAC-411F-B418-74599EE7E4A0.png)

我常用的备份命令

这些命令可以写在同一份 Shell 脚本中，每条命令换一行，运行时就可以一次备份完全部的文件。rsync 还可以和自动化动作结合起来，我就为 iCloud 和 U 盘各做了一个备份动作。

![](.evernote-content/8250306C-91D0-4E9B-8B52-0B7B224E2591/F9981B25-E05A-4D24-84B3-3A108978974F.png)

结合了 LaunchBar 的备份脚本

由于 rsync 没有版本记录，相当于只保存最近的一版，空间不大的 U 盘也可以用来承载这点备份。

可以看出，这些文件散在不同地方，有的在本地、有的在云端，利用 rsync 还能起到把它们汇集于一处的作用。对于常用文件的备份，rsync 可以说兼顾了效率和便利。

结合 Automator 实现定时备份
-------------------

其实比起备份的技巧，养成备份习惯是更难的。Ulysses 教程作者 @sainho 曾[分享](https://sspai.com/post/39591)过他教朋友使用 Time Machine 的经历，对方惊叹这个工具备份如此方便，但过段时间照样把备份工作抛到九霄云外。

我们还是借助一些工具自动备份，这样比较牢靠。上一小节的 rsync 脚本可以和各种自动化工具结合，来实现自动备份功能。我们以 macOS 自带的 Automator 为例，看看如何操作。

iMac、Mac Pro 和 Mac mini 这类台式机型可能会长期接通电源，基本随时都能叫起来工作，它们就可以享受定时的自动备份。

首先我们要创建一个 Automator 日程提醒，因为单独的脚本是很难直接运行的，最好借助 Automator 套上一层壳。

![](.evernote-content/8250306C-91D0-4E9B-8B52-0B7B224E2591/D7B521CA-C262-4127-971E-6FA065015458.png)

创建日程提醒

在 Automator 里添加一个运行 Shell 脚本的模块，再把 rsync 命令写进去，备份的动作就完成了。实际上这个日程提醒是个 Application 文件，双击运行就可以开始备份，不过它会藏在系统文件夹里，平时不碍眼。

![](.evernote-content/8250306C-91D0-4E9B-8B52-0B7B224E2591/9E9C00A9-03BE-4C1B-8682-2843918BB7ED.png)

添加定时备份命令

取个名字，比如「定期备份」，保存这个日程提醒动作后，Automator 会打开原生日历应用，在里面显示刚刚一个名为「定期备份」的日程。接下来就像设置普通日程一样，调整它的开始时间（即开始备份的时间）、重复频率，刚才的备份动作就会定期运行了。时间一般设置成深夜，夜深人静让 Automator 自己慢慢备份，免得打搅日常工作。

![](.evernote-content/8250306C-91D0-4E9B-8B52-0B7B224E2591/C73D7E5C-FA54-42EC-A05E-C90FE0B838E1.png)

设置定期备份

这套定期备份比较基础，大家用过也许觉得还能改进，或者暂时不想用 —— 需要修改和删除 Automator 动作的话，都可以前往这个文件夹：

```
~/Library/Workflows/Applications/Calendar
```

![](.evernote-content/8250306C-91D0-4E9B-8B52-0B7B224E2591/50302E1F-1DCB-4A08-97D1-058D0FA908A4.png)

日程提醒藏在这里

小提示：删除日程并不会删除对于的日程提醒 Automator 动作，不想用时也可以在这个文件夹中把动作清理干净。

当然了，全自动备份的前提除了电脑本身开机，还需要备份所在的磁盘 / 网盘一直连着：如果你备份到网盘，电脑就得保持联网；如果保存到磁盘，则需要持续连接。这些网络和硬件条件能满足的话，也就可以享受全自动的备份。

至此，我们已经从备份单个应用配置，拓展到批量备份所有自动化工具的配置，最后还实现了自动备份到云端。

* 1 指没有安装任何自动化工具，基本只是装了几款「装机必备」的办公和聊天工具的电脑。
* 2 拥有多台电脑的读者还可以通过网盘保持各个电脑上的自动化环境一致。
