# 突发！OpenAI高层巨震，ChatGPT与CodeX或合并，超级Agent来了！

根据《连线》5 月 15 日的独家报道，在最新一轮组织调整中，OpenAI 总裁 Greg Brockman 正式接替仍在病休的 Fidji Simo，掌管公司产品战略；原 Codex 负责人 Thibault Sottiaux 则升任核心产品和平台团队负责人，统一管理 ChatGPT、Codex 以及面向开发者的 API。

![](.evernote-content/637BC692-BB07-466E-8350-92E3AAF3E6A7/A5BFC5BD-58C8-4A56-A2F9-BCF828B3E4F4.png)

Altman 背后的男人就是 Greg Brockman，

图片来源：《连线》

更重要的是，Sottiaux 也是 OpenAI 开发「超级应用」的负责人之一，同时 Brockman 已经在内部备忘录里明确表示：

**OpenAI 决定把 ChatGPT 和 Codex 合并成一个统一体验，更聚焦 AI Agent 的未来。**

如果说 ChatGPT 解决的是「问 AI」的问题，Codex 解决的则是「让 AI 做事」的问题，前者更多面向普通用户，后者更多面向开发者以及其他专业用户。

现在 OpenAI 要把两者合并，除了整合三个产品团队集中资源，大概率也是希望周活超过 9 亿的 ChatGPT 用户更多从「问 AI」到「让 AI 做事」，把更多真实任务交给 AI 完成。

毕竟这一轮 AI 的竞争，已经不只是模型能力之争，谁能成为用户每天处理任务的入口，谁才更有机会成为 AI 时代的超级应用。

**从实际表现看，Codex 也有这个能力。而另一方面，这也意味着那个统一 Codex、ChatGPT、Atlas 浏览器的超级桌面应用，应该离我们不太远了。**

![](.evernote-content/637BC692-BB07-466E-8350-92E3AAF3E6A7/85D103EE-C35C-42FE-B8FB-09418373E4E2.png)

图片来源：雷科技

**ChatGPT 和 Codex 合并，**

**超级Agent的雏形**

合并 ChatGPT 和 Codex 这件事，算不上突兀。

Sottiaux 早前就承认了 OpenAI 超级应用的存在，就在前两天，移动版 Codex 直接在手机里的 ChatGPT App 上线了入口。具体来说，你在手机上的 ChatGPT 就可以直接进入 Codex，连接桌面端运行中的环境，然后在手机上查看桌面端 Codex 的任务进展，推动长任务继续或者发起新的需求，让桌面端 Codex 去完成。

![](.evernote-content/637BC692-BB07-466E-8350-92E3AAF3E6A7/E53E0F81-C1A5-4481-B929-0CE0B81FCA25.jpg)

用安卓上的 Codex 连接了 macOS 上的 Codex，

图片来源：雷科技

之所以没有选择推出单独的 App，**既是因为 ChatGPT 在用户规模上已经有了巨大的优势，也在于移动版 Codex 从定位上也不是一个比较重的产品，更多还是桌面端 Codex 在手机上的延伸，而非迁移或者复制。**

第一点在桌面端上同样成立，尽管 Codex 非常成功，但用户和传播度还是更多局限在开发者、爱好者以及其他专业用户中。但既然 OpenAI 已然笃定 AI Agent 的未来，通过简化用户体验，降低更多用户接触、使用 Codex 能力的门槛和成本，对于更早占据用户心智就很重要。

这也是 OpenAI 超级应用的第一个雏形，最终用户看到的可能只是一个 ChatGPT 入口，背后却已经是一套跨设备、跨任务、跨工具的执行系统。

**但整合 ChatGPT 和 Codex 也不是 OpenAI 超级应用的全部，他们还计划把 Atlas 浏览器进一步整合起来：**

- ChatGPT 提供人与 AI 之间最自然的入口，用户已经习惯把问题、想法和任务丢给它；

- Codex 提供执行长任务的框架，尤其是规划、拆解、修改文件、运行命令这些能力；

- Atlas 浏览器则解决 AI 进入网页世界的问题，网页依然是现代工作最重要的操作界面之一，资料检索、SaaS 后台、办公系统、购物网站、企业管理平台，大量任务都发生在浏览器里。

![](.evernote-content/637BC692-BB07-466E-8350-92E3AAF3E6A7/FE4C54CF-7867-40E8-A19B-A127990B33D2.png)

图片来源：雷科技

OpenAI 的超级应用，大概率不会只是把三个图标合到一个窗口里。更合理的推想是，它会把 ChatGPT 变成主要的入口，把 Codex 扩展成通用执行引擎，再把 Atlas 作为网页和本地工作环境的桥。

用户输入一句话，系统可能先在 ChatGPT 里理解意图，再由 Codex 拆成步骤，涉及网页资料就交给 Atlas，涉及代码、数据、文档就调动本地环境或云端沙盒，涉及企业系统就通过 API 和连接器完成。

这也解释了为什么 OpenAI 会先强化 Codex。Codex 最初看起来是开发者产品，但它真正沉淀下来的东西，恰恰是 Agent 时代最稀缺的产品能力：长时间运行、多步骤规划、上下文保持、权限确认、版本回滚、结果验收。

写代码只是其中一个场景，分析数据、整理文件、生成报告、检查网页、迁移内容、处理表格，本质上都需要同一种自主代理能力。

**从这个角度看，Codex 很可能就是这个超级应用的核心发动机。**

![](.evernote-content/637BC692-BB07-466E-8350-92E3AAF3E6A7/2E7B43F1-E64B-48FB-90AB-EA51E76D31AA.png)

图片来源：OpenAI

更重要的是，OpenAI 的超级应用，可能不像传统超级 App 那样用支付、社交和内容聚合用户，而是覆盖用户大部分的工作和知识场景。**用户每天打开它，让它写代码、一起浏览网页、完成一系列工作任务，实际体验越好，用户越不愿意离开这套系统。**

当然，这条路也不容易。OpenAI 要做成超级应用，最难的不是把功能塞进去，而是怎么把体验磨到用户愿意长期依赖，这一点在 AI Agent 时代并不会改变。

但无论如何，OpenAI 必须这么做。除了简化用户体验的「官方理由」，与 Anthropic 和 Google 在企业、消费者市场的双线作战大概也是关键的外部原因，所以这既是一次主动的集中出拳，也可以看作一次被竞争倒逼的产品聚合。

**AI 时代的超级应用，**

**不会只有一种**

OpenAI 要做超级应用，是因为它还没有一个真正意义上的超级应用。回过头来看，国内巨头面对的问题就有所不同，它们手里本来就有超级 App、交易系统、支付体系、内容平台、办公协同和手机生态。它们的 AI 产品战略，不可能也不会照搬 OpenAI 的路径。

阿里最典型。千问 App 不只是千问大模型的 C 端入口，更是在某种程度上阿里 AI 超级应用的产品战略。阿里把淘宝、支付宝、高德、飞猪、淘宝闪购等能力接进千问，是为了让千问从「回答问题」走向「完成任务」。用户也不只是问它去哪玩、买什么、怎么走，还能直接让它规划、比较、下单、支付、改签、售后。

![](.evernote-content/637BC692-BB07-466E-8350-92E3AAF3E6A7/EA2A1ABB-315C-4E0D-8C01-D7661863A7EC.jpg)

图片来源：雷科技

**甚至从接入各大航司提供订机票等服务开始，千问的想象空间就已经不只限于阿里生态内部。**阿里生态当然是它最强的后厂，但千问想争的是 AI 时代的第一入口，要做的是把模型、工具、服务、商品、内容和外部任务连接起来，成为用户面对真实世界任务时的默认起点。

腾讯的路子又不一样。除了继续推元宝，腾讯还基于 OpenClaw 的产品形态推出了 QClaw、WorkBuddy 等 Agent 产品，让 AI Agent 以微信联系人或插件形式出现，用户可以在微信里向它发出指令，远程控制电脑完成传文件、发邮件等任务。

![](.evernote-content/637BC692-BB07-466E-8350-92E3AAF3E6A7/29ED0E17-256D-4B00-AE13-F463E950E188.png)

图片来源：腾讯

微信本身就是中国最强的数字生活入口，腾讯甚至没必要把所有人导向一个新的 AI App。更现实的做法，是让 AI Agent 出现在用户已经习惯的会话体系里。你可以像找同事、朋友、文件传输助手一样找一个 AI，让它帮你处理电脑上的任务。

**而微信还没有正式出手。**最近的财报电话会议上，腾讯基本也明示了微信 Agent 的存在，但短期内我们大概还看不到它的上线——微信往往只会在产品体验足够成熟时才端上桌，这是小龙一以贯之的产品观。。

字节则更像是另一种路径。

作为国内用户规模最大的 AI 应用，豆包把对话、搜索、语音、多模态和陪伴体验做到足够轻、足够高频。很多普通用户开始使用 AI，就是为了问问题、写文案、生成图片、语音聊天、辅导孩子、总结网页，这些更靠近日常场景的需求。**与此同时，豆包也在不断完善购物、支付等功能，产品路线上明显在和阿里的千问靠拢。**

![](.evernote-content/637BC692-BB07-466E-8350-92E3AAF3E6A7/6221BFF2-C29F-4DFB-AE93-101CE49AA635.jpg)

图片来源：雷科技

另一方面，除了豆包在占据国内大众的 AI 心智，字节在专业 AI 产品线上可能也是国内巨头中最完善的一家，即梦和剪映负责 AI 内容生产（SeeDream,SeeDance……），Coze 负责 Agent 搭建，Trae 负责 AI Coding，火山引擎负责企业模型和 Agent 商业化。

![](.evernote-content/637BC692-BB07-466E-8350-92E3AAF3E6A7/1E7475A1-CA6F-4C57-B9D2-BD80539DC4FF.png)

**写在最后**

回到三年前，ChatGPT 可以说最初定义了大家构想中 AI 时代的超级应用。但过去三年的变化实在太大，尤其是过去一年的 Claude Code、 OpenClaw，让我们都看到 AI Agent 的未来。我们需要的超级应用，其实更准确的名字应该是超级Agent，这正是OpenAI们在努力搭建的。

但未来的产品、未来的人机交互形式会怎么变，还是一个开放题。在这个背景下，OpenAI 的答案，是把 ChatGPT、Codex 和 Atlas 往一个统一的超级应用里收。阿里的答案，是让千问成为连接模型、服务和交易的 AI 入口。腾讯的答案，是让 Agent 进入微信这样的超级 App。字节的答案，则是用豆包占住大众心智，再用更完整的 AI 产品矩阵覆盖创作、开发者和企业场景。

路径不同，但终点很接近。
