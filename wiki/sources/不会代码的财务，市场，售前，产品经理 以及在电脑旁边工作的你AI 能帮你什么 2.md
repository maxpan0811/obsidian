1️⃣ 财务 · 李姐的私信
--------------

*李姐 · 某中型公司财务主管 · 她写道:*

**「我每月最累的就是月底两周:报表跑出来 · 我得做口语化分析给老板 · 还要追应收逾期 · 看新出的合规政策。AI能帮我什么?我连函数都不太会写。」**

### 我的回信

李姐,4件事AI全都能帮 · **不需要会写函数**:

· **月报口语化分析** · Excel导PDF → 「从老板视角找5件值得关注的事」 → 20分钟出活(原2小时)

· **追应收逾期** · 应收表丢给它 → 「30/60/90天分别多少 · 哪几个大户」 → 直接出优先级表

· **合规政策** · 国税公告PDF → 「对我们哪些业务有影响 / 要不要改流程」 → 白话总结

· **Excel公式** · 描述要做啥(「计算每客户逾期天数 · 数据在B和C列」)→ 直接出公式

━━ 李姐 的对话 ━━

帮我从老板视角找5件值得关注的事

1. 营收 +18% · 但毛利率降3pt 2. 应收30天 +¥230万 · 集中3个客户3. 销售费用 +28% · 主要投放4. 现金流 -¥80万 · 6月有回款5. ¥45万设备款下月到期

一句话总结给老板

本月营收增长但毛利承压 · 应收风险集中3大户 · 下月现金流压力可控

2️⃣ 市场 · 小程的私信
--------------

*小程 · 30岁 · 某消费品公司市场专员 · 她写道:*

**「我老板每次活动都要30版海报文案 · 我一周写不完。AI能帮我吗?但我又怕写得太套路 · 客户一看就是AI写的。」**

### 我的回信

小程,直接让AI写30版 · 大概率是30版废话。正确做法 —— **把它当副驾驶 · 不是自动驾驶**。3步法:

· **第1步** · 你自己写一版最满意的(哪怕只是「调性是学生省钱 + 日常自嘲」一句话)

· **第2步** · 让AI「按这个调性写10条不同角度 · 每条留钩子」

· **第3步** · 你挑3条 · 让AI各微调3个版本 → 30条都带你的底色

客户看不出AI味 · 因为**判断是你的**。

━━ 小程 的对话 ━━

调性「学生省钱·日常自嘲」 写10条200字内文案 · 每条留钩子

1. 这月生活费快光了 · 但还想喝奶茶 · 9.9元套餐刚好2. 房租交完 · 今天点外卖犹豫了20分钟 · 你呢? 3. ……

第1条好 · 再来3个不同钩子的变体

1A 9.9 = 半节课不睡觉的回报1B 9.9 = 学校楼下烤肠的两倍 · 但更顶1C 9.9 = 今天唯一不亏的消费

3️⃣ 售前 · 老周的私信
--------------

*老周 · 40出头 · 某B端解决方案公司售前总监 · 他写道:*

**「我们这行每周都要写方案PPT、做标书、回客户技术问题。我经常凌晨2点还在改。AI真能减负吗?客户问的问题很专业 · 我怕它瞎说。」**

### 我的回信

老周,安全姿势 —— **AI不替你判断 · 但替你干体力活**。4件事:

· **标书改写** · 招标文件 + 你5个历史项目 → AI按章节 / 评分点组初稿 → 你改 → 3天活变1天

· **方案PPT大纲** · 客户场景 + 你产品能力 → 15页大纲 + 每页要点 + 配图建议

· **客户技术答辩** · 30个技术问题 → 按「问题 / 思路 / 可能追问 / 风险」出答辩手册

· **竞品对照表** · 你能力清单 + 竞品公开资料 → 横向对照 → 你审一遍就能用

一句最重要:**AI出的内容都要你审 · 但审一遍比从零写快5倍**。

━━ 老周 的对话 ━━

招标文件 + 我们5个类似项目 · 出投标初稿

67页初稿 · 按招标8章节 · 对应评分点 · 替换为你5个项目案例 · 标红12处需确认数据

30个技术问题 · 整理答辩手册

4类:架构8 / 数据7 / 安全9 / 交付6每题给:标准答 / 可能追问 / 风险点 / 建议案例 标红5处需你跟产品确认

4️⃣ 产品经理 · 阿琳的私信
----------------

*阿琳 · 28岁 · 互联网公司产品经理 · 她写道:*

**「我数学不好 · 但老板每次都要『数据驱动决策』。我写PRD也写得磕磕巴巴。AI能帮我什么?但PRD这种东西它能懂我的产品吗?」**

### 我的回信

阿琳,AI的强项恰好是**「把模糊变清晰」**。4个场景:

· **PRD磕巴** · 大白话100字描述需求 → AI按标准框架(目标 / 用户故事 / 验收 / 边界 / 风险)展开8页

· **用户访谈整理** · 10篇文字稿 → 「共性痛点 / 个性反馈 / 优先级 / 原话」一张表 → 30分钟搞定

· **竞品功能矩阵** · 3-5个竞品资料 → 横向对比表(功能 / 定价 / 定位 / 差异化)

· **数据读懂** · 数据表 + 「转化掉8% · 帮我看原因」→ 归因思路图

━━ 阿琳 的对话 ━━

「批量导出」功能 · 帮我写PRD

【目标】单次下载多文件 · 响应 ≤3s · 错误率 ≤1% 【用户故事】客户经理一次下载本月所有合同 · 不用点50次 【验收】勾选行批量下载 / 单批100个 / 显示进度 / 失败重试 【边界】不影响单文件 / 不改权限 【风险】带宽峰值 / 大文件超时

本周转化掉8% · 看原因

3个最可能: 1. 注册-激活步骤4掉12pt(UI改版?) 2. 移动端占比涨 · 移动端原本低5pt 3. 新增渠道15% · 该渠道历史转化偏低 建议先看1

收尾 · 不是教程是态度 · 3件你该记住的事
-----------------------

看完5个回信,你可能发现一件事 —— **5个岗位用AI的方式不一样 · 但底层逻辑是一样的**。

总结成3件你能马上记住的事:

### 一 · AI不挑岗位

凡是「写字 / 看数 / 想方案 / 答客户」的工作 · AI都能帮上。

财务的报表 · 市场的文案 · 售前的标书 · 产品的PRD · HR的JD · 法务的合同 · 运营的复盘 · 销售的客户回邮 · 行政的会议纪要 …… 没有哪个不能用。

反过来:那些**「拧螺丝 / 跑腿 / 端盘子」**的工作AI暂时帮不上 —— 那些事不靠脑子。

### 二 · 用法只有3种 · 没有第4种

记住3个动作就够:

· **问问题** — 不知道的事直接问 · 把它当一个无限耐心的聪明同事

· **拍照** — 你看不懂的报表 / 合同 / 政策 / 体检 / 截图 · 拍下来让它解读

· **让它写** — 邮件 / 文案 / PPT大纲 / 报告 / 总结 · 你给方向 · 它出初稿

其他用法都是这3种的衍生。不需要学新东西。

### 三 · 把AI当你旁边那个聪明同事

这是态度问题 · 也是最重要的一点。

不要把它当工具 —— 工具是「我点一下你出结果」。

要把它当**「能问问题的聪明同事」** —— 你跟它来回聊 · 不满意就继续问「为什么 · 再换一个角度 · 用人话说 · 给我例子」。

越问越熟 · 越聊越精准。
