学得快的人，和一直卡住的人，差距通常不是智商。

甚至也不是时间。

## 1. 用20小时入门任何东西

有个真相，很多人不太想听。

学习任何新东西，最重要的其实是最开始的 20 小时。

这 20 小时，会把你从完全不懂，推进到“真的能上手”的状态。

但大多数人，会把这 20 小时浪费在错误内容、错误顺序和错误重点上。

Claude 可以立刻修正这件事。

提示词：

我想在20小时内学习【主题】。我是完全的新手。
请帮我拆成一个结构化学习计划。
告诉我：
1. 一开始最应该专注学什么
2. 哪些内容可以暂时忽略
3. 新手最容易犯哪些错误
4. 每个阶段应该练什么
5. 如何判断自己已经入门
请按优先级输出，让我可以直接照着执行。

你会得到一张路线图。

不是那种“多看看资料、多练练”的废话路线图。

而是一份真正有优先级、照顾新手视角的学习计划。

它会告诉你先学什么，暂时别碰什么，以及新手最容易踩哪些坑。

光这一点，就能帮你省下好几天。

## 2. 做一张一页速查表

你的大脑能吸收的东西，比你想象得多。

真正的问题不是记不住。

而是用的时候想不起来。

当你正在实际应用某个知识时，你经常会忘记信息放在哪里，然后开始翻笔记、找网页、查书签。

一张一页速查表，可以直接解决这个问题。

提示词：

请为【主题】创建一张一页速查表。
内容包括：
1. 核心概念
2. 关键框架
3. 常见错误
4. 必须记住的公式 / 步骤 / 方法
5. 一个我能在30秒内快速浏览的速查区
要求：
语言简洁，结构清晰，适合打印或放在屏幕旁边随时查看。

打印出来。

钉在旁边。

或者直接常驻屏幕。

你会发现，自己用到它的次数，比想象中多得多。

## 3. 一直考我，直到我崩溃

阅读会让人产生一种错觉：

我在学习。

但很多时候，并不是。

至少，不完全是。

坐在那里看书，让眼睛一行一行扫过文字，其实是非常被动的学习方式。

相关研究早就很清楚了：检索练习，也就是主动从记忆里提取信息、回答问题，效果远远超过反复阅读。

问题是，设计好问题本身很累。

Claude 可以替你做，而且做得不错。

提示词：

请围绕【主题】来测试我。
规则：
1. 从简单问题开始
2. 如果我答对，就逐渐提高难度
3. 如果我答错，请解释我错在哪里
4. 然后继续追问一个相关问题
5. 不要一次性给我答案
6. 一直持续，直到我连续答对10道题
请像一个严格但耐心的学习教练一样带我练习。

光是这一个提示词，就能暴露出你理解里的所有漏洞。

很多漏洞，你之前甚至不知道它们存在。

## 4. 搭一架学习梯子

按

大多数人放弃学习新东西，不是因为它真的太难。

而是因为他们没有走楼梯。

他们直接去爬墙。

基础还没稳，就冲进高级材料。

然后开始迷糊。

接着开始烦躁。

最后放弃，并怪自己不适合学这个。

但真正的问题，往往只是学习顺序错了。

学习梯子，就是用来修正顺序的。

提示词：

请为【主题】搭建一套学习阶梯。
要求：
1. 从零基础开始
2. 一步一步进阶到高级水平
3. 每一级都要建立在上一级基础上
4. 每一级说明我需要掌握什么
5. 每一级给出练习任务
6. 标出哪些概念必须先学，哪些可以后学
请用“第1级、第2级、第3级……”的形式输出。

从那一刻开始，你遇到的每个概念，都有了自己的位置。

它不再是随机砸到你脸上的信息碎片。

而是一级一级往上走的台阶。

## 5. 找到最好的学习资源

不是所有学习资源都一样。

这不是观点。

这就是事实。

有些书会彻底改变你的思考方式。

但大多数书，也就一般。

有些 YouTube 频道能帮你少走几个月弯路。

但更多内容，只会浪费你一个下午。

最麻烦的是，在你投入时间之前，你根本不知道哪个值得看。

Claude 处理过的信息，远远超过任何一个人一生能读完的量。

所以，用它。

提示词：

请帮我找出学习【主题】最好的资源。
请按类型分类：
1. 书籍
2. 免费课程
3. YouTube频道
4. 播客
5. 网站或文档
6. 实战项目
每个资源请说明：
- 适合什么阶段的人
- 为什么值得学
- 我应该从哪一个开始
- 学习时要注意什么
请不要只列清单，要告诉我选择顺序和原因。

这里最关键的是最后两个字：

为什么。

因为“为什么先看这个”，比单纯给你一串资源名单重要得多。

它能帮你判断，这个资源到底适不适合你现在的位置，而不是只适合那个未来已经很厉害的你。

## 6. 使用费曼学习法

Richard Feynman 一生都特别执着一件事：

如果你不能把一个东西解释得很简单，那你其实还没有真正理解它。

你只是以为自己懂了。

这个方法会让人有点不舒服。

但重点就在这里。

提示词：

我会尝试把【主题】解释给一个12岁的小孩听。
你的任务是：
1. 当我说得不清楚时，立刻打断我
2. 当我使用没有解释的专业术语时，指出来
3. 当我的逻辑跳跃时，提醒我
4. 当我解释完后，告诉我哪些地方说对了
5. 哪些地方说错了
6. 还有哪些理解漏洞
请不要为了礼貌而放过问题。我要的是诚实反馈。

然后你就开始讲。

可以打字。

也可以口述。

怎么自然怎么来。

Claude 会在你解释不清楚的地方打断你。

它会指出你无意识使用的术语。

它会发现你一带而过、其实并没有讲明白的漏洞。

等你说完，它会给你一份诚实、结构化的反馈：

哪里理解得很稳。

哪里说错了。

哪里还有空白。

这种反馈，大多数人平时根本拿不到。

因为现实里，大多数人太礼貌了。

他们不会直接告诉你：

你这里其实没讲清楚。

Claude 不会那么客气。

而这正是你需要的。
