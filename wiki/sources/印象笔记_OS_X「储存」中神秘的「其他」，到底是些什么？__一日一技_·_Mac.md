---

![](.evernote-content/F9E903A0-69DA-4085-AC38-5AE3CC59D811/1DCD58E7-ABD6-4B84-8C41-7A68EE864696.jpg)

需求
--

笔者经常被问到的一个问题是：在 Mac 的「关于本机 - 储存」中的「其他」到底是些什么？它们是垃圾文件么？我可以删掉它们，为内置磁盘腾出一部分存储空间么？

![](.evernote-content/F9E903A0-69DA-4085-AC38-5AE3CC59D811/FD9D5559-93DF-45DF-A213-87E7CFB45ADB.jpg)

需要明确的是，当我们连接外置磁盘空间或 iOS 设备到 iTunes 时，也是能够看到「其他」这一类别的。因此，初看起来，「其他」存储中的文件和一般意义上的「垃圾文件」肯定是有着相当的距离的。

![](.evernote-content/F9E903A0-69DA-4085-AC38-5AE3CC59D811/34A229F9-23E3-485F-8611-3C24C466F17E.jpg)

在本文中，我将具体解释「其他」存储出现的原因、其中所包含的主要文件类型，并在最后就是否需要对其进行删除这一问题给出建议和解答。

磁盘占用统计如何得出？
-----------

我们在「关于本机」中查看到的系统「储存」简报信息，其实是通过 Spotlight 所倚赖的一套元数据索引（Metadata Index）机制而实现的。简单说，元数据可以理解为对整个文件系统（也可以是外置磁盘中的文件）所做的快速分类和索引。举例而言，一张照片的元数据信息，就包括其大小、图片尺寸、分辨率、拍摄时间等等。

通过对不同文件类型的大小进行统计，自然可以快速统计出各类文件所占用的磁盘空间。而那些不被 Spotlight 所识别的文件、数据类型，也就自然会被归入到「其他」中去。

当我们以安全模式启动 Mac 后（具体方式是：听到启动声后，立即按住 Shift 键），再去查看 OS X 的「关于本机 - 储存」信息，就会发现所有已经被使用的磁盘空间，均被归类于「其他」。这是因为，当 Mac 以安全模式启动时，系统会跳过启动登陆项目和其他应用程序，而只加载必须的驱动和应用，因此也就不再具体统计不同文件类型在内置磁盘中的占用情况了。

![](.evernote-content/F9E903A0-69DA-4085-AC38-5AE3CC59D811/B8FC27AC-2E27-4C66-BBDE-6A79BBC1BAA0.jpg)

「其他」存储中有些什么？
------------

总的来说，那些不能够被归入应用、音频、影片、照片、Time Machine 备份的文件，都会被标记为「其他」存储。

下面是一些被归入「其他」存储中的典型文件类型示例：

* OS X 系统本身；
* 在 OS X 系统文件夹中的内容，例如系统和「资源库」（Library）文件夹；
* 暂存文件，例如系统交换文件以及你的 Mac 睡眠镜像（Sleepimage）文件；
* 应用缓存、系统缓存以及崩溃日志；
* Spotlight 和自动保存的版本数据；
* 由应用创建的非 Time Machine 备份文件（例如由 iTunes 创建的 iOS 设备备份，1Password 自动备份的内容等）；
* 个人信息，例如你的邮件、联系人及日历信息等；
* 在每个用户文件夹中的内容；
* 应用插件和应用扩展；
* 无法被 Spotlight 搜索所识别的文件类型；
* 偏好设置和用户设置文件；
* 在「下载」（Downloads）文件夹中的内容；
* 浏览历史、书签、Cookies 和网站图标；
* 钥匙串和插件（Widgets）；
* Apple 或其他第三方应用的支持文件；
* 其他系统文件，例如目录、磁盘目录等；
* iCloud 文稿和数据同步文件夹。

「其他」≠ 无用文件
----------

从上述「其他」存储所包含的文件类型中，我们不难想见其体积缘何如此巨大的原因。同时也不难看出，除了部分可以使用清理应用（如 [CleanMyMac 3](http://sspai.com/tag/CleanMyMac)）加以处理的缓存文件等之外，其他文件类型（大）都是系统正常运行的重要文件。因此，笔者并不建议大家随意对其进行删除等操作。
