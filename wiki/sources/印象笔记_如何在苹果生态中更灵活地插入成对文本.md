如何在苹果生态中更灵活地插入成对文本
==================

| 本文为付费栏目文章，您已订阅，可阅读全文 |

前不久有读者在群内提问，**如何在文本两端快速插入括号。**仔细一想，平时插括号的方式还真不方便，除了来回挪光标，可能还要切输入法，打断写作思路。

有些代码编辑器具有自动补全引号、括号的功能，比如 CodeRunner，你输入一个引号 `"`，它就补上一个引号 `"`，并且把光标移到两个引号中间，等待你继续输入。

![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/3724ACFA-FE07-41BF-BF6E-27212C5F237B.gif)

自动补全引号

而在日常工作中，「插引号」还涉及更多场景，简单到括号、引号、部分 Markdown 符号，复杂如网页的 HTML 标签、博客的 Meta 标记…… 我在 Mac 和 iOS 设备上各实现了一套插入成组符号的动作，模仿代码编辑器的**补全**功能，简化这类**成组成对**符号的输入。

![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/A6125170-18EA-4035-B332-84E501A10781.png)

两个平台的上括号插入动作

macOS 最优解决方案：Keyboard Maestro
-----------------------------

macOS 上插括号的需求可以通过 Keyboard Maestro 来满足，不仅不用面对脚本，同时可以很好地认识 Keyboard Maestro 中的两个重要概念：

1. **变量 Variable**：用于临时存储数据，我们将用它暂存剪贴板内容，避免影响剪贴板的原有文本。
2. **菜单 Palette**：在 Macro 过多时可以通过 Palette 把它们收纳在一起，方便取用。

想用到 Keyboard Maestro 进阶功能的话，这两个知识点几乎绕不过去。于是就有了这篇文章，一方面直接解决各种「插括号」的问题，另一方面也借机讲讲 Keyboard Maestro 里这两大重点概念。除了 macOS，文章后半部分也会讲讲 iOS 的解决思路和自定义方法。

暂时只推荐 Keyboard Maestro，注意是考虑到它操作灵活、实现简单 1，后期自己改动也比较方便。

当初群内讨论插括号问题时，就有读者提到 PopClip 也可以实现，唯可惜不能全键盘操作。就我自己的体会来说，Keyboard Maestro 在操作方式上的确也更舒服，有 Keyboard Maestro 的读者推荐直接采用本节的方案。

简明起见，文章统一以「插括号」为例，需要其他符号时自行修改两边文本即可。

### Macro 使用

[> 动作下载](https://github.com/BlackwinMin/Keyboard-Maestro-gallery/tree/master/Insert%20Parentheses.kmmacros.zip)

诸位下载的其实是一组动作（在 Keyboard Maestro 中称作「Macro」），包含了 5 种预设的常见符号，随后也将介绍自行修改的方法。

![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/A5EBA2E2-426E-4816-A5EC-E81DFC91EB1D.png)

装好后的 Macro

经由快捷键呼出后，可以任选想插入哪种括号或其他标记：

1. 在任意编辑器内选中需要加括号的文本（不选也可以），再通过快捷键 `⌃Control-⌥Option-⌘Command-K` 呼出 Keyboard Maestro Palette 菜单；
2. 选择动作，可以挪鼠标点，也可以按动作首字母直接触发。 Palette 上有当前可选动作的提示（下图中颜色较暗的字母），比如按 `H`，就能触发 Insert HTML Label 动作。这提示我们要可以多采用字母加数字的命名方式，以便全键盘操作。   

   ![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/2D2C0C02-2717-4FA6-8FBC-87BB5ADC9ED0.png)

   提供可选符号的菜单
3. 根据所选文本插入不同内容：
   1. **选中文本时**：在文本两端插入括号。修改文章时我经常使用这个动作，比如发现了需要用着重强调的用词，就可以选中文本后快速插入成对引号，而不必在文本两边分别手动插入。  

      ![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/0DC4A60A-88B8-4913-BD5D-91C8F9DD0C05.gif)

      在文本两段插入括号
   2. **没选文本时**：直接插入整组括号，并把光标移到中间等待进一步输入。写文章时这个动作更加常用，有时候需要插入 HTML 代码来展示特殊效果，就可以先插一段代码再接着打字。

      ![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/A1B3E3A8-87E8-49E2-90F1-83B20A810857.gif)

      原地插入整组括号
4. 整个插入过程结束后，恢复原先的剪贴板内容。这是一个小细节，尽量避免插入操作对剪贴板的影响。  

   ![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/1D096119-82A9-4E31-86DF-E562C8C32278.gif)

   插入括号后不影响原有内容

通过 Keyboard Maestro 实现的这套「插括号」动作，相比使用其他方案有着明显的好处：

* **可以全键盘操作**，手不离打字区域即完成插入。
* **方便自定义备选括号样式**，比如添加 `[]`、`【】` 等括号。我提供的动作中包括了两种括号、两种引号和一对 HTML 标签。
* **完全用 Keyboard Maestro 自带模块实现**，无需掌握额外的脚本知识，自己改起来无门槛。

当然在核心的「插括号」功能上，其他动作也不逊色。我们先从最简明的 Keyboard Maestro 实现开始，看看这些括号到底是怎么打出来的。

### 实现原理和自定义方式

我们先来了解单个 Macro 动作，熟悉之后再批量生产用于插入其他符号的「改款」。整个 Keyboard Maestro 动作流程分 4 大步：

![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/0FD55EFD-15C3-49C7-8B86-F27E299D6134.png)

Keyboard Maestro 的整个流程

1. 把原有剪贴板内容暂时保存在 `temp` 变量中，稍后取用。
2. 拷贝当前选中的文本，即便没选也强行拷贝一下。
3. 根据是否选中文本，用不同的形式插入括号。
4. 恢复原先的剪贴板内容。

**知识点：Keyboard Maestro 中的变量**

所谓变量，就像一只只有名称的**篮子**，里面可以存放文本、链接、图片等各类内容。在自动化工具中，变量的典型用法就是暂时保管一些内容，后续步骤中要用的时候再把内容物倒出来。

Keyboard Maestro 对于变量的处理比较友好，不需要写代码，在图形界面中通过特殊样式的文本就可以使用变量。在 Keyboard Maestro 里，变量主要有两种写法：

1. Keyboard Maestro 自带变量：两边用百分号包裹起来，比如系统剪贴板就是 `%SystemClipboard%`。
2. 用户自定义变量：形如 `%Variable%变量名%`，名字可以自己设置，我们稍后用于临时保管系统剪贴板内容的就是 `%Variable%temp%`。

除了把文本「装进」变量，我们也可以直接把一个变量里的内容赋予另一个变量，一会儿我们就会把 Keyboard Maestro 内置剪贴板变量中的文本暂存到自定义变量中，从而临时保存剪贴板内容，避免它被插入括号的操作影响。

整个流程中需要绕弯的是「判断是否选中文本」这一环。macOS 中没有「Current SeletedText（当前选中文本）」这一概念（关心原理的可以看我 [以前这篇关于 macOS 剪贴板的文章](https://sspai.com/article/44129?series_id=9)），我们得靠剪贴板来曲线实现一下 2。

大致想法是这样的：不论如何，先复制一下（按一下 `⌘Command-C` 键），如果当前剪贴板里的内容和 temp 不一致，就证明复制到了新东西，倒推过来说明**肯定选中了文本**；反之，如果剪贴板没动静，一般意味着没选中任何文本 3。

把思路反映到 Keyboard Maestro 中，就是添加一个 if 判断模块，把两种情况分开来考虑：如果没选中文本，就插入 `[%|%]`，其中 `%|%` 表示光标位置，即插入一堆 `[]` 后把光标移到中间；选中文本的话，则通过 `[%SystemClipboard%]` 把文本连带两侧括号一起贴上，看上去就和在两端各加了一个括弧一样。

![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/AE9CA805-FC6B-4048-A7DA-E6B69307A54C.png)

分情况插入不同内容

当然，不管最后贴上了什么，为了不影响剪贴板原有的内容，我们都要在结束 if 判断及后续操作后，把 temp 变量中暂时保管的内容还给 `%SystemClipboard%` 变量。

![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/F981A8F2-34B5-4AC6-9454-169995149F2C.png)

恢复原有剪贴板内容

单个动作就此完成。接下来我们可以为这个 Macro 制作一堆副本，满足不同类型符号的插入需求。

随便挑一个副本，首先改动其中的符号。直接跳到第三大步中的**「Insert Text by Pasting」**模块，可以看到两种情况下粘贴的分别是 `[%|%]` 和 `[%SystemClipboard%]`。

![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/4B0BF64A-AE85-4217-A392-A02BC77C1943.png)

修改这两处的括号

中间部分不用变，直接把两侧符号换掉即可。除了括号，其实还有很多拓展用法，比如换上 HTML 标签，变成这样：

```
<font color="grey">>%SystemClipboard%</font>
```

如此一来便可以把文字渲染成灰色，写文章时插一些编注和补充说明的文字很顺手。当然具体颜色可以自由指定，能设置的文本属性也不限于颜色，这就留给和 HTML 打交道的读者自行发挥。

![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/C5200D21-DAF6-41CF-85F6-6367BB1C85D2.png)

灵活插入成组 HTML 标签

修改括号后，工作就此结束。特别注意，不必改动副本 Macro 动作的触发方式，要得就是它们都用**同一组快捷键，**这样可以引发一个特殊 Palette：**冲突菜单。**不过具体的快捷键各位可以随意改，只要保持插入符号类的触发方式一致即可。

![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/E99DF37B-7BBF-44C2-BA87-82CE77EF9C46.png)

引发冲突菜单

上图黑色的浮窗就是冲突菜单，其中包括了所有可选的 Macro 动作。这样的好处显然就是节约键位，用一组快捷键就能管所有需要插入的成组符号。平时用到的符号再多，也不用增加记忆快捷键的负担。

注：起初的动作完全靠脚本实现，但因为 Mojave 权限问题运行得很蹩脚，大家也学不到太多能直接用的知识。碰巧发现 Keyboard Maestro 可以更方便地完成，我们就优选此方案。

iOS 解决方案：Drafts
---------------

### 动作使用

考虑到需要选中文本，建议直接在键盘中使用这一动作，不致于遮挡所选内容。

![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/3A1B711D-8695-4C53-B3CA-79DCE83327D3.jpg)

建议在键盘中使用本动作

选中文本并运行 Drafts 动作后，可以看到预设括号选项，目前我准备了 **3 种样式（包括一对儿书名号）**，后续大家可以自己增减。和 Keyboard Maestro 中的操作类似，Drafts 版也会视情况插入不同内容：

1. **选中文本时，在文本两端插入括号。**其实和 Markdown 加粗符号的实现原理类似，就是把用左右两侧符号加上所选文本，整体替换掉当前选中的内容，看上去就像在两侧插了符号一样。   

   ![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/D8888BE3-C569-4AF7-9482-25824DA7126D.gif)

   选中文本时
2. **未选文本时，直接插入整组括号，并把光标移到中间。**配合 iPhone 上的重压手势或 iPad 的双指手势，还可以快速移动光标，在打完内容后马上跳出括号，继续输入别的东西。   

   ![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/DC7E416A-E848-4CE6-8515-CE238E66C001.gif)

   未选文本时

Drafts 版的各组可选符号都集成在一个动作中，直接在**弹窗（Prompt）**中进行选择即可，这点比 Keyboard Maestro 更加方便，也更符合移动端的操作（毕竟设备屏幕空间局促得多，整一堆动作按钮并不明智）。接着，就看一下如何自定义符号，把得动作打造更加个性化。

### 自定义符号和键位

整个 Drafts 动作由一个弹窗和一段 JavaScript 代码组成，弹窗提供符号备选项，乐意加多少就加多少；代码负责解析所选符号，并视情况插入不同文本。

![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/400C25E4-3265-4229-ABDF-5586C8F773B1.png)

Drafts 动作的组成

主要的自定义部分在弹窗步骤 **BUTTONS** 部分，可以看到预设符号是写成这样的：

```
(Text)|[Text]|《Text》
```

用 `|` 分隔开的 3 组文本，表示可选的 3 组符号。如果需要添加其他符号，比如**引号**，就可以继续在后面加上 `|"Text"`（别忘了加上 `|` 分隔符）。其中的 `Text` 务必带上，因为后面的 JavaScript 脚本需要以它为记号，把选中的文本替换上去 4。添加相对复杂的 HTML 代码也是同样的操作，用 HTML 标签把 `Text` 包裹起来便是，形如：

```
<font color="grey">Text</font>
```

![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/590B3A5D-80A5-4F64-93A5-0671E2273824.jpg)

在 BUTTONS 中自定义符号

至于 JavaScript 代码实现，其实比 Keyboard Maestro 更简洁：由于 Drafts 中具有 SelectedText（所选文本）对象，剪贴板一开始就不会被影响。整体思路是差不多的，也没有什么特别值得改动的，这里不展开介绍。

此外我还为这个动作绑定了快捷键 `⌃Control-⌥Option-⌘Command-K`，和 Mac 上保持一致，方便 iPad 用户使用。美中不足的是弹窗中的选项不能直接通过键盘选中，还是要手工点一下。

![](.evernote-content/5D33E999-7E61-4649-8894-32303B8822A4/388C7F61-FD64-4136-86EC-AFFE444596D8.jpg)

为动作设置和电脑一样的快捷键

具体键位各位也可以在动作编辑界面中自行修改，总之最后自己用得舒服最重要。
