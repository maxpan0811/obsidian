用 Drafts 和 Evernote 做更漂亮的摘抄
===========================

| 本文为付费栏目文章，您已订阅，可阅读全文 |

不少人都有做摘抄的习惯。

但是摘抄这件事隐藏了两个难点，其一是**能不能快速摘录**，其二是**有没有美观的保存工具**。问题在于，摘录时的简便快速和回顾时的整齐好看似乎不能兼得 —— 如果你没有坚持下来，一定程度上确实要怪缺少合适的摘抄工具。

我虽然平时也在做摘抄，但对于自己的工作流并不满意（我直接用 Markdown 格式来记录，展示效果很单薄，可能采用类似方法的读者也有同样的不满）。直到 @Hum 和我分享了一套**「用 Drafts 记录、发送到 Evernote 保存和展示」**的摘录流程，我才找到比较理想的方案。

![](.evernote-content/FF817F5E-C4F0-40D0-8373-88993BAB303D/B08D3AF1-8FCD-472E-92DC-FE84529E4B79.png)

从 Drafts 发送摘录内容到 Evernote

[> Drafts 5 动作下载](https://actions.getdrafts.com/a/1Wp)

这个方案涉及的动作用起来非常简单，就像往常一样在 Drafts 里记东西，然后一键发送到 Evernote，就能把摘录内容带上好看的样式加进笔记里。

注：本动作不支持 Drafts 4。Evernote 和印象笔记的使用效果一致，具体设置方式下文会有体介绍。

动作使用
----

摘录的动作使用起来基本就是一键完成，完整的流程是：

1. **在 Drafts 中记录**：第一行是小标题，换行后都是摘录的具体内容。
2. **选择放到哪条笔记**：根据 Drafts 弹窗提示，选择一个要收录新摘抄的笔记。
3. **发送到 Evernote**：草稿会被渲染上样式（默认是少数派的主题），带着小标题、摘录内容和当前日期时间追加到指定的 Evernote 笔记后面。发送成功后草稿会被归档。

![](.evernote-content/FF817F5E-C4F0-40D0-8373-88993BAB303D/998C51A2-DAC0-4BD4-8CFD-04D4FD01081E.gif)

用 Drafts 做书摘到 Evernote

动作模板里的笔记是根据我自己所在场景设置的，诸位的情况当然各不相同，我们可以稍微改动一下，好让 Drafts 顺利找到各人的笔记。在这一 Drafts 动作的编辑界面中找到「Steps-Prompt」，拉到最下面就能看到用 `|` 分隔开的几条预设笔记。

```
书摘|网页|会议|台词|歌词
```

把 `|` 间的笔记标题换成你自己在用的，就能把草稿准确追加进去。如果你有专门的摘录笔记本，也需要去「Steps-Evernote」步骤中把「Notebook」填上（默认是不添加到指定笔记本的，而且因为 Evernote 是个在线服务，新建笔记本后需要过几分钟才能正常运行添加摘抄的 Drafts 动作）。

![](.evernote-content/FF817F5E-C4F0-40D0-8373-88993BAB303D/57C2CB67-A3BB-4A49-B529-DA577EB8495F.png)

添加预设笔记

**小技巧：在 Drafts 选择登录 Evernote 或印象笔记**

Drafts 5 同时支持 Evernote 和印象笔记，在第一次运行相关动作时 Drafts 就会提示进行授权。特别注意，**不同网络环境下能够选择的服务会有不同**，一般在授权界面左上角有切换国内 / 国际版的按钮，而在使用某些网络优化工具后就只能看到国际版（Evernote）。如果想登录印象笔记，需要的境内正常网络下操作。

![](.evernote-content/FF817F5E-C4F0-40D0-8373-88993BAB303D/64CA595B-CA90-4131-8766-9E40B3656E39.png)

在 Drafts 中提供 Evernote 授权

Evernote 分拆后不少读者可能同时拥有两个帐号，如果在 Drafts 中授权后需要更换，需要在「Settings-Credentials-Evernote」中退出原有账户。

![](.evernote-content/FF817F5E-C4F0-40D0-8373-88993BAB303D/B511E5D7-6FE9-498C-91C9-889846ECEA0A.png)

退出 Drafts 中的 Evernote 账户

分几个不同笔记是自然是为了区分具体场景，多数人在上学时应该就有好几个笔记簿用来记录不同学科的内容。用 Evernote 收录的摘抄也一样，可以（也应该）分不同笔记。作为参考，我的分类是：

* **读书摘抄**：阅读时的好句好段，包括关键概念、精妙比喻和一些日后写作可能用上的案例。本文介绍的动作最适合做这种需要手打一部分内容的摘抄。
* **网页摘录**：在线文章里读到的好内容，代码、软件技巧另有专门笔记本，一些评语、生活技巧就放这儿了。Hum 在《[捷径：由浅入深完全指南](https://sspai.com/series/68)》里有个专门的 [捷径动作](https://www.icloud.com/shortcuts/6506b5a238b84d50a18e099d08168dc3)，可以摘录内容加上带超链接的标题。
* **会议点子**：开会时同事会提出很多好点子，尽管不会全部被当即执行，但总得有个地方统一收集起来，哪天机会来了就可以实施。由于需要手动录入，这些内容也适合用 Drafts 来捕捉。
* **妙语汇集**：这个来源和内容就比较杂，可能是社交网络上某位朋友发的点评，也可能是播客节目里一段恰到好处的讽刺。
* ……

部分读者可能对「追加」的概念少有疑惑，为何不直接新建笔记？的确，如果要精读一本书或一篇文章，建议还是老老实实开一个新笔记文档，甚至有必要打印一版 PDF 出来直接做手写笔记；但摘录不一样，这些零散句子甚至关键词的最好归宿还是**统一的笔记**，回顾效果要好于一大堆碎片。以前我每会为条摘录创建一个新 Markdown 文档（搭配 nvALT 做速记），只管记得快、不管看得舒不舒服，积重难返，很多内容基本算是荒废了 —— 直到换回 Evernote。

![](.evernote-content/FF817F5E-C4F0-40D0-8373-88993BAB303D/BD2F55F3-5703-4F84-88DE-5FA43CCC1E03.png)

统一的摘录和分散的摘录

原理解析
----

开头我们提到过，摘录的关键在于**记录**和**展示**这两个环节，在原理解析部分我们会从最终呈现的效果开始，倒退回来逐个击破。

### 在 Evernote 中创建带格式的文本

摘录内容的保存工具务必要有一定的格式（即**支持富文本**），首先是为了提高**可读性**，方便一眼区分标题、摘录内容和日期，在此基础上上也可以追求一些美观。我曾经试图让自己相信格式不重要，但事实上，尽管 Markdown 源码也看得懂，但我仍然没太大兴趣面对那些干巴巴的纯文本。同时，这些样式也不能丢给做摘抄的人手动添加，所以最后的保存工具还要允许一定程度的**自动化**。满足这两项的工具里，最合适的就是 Evernote。

一套比较理想的格式是：

1. 小标题居左，加粗
2. 摘录内容居左，加上引用效果
3. 日期时间靠右，不用其他特殊样式 1

其中引用效果套用了少数派的主题样式，保证视觉上有一定区分度的同时比较简洁。当然在介绍完摘录模板后，你也可以换上其他样式。现在打开 Drafts 动作编辑界面里的「Steps-Evernote」，可以看到已经设置好的 TEMPLATE（模板），上面的摘录样式就是由这堆 HTML 格式的文本控制的。

![](.evernote-content/FF817F5E-C4F0-40D0-8373-88993BAB303D/8687C9BF-40FF-4969-8FAC-773DB21DE935.png)

模板和 Evernote 中摘录内容的对应关系

我们暂时不管其中 `[[title]]` 之类的东西，先关注一下怎样设置样式。模板中的 3 个 HTML 标签（形如 `<标签名>` 的代码）设置了摘录中各部分的样式，第一个 `<p>` 渲染小标题，中间的 `<blockquote>` 复杂内容，第二个 `<p>` 控制日期。如果你想自己修改，只需要去找个 HTML 模板（可以看我以前整理的 [这篇](https://sspai.com/post/43873)），扒出其中对应标签的代码替换上去。

至于下方的输出格式，除非你很熟悉 Evernote 自己的 ENML 语法（一种类似 HTML 变种的东西），不然选择 Markdown 就行。在我们这个摘录场景中也暂时用不到 ENML 的样式。

**知识点：在线服务的优势**

如果只考虑富文本和自动化，有一批支持 URL Scheme 的笔记工具倒也凑合，但 Evernote 的**服务属性**让它从中脱颖而出。所谓的服务，主要表现就是能够通过 API 来传递数据，而不用走本地的 URL Scheme。尽管对于网络有一些要求，但服务的功能往往更全面，而且可以**在后台隐形运行**，不需要跳到应用里面。之前 @Hum 介绍过一个用 Drafts 添加 Todoist 任务的动作，同样借助服务实现了一键添加。

![](.evernote-content/FF817F5E-C4F0-40D0-8373-88993BAB303D/63799B9B-B2DF-4D89-B737-2CB8D527E9BC.jpg)

带有格式 Evernote 摘抄

在摘抄带上各种格式之后，对于记录的要求就高了很多 —— 而且，相信多数人都同意手动设置是非常耽误功夫的。这时候就要轮到 Drafts 出场，让格式设置自动完成。

### 将笔记模板结合到 Drafts 里

搞明白摘录格式怎样生成之后，我们再回过头来解决 Drafts 和 Evernote 之间文本传递问题。照例梳理一下，可以发现一共有 4 组数据要传给 Evernote，它们的在 Drafts 的 Evernote 步骤中都有对应的**标签**（即上一节中暂时跳过的 `[[标签名]]` 部分）来表示：

1. `[[prompt_button]]`：上一步弹窗中选择的按钮名称，负责指定我们预设的笔记名。
2. `[[title]]`：草稿的第一行，在 Drafts 里视作草稿标题，也将作为摘录内容的小标题。
3. `[[body]]`：草稿第二行开始的内容，都会作为摘录内容并套用引用格式。
4. `[[time]]`：运行动作时的日期和时间。

![](.evernote-content/FF817F5E-C4F0-40D0-8373-88993BAB303D/34EDA3FB-814E-4621-AE5D-CFE8A013EA23.png)

Drafts 标签和 Drafts 草稿的对应关系（部分）

如果希望摘抄模版能够渲染 Markdown 格式的草稿，可以在 `[[body]]` 两端加上百分号变成 `%%[[body]]%%`，这样 Drafts 就会自动把草稿的主体内容部分转换成 HTML。不过这个自带的渲染方式似乎对英文符号支持不是特别好，我在摘录一些书名带英文冒号的文章就发现动作无法顺利运行，各位可以视自己的阅读环境决定要不要改动渲染格式。

![](.evernote-content/FF817F5E-C4F0-40D0-8373-88993BAB303D/94F2D206-1536-485B-91DC-FBE58C3E9A47.png)

渲染 Markdown 列表

通过这 4 个标签，Drafts 草稿中的数据就能被成功传进 Evernote。可以灵活组合标签和 HTML 模板是我喜欢 Drafts&Evernote 摘录方案的原因之一，不需要理解兜圈子的代码或 URL Scheme 跳转逻辑，简单拼接就能实现格式设置和笔记追加。

小结
--

能做摘录的工具不在少数，但是真正同时照顾好的「速记」和「回顾」两点的实在求之不得（至少在 iOS 是这样），相当一段时间里我只是组合使用 Drafts 和 Byword，用 URL Scheme 和 Markdown 文档勉强记录零星内容。但是就像文中不断提到的那样，这套方法的展示效果实在令人无法满意。

Hum 给我的这个动作一石二鸟地解决了记录和回顾展示问题，让摘录的整个过程都变得很舒服。另外，**不订阅 Evernote 进阶或高级版并不影响使用这一工作流**，只在两台设备上同步的话不想额外花钱也没关系。
