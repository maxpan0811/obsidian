# 在电脑上用你喜欢的方式管理 iOS 设备：数据管理软件 iMazing

---

![](.evernote-content/90726571-7B49-4F03-81EB-A577069B42C6/E65F99C1-8D51-4AFB-BF34-226B165004DA.jpg)

定期备份 iOS 设备资料是一个好习惯，因为你不知道你的设备什么时候被偷被抢，或者被你鼓捣成一堆金属废料。

而作为 iOS 官方备份工具的 iTunes 却一直广受诟病，主要原因也很简单， 就是因为它太简单——或许是因为我们都太向往自由，所以我们更希望摆脱 iTunes 强加给自己的条条框框，用一种更个性化的方式去存取或管理自己的设备资料。

而除了自由，不少用户还反应 Windows 上的 iTunes 简直是灾难，各种卡壳停滞重复操作等问题一直不被解决，以至于不少 Windows 用户都放弃了将数据备份在电脑上这件事。

不过苹果也并非那么「不近人情」，根据一些被开放的 API，在不越狱的前提下，我们仍然有除了 iTunes 外的管理工具选择，在这类软件中，本文的主角 iMazing 则被广泛推荐，它的前身 DiskAid 紧随第一代 iPhone 的发售而问世，是直至目前仍在不断升级革新的少有的元老级软件。

![](.evernote-content/90726571-7B49-4F03-81EB-A577069B42C6/FF3E6103-8586-4A96-9C16-8FCEE5DBAFC1.jpg)

查看设备信息
------

与其描述 iMazing 的界面外观，不如来聊聊它的一个贴心功能——查看设备信息。相比臃肿的 iTunes，专注于数据管理的 iMazing 在界面上则显得非常清爽简洁，与此同时还能从首界面中看到设备的重要信息，除了系统版本、上次备份时间和存储空间占用等一些基本信息外，还能直接跳转查询 iOS 设备的保修范围和电池健康状态。

![](.evernote-content/90726571-7B49-4F03-81EB-A577069B42C6/387B2D36-029E-43D6-A64A-D4DBB8AA8033.jpg)

同样的备份，更精细的恢复
------------

### 数据备份

在介绍 iMazing 功能之前，我想先强调一点的是 **iMazing 只是借助了 iTunes 的开放 API 接口以实现数据管理，我们可以把 iMazing 当成是 iTunes 的一个主题皮肤或是小插件，而不能指望它拥有超过 iTunes 权限的数据管理功能。**

![](.evernote-content/90726571-7B49-4F03-81EB-A577069B42C6/BD7A9B64-7C8E-40D3-B1EB-2DCCA3957B56.jpg)

在最基础的备份上，iMazing 和 iTunes 同样支持有线备份和 Wi-Fi 无线备份，后者在速度上则稍微弱一些。而在备份这一功能上，iMazing 拥有一个非常实用的「备份存档」功能，类似于 macOS 上的 Time Machine，即新的备份数据不会覆盖掉旧的备份数据，将两者的数据差以「存档」的形式保留，这样的好处是当你进行了某些错误操作或删除时，可以方便的追溯还原相关数据。

![](.evernote-content/90726571-7B49-4F03-81EB-A577069B42C6/3D47D1CB-4E5D-4F18-B0B3-603D07B0F33B.jpg)

### 备份恢复

而在恢复备份上，iMazing 除了有 iTunes 的「一股脑」恢复模式，还支持更个性化的「按需备份」，你不仅可以自由选择是否恢复短信、通话记录、照片等数据，还可以任意选择想要恢复的应用。

![](.evernote-content/90726571-7B49-4F03-81EB-A577069B42C6/3FCBE803-1FD1-439F-BF8F-BD1ECF2928FE.jpg)

除此之外，通过 iMazing 我们还可以很方便的对单个应用进行导入和导出，如果你不信任或是没有使用 iCloud 的应用数据备份，那么通过 iMazing 的应用数据本地备份，你可以更放心的保留某个应用内的重要数据，用官方的视角来看，这个功能可能更多的体现在游戏存档上，我身边就遇到不少人因为误操作而让辛苦了很长时间的游戏存档全被覆盖和抹掉。

![](.evernote-content/90726571-7B49-4F03-81EB-A577069B42C6/1B355535-CB76-4C85-B9F0-96F3D5A6E423.jpg)

数据传输
----

备份之外，便是 iMazing 取代 iTunes 成为不少人选择的重要原因——数据传输。这里的数据传输则分为两种形式，一个是「传进去」，另一个是「输出来」。

### 导入资料到你的 iOS 设备

「传进去」顾名思义，就是将电脑上的数据传递到手机里，虽然 iCloud 或第三方云盘也能实现，但基于速度的考量，我还是更倾向于直接通过数据线或 Wi-Fi 进行一对一导入。

举个例子，当我想要将电脑上的某个 PDF 文件导入到 iOS 上的 PDF Expert 应用中打开，如果是 iTunes，我需要进入应用页面，找到 PDF Expert，再通过下方的添加按钮从电脑中定位到该 PDF 文件选择确认。

而在 iMazing 上，我们都可以文件夹的形式打开每个应用，然后通过拖拽或是复制粘贴直接添加文件到对应应用中即可。至于音乐和视频文件，同样可以直接拖拽添加。当然因为权限限制，我们并不能通过 iMazing 看到每个应用里的所有数据。

![](.evernote-content/90726571-7B49-4F03-81EB-A577069B42C6/01AF671C-CBD6-4EDC-8CC5-C4AA238AE7E4.jpg)

### 将 iOS 设备中的数据保存到电脑上

iMazing 的「输出来」就更为实用了，它可以将信息、电话、通讯录和备忘录逐条或全部导出到电脑上，根据不同内容可以导出为包括 PDF、CSV 和文本等格式，包括信息中的附件也可以直接导出。

![](.evernote-content/90726571-7B49-4F03-81EB-A577069B42C6/135AE2B1-C7E7-41B4-83E0-C4CB78439110.jpg)

备份是个好习惯
-------

作为一款老牌且功能较专一的软件，本文已不必再如说明书一般赘述 iMazing 的使用方式，而是从「与 iTunes 相比」的角度去介绍它。在前期的试用及文章的撰写过程中，因为使用了很多年的 iTunes，我一边感慨 iMazing 的「细致入微」，一边却又思考在「备份」这件事上，我真的有这么多需求吗？

iTunes 需要一定的学习门槛，但是熟练使用后你会发现它真的有属于自己的独特「智慧」，一种「插上就自己动」的行为解放；而 iMazing 就像一辆手动挡汽车，也许是出自对「自动化」的不信任，必须得自己敲敲点点才觉得完成了工作。

不过无论如何，工具始终是工具，重要的是要养成定期备份的好习惯，别等到经历了血与泪的洗礼才后知后觉。
