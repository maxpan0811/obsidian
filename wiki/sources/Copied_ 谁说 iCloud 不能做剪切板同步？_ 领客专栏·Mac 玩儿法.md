# Copied: 谁说 iCloud 不能做剪切板同步？| 领客专栏·Mac 玩儿法

---

Copied: 谁说 iCloud 不能做剪切板同步？| 领客专栏·Mac 玩儿法
-----------------------------------------

*2015-11-17* *应用推荐第一媒体*

我们介绍过无数的剪切板工具，在 Mac - iOS 端双向同步 Clipboard 的著名软件有 Pushbullet 和 Command-C，不过前者网络时常出问题，后者同步的即时效率不高，这类软件我个人认为无论从设计还是实现已经没有任何秘密和独到之处了，反倒是内容同步速度成了关键。

说实话大公司提供的云同步服务还是最靠谱的，虽说 iCloud 在国内的速度不是很理想，但对于 Clipboard，iCloud 还是能完美胜任的。

今天要介绍的主角是新秀产品：Copied，就是一款站在巨人肩膀上的聪明人。有些朋友连用都没用就说 Copied 站错了队，但事实上她却给我留下了最好的印象：稳定、顺滑。

Copied 的设计紧随 El Capitan 风格，提供了黑白两色主题，这种设计同样出现在了 iOS App 上，每一个文本、超链或是图片都能被 Copied 记录下来，如果你在其他设备上也安装了 Copied ，那么剪切板内容就将通过 iCloud 实现同步，速度在 3 秒 到 7 秒不等，需要看剪切板内容的大小。

虽然不像 Command-C 那样「秒同步」（实际上两款软件的差距真不是太大），但表现非常非常稳定，没有卡死，闪退或者无法同步的囧态发生，与 PopClip 配合默契，对于日常同步邮箱地址、手机号或者简短备忘录等常见操作需求来说绝对是够够的。

实测情况中，Copied 对图片的记录存在 Bug，许多情况下拷贝图片后剪切板竟然不能记录图片…

iOS 版能够释放的操作非常多样化，你可以通过 Safari 的 Extension Action 获取内容，同时也能使用 Copied Keyboard 来快速应用 Clipboard 文本/超链内容，甚至是在 Today Widget 上获取。

iOS 的设计趋近传统，单一 Clipboard 记录上左右滑动可对其进行 Copy，归入 List，分享或是删除，这里要提一下 Lists 功能，用户可以将所有 Clipboard 内容根据种类进行分组管理，比如右下图里收集了 GIF、代码片段、文章、网页书签等不同形式的剪切板内容列表。

在 Mac 版上，我们可以对任意软件设置 Copied 的应用范围，比如默认情况下 Copied 可以从所有应用程序内收集 Clipboard Items，但你也可以将不希望记录剪切板内容的目标软件进行排除，或者是将某个软件的 Clipbaord 内容归入指定 List 管理。

对于一个网络书签（超链）来说，用户还能将其转换为纯文本、所在网页的 Title、HTML 格式或是 Markdown 外链等等格式，满足某些用户的文本处理需求。

另外你也可以在后台设置添加更多形式的文本格式。

最后，Copied 设计了组合热键以满足调用主窗口、选择 List，Copy 项目等操作热键。

总的来说，Copied 的功能服务点满足了大部分 Mac 用户的需求，你不必再使用 QQ，备忘录甚至是 Mail.app 来同步文字或是图片，仅仅依靠 iCloud 就能搞定大部分工作。

Mac 版售价目前为 45 元，iOS App 是免费的，但如果你想实现与 Mac 端即时同步 Clipboard 的话还需要花费 6 元来解锁此服务，对于这样的销售策略我持怀疑态度，毕竟已经在 Mac 端向用户收费一次了，为何不能在 iOS 端免费服务呢？其他同类产品也都是这种策略，更别提全免费的 Pushbullet 了，我想作者在这个问题上还需要做进一步调研再做定夺，这样才能拉拢更多的目标用户。
