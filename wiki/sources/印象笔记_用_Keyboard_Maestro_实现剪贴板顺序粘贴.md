用 Keyboard Maestro 实现剪贴板顺序粘贴
============================

| 本文为付费栏目文章，您已订阅，可阅读全文 |

大家如果有过填写申请表的经历，就知道大量复制粘贴也是个体力活。比如填**入境申请单**，就需要从个人信息表里拷贝出各项信息，一条一条填到对应的位置，不断检查自己有没有**填错、漏填**，对于第一次填的人来说，心理压力尤其大。

需要来回复制粘贴的场景还有：

* **保存知乎回答**，发现不能拷贝整段文字 1，只好小半句小半句来回复制粘贴。
* **搬运资料到网站后台**，比如以前整理 Shortcuts Gallery，我就需要手动从表单里把功能介绍、下载连接等七八项信息一一复制到后台的每一个框里。
* ……

这些窘境，其实有些剪贴板工具已经在尝试解决了。在国外，有一类功能专门就叫**「Paste Queue（顺序粘贴、队列粘贴）」**，顾名思义，就是**先依次复制所有内容，再按顺序全部粘上**，免去来回切换跳转，方便之余也降低出错的可能。

![](.evernote-content/791505A7-BDB6-457C-92AA-4F278E369570/372E7147-7AE0-4EE3-8746-5F71225477D2.png)

Pastebot 的顺序粘贴功能

不过多数顺序粘贴功能需要设置专用快捷键，偏偏顺序粘贴使用率没那么高，键位容易忘暂且不说，按起来也不舒服；还有些剪贴板工具（比如 Copied）的顺序粘贴，则直接因为系统限制（沙盒机制）而报废。最后我的解决方案是，通过 Keyboard Maestro 来实现，可以沿用 `⌘Command-C` 和 `⌘Command-V` 这组常用键位，还不用担心系统权限限制。

Keyboard Maestro 的顺序粘贴效果

使用流程是这样的：

1. 按下快捷键 `Hyper-P`，呼出 Keyboard Maestro Palette（一个浮窗），表示此时进入顺序粘贴模式。
2. 连按 `⌘Command - C`，进行一系列复制，复制的顺序就是粘贴的顺序。
3. 连按 `⌘Command - V`，进行一系列粘贴，刚才复制的内容会按顺序粘上，比如视频中最开始复制了 Ann 的电话，那么第一个粘上的即是她对应的号码。
4. 再次按下快捷键 `Hyper-P`，或者直接关闭 Palette，结束顺序粘贴

注：Hyper 键指 ⇧Shift、⌃Control、⌥Option、⌘Command 这 4 颗键一起按下去。

用动作组充当顺序粘贴模式的「开关」
-----------------

我们提到过，多数现成工具的 Paste Queue 功能都要单独另记一组键位，属于典型的繁琐操作，为了解决一个复杂问题，又引入了另一组复杂操作。理想的使用状态是**直接把惯常的 `⌘Command-C` 和 `⌘Command-V` 设为顺序粘贴的快捷键**，减轻记忆和操作的复杂度。这就要求设置一个顺序粘贴模式的**开关**，只在进入该模式后把 `⌘Command-C/V` 变成顺序复制和顺序粘贴；退出顺序粘贴后，`⌘Command-C/V` 又能变回正常的复制粘贴。

Keyboard Maestro 中的 Palette（浮窗）就可以作为顺序粘贴模式的开关。我们可以把动作放在一个专门的组（Group）里面，只有先呼出包含这组动作的 Palette，才会激活顺序拷贝和粘贴的动作，不影响日常使用 `⌘Command-C` 和 `⌘Command-V`。

具体设置是，两个动作放在同一个组里，组设置为**「Shows/hides a palette when: The hot key 你喜欢的快捷键 is pressed」**，并且勾选**「Display when toggled」**，这样每次复制 / 粘贴后 Palette 不至于自动消失，你可以放心地连续拷贝数据。

呼出动作组后才会开启顺序粘贴模式

我一般会把 3 颗甚至 4 颗修饰键（`⌘Command`、`⌥Option` 这些）一起用上拿来开关 Palette，这样的好处是**不容易和其他快捷键冲突**，而且也**不用特意去区分快捷键，全按下去就行**。

到此为止，顺序粘贴动作组已经设置好，具体的顺序拷贝和顺序粘贴两个动作利用 Keyboard Maestro 中的**变量（Variable）**来实现临时存储复制粘贴的所需的数据，下面我们分别介绍拷贝和粘贴的两个动作的具体实现。

顺序拷贝：把剪贴板文本存到变量中
----------------

分析操作前，先来理一下顺序拷贝和粘贴的基本思路。整套动作实质上就是把数据存进变量、再从变量中取出，**和变量的交互**变成了动作的核心：

* **顺序拷贝**：每次拷贝，就把新文本粘贴到变量**最下方一行**。
* **顺序粘贴**：每次粘贴，都从变量中获取**最上方一行**放到剪贴板，并且把这行用过的文本从变量中删除。

变量的概念可能有些晦涩，可以把它简单理解成一个存放各种数据的「纸箱」。把顺序拷贝的过程画成示意图的话，就是像下面这样，从左边文档中取一段文本、再放进右边的变量中。变量在顺序拷贝和粘贴中起到了**暂存文本**的作用。

顺序拷贝示意图

理清逻辑之后，正式进入动作的具体步骤介绍。

「顺序拷贝」的触发方式（Trigger）是快捷键 `⌘Command - C`。这样的触发方式是特意为之，而且使用中因为需要开启顺序粘贴模式后再使用，实际上不会和日常的复制操作冲突。

拷贝动作

在操作中，第一部分是**模拟拷贝操作**。因为在设置触发方式时，为了让顺序拷贝和系统的拷贝快捷键保持一致，我们把 `⌘Command-C` 设成了启动顺序拷贝的键位，所以现在还需要进行一次真正的拷贝操作。

**知识点：在 Keyboard Maestro 中自动创建变量**

Keyboard Maestro 中的变量不需要特意手动创建，如果没有的话，**用到时就会自动新建**。比如，我们的动作中有一步检查 `pasteQueue` 变量内容的操作，如果运行至此 Keyboard Maestro 发现你还没有这个变量，它就会默默帮忙创建一个。

第二部分则需要先对变量中的内容进行判断，视情况用不同方式把文本**追加（Append）**到变量最下方。一般刚开启顺序粘贴模式后，变量中理应是没有内容的，此时直接追加即可；如果检测到变量里有内容，需要**换行**之后再进行追加，通过行来区分各次拷贝下来的内容。

在追加文本部分，我们用到了两个 Keyboard Maestro 内置变量，它们是 `%SystemClipboard%`（系统剪贴板）和 `%LineFeed%`（换行符号）。自己制作或编辑动作时，在任何支持使用变量的地方（几乎所有文本框）都可以通过「Insert Token」来插入变量。

插入变量

上面的设置都结束后，动作就完成了。每次按下快捷键，Keyboard Maestro 就会把内容存到临时变量的**最后一行** —— 顺序很重要，它关系到下一节使用顺序粘贴时能否按照拷贝的顺序进行粘贴。

我们在 Keyboard Maestro 动作编辑界面按一下 `⌘Command-,` 打开设置界面，在 Variables 栏中检查一下 `pasteQueue` 变量。现在，按下 `Hyper-P` 开启顺序粘贴模式，试着找一个文档或网页复制一些文本，可以看到复制的内容**一行一行**出现在了变量中。

检查顺序拷贝，文本依次被追加到变量中

现在已经实现了依次拷贝。

顺序粘贴
----

第二个「顺序粘贴」的动作稍微复杂一些，触发方式设置的道理和顺序拷贝一样，同样是用 `⌘Command-V`「挤占」系统的粘贴快捷键。下面重点介绍操作步骤，看看它怎样实现**依次取用**刚才拷贝下来的文本。

顺序粘贴示意图

开始运行动作后，还要分情况讨论。正常来说，只要变量中还有文本，Keyboard Maestro 就会不断取出来供我们使用；如果其中文本已经粘贴完毕，就弹出提醒通知（方便起见，通知中还会显示最后一次粘贴的内容是什么，比如下图中就提示了最后粘贴的是 `到此为止啦`）。

![](.evernote-content/791505A7-BDB6-457C-92AA-4F278E369570/FB82815F-F2FB-43BD-B06D-86070C638A22.png)

顺序粘贴结束后会发出提醒

重点在于**如何判断内容已经粘贴完毕**。我使用的方法是检查 `pasteQueue` 变量中的内容，如果**残留文本和当前剪贴板一致**，说明变量里的内容都已经使用过了，一轮顺序粘贴也结束了。

到此为止，剩下的事情就是在变量中尚有内容时，如何解决**依次取用**和**删除用过的内容**两大问题。本文动作通过一段 AppleScript 脚本和一条正则表达式分别解决了两个难点：

* **用 AppleScript 提取第一行**：提取第一行文本并且拷贝到剪贴板。AppleScript 也有删除文本的功能，但是它对于换行符号的处理不佳，会造成 Keyboard Maestro 无法对 AppleScript 处理过的文本正确分行，所以我们用下面的方法完成文本删除。
* **用正则表达式删除第一行**：删除的要求是只删用过的第一行，这个需求可以通过表达式 `^.*[\n\r]` 完成，它表示「从文本开头到第一次换行」，也就是**第一行**，把这个表达式表示的内容替换成空白即是删除。这正则技巧来自 Keyboard Maestro 论坛上的一则讨论（[Remove the top line of a text file or variable](https://sspai.com/post/56648#)）。

提取和删除文本后，别忘了接上一个模拟快捷键 `⌘Command-V` 动操作，至此才彻底完成顺序粘贴。和检查顺序拷贝时一样，正式使用前先在 Keyboard Maestro 设置的变量界面中检查一下顺序粘贴的运行情况，确认它可以依次取用、删除变量。

检查顺序粘贴，文本依次从变量中被提取出来
