# 刚学会的Markdown，这就凉了？

> 来源：HiddenState
> 基于Anthropic Claude Code负责人Thariq的推文（1500万阅读）+ Karpathy转发认可

## 一句话
Claude Code团队主张"HTML胜于Markdown"——当写文件的是AI而非人类时，HTML的信息密度、可读性和交互性远优于Markdown。

## 5大HTML实用场景

1. **方案对比**：让Claude做网格并排HTML，一眼看出优劣
2. **代码审查**：HTML审查报告=行内批注+颜色编码严重程度+流程图
3. **项目规划**：时间线+可折叠任务+红黄绿风险标注
4. **一次性小工具**：拖拽看板/并排编辑器/滑块调整参数，用完即扔
5. **研究报告**：整合代码库+git+网络信息，带目录导航+图表+引用

## 代价
- HTML生成消耗2-4倍Markdown的token
- 年花费约增加$5000
- 版本控制：HTML diff较嘈杂

## 作者建议
- 日常笔记/CLAUDE.md/README → 用Markdown
- 给别人看/需要对比/一次性工具 → 用HTML
