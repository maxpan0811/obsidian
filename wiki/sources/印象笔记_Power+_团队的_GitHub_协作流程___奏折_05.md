### GitHub：私有仓库免费，最多支持 3 人协作

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/7C950A78-118A-45B9-A3CD-318570CA57E9.png)

**GitHub**

[网站链接](https://github.com/)

#### Git 和 GitHub

Git 是一个分布式的版本控制系统 1 ，GitHub 则是基于 Git 的软件源代码托管服务。不过，由于版本控制和社交互动方面的优秀表现，很多非程序员也会把项目放在 GitHub 上，比如我们 Power+ 小队。

#### Power+ 是怎样用 GitHub 协同写稿的

**@Minja：**掐指一算，现在已经是 Power+ 小队用 GitHub 协作的第 3 个年头了。除了两期 Power+，不少付费教程原稿、自动化工具库也是在 GitHub 上沟通的。

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/9DA58949-D6CB-4C62-A82C-DC32B83F41CB.png)

部分放在 GitHub 上的项目

付费栏目毕竟收了各位看官银子，肯定要用私有仓库把文章素材保护起来，以前 Hum 和我前仆后继地提供学生账户；这次微软直接发了免费的私有仓，的确大大减轻了小团队、小项目的资金压力。

Power+ 选用 GitHub 作为协作工具，正是看重其版本管理功能。一方面，由于所有成员完全分散在世界各地，我们常常需要分头写作、统一合并；另一方面，文章总是要一改两改再改的，我们希望保存每一次的修改记录。虽说很少涉及代码，但 GitHub（或者说 Git）刚好满足这些细致的需求。

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/1EFB0754-24DC-4878-82AD-E3C4D19D94E5.png)

Power+ 内容生产流程

一般每周的选题会议后，大家就开始分头写自己的奏折部分。当然，并不是必须在 GitHub 客户端里写东西 2 ，我所有设备上就都不装官方客户端，仅仅把数据搞到本地，然后用更顺手的工具来写作、同步。

以 iOS 端为例，我们使用的是 [Working Copy](https://workingcopyapp.com/)，它本身就有完整的编辑、预览功能，而且还支持 URL Scheme，可以跳到 Ulysses 里舒舒服服写完文章，再跳回来提交修改。

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/688BA8B1-81E8-4AD0-A8AA-A4CA8257A38E.png)

用 Working Copy 移动办公

查看修改记录方面，GitHub 也非常方便。

以前我们试过用 Quip 协作，它倒是能记录修改历史，但全都是平铺开来的，没有版本的概念，不便于比较两次修改。GitHub 可以比对两次修改的差异，下图红色表示删除，绿色表示新增，一目了然。

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/6C543730-7486-4941-B74D-276DD27A5767.png)

一段修改记录

另外，和常见的网盘不同，在 GitHub 里可不能闭着眼睛一键同步，万一你把同事的劳动成果覆盖掉就麻烦了。也许还有人记得 Git 那反直觉的同步方式：先 `git add.`，再 `git commit`，接着 `git pull`，最后 `git push`…… 这推推搡搡（pull push）的都是什么东西？

GitHub 客户端本身就有一个快速同步功能，基本绕开了上面这堆命令行；我还自制了一个 LaunchBar 动作，连客户端都不用打开，输入修改的注释就能直接提交。

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/56A54C4D-29E6-4386-BC94-B66EB9A5AC98.png)

用 LaunchBar 快速提交奏折

Power+ 小队完整的协作流程、大量的原创自动化 tricks，将会在未来的文章里和大家见面。

用 LaunchBar 翻译当前网页

总之，你看不到我正儿八经打开 Google Translate 来翻译。一般我们习惯边看资料边查翻译，在屏幕上毕竟不像书桌上那么铺张，翻译器的界面是首先需要压缩的，要么给它留小半个屏幕、分屏使用，要么通过自动化工具随时呼出，而不能占据太大的屏幕空间。

### Instashare 更新：无线文件传输应用现状

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/3811ADD4-5DAF-47E0-8BDA-E656C1CCF814.png)

**Instashare 2.5.0（2019–01–06）**

[下载链接](https://itunes.apple.com/us/app/instashare-air-drop-second-ed/id638672344?mt=8&ign-mpt=uo%3D4)

**@Fairyex：**老牌的跨平台无线文件传输应用 Instashare 在本周迎来了一次时隔 3 年的久违更新。这次更新主要内容：

* 适配了新设备
* 添加了对 iPad 文件拖放传输的支持
* 支持新的 HEIC 图片格式
* 各种 Bug 修复

作为一个支持主流系统间相互无线传输所有格式文件的应用，Instashare 曾经是很多人的「装机必备」。不过随着 iOS 系统自带的 Airdrop 功能越来越方便完善，这类应用渐渐地显得不再那么必要，所以大多数文件传输应用的最后更新都停留在了几年之前。这样看来，Instashare 这次也算得上是一次良心更新。

而对于安卓用户来说这个问题的答案刚好相反，由于安卓系统本身并没有和 Airdrop 类似的功能，大多数用户依然是使用微信或者 QQ 来在个设备之间无线传输文件。单个文件或者小文件还好，一旦涉及到多选文件或者文件需要另存为等操作就显得十分的繁琐不便。这时候像是 Instashare 这类的应用就显得十分地有必要。

不过 Google play 上 Instashare Android 版的最后更新日期还是在 2014 年，这显然不适合在 2019 年推荐给大家下载使用。接下来给各位 Android 用户推荐一个特别好用的无线文件传输应用，这个应用就是**文件闪传**。

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/5995E683-3C92-4A09-93A5-7A4751A92CD9.png)

**文件闪传 2.8.2**

[下载链接](https://play.google.com/store/apps/details?id=app.eleven.com.fastfiletransfer)

这个体积精悍的应用虽然只有 Android 版，但是它几乎能跟所有系统实现互传，因为它使用 HTTP 的链接方式实现文件互传。这样子的优点是手机上打开了文件闪传，其他设备只要有浏览器就可以实现浏览文件、上传下载、同步剪贴板等等操作，不需要安装任何额外的软件插件。

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/07C7A5A6-5C05-4C3A-8F57-3681D92AF397.jpg)

文件闪传

文件闪传的其他优点：

* 分类浏览图片、视频、音乐、下载目录和剪切板
* 完整的文件目录浏览
* 上传文件到手机，文件保存到正在浏览的目录下
* 通过浏览器投影手机屏幕（适合演示时使用）
* 手机上复制文本，自动发送到网页端
* 网页端也可以发送文本到手机剪切板

### Kindle for macOS is back

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/2864EA67-513C-45D9-81D1-E06D8A851FB3.png)

**Kindle for Mac 1.25.2（2019–01–8）**

[下载链接](https://itunes.apple.com/hk/app/kindle/id405399194?l=en&mt=12)

#### 应用简介

Kindle 是亚马逊旗下的跨平台阅读服务，除了实体阅读器 Kindle 系列硬件，也在 Windows、macOS、iOS 和 Android 等主流操作系统上推出了客户端。

#### 闪退 bug 终于消失

**@Minja**：Kindle 商店中书籍资源丰富，版式较为工整，加上各种促销和 Kindle Ulimited 包年会员，越来越多人养成了购买正版的习惯。

但是，Kindle for macOS 长期以来都存在莫名其妙的闪退，加上 Kindle Ulimited 借来的书看完归还后不能保存批注 3 ，不少资深书虫宁可掏钱向亚马逊买书，再自己破解自己的数字财产，放到其他阅读器里追求完整的阅读体验。所谓「花正版钱，看盗版书」，大概就是这种感觉。

Mac App Store 里的 1 星评价，真的是一点都不冤。

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/74EADD42-AC14-425E-8504-C92BB55AB4D5.png)

Kindle 的评分只有一星

我们花钱买的不仅仅是书，还有一整套的服务 —— 笔记同步，批注云存储，等等 ——Kindle 以前不争气，也就无怪各路高手八仙过海各显神通了。好在最近的更新里，**闪退 bug 似乎已经消失**，如果有读者因为这个原因离开 Kindle，可以再给它一次机会。

话说回来，Kindle for macOS 长期以来的不争气，倒是培养出我「不挑食」的阅读习惯。之前 Mac 版几乎处于不可用的状态，导致我在手机上用 Kindle 看书，回电脑上则换成 iBooks。工具都不一样了，阅读、笔记、批注的操作也迥异，就迫使我想出一套通用的阅读 - 批注方法：**基于颜色的批注**。

1. **蓝色：本书理论；**
2. **橙色：经典案例；**
3. **红色：关键词、专有名词；**
4. **紫色：对我有特殊意义的内容。**

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/3BB57E3D-E004-436C-846F-E06825C2596C.png)

用 4 种颜色做批注

这周讨论奏折选题时，Hum 也分享了他的颜色批注方案，我们的分类大致是一样，不过他还提到一类「写得好的句子」，也值得单独标出来。可见每个人的批注方法不同、所用工具各异，但仍然可以用几种基础的颜色来满足。

### Overcast：播客客户端可能是听音频的最佳方式

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/B7F0BA39-72C8-4FAB-8D15-EAF4B3B14A25.png)

**Overcast 网页端**

[下载链接](https://itunes.apple.com/us/app/overcast-podcast-player/id888422857?ls=1&mt=8)

#### 应用简介

知名开发者 Marco Arment 开发的播客客户端，主打 Smart Speed 和 Voice Boost 两个特色功能，前者可以智能缩短节目的沉默时间，后者可以均衡节目的音量。主体功能免费，移除广告、上传自定义音频文件需要订阅。

#### 支持批量上传用户音频

**@Minja**：一个播客客户端，却把 [上传自定义音频](https://thesweetsetup.com/upload-mp3-files-overcast/) 作为付费功能，可能有些人要看不懂了：我又不是播客主播，哪里用得着上传音频呢？

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/38D24293-506E-4CF9-AF16-EF508DE46DB2.gif)

Overcast 支持上传音频

**其实，只要是个音频文件，不管是歌曲、播客、电子书还是听力材料，用 Overcast 来听都是最自然的。**

现在还活着的、能播放音频的工具，操作性大多已经很孱弱了，你基本是躺那儿，也不需要动手，就等音乐自己推荐过来。以前听歌可不是这样，老机器上有倒带按键，有快进快退按钮，有 A-B 循环选项……

一句话，现在听点东西，就像临时呆在别人家的酒吧，你最多只能点歌罢了。而播客客户端，却难得保留了古早的聆听习惯，直接把留声机、复读机或者说收音机交给你操作：

* **可以快进快退**，听漏了一个知识点能瞬间返回
* **可以倍速播放**，而不用等老教授慢吞吞讲上几个钟头
* **可以竖屏操作**，诚然现在很多视频播放器的操控性也很棒了，但捧着手机总不如随心一握来得舒服
* ……

曾经有备考的读者希望我推荐一款播放器，用来听 VOA4  听力素材。不是考虑付费因素的话，我当时肯定推荐 Overcast。我自己用原生 Music App 听歌时，有时还会习惯性想按快进键，可惜一去不复返。

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/B62A4DBB-48F2-4898-B071-9E59B97A9229.jpg)

快进功能曾经出现在 iPod 上

美中不足的是，Overcast 是一个服务，需要将音频上传到其服务器中，容量和速度都有限。这回支持批量上传，也把存储空间提升到 10GB5 ，多少算一次进步；有朝一日能本地播放、AirDrop 直接投送的话，才是最理想的。

### Due 3.2：继续优化「黑科技」般的通知交互

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/88BD0276-2219-43ED-A427-99276843E131.png)

**Due 3.2（2019–01–07）**

[下载链接](https://itunes.apple.com/cn/app/id390017969?mt=8)

#### 应用简介

Due 是一款由新加坡开发者 Lin Junjie 开发的提醒事项应用。它最大的特点是会不断地给你弹通知，直到你把任务完成。如果 Due 弹通知的时候你确实没空，它也有几种快捷的操作让你推迟任务。此外，定时器功能、丰富的设置选项和手势操作、自然语义输入、URL Scheme 等等，也是 Due 的特色。

#### 值得注意的更新内容

### Google Message 可以拦截垃圾短信了

**@Fairyex：**国内的 Android 用户可能对短信拦截已经习以为常了，甚至觉得这个标题像是上个世纪的新闻，但这的确是谷歌最近才在原生 Android 系统的短信应用上添加的功能。

这项新功能现在只对部分用户开放，并且在提示之后是默认开启状态。目前还不清楚谷歌什么时候才向全体用户开放这项功能，只要开启了这项功能，短信应用就会后台默默分析并拦截疑似垃圾短信的信息内容。

拦截垃圾短信功能的提示

### Google 确认 Fuchsia OS 将支持 Android 应用

**@Fairyex：**Fuchsia 是 Google 正在开发的新系统，不同于现在自家使用 Linux 内核的 Android 与 Chrome OS，Fuchsia 基于一个全新设计的微型内核。这使得它更加轻量，节省资源，也意味着它可以运行在更多的设备和平台上。

不过不同内核导致 Fuchsia 有很大可能将不支持之前的 Android 应用，不过根据著名网站 9to5Google 的[报道](https://9to5google.com/2019/01/02/android-runtime-app-support-fuchsia/amp/)，谷歌最近确认了 Fuchsia 支持 Android 应用的可能性。

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/45464ED0-94D8-4BB5-B22A-BEDCA08D3D36.jpg)

（图：9to5Google）

谷歌最近在 AOSP（Android Open Source Project，Android 开源项目）里面提交了两个和 Fuchsia 相关的库，并在其中一个的读我（ReadMe）文件中提到了将会为 Fuchsia 设计一个特别的 ART Android 运行虚拟机制来运行 Android 应用，不过不是传统的 .apk 文件而是 Fuchsia 专用的 .far 文件。

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/65576FBA-94B9-4BF4-9A3C-6F0E1571454E.png)

（图：9to5Google）

这次更新我们也能看出谷歌原本的野心。手拥两个用户数量庞大的操作系统（Android 与 Chrome OS），Fuchsia 的出现和兼容 Android 应用就是为了整合这两个系统的用户并且将适用范围继续扩大，为将来一定会流行的各种物联网设备铺路。虽然现在 Fuchsia 还有很久才能和大众见面，但是只要谷歌成功地将这个计划执行，那么这个新系统的地位将会比曾经 windows 的垄断地位更加高。

### 推荐文章：采访 GTD 的作者 David Allen

**@文刀漢三：**自动化服务 Zapier 的官方博客，最近发布了一篇对 GTD（Getting Things Done）作者 David Allen 的采访。

除了对 GTD、效率、创意等问题的讨论，访谈最后还聊到了 David Allen 对 GTD App 的看法。

市面上很多优秀的 GTD 工具，比如 Asana、Things、OmniFocus 等，David Allen 都用过，但是并没有取代他现在的系统。访谈里虽然没有具体提到了他在用的 GTD 工具，但是我们从其它的[信息渠道](https://lifehacker.com/5955235/the-tools-david-allen-uses-to-get-things-done)可以查到，David Allen 其实还是在用以纸笔、实体文件夹为主，少数 App 为辅的 GTD 系统。

其中他具体提到了一款叫做 IBM Notes 的软件，这是一款集成了邮件、日历、联系人、任务管理、通讯等多种功能的办公软件。但他在用是朋友帮他改造过的特制版。换言之，我们还是不知道他在用什么。不过他提到了一个具体的使用细节：当他发送一封邮件给别人时，如果需要等他的回复，那么他可以按一下「按钮」，邮件就会自动添加到 Waiting For 清单，自动将收件人的名字填在上面，并且自动填上日期。

David Allen 曾试过两次用完全用 App 来建造 GTD 系统，但都失败了。也曾和一家叫做 Intentional Software 的公司一起打造 GTD 应用，也失败了。

访谈最后 David Allen 还推荐了一些他在用的 App，有：

* Word、Excel
* [Snagit](https://www.techsmith.com/screen-capture.html)：截图工具
* IBM Notes：David Allen 对这个工具评价很高
* [The Brain](https://www.thebrain.com/)：脑图工具
* [Zoom](https://zoom.us/)：视频会议工具
* [Quicken](https://www.quicken.com/)：个人财务管理工具
* Words With Friends：单词游戏

原文链接：[Quality Over Quantity: An Interview With GTD’s David Allen](https://zapier.com/blog/interview-with-david-allen/)

### 推荐网站：高质量技巧网站 MakeUseOf

**@沨沄极客**：今天要为大家推荐的是一个国外的高质量技巧网站：[MakeUseOf](https://www.makeuseof.com/)。正如它的 Twitter 自我介绍，MakeUseOf 的主旨就是「Get even more value out of your tech with our daily articles, tutorials, tips, and reviews.」。

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/16F0A099-6747-4FDD-B3DC-6101EC836688.png)  

MakeUseOf 的 Productivity 频道

它涵盖的内容非常丰富，整个网站分为 PC & Mobile、LifeStyle、Hardware 这三部分，从 [Windows](https://www.makeuseof.com/service/windows/) 到 [macOS](https://www.makeuseof.com/service/mac/)，从[科技新闻](https://www.makeuseof.com/service/news/)到[创作频道](https://www.makeuseof.com/service/creative/)，从[设备购买指南](https://www.makeuseof.com/service/buying-guides/)到[智能家居](https://www.makeuseof.com/service/smart-home/)，你总能找到感兴趣的话题。

有关应用和技巧的文章通常以合集的形式出现，比如「[10 个节省时间的 Google Docs 小技巧](https://www.makeuseof.com/tag/10-google-docs-tips-take-seconds-save-time/)」和「[在 Windows 上查看 Google 日历的 7 种方法](https://www.makeuseof.com/tag/google-calendar-windows-desktop/)」，你可以在很短的时间里了解到许多实用的技巧，或者一次发现多款符合需求的工具。

![](.evernote-content/D3E0A009-16DA-402D-AED2-5F6E1BE0AEB0/4DC85B62-5E53-4520-91ED-A18BB60D9062.jpg)  

MakeUseOf 手机版

而 MakeUseOf 的手机版应用也与大多数新闻 App 不同，它根本没有时间流的概念，而是将大量的 Tag 列举出来，让你自行选择订阅哪些 Tag。

除了文章内容，MakeUseOf 还有大量的交流频道，如果对某个话题感兴趣就可以加入相应的频道，频道内会自动推送与之相关的新闻，也会有人正常聊天，和 Telegram 有点类似，只不过这里大多数都是官方频道。

MakeUseOf 的 RSS 订阅方式也很有意思，特定栏目 RSS 只需要在网页后加上 `feed` 就可以生成 RSS，分别可以订阅特定分类和特定 Tag，比如：

Windows 频道的 RSS：`http://www.makeuseof.com/service/windows/feed/`

Microsoft Tag 的 RSS：`https://www.makeuseof.com/tags/Microsoft/feed/`

从而使读者可以根据自己的喜好订阅任意一类文章，不至于被过多的内容淹没。

正是因为这种一切从实用角度出发的理念，让我受益良多。它从被我无意间发现开始，逐渐发现它的好文章产量很高，如今早已被我加入到 RSS 的重点关注列表里，也推荐给大家。
