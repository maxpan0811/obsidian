我一直用 Markdown 来写作。

Markdown 并不是什么高级的东西，它本质上——跟 txt 一样——就是纯文本，只不过加入一些约定好的「语义化标记」，比如一行文字前加上 `##`代表这是一个二级标题，用 `** **`围起来的文字代表强调。

![](attachments/d00eb1b9479d7b3f.png)

纯文本有着强大的生命力，它可以跨越系统、跨越软件、甚至跨越时间。

手机电脑会过时，操作系统会停止更新，写作软件也终有死掉的那一天，但是纯文本不会。不管你用一台二十年前或者二十年后的电脑，不管你用 Windows、macOS、Linux、iOS、Android，不管你用什么文本编辑软件，都能打开它。

![](attachments/87c2b48245871282.png)

而且纯文本的体积很小，一篇文章只有几或十几 KB，存下整个文库往往也只需要一点点存储空间（不考虑图片的话），这意味着管理它们的成本极低。

与之相对的是 Microsoft Word、Pages 这样的富文本格式，它依赖平台、依赖软件，甚至同一个软件的不同版本对格式的支持都不完全一样，甚至不兼容。你会被格式所捆绑，而且可能要不停的付费，文章的迁移成本很高。

我了解的很多长期主义写作者都在用纯文本（.txt 或 .md）写作。

Derek 在它那篇《Write plain text files》里写道：

我写下的文字是我最宝贵的财富，也是我人生的见证。正因如此，我只使用纯文本文件。 这是最可靠、最灵活且最持久的选择。

……即使在数百年后的未来，后人依然能够阅读这些纯文本文件。

https://sive.rs/plaintext

……这款应用（Obsidian）终将过时。**真正能经久不衰的，是我创建的那些纯文本文件**。除了我之外，或许没人愿意阅读它们，但未来的我已足以成为一个值得为之写作的读者。

我曾写过一篇[不想再被捆绑，我放弃了 Ulysses。](https://mp.weixin.qq.com/s?__biz=MzI2MTAzMzcyNQ==&mid=2648567117&idx=1&sn=f08a840446612042a84f835d51202729&scene=21#wechat_redirect)

Ulysses 是一款体验非常棒的写作工具，并且支持 Markdown 格式写作。但它并不是用 Markdown 纯文本格式来储存，而是有一套私有格式。这使得我用 Ulysses 写的文章只能用 Ulysses 打开，而我需要每个月给 Ulysses 付一笔「租金」。

所以我把 Ulysses 换成了 iA Writer。

iA Writer 只专注于「写」，至于怎么「存」，它认为应该交给写作者自己。想存哪、想用什么格式（.txt 或 .md）、想用什么文件名都是我说了算。

我随时可以抛弃 iA Writer 换成其他 Markdown 写作工具，比如 Typora（这个我也买断了）或者 Obsidian 而没有任何迁移成本。这给了我一种自己掌控数据的自由。

而且 iA Writer 和 Typora 是买断制，Obsidian 是免费的，我不用每个月给它们付租金。

软件应该服务于长期保存知识，而不是把用户锁定在某个平台里。
