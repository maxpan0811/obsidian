Spark 2.0
---------

![](.evernote-content/64CEE91A-708E-42D2-93A7-780FAE155770/7AA6A6D1-7922-48A2-9BF3-C000E4AE452B.png)

**Spark 2.0（2018–05–23）**

[下载链接](https://itunes.apple.com/cn/app/spark-by-readdle/id997102246?mt=8&ign-mpt=uo%3D4&at=10lJSw)

#### **应用简介**

Spark 是 iOS 上的邮件客户端中竞争力最强的一款。在继承了老一辈的滑动手势以及延时设定之外，还拥有智能收件箱（根据邮件来源来区别紧急程度）、智能通知以及过滤等这一代 iOS 邮件客户端才有的强竞争力功能。除此之外它的设计出众，体验稳定。

#### **值得注意的更新内容：2.0 大版本更新**

**@Hum：**这次的 2.0 更新，Spark 没有做什么界面上做的改变，反而从底层开始，对邮件这个东西的编写与查阅方式，做了一个大的调整。这个大的调整，简单来说，是给邮件搭出了一个 Web 的界面，从而实现了多人共同编写，以及用链接分享邮件的功能。

多人协同功能（需订阅付费）：

这个功能对我本人来说不太重要，因此我想侧重地说明一下邮件分享的实用价值。

大多数客户端在使用邮件分享的时候是用本地链接的形式，比如 macOS 上的官方邮件客户端，或者是 Airmail，它们给每一封邮件生成一个本地的 link，我们从其他软件里点这个链接，就会跳转到这封邮件上。这就让我们可以把这些邮件放到任务管理工具里，作为任务流程的一部分。

![](.evernote-content/64CEE91A-708E-42D2-93A7-780FAE155770/0EF665BE-6584-433D-861F-CFCD4388A834.png)

Airmail 与 Spark 中分享出的链接

然而 Spark 超越了这个思维，把一封邮件做成了一个 Web 界面，生成了这个 Web 界面的链接。我们可以看上图中，Airmail 的链接是 `airmail:` 开头的，但 Spark 的链接是 `http:` 开头的。这样这封邮件我们就可以分享和展示给任何人。与此同时，它还利用了 Universal Link 这项技术，当这个邮件是我们自己 Spark 中的邮件的话，就可以直接将其在 Spark 中打开，进行回复。

![](.evernote-content/64CEE91A-708E-42D2-93A7-780FAE155770/46D62C63-DA87-445E-9D98-F22FA70E2D4A.png)

在 Spark 中生成别人可以查看的邮件网页

因此它既可以成为本地任务，也可以分享展示给其他人，是邮件分享的新的一步。眼看着 Airmail 团队在打磨另一款编辑器类的软件了，第三方邮件客户端领跑的作品里，Spark 看来要拉开与其它作品的距离了。

最后两点注意：

1. 生成分享链接需要登录 Spark 的服务，也就是选择一个你的邮箱来同步信息，如果没有这么做就无法生成分享链接；
2. 目前邮件分享出的链接不会隐藏收件人的地址，所以分享的时候还是注意一下。

Castro 3
--------

![](.evernote-content/64CEE91A-708E-42D2-93A7-780FAE155770/BD300663-4D1A-4A37-8E32-D2FA14B4A700.png)

**Castro 3（2018–05–22）**

[下载链接](https://itunes.apple.com/app/id1080840241?mt=8&ign-mpt=uo%3D4&at=10lJSw)

#### **应用简介**

Castro 是一个泛用型播客客户端，主打 GTD 式的播客收听方式，用播放队列、收件箱和归档来给播客分类。用户可以通过应用内搜索或从其他播客应用导入的方式来订阅播客。

#### **转向订阅制，老用户免费使用原有功能**

**@Minja：**本周二凌晨的 Casro 3 并没有发布独立新应用，而是采用了应用内直接升级的方式，并带来了不少的提升：

* 更简洁的界面，采用更具 iOS 11 风格的卡片设计；
* 双击播客封面收藏当前正在播放的界面；
* 更快的在线播放速度；
* AirPlay 控制；
* 具有简单播放功能的 Apple Watch 应用，但是暂不支持调整音量。

![](.evernote-content/64CEE91A-708E-42D2-93A7-780FAE155770/7A1E7556-7B1B-486F-B17B-4ADDB1CD4D93.png)

Castro 3 的播放界面和播放设置

我最在意的就是在线播放的速度确实变快了，而且 Show Notes 几乎也是瞬间打开，不会再让用户因为转个不停却不见动静的播放键而感到沮丧。就像流媒体音乐一样，播客工具的基本功就是「稳」，卡顿一下、加载半天自然不能接受。

升级后的 Castro 成了一个免费加内购的应用，用户可以免费下载、使用，高级功能则需要付费解锁，比较重要的有这些：

* 跳过空白、人声增强；
* 选听播客章节（需要节目本身支持）；
* 调整播放速度；
* 跳过开头；
* 黑夜模式。

对于老用户，Castro 也准备了一份回礼 —— **人声增强和黑夜模式可以永久免费使用**。官方的解释是，老用户在买断制时代就为这些功能掏过钱，以后也就不需要再次付费。我很欣赏这种向老用户开放部分付费功能的做法，它比单纯打折让人觉得更舒服，毕竟我们往往是冲着应用的某些功能去的，再贵也不得不买；而能够免费使用一些高级功能，用户在决定订阅前就有个缓冲期，不至于马上掉头走人。

又如 Timepage 给老用户免费账户、Sketch 停止订阅后当前版本可以继续使用，我认为都是转订阅制探索中的正面例子。

Castro 3 订阅价格为每 3 个月 18 元，每年 58 元，提供一周的试用。

Working Copy
------------

![](.evernote-content/64CEE91A-708E-42D2-93A7-780FAE155770/AD0BE0D5-B1B6-4280-89AF-F645D22765C4.png)

**Working Copy 3.3.7（2018–05–24）**

[下载链接](https://itunes.apple.com/cn/app/working-copy/id896694807?mt=8&at=10lJSw)

#### **应用简介**

Working Copy 是一个 Git 客户端，不同于同类产品，它大量使用手势控制，操作人性化，非编程专用的用户也可以使用，例如用于文稿、设计稿的版本管理。

#### **更深度的 Files 集成**

**@Minja：**一直以来，我们都喜欢称赞那些支持 iOS 11 Files 的应用「宛如原生」，其实它们不仅在界面上和谐统一，还拥有一些特殊的功能。前不久从 MacStories 的一篇文章《[iOS 11’s Streamlined, Yet Extensible File Management](https://www.macstories.net/ios/ios-11s-streamlined-yet-extensible-file-management/)》中，我得知原来支持 Files Provider 的应用，**还可以在文件长按菜单里加入「插件」功能**，比如：

* 导出不带批注的文件（[PDF Viewer](https://itunes.apple.com/us/app/pdf-viewer-by-pspdfkit/id1120099014?mt=8)）；
* 导出不可修改批注的文件（PDF Viewer）；
* Commit 一次文件改动（Working Copy）；
* ……

![](.evernote-content/64CEE91A-708E-42D2-93A7-780FAE155770/81F797AB-C108-4160-8832-8FD0E399DCA3.png)

PDF Viewer 中的「右键菜单」

这是很不起眼的功能，甚至很多人都不会注意到长按后弹出的菜单里还有一番玄机。但我看来，把文件整理、导出相关的功能「藏」在这里，其实是一件很自然的事 —— 就像 macOS 的右键菜单。

而 Working Copy 这次的更新还用上了 File Provider 的另一个特性：**标签**。所有有过修改但是还没提交（Commit）的文件，都会被标上黄色的 `Modified` 标签（和 Working Copy 应用内表示已修改过动文件用了一样的颜色），这样不用进入 Working Copy 就能知道有没有修改需要提交，而不用打开应用查看；如果你用其他支持 File Provider 来编辑文档（比如 Pretext），改完在应用内就能看到 Working Copy 文件夹里的黄色待提交标签，颇为显眼。

![](.evernote-content/64CEE91A-708E-42D2-93A7-780FAE155770/46159456-B964-43ED-9649-2F24E316ECD9.png)

自动添加的标签

在众多应用中，像 Working Copy 这样积极拥抱 File Provider 的，并不多见。官方 Twitter 连续一个月都在介绍 File Provider 相关的功能，还几乎不带重复的，这样的热情也是罕见。虽然 PDF Expert、Scanner Pro、Office 这些主流的效率工具对 File Provider 仍然支持有限或者压根爱搭不理，但我们也看到 Working Copy、PDF Viewer、[Pretext](https://itunes.apple.com/us/app/pretext/id1347707000?mt=8)、[Textor](https://itunes.apple.com/us/app/textor/id1330406995?mt=8) 等一票应用迈出了第一步。

1Password 7 for Mac
-------------------

![](.evernote-content/64CEE91A-708E-42D2-93A7-780FAE155770/7EE3FA84-3147-491E-9581-D864282BB794.png)

**1Password 7 for Mac（2018–05–22）**

[下载链接](https://itunes.apple.com/cn/app/1password-7/id1333542190?mt=12&ign-mpt=uo%3D4&at=10lJSw)

#### **应用简介**

1Password 是苹果生态里最出名的密码管理工具。

#### **值得注意的更新内容**

**@文刀漢三：**这次更新有几个我认为比较重要的点：

1. **重新设计的 1Password mini：**可以看出 1Password mini 的功能更丰富了，更接近完整版应用。其实大部分时间我们都是在和 1Password mini 打交道，而完整版应用一般是在整理时才会打开。现在 1Password mini 的形态看起来很像系统自带的 Spotlight，窗口尺寸也差不多，相信使用起来会更方便；  

   ![](.evernote-content/64CEE91A-708E-42D2-93A7-780FAE155770/F86CDAD3-9B65-4365-8791-F113F4DFD334.png)

   1Password mini
2. **应用界面重新设计：**最主要的是突出了每个账号的用户名和密码区域，让我们能一眼就看到最重要的信息，不像以前那样把所有内容都混在一起。  

   ![](.evernote-content/64CEE91A-708E-42D2-93A7-780FAE155770/986AE8C2-3C71-4746-B8AA-C1D34CC01345.jpg)

   1Password 主界面
3. 备注和备忘录支持 Markdown：支持 Markdown 其实是非常小众的需求，说真的就算对于我来说也是可有可无的功能，但 1Password 团队还是支持了。我个人会在 1Password 里存一些文字信息，比如设备序列号、密保提问等，支持 Markdown 的话也可以让我更方便地用列表等语法来排列内容。  

   ![](.evernote-content/64CEE91A-708E-42D2-93A7-780FAE155770/F7C955D4-088A-4F63-B231-71387FC0160C.jpg)

   支持 Markdown

官方博客对这次更新的详细介绍：[1Password 7 for Mac: The Best Ever](https://blog.agilebits.com/2018/05/22/1password-7-for-mac-the-best-ever/)

这次更新对于订阅用户来说没有任何影响，可以直接升级。对于 1Password 6 的用户，则需要付费升级。你可以选择以 49.99 美元（之后会涨价到 64.99 美元）的价格在官网进行买断，但之后再有大版本升级则可能需要重新付费；或者以每年 233 人民币的价格加入订阅版。

Amazon Kindle for iOS 6.7
-------------------------

![](.evernote-content/64CEE91A-708E-42D2-93A7-780FAE155770/EDDAEAFB-519D-4DCC-B5BD-EF859D22F6B0.png)

**Kindle for iOS 6.7（2018–05–21）**

[下载链接](https://itunes.apple.com/cn/app/amazon-kindle/id302584613?mt=8&ign-mpt=uo%3D4&at=10lJSw)

#### **应用简介**

Kindle 是亚马逊公司推出的图书阅读应用。

#### **值得注意的更新内容**

**@文刀漢三：**由于习惯了在网页、手机上阅读，我会更喜欢「连续滚动」这种阅读模式，而不是那种模拟真实书籍的「翻页模式」。因为连续滚动在触摸屏上体验更自然，也更适应图片、视频、代码块等内容。

苹果自家的 iBooks 应用一直都支持连续滚动，但无奈国区 iBooks Store 停止运营了。Kindle for iOS 后来也只支持了连续滚动，但却不支持在连续滚动模式下选中文字，也就导致无法进行复制、查词和高亮等操作。最新一版终于修复了这个问题，推荐大家可以试一试连续滚动这种阅读模式。

![](.evernote-content/64CEE91A-708E-42D2-93A7-780FAE155770/50AAB6AF-F9BB-4E83-8B9C-FD5D5788B242.jpg)

Kindle 的连续滚动模式
