# 微信读书官方 Skill 和 CLI 出来后，我的一个明显感受是：

微信读书官方 Skill 和 CLI 出来后，我的一个明显感受是：

个人知识库这件事正在变得更“正统”了。

之前我也折腾过微信读书 MCP。

MCP 的好处是开放、灵活，能把书架、划线、笔记接进 Claude、OpenClaw 或其他 Agent 工作流里。

但问题也很明显：配置门槛偏高，稳定性取决于实现方，接口变动、登录态、Cookie、数据结构这些细节，都可能让普通用户望而却步。

这次微信读书官方 Skill 和 CLI 给我的感觉不一样。

CLI 负责把能力封装成稳定命令，比如搜索书籍、查看书架、读取阅读进度、导出划线笔记、查看阅读统计；Skill 则负责告诉 Agent：这些命令该在什么场景下用、字段怎么理解、结
