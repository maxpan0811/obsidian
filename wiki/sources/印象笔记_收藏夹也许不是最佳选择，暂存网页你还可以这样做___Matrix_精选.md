大多数人在浏览网页的过程中，都会不知不觉打开了大量的标签页，有时候还不止一个窗口。这些页面，少部分是需要永久保存的，而更多只是目前这个状态下你需要的资料，看完即可关闭。这种情况下如果浏览被打断，你会选择用什么方式暂存现在的阅读状态以便稍后查阅？

我是一名产品经理，日常工作中经常需要跨不同主题研究，且往往会被其他新主题所打断。如果不能平衡好不同主题之间的关系，那最终将导致打开的页面太多，没有欲望再次进行整理或阅读。当然以上场景还涉及到其他时间管理方面的技巧，但于每位效率工作者而言，最需要解决的是：

1. 我如何能完美地「冻结」我现在的工作状态？
2. 我如何能完美地「还原」我的工作状态？

标签页管理插件
-------

我用 Chrome 的标签管理插件解决以上问题，这类工具可以暂存所有打开的标签页，并且方便地一键恢复到暂存时的浏览状态。

如果没有使用这类工具的习惯，你可能会疑问：标签页有什么好管理的？存收藏夹不就好了吗。再不然就稍后读服务，或者印象笔记剪藏。回到文章开头的场景，如果打开的标签页数量比较多，比如好几十个，你还能淡定地面对这满屏的标签页，然后一个一个保存下来吗…….即便是有耐心保存，它们也可能会让收藏夹显得非常乱。

比起收藏夹和稍后读，标签页管理工具有着更好地批量处理功能，更高地自定义性（如存哪些不存哪些），一键恢复多个浏览页。从最初接触 Chrome 开始一直到现在，我前后试用了很多不同的插件，包括 [Stash](https://chrome.google.com/webstore/detail/stash/bnhjedgfogckebfhnlicnkbdjlmpibck)、[Session Buddy](https://chrome.google.com/webstore/detail/session-buddy/edacconmaakjimmfgnblocblbcdcpbko)、[FreshStart](https://chrome.google.com/webstore/detail/freshstart-cross-browser/nmidkjogcjnnlfimjcedenagjfacpobb?hl=zh-CN)。

### Stash

这是一个 iOS 和 Node.js 开发者做的一个 Chrome 扩展程序，有着基础的暂存和恢复功能。这个插件 2012 年就有了，当时 Chrome 还没自带批量存标签页的功能，Stash 算是满足了一个痛点，现在来看它的功能就显得比较单一了。

### Session Buddy

这也是一款方便易用的标签页管理工具，功能在三款工具之中来说是最强大的，除了暂存和恢复标签页外，它还有以下功能：

* 可以集中浏览所有标签页
* CrashRecorvery，当浏览器挂掉后，可以快速恢复

标签页管理应满足什么需求？
-------------

Stash 和 Session Buddy 因为各种原因，达不到「相对完美」，最终只有 FreshStart 能满足我的需求。何谓「相对完美」呢？就以上三款插件，详细聊下几点要求：

### 1. 支持云同步

这点非常非常重要，尤其对于我这种多设备的人来说，没有云同步就意味着我要手动进行导入导出，这是非常难以接受的。我之前主力用的第二款 Session Buddy 其他方面几乎完美，但就因为不支持云同步，故最后只好转入 FreshStart 的怀抱…..

那为什么 FreshStart 和 Stash 支持云同步，而 Session Buddy 不支持呢？这要从两者的存储原理差异性来谈起了。

FreshStart 和 Stash 都是通过在 Chrome 收藏夹中，建立一些以时间点为名的文件夹来储存 标签页的。该文件夹包含了此次所保存的标签，再通过 Chrome 的原生收藏夹同步，从而实现了拓展数据库的同步。而 Session Buddy 就是自有数据库了，自有数据库格式的好处是可以进行更为强大的功能扩展（如 Json 导出导入、格式定制等），但坏处就是要自建通道，非常麻烦。

我曾捐赠过 SessionBuddy（因为确实好用，功能设计是同类产品最好的），也在捐赠后去找是否有同步的解决方案，结果还真的有人整了一篇[教程](https://groups.google.com/forum/?fromgroups#!topic/sessionbuddy-discuss/OQKUJG3bFnc)，非常详尽，但这折腾成本真是……

### 2. 可选择性保存标签页

其实就是保存的时候可以自定义哪些标签页需要保存，哪些不需要。这个特性非常重要，因为在很多时候保存标签页时，所面临的格局是：特定主题的网页+其他中转页。「中转页」意思是当你在进行搜索或探索行为时，由一个关键词进行多层级关联后遗留的页面。当你找到最终页面时，你往往不再需要这些中转页了。但是如果你只能保存所有标签页为一个分组，那久而久之，你需要付出的整理成本将越来越大，越来越不想整理，这就违背了 标签页 管理的初衷。因为这个原因，我抛弃了 Stash。

在 FreshStart 中保存标签页

### 3. 多窗口保存

这点就不赘述了，就字面意思，可以同时保存多个窗口下的标签页，这个特性对于我这种双屏、三屏党（趋势:P）而言是最需要的。FreshStart 是我尝试过的，对这个特性支持得最好的管理插件。

### 4. 可单独编辑某分组下的标签

有时保存标签页时往往是一瞬间的事，这意味着你可能并没有太多的时间来一一筛选；而当你打算进行分组二次编辑或恢复时，你需要的仅仅是该分组下的某个标签页，而该分组下却有几十个标签，这时你该如何处理？这也是我抛弃了 Stash 的原因。

![](.evernote-content/C510AA6E-80FA-4E43-82DC-10230ECDF457/6895B090-89D0-47BB-ABD0-6A84B72978AA.png)

在 FreshStart 中编辑分组标签

### 5. 可给标签页分组命名

也是字面意思，该特性很重要，Stash 默认是以最前一个标签页的标题页当做分组名字，导致每次看起来总是怪怪的，而 SessionBuddy 和 FreshStart 对此特性支持非常好。

总结
--

FreshStart 的交互做得一般般，具体表现为当你编辑超过 10 个标签的分组时，空间会很捉急。功能自定义也没那么强大（相较于 SessionBuddy 而言），但有个非常好的特性是CrashRecovery，能每隔 5 分钟自动保存标签，非常实用。

所以，如果你需要一款支持云同步的标签页管理插件，我首推 [FreshStart](https://chrome.google.com/webstore/detail/freshstart-cross-browser/nmidkjogcjnnlfimjcedenagjfacpobb?hl=zh-CN)。
