我之前一直觉得，学 Claude Code 主要靠两件事：自己多用，或者去翻官方文档。

后来才发现，Claude Code 里面本来就有一个学习入口："/powerup"。

这个命令不负责帮我们写代码、改 bug。

它更像是一组放在 Claude Code 里的互动演示，带我们看看一些平时容易忽略的能力。

很多时候我们学一个新工具，很容易带着旧习惯去用它。

以前怎么写代码，怎么查问题，现在就怎么让 AI 帮忙。

用了一段时间之后，表面上是在用新工具，实际还是老一套工作方式。

"/powerup" 比较有意思的地方在这里。

它会让我们重新看看：Claude Code 到底还支持哪些工作方式。

很多能力不一定马上用得上，但至少会知道，原来它可以参与任务规划、上下文管理、工作流拆解，以及更完整的开发过程。

所以我觉得，刚开始用 Claude Code 的朋友，可以花几分钟体验一下，已经用了很久的朋友，也值得花几分钟看一看。

有时候真正限制我们的，是旧习惯。工具变了，使用方式也该跟着变一变。
