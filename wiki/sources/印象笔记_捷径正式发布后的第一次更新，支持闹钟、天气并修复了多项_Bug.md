这次是捷径应用发布后的第一次更新，包含了一系列新功能和 bug 修复。

新增功能
----

首先是这几类操作有了更新：

* 天气类：获取当前天气、获取天气预报
* 闹钟类：创建闹钟、切换闹钟、开始计时
* 计算类：测量、转换测量
* 照片类：获取最新导入

### 天气操作

之前获取天气信息是个比较复杂的工程，需要用到一些服务的 API。但现在获取天气在捷径中被简化，通过「获取当前天气」和「获取天气预报」这两个新操作，就可以获取当前和未来一段时间的天气，不仅是温度和晴雨，还包括日出日落时间等等，我们可以轻松地把这些操作作为其它捷径的一部分进行利用。比如说 Power+ 就有读者曾经要求给他做一个捷径，运行后会给异地恋的女友发短信告知对方天气情况，提醒她带伞穿衣，十分温馨。

如果你已经升级到 iOS 12.1，这两个无比传神的 emoji 也能刚好派上用场：

![](.evernote-content/21C86725-F392-42F2-9D58-6CCC7E96BB16/6447BF97-5D1F-4668-A17A-ABF9285B4727.png)

寒冷和酷暑的表情也可以用来代表天气

### 闹钟操作

闹钟方面，捷径新增了「开关闹钟」、「创建闹钟」和「开始计时」3 个操作。

我不久前做过一个「[自定义勿扰模式时长](https://www.icloud.com/shortcuts/b6b8ba9bc671415f9b27cecbdc95fcef)」的捷径，可以自己调整勿扰时间的长短，午休时拿来用很灵活。捷径应用更新到 2.1 版本后，我马上改进了勿扰模式的捷径，给它增加了一个闹钟提醒，在勿扰模式结束时可以把你叫醒——万一你睡过头的话。

![](.evernote-content/21C86725-F392-42F2-9D58-6CCC7E96BB16/891CA97F-124F-4771-B2E5-23CF68361256.png)

勿扰模式的捷径现在能够叫醒你了

此外，你也可以分工作日和假期来管理不同的闹钟，一键开关度假模式。这个比较简单，留给大家自己研究，有成果的话欢迎在评论里分享。

### 单位转换

相信各位读者都知道，Spotlight 已经支持了单位转换：

![](.evernote-content/21C86725-F392-42F2-9D58-6CCC7E96BB16/2518C020-AEB8-4988-BCF0-DBCD0673AF2B.png)

捷径的单位转换比 Spotlight 丰富得多

但是我们在捷径中，偶尔还是会需要用到单位转换这类操作。捷径这次的更新，**把原本要好几步的操作化简成了一步**。

捷径中有很多这样缩减工程量的操作，比如「图片蒙版」、「计算统计数据」、「获取 App Store 应用的详细信息 」、「从输入中获取 URL」等等，都属于这种「清障」型的操作，玩家可以不必再怼着算法和 API 自己造轮子。

作为作者和编辑，平时经常和图文打交道，如果手边不巧没电脑，难免就要在手机或 iPad 上压缩图片，于是千辛万苦搜到一个调用 TinyPNG API 的捷径，如获至宝。但是上期 Gallery 投稿里忽然冒出一个压图的捷径，调用的是捷径自带操作，效果奇好，还可以自己调参数。从压缩的效果来看，估计捷径和 TinyPNG 用的是同一套算法（实际上业界基本都是这套算法！），但是捷径把原本复杂交互打包了，带来的便利不言而喻。

### 停止 iCloud 分享

最后一个新功能是管理已经分享出去的捷径链接。只要是你自己创建的 iCloud 分享链接，你可以随时停止分享它们。虽然一般我们没啥必要停掉自己分享出去的捷径，但如果不慎把包含了某服务中自己的授权密钥的捷径公开了，那通过 iCloud 把它停用倒是能够减少损失。

![](.evernote-content/21C86725-F392-42F2-9D58-6CCC7E96BB16/E9134570-95B6-469D-A222-896E71F09A1C.png)

现在可以停止分享捷径

不过，我更希望的却是「更新现有捷径」之类的功能——如果捷径能像原生应用一样收到更新提示，比如原作者修复了出问题的捷径后可以直接发布给所有使用者，提示其更新，那就非常棒了。

修复功能
----

捷径初版发布后出了一些问题，比较棘手的有闪退、App Store 和 iTunes Store 相关动作异常……这些都很闹心。随着这次更新，大半问题都得到了修复，下面是比较重要的几项。

* 修复闪退，原文是「修复了打开编辑器时“捷径”应用可能会崩溃的问题」
* 修复 App Store、iTunes Store 操作运行时显示错误代码的问题。所以，现在 Gallery 里的「[图标收割机](https://www.icloud.com/shortcuts/aded18b3ecf24eb0aad986eb51ccb51c)」等受灾捷径终于恢复正常了。
* 修复同步捷径时出现的重复。选择困难症的朋友，有没有遇到过选了半天结果选了个旧版捷径的情况？现在应该没事了。如果你心有余悸，可以用「[备份捷径库](https://www.icloud.com/shortcuts/f13fc46f57de4ba8b681d2c7c682d618)」来上一道保险。
* 修复了登录 Todoist 可能会现实错误信息的问题。

不过有些 bug 依旧存在，比如：

* Apple Music 相关操作没有完全修好，「获取当前歌曲」操作会提示「项目无法导出……未同步到这台设备」，不过我们仍然可以获取音乐标题。
* 还是 Apple Music，「获取当前歌曲」真正获取的是上一首歌。
* 捷径同步排序问题：这也是个老问题，经常同步时捷径顺序会被打乱，我在审核 Shortcuts Gallery 投稿时本地捷径就乱成了一团。

小结
--

暇疵归瑕疵，这次的更新透露出了 3 个比较重要的信息：

1. **捷径会作为单个应用更新：**只是不知道这次随着 iOS 12.1 更新是偶然还是苹果造势。
2. **捷径的更新更加深入系统：**先前已经支持了包括勿扰模式在内的一种系统开关，现在又闹钟和天气，以后应该可以期待和系统更深度的结合。
3. **捷径更新的调性未变：**依然是简化过去繁琐的操作，降低新人门槛，把原来复杂的需要使用多个步骤才能实现的功能转化为一个操作。

作为正式发布后的第一次更新，捷径没有体现出之前的担忧，它还是我们熟悉的那个工具，仍在继续开辟 iOS 的自动化的可能性。

作为一款被苹果收购的 iOS 自动化应用，捷径同时兼顾了易用性与可玩性。如果你想全面了解捷径的用法，可以订阅由少数派出品的**《[捷径：由浅入深完全指南](https://sspai.com/series/68)》（7.5 折促销中）**。教程会通过大量独家原创捷径实例，并且覆盖本次捷径应用的更新内容，教你成为捷径高手。
