Claude Code 提供了一‍个官方的 GitHub 集成，让 Claude 可以在 GitHub Actions 中运行。这个集成提供两个主要工作流程：﻿对 issues 和 pull requests 的提及支持，‌以及自动 pull request 审查           ﻿

设置集成
----

要开始使用，在 Claude 中运行 /install-github-app。这个命令会引导你完成设置过程：VprbsNNi06uIawNJLcFbxOcV+1AAkBa39eKpDvimHqc=

1. 在 GitHub 上安装 Claude Code 应用
2. 添加你的 API 密钥
3. 自动生成一个包含工作流文件的 pull request

生成的 pull request 会向你的仓库添加两个 GitHub Actions。合并后，你将在 .github/workflows 目录中拥有工作流文件。

默认 GitHub Actions
-----------------

集成提供两‍个主要工作流程：          ﻿         ‌         ﻿

### 提及 Action

你可以使用 @claude 在任何 issue 或 pull request 中提及 Claude。当被提及时，Claude 将：VprbsNNi06uIawNJLcFbxOcV+1AAkBa39eKpDvimHqc=

1. 分析请求并创建任务计划
2. 以完全访问你的代码库的权限执行任务
3. 直接在 issue 或 PR 中回复结果

### Pull Request Action

每当你创建 ‍pull request 时，Claude ﻿会自动：       ‌           ﻿              rqSLrC9+VsupyuuN5CJXBcTogV3TB0PLFwn6Ar40W6Y=

1. 审查提议的更改
2. 分析修改的影响
3. 在 pull request 上发布详细报告

合并初始 ‍pull request 后，你可以自定﻿义工作流文件以适应你项目‌的需求。以下是如﻿何增强提及工作流程：9chWMQ7We23/Q9LjGF84uV9QonUxGOU3a5J+eWKzipk=
