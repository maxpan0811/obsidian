每天早上醒来，打开手机或电脑都会有大量的信息扑面而来，这就是如今我们的生活。开始你会兴奋，发现知识的海洋真正向你张开了怀抱，你能轻松地获取到大量想要的信息。但渐渐的，你会变得焦虑，面对海量的信息感到迷茫。

这时候，如何管理自己的资料就变得尤为重要。今天我会站在一个设计师的角度，来给大家分享一下自己的方法。

对于设计师，接触到的资料类型有很多。我将它分为图片、链接和文档。

先来说说图片，图片素材应该是设计师最为重要的资料也是最占空间的资料。十年前我不断尝试用 Windows 文件夹来进行分类管理，但便利性始终很低。转到 Mac 后，你会发现这完全是另一片天空。虽然没有达到想要的标准，但整体上已经上了一个台阶。

先聊一下目前我对图片管理的需求：

1. 轻便、快速；
2. 基于文件夹和 tag 交叉管理；
3. 支持网页截图；
4. 支持订阅。

一年之前，我向身边的设计师们推荐 Ember。它的出现就像一头机器怪兽秒杀了其他所有同类应用，Ember 集成文件管理器、浏览器、订阅器，再加一个图像设计工具设计师们就可以扎里面不出来了。49.99 美金我乖乖地交了，可怪兽渐渐臃肿，迈不开脚步也张不开嘴。而就在我不堪忍受其糟糕性能的时候，Inboard 正好接棒了。

Inboard 我非常喜欢的一点在于做减法，它只关心怎么把图片素材管理做到最好。对啊，浏览网页的事儿应该留给 Chrome 和 Safari 去干，订阅的事儿应该留给 Reeder 去干。

用来一段时间的 Inboard，我的评价如下:

1. 满足文件夹、tag 交叉管理；
2. 轻量、速度快；
3. 网站一键截图保存（保留原始 URL）；
4. 支持 Dribbble like 订阅（稍后会增加 Behance 的支持）。

接下来聊聊链接，自从 Yahoo 接手 Delicious 后，这款曾经笑傲书签界的应用就日渐没落。还好让我找到了 Pinboard，一个更为神奇的书签应用。

![](.evernote-content/A3A1704B-EE8B-43D2-A88B-B36E3F5B2B9A/29DD08B2-28CE-48C3-AEDD-0601378F4880.jpg)

很多人第一眼看到它就会关掉。是的，Pinboard 的页面几乎没有设计，但它却做了让我觉得挺了不起的事情。

1. Pinboard 只关心如何将收藏夹这件事做好，没有其他；
2. Pinboard 没有做客户端，开放丰富的 API 供其他开发者来让丰满自己的产品；
3. 递增式的收费方式，越早加入年费越低。4 年前加入时年费应该是 9 块多美金，现在已经 11 美金一年了。采用这种收费方式且还活着的，我目前只知道这一家。

正如前面提到的，Pinboard 在 OS X 和 iOS 上的应用都非常丰富。目前在 OS X 上我首选的是 Spillo。

![](.evernote-content/A3A1704B-EE8B-43D2-A88B-B36E3F5B2B9A/5838A9CA-CF8A-4A03-A265-E2BB973654F8.jpg)

标准三栏式设计，能够替代网页来进行书签管理。同时还可以进行订阅书签的查看（这个在大多数应用中都没有实现）。至于新加收藏，我还是更喜欢使用 Alfred Workflow 来完成。

![](.evernote-content/A3A1704B-EE8B-43D2-A88B-B36E3F5B2B9A/80F8C5D5-88A4-4B96-9EFF-00B076D2F84A.jpg)

最后来说说文档，这也是我目前唯一没有满意解决方案的一块。我不得不提 Evernote，除了对有用的 URL 进行收藏，有时候还需要对一些图文进行摘要保存，在这方面 Evernote 有丰富的插件可以来帮助提升效率。但如果要说到日常记录和码字，我非常不喜欢 Evernote，因为它过于臃肿花哨。

还好我们有 Ulysses。

Markdown 语言由于其轻便易用的优势已经成为很多人的首选。OS X 上的 MD 应用层出不穷，但在文档管理这块 Ulysses 当仁不让领先于其他选手，而文档的管理也是我们今天一直在讨论的话题。当然，在书写的体验上 Ulysses 也不亚于 Mou、iA Writer 等 Markdown 应用。

![](.evernote-content/A3A1704B-EE8B-43D2-A88B-B36E3F5B2B9A/46B0F370-EECC-4570-8D7B-3BDBF688407C.jpg)

既然是个人资料的管理那就需要面对数据的备份问题，好在处在云数据今天这些都不是问题。

1. Inboard 可以将图片存储路径改为 Dropbox 来完成数据的云同步；
2. Pinboard 可以通过 IFTTT 将每条添加的收藏网址保存到 Google Drive 中；
3. Ulysses 可以通过 iCloud 进行文档数据同步。

这两年我不断删除旧的图片素材，因为有很多多年都未曾再次打开。那些曾经有用的文章早就应该成为你脑海中的一个知识点而不是一个备份，工具永远只是工具，它不能替代你思考。如果不去真正去吸收，它终将变成另一个「海洋」——犹如当年 RSS 阅读器上的那个 999+，你再也不愿意去点开。
