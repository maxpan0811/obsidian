# 法律人的Claude Code教程（七）（教不会我吃民法典！）：Hooks — 给AI立铁律

Original两眼一睁就是学呀 智律积成

上一篇我们学了 Skills，把工作经验封装成"一键操作"，没看过的同学赶紧补补课。

你的 Claude Code 现在应该已经很好用了。但你有没有遇到过这样的情况：

- 你在 CLAUDE.md 里写了"完成任务后通知我"，它忘了
- 你告诉它"每次修改文件都要记录日志"，它跳过了
- 你让它审一份长合同，切到别的窗口干活，半小时后想起来，它早就审完了，一直在等你

***我都写在规则里了，你怎么还是会忘？***

**这不是它"不知道"该怎么做，而是"知道"和"每次都做到"之间，差着一个东西。**

这个东西叫**确定性**。

---

## 法律人最懂确定性

你在合同里写违约条款，不是因为"希望"对方不违约，而是，**万一违约，后果是确定的。**

你不会跟对方说"请你一定要按时付款哦"，然后指望他自觉遵守。你会写：

***"逾期付款的，每日按未付金额的万分之五支付违约金。"***

不靠说服，靠规则。Hooks，就是你给 Claude Code 写的"违约条款"。

- CLAUDE.md 是"嘱咐"，它可能忘
- Skill 是"教招式"，它可能跳步
- **Hook 是"立铁律"，触发了就自动执行，没有商量余地**

![](attachments/8709d581c783531f.png)

---

## Hooks 到底是什么？

先说人话版结论：

***Hooks 就是你提前设定好的"如果……就……"规则。***

如果任务完成了，就播放提示音。如果修改了文件，就自动记录日志。如果碰了原始合同，就拦截。设好之后，这些规则**每次都会执行**，不取决于 AI 那天"状态好不好"。

还记得我们的武侠线吗：

| 教程 | 能力 | 武侠类比 |
| --- | --- | --- |
| 教程一~二 | 安装、基本操作 | 拜入师门、扎马步 |
| 教程三 | 工作模式、权限 | 内功心法 |
| 教程四 | CLAUDE.md | 调息吐纳 |
| 教程五 | MCP | 倚天剑屠龙刀（兵器） |
| 教程六 | Skills | 太极拳法（招式） |
| **教程七** | **Hooks** | **护体真气（自动防御）** |

Skills 是招式，你得主动使出来。**Hooks 是护体真气，有人偷袭，自动弹开，不需要你运功。**

---

## Hook 触发之后能做什么？

规则被触发之后，可以执行四种不同类型的动作：

| 类型 | 能做什么 | 法律场景举例 |
| --- | --- | --- |
| **执行命令** | 在你电脑上跑一条指令 | 播放提示音、写日志、拦截危险操作 |
| **发送请求** | 把信息发到一个网址 | 任务完成后自动通知团队协作平台 |
| **让 AI 判断** | 让 Claude 做一次单独评估 | "这次审核是否覆盖了所有必查项？" |
| **派出子代理** | 派一个小助手去检查文件 | "检查输出的审核意见格式是否规范" |

大多数情况下，前两种就够用了。后两种适合需要"理解内容"才能判断的场景，比如检查合同审核是否遗漏了关键条款。

你不需要记住这些类型的名字。配置的时候，**跟 Claude Code 描述你想要的效果就行**，它会自动选择合适的类型。

---

## 先看效果

老规矩，先不讲怎么配，直接看配好之后是什么体验。

### 效果一：任务完成，自动播放提示音

我让 Claude Code 审一份合同，然后切到别的窗口干活。审完了，电脑自动响了一声。我听到声音，切回来，审核结果已经等着我了。再也不用隔几分钟就切回去看一眼。

### 效果二：每次修改文件，自动留痕

我有一个韦伯社会理论研究项目，让 Claude Code 帮我整理资料、写研究笔记。问题是：它改了哪些内容？改之前是什么样？

如果没有这份日志，我可能根本不会注意到。等发现的时候，内容已经找不回来了。**但有了审计日志，打开一看就知道：什么时候删的、删了什么、改成了什么。**

（聪明的你应该能想到其他方法也能实现留痕，但 Hook 的好处是：你不需要记得去做，它自动帮你记。）

## 怎么配？

和教程五配 MCP、教程六写 Skill 一样：**直接跟 Claude Code 说。**

### 配提示音

帮我配置一个全局 hook：当你完成任务的时候，自动播放 macOS 系统提示音

完了。它会自动写好配置，你不需要碰任何文件。想换个声音？继续说：

把提示音换成 Hero，听起来更明快一点

### 配审计日志

在项目目录下打开 Claude Code，说：

帮我给这个项目配置一套审计 hook：
1. 每次会话开始时，记录开始时间和项目名
2. 每次编辑或创建文件时，记录操作类型、文件路径、修改前后的内容
3. 每次会话结束时，记录结束时间
日志写到 .claude/research*\_log.txt*

**你只需要说清楚"我要记什么"，它来写规则。**

### "全局"和"项目级"怎么区分？

和教程三讲的配置层级一样：

| 你想要的效果 | 怎么跟 Claude Code 说 |
| --- | --- |
| 所有项目都生效（比如提示音） | "配置一个**全局** hook" |
| 只在当前项目生效（比如审计日志） | "给**这个项目**配置 hook" |

Claude Code 会自动写到对应的配置文件里。

---

## 还能用 Hook 做什么？

上面两个是我自己在用的例子。其实 Hook 的思路很开放，你可以根据自己的工作场景自由组合。随便举几个方向：

- "每次审核完成后，检查是否覆盖了所有必查项"，防遗漏
- "执行命令之前，先检查是不是涉及删除操作"，防误删

**你不需要提前想好所有规则。碰到问题了，跟 Claude Code 说一句，加一条 Hook 就行。**

---

说实话，Hooks 这个功能本身并不复杂。真正重要的是它背后的思维方式：从"希望 AI 做对"到"确保 AI 做对"。

这个转变，适用于你使用任何 AI 工具。而法律人天然就懂这个道理。毕竟，我们每天都在做一件事：

**用规则创造确定性。**
