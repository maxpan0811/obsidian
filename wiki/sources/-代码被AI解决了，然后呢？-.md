Fiona Fung 这期访谈，表面在聊 Claude Code 和 Cowork，真正有价值的是另一件事：当 AI 已经能把代码产出放大很多倍，团队到底靠什么继续拉开差距？

她有一句话很准：coding is salt。代码像盐，很重要，但不是主菜。

这句话背后有三个变化：

1. 代码变快以后，验证更稀缺。

模型可以生成代码，但不能替团队定义“什么算好”。企业如果只统计代码行、PR 数、token 消耗，很容易把动作误认为进展。

2. 高主动性要绑定高问责。

coding agent 降低了行动成本，也降低了乱动成本。一个人可以很快开很多分支、试很多方案。真正重要的是：每个动作有没有假设、验收和复盘。

3. 团队要重新设计共同现场。

Agent 越强，个人工作流越私人化。每个人都有自己的 prompt、routine 和工具用法。短期看效率更高，长期看组织学习可能被锁在个人电脑里。

所以，AI 编程的下一步，已经不只看谁会写代码。更大的分水岭在于公司有没有三套系统：

- 质量系统：把 what good looks like 写进文档、仓库和评测机制；

- 责任系统：让主动性和结果绑定；

- 学习系统：通过 dogfooding、pair lunch、内部 demo 和失败复盘，把个人经验变成团队资产。

代码越来越像基础材料。真正难替代的，是定义问题、判断质量、验证结果，以及把工具能力变成用户价值的人。
