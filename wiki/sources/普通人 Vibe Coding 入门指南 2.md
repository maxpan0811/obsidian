# 普通人 Vibe Coding 入门指南

---

Vibe Coding 大概是从去年开始叫响的。这个纯粹的英文单词中文翻译过来叫氛围编程。我觉得不太贴切。编程和氛围有啥关系。

它想表达的概念容易理解。编程不再是需要天分，需要经年累积的经验，需要苦大仇深地坐在电脑前把头发薅秃才能勉强入门的技能。现在有了 AI，你可以喝着茶，品着酒，聊着天，听着音乐，聊天的同时，顺带手把活儿干了。

真有那么简单吗？一半一半吧。

不过我想，编程这种曾经有点高级的技术活儿，对很多人来说可望而不可及。现在 AI 降低了进入这一领域的门槛，而现代社会的根基之一便是运行各种程序，App 的设备，所以，通过 AI 掌握一定编程技术，很多人会感兴趣，蠢蠢欲动。这是我想写一篇入门指南的初衷。

在描述进入 Coding 领域之前。我认为最重要的不是学习 AI，而是对这件事是否有足够强的好奇心，足够强的动力。否则，即使是最初级，最平缓的学习曲线，也会让人放弃。

诚如 Dan Koe 所言，好奇心驱使我们探索，探索让我们深入学习技能，掌握技能的快感足以弥补这一过程中消耗的心力，精力。

如果动力不足，必然找不到快感，也就无法继续下去。

Vibe Coding 的第一步，是学习与 AI 对话。

豆包的崛起让用户觉得和 AI 对话并不难。用户表达和 AI 理解是双向奔赴，两者之间的距离会越来越短。我同意。但现实是，即使两个人类交流，一样会有信息损耗。

最善于表达的人，也无法保证每次表达都能 100% 精准输出自己的思想。接受信息的人，基于自己对事物的认知，又会对同样的文字做出不同的理解。信息在传递中被扭曲是必然的。

我和 AI 沟通的经验是，不要怕重复，不要怕啰嗦，不要嫌麻烦。要把想做的事，想要完成的目标，尽可能详细地描述出来，告诉 AI。把 AI 当做一个具备成人知识的幼儿园小朋友，把话说明白。

现在的AI已经大有提高，甚至能听出你的话外音。但是猜测不一定准确。好的沟通是不要让对方猜，而是明确表达。少用代词，不断重复，分部确认，这样才能让 AI 做事尽可能不偏离要求。

学会和 AI 对话之后，接下来的步骤就有了 AI 加持。任何不懂的问题都可以交给它，让它给你答案。

如果只是想完成一个非常简单的算数程序或者检索 excel 表的 python。那么，有对话框的聊天 AI 足矣。

即使你不会安装 Python 环境，也可以通过请教 AI 来完成具体操作。

此时还谈不上 Vibe Coding，也不用挑什么模型。海内外几乎所有模型都能满足初阶代码需求。

真正进入 Vibe Coding，至少要学会一些简单的工具使用。比如 IDE 工具。

所谓 IDE，全称是 Integrated Development Environment，集成开发环境。最常用的工具是 VS Code。VS Code 在 Windows 和 Mac 上都有，全图形界面，一边显示文件夹，一边显示代码。

初哥不需要学习 VS Code 的详细用法，只要会安装，会打开即可。VS Code 支持 AI 插件，安装 AI 插件之后，编程就像聊天一样，在对话框里完成。

我现在最喜欢用的 IDE 是谷歌的 Antigravity，中文叫反重力。

Antigravity 集成了谷歌的 Gemini 3.0 Pro 模型和 Claude Opus 4.5，业界领先。但 Antigravity 的使用有点麻烦。它要求用户的 IP 在美国，而且产品流量并不全走系统代理。这导致很多人卡在登陆那一步。

路由器转发流量和电脑上软件开启 TUN 模式，都可以解决这个问题。这需要用户具备这两项技能。

所以我更推荐 VS Code。

有了 VS Code，编程不再是一个个独立的代码程序，而是存在文件夹里，有结构，有对应关系。再来 Vibe Coding，天花板要比局限在聊天窗口里高了不少。

过去一年里，Vibe Coding 技术进展非常大。早期我用 AI 编程，越往后越不敢用。我总有用火柴棍搭埃菲尔铁塔的感觉，不知哪一次修改就会让衣衫褴褛但运行无碍的代码崩溃。事实上，一年前不管是哪个模型，都捅过类似的篓子。

可谓编程 1 小时，排错 1 整天。

自从 Gemini 3.0 发布，海外一线模型在代码完整性，可靠性上有了极大的提高。极少会出现早期那种越改错误越多的毛病。

Antigravity 甚至在我要求它将代码推送到我的 github 仓库之后，形成了记忆，每次修改完成，都自动推送到 github。

Github 是开源代码仓库。个人开发本不需要它，但代码仓库提供了版本控制。每次修改即推送，相当于保留了代码时间点。如果新的修改有问题，改来改去还不如原来，可以直接回滚，回到过去。

去年新手想和 Github 互动，还得在 AI 聊天框里学习命令，自己在终端里运行。现在，不管是推送，拉取，同步，回滚，都可以让 AI 自动操作。你只要在 VS Code 里点击允许即可。

这叫 AI Agent。

Agen 相当于 Vibe Coding 的手，可以直接修改文件，进行各种本地操作。Antigravity 甚至会在程序修改之后，自行编写一个测试程序，先测试功能是否正常，再通知用户。如果有错误，它会自己根据报错直接改，改完再叫你。

玩到这一步，基本已经入了 Vibe Coding 的坑。任何你不懂但又想知道的问题，想要深入学习的技能，都可以通过和 AI 对话搞定。

只要有好奇心，你完全可以和 AI 一起成长。

模型的选择在初期依然不重要，大部分模型都能满足初级开发需求。不过，我观察到的现象是，越是初哥，想要做的事情越大。因为他不知道产品背后的逻辑是什么。在不懂的人看起来，一个 App 几句话就能说明白。“给我做个抖音”，“给我写个 Photopshop”，诸如此类。

不懂拆解产品的人是做不出好产品的，给他再强的 AI 也没用。

能把产品需求拆解成页面，并且用语言将页面描述清楚，是 Vibe Coding 的基本能力。好在现在大模型有视觉能力，你说不清楚可以直接放截图。让 AI 照猫画虎。

我的使用经验里，Gemini 在 UI，也就是用户看得见的使用界面上，编程效果比较好，还原能力强。OpenAI 的 Codex 则对后端数据处理逻辑和复杂问题的处理上更好。

比如我的交易机器人，用 Gemini 写完之后，交给 Codex review，针对我不太满意的地方出整改方案。Codex 能够厘清复杂需求，将之逐条反应在代码上。效果非常好。

如果用不了海外双雄，朋友们反映，智谱的 GLM，千问 Coder，Minimax 代码生成，也能用。

再进一步，Vibe Coding 的工具就要变一变了。

VS Code 这种图形化工具在高阶用户面前，有点小儿科。目前高级码农们推崇 Claude Code，Codex CLI 或者 Gemini CLI。

它们仨都是命令行界面，Claude Code 水平最高。水平高，不是高在模型上。Claude Code 核心价值是 Agent，也就是执行 AI 指令实际操作电脑的能力。在命令行界面里，Claude Code 几乎无所不能。多线程工作，审查，修改，运行代码。

它之所以是高阶工具，一方面是普通人厌恶命令行界面，不直观。一方面是它的能力太过不受限，稍不留神，删光你硬盘数据也不是不可能。

我安装了 Claude Code，主要用它使用 Skills。Claude Code 理论上必须搭配 Claude 订阅。实际上，智谱 GLM 提供对等接口，指导文档，可以假装 Claude Sonnet 模型接入到 Claude Code 里。只是使用 Skills 的话，区别不大。

能掌握到这里，我认为是业余玩家的深水区，够了。Vibe Coding 是未来 AI 时代的实用技能，得会，但不用精通。

最重要的不是学会 Vibe Coding，而是用 Vibe Coding 做了你以前做不了的事。

PS：

今天研究了一下预测市场，感觉大有可为。以前不会编程，无法用数学模型去计算概率。现在有了 Vibe Coding，万事皆有可能。
