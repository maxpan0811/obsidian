关注我

朋友们，今天要跟大家坦白一件挺丢人的事。我当初用 Claude Code 写代码，一直就是干巴巴地在对话框里打字，让它改这改那。改完了什么样，我就看它嘴上说的。它说改好了，我就信了。结果有一次，一个登录流程的重构，它顺手把我三个不相关的文件也动了。我压根不知道，直到上线前测试的同事跟我说，你这接口怎么返回值变了。

我当场就懵了。翻了半天对话记录才搞明白怎么回事。

后来才知道，Claude Code 里有一个 /diff 命令，一敲就能看到它到底改了哪些文件、改了哪些行。我这大半年，等于是闭着眼睛开车。

我对自己这种用法的评价是：纯纯的原始人。

![](.evernote-content/38CDC4A9-514C-43AC-9F51-D571D0BBFD52/237FB4EB-E5BF-43E4-AAB4-01D3C47F34A7)

一、先搞懂这件事：斜杠命令不是用来让AI变聪明的

在 Claude Code 会话里输入一个 /，会弹出一堆命令。

/help、/model、/init、/plan、/diff、/compact、/doctor，一大串。

很多人第一反应是，这些是不是能让 Claude 写代码更牛。不是的。这些命令干的事，是让你能伸手控制这场对话，而不是被 Claude 牵着走。

上下文太长了它开始瞎编，你用 /compact 压一压。让它先想方案它偏要直接动手，你用 /plan 拦一拦。改完了不知道动了哪，你用 /diff 查一查。

这套东西不是锦上添花，是缺了它你根本看不清 Claude 干了什么。

二、新手先记这10个，别的都能先放一放

朋友们别慌，40个命令不用一次背完。先把这10个焊在脑子里：

/help 查看帮助和当前可用命令/init 初始化项目，生成 CLAUDE.md/model 切换模型/plan 先规划，再动手/permissions 管理工具权限/diff 查看本次改了什么/compact 压缩上下文/status 查看版本、账号、连接状态/resume 恢复之前的会话/doctor 诊断安装和环境问题

记不住顺序的话，就记这句：开工前 /init，大改前 /plan，改完后 /diff，上下文长了 /compact，出问题 /doctor。

我把这句话抄在便利贴上，贴在屏幕边上整整一个月。

三、这几个命令，我现在天天离不开

1. /plan：省下的回滚时间比命令本身值钱

我之前重构一个支付模块，图省事直接跟 Claude 说"帮我重构一下"。

它上来就是一顿改，改完发现方向完全不是我想要的，撤回去又花了半小时。

后来学会了先用 /plan，让它把方案说清楚我再点头，一次性改对。

我对这个命令的评价是：省下的时间，够我多喝三杯咖啡。

2. /diff：验收环节，谁说都不如自己看一眼

这个就不多说了，前面那个丢人的例子已经说得很清楚。

Claude 说改好了，不代表真的改对了。/diff 是你唯一能相信的东西。

3. /model：小任务别拿大模型硬扛

简单的任务，比如改个文案、调个样式，我现在都会先切一个轻量模型。

复杂任务再切回强模型。省下来的额度，够我多跑几个正经任务。

4. /permissions：这个真的别偷懒

删除文件、装依赖、跑危险命令，这些权限我是从来不一次性放开的。

图一时方便，出了事收拾起来更麻烦。

5. /compact：上下文不是越长越好

对话拖太久，Claude 会开始忘事、答非所问。这时候别硬撑，/compact 一下清爽很多。

四、老手才会用到的那一批命令

/login 和 /logout 用来切账号。/memory 用来维护项目的长期记忆，跟 CLAUDE.md 挂钩。/hooks 查有没有配自动格式化、自动测试的脚本。/rewind 是我最喜欢的一个，改坏了直接回到之前的检查点，不用手动救火。/recap 适合任务做到一半，想快速理一遍现在进展。/skills、/plugin、/mcp 分别管技能、插件和外部服务的接入，出问题先看这三个。

这一批不着急记，用到的时候现查也来得及。

五、按场景记，比死背40个靠谱多了

第一次进新项目：/init → /memory → /permissions → /status。准备做复杂任务：/plan → /model → /permissions。Claude 改完代码后：/diff → /simplify → /recap。上下文太长：/compact、/focus、/clear 挑一个用。遇到异常：/status → /doctor → /debug → /feedback。接着上次的活儿干：/resume → /rename → /export。

![](.evernote-content/38CDC4A9-514C-43AC-9F51-D571D0BBFD52/035F6E8F-7D15-40B3-91C6-EF80667BA309)

六、三个坑，我踩过所以提前告诉你

第一，不同人看到的命令不完全一样，跟平台、账号套餐、版本、插件都有关系。最准的办法永远是自己输入 / 看一眼当前环境有什么。

第二，斜杠命令必须放在消息最开头。直接打 /model 才算数，写成"请你 /model 切换一下"是不会被识别的。

第三，这些命令不是万能按钮。/plan 给的方案还得你自己判断对不对，/diff 看完改动还得你自己验收，/permissions 权限放太开照样有风险。

工具再好用，最后拍板的还是人。

朋友们，赶快去敲一下 / 看看你现在的环境里有哪些命令吧，说不定藏着你从来没用过的宝贝。
