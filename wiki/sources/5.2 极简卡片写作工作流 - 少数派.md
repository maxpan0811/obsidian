# 5.2 极简卡片写作工作流 - 少数派

---

5.2 极简卡片写作工作流
=============

你好，我是 Louiscard，这是你打造信息管理系统的第 19 天，上一节课我们讨论了如何打破对写作的固有认知，算是输出部分的理论篇，这一部分我们更进一步，聊聊写作的基本流程，算输出部分的实战篇。

今天要给你介绍的方法是以时下最流行的卡片写作理念为基础，结合我自己多年的经验形成的标准化工作流，按照这几个步骤，我相信你可以用最小的成本迈出自己稳定输出的第一步啦。

第一步：确定主题
--------

我们在前面反复提到了「以输出为导向的学习方式」，不管形式如何，是写一本书、打造一门课、设计一次演讲都可以。

关键点在于：尽早确定主题。

比如我真正决定开始系统研究个人信息管理相关的主题之后，第一件事就是在 Ulysses 创建了写作项目，用几天把主题和大纲搞定。

![](.evernote-content/0A85C7AC-2114-472C-8302-810BF4CC305F/CB3951A8-5324-42EE-9611-CE29E754372C.0)

老话讲「万事开头难」，只要舍得花几天时间把要写的主题和大纲「立住」，后面的工作会越来越顺利，不知不觉就是好几万字。

除了确定一个项目的主题，每周聚焦在一个篇章级主题也不容忽视，否则依然会行动瘫痪。我曾经尝试过不确定具体主题，推崇「想到什么，随便写写」的灵感型策略，效果非常糟糕。

我大大低估了「完成」的价值。

「完成」这件事不仅能获得一种正向反馈，更重要的是它让你能够产生「放下」的感受。与 GTD 的底层逻辑相同，如果有 100 个齐头并进的主题在推进，有些才起好题目，有些暂时搁浅，有些达到了 90% 的完成度，听起来非常美好，但实际上，这千头万绪的状态会让你陷入一种强烈的未完成感，几个月都未必能交付一篇 3000 字的文章。

为了不被这种负面感受拖垮，我采取一种新的思路：「多更新，多发版」。

虽然这两年来少写「公众号」类型的内容，但投入到教程上的时间可真不少，我的策略就是每周聚焦一篇文章，一周完成一篇内容并交付编辑，这就等于产品发布上线了一个「版本」，下周可以开始推进新的内容或者启动这篇文章的第二个版本的更新，用清晰的版本保持良好的节奏，不给自己增加额外的管理成本。

第二步：撰写卡片
--------

首先，不得不提到「卡片写作」（index card writers）和首席布道师大作家纳博科夫。

![](.evernote-content/0A85C7AC-2114-472C-8302-810BF4CC305F/E64E531E-1044-4043-9338-DABEE1C5E16B.0)

纳博科夫是 20 世纪最伟大的作家之一，他基于卡片的创作方式影响了非常广泛的创作者。纳博科夫曾在接受 BBC 采访时分享自己的创作方法：

> 我发现索引卡片确实是写作最好的纸张，我可以用它们来完成协作。 我不会从开始逐章写到结尾。我只是在填补拼图中的空白，这个拼图在我的脑海中非常清晰，从这里和那里挑出一块，填满一部分天空，一部分风景，还有一部分 —— 我不知道，那些寻欢作乐的猎人。
>
> 1

除了纳博科夫，像钱钟书、李敖这些作家也是卡片写作的拥趸，而且不仅是作家，现在像奥斯卡最佳原创剧本获得者 Dustin Lance Black 这样的好莱坞编剧也将卡片用在自己的创作系统中。

具体怎么做呢？

简单来说，就是围绕主题，不考虑逻辑结构，将想到的内容以「卡片」的形式记录下来，非常发散地形成一个个相对独立的信息组块，最后再将其拼接形成完整的作品。

![](.evernote-content/0A85C7AC-2114-472C-8302-810BF4CC305F/B40A3E86-187F-44D7-8D8B-636AD39650EE.0)

我在跟 Hum 交流时，他提到一种写作状态很有意思：

> 喝醉了写作，清醒时修改。

大体意思也是如此，我们确定一个主题开始撰写卡片，不要在意所谓的条理有序，你大可不必真的像纳博科夫一样使用纸质的索引卡片来做这件事，选择一款笔记工具，比如 Ulysses、备忘录或 Evernote，或者更专业的 [Gingkoapp](https://sspai.com/post/gingkoapp.com) 或 Scrivener 把你想到的例证、数据、结论、观点、比喻等等尽数写下来就好。

比如前段时间刷抖音看到社科院的一个副院长分享关于专业的理解，我顺手记了张卡片，表述都非常简单，完成度很低，但在卡片撰写阶段已经足够，这部分就有机会成为专栏中的补充素材。

![](.evernote-content/0A85C7AC-2114-472C-8302-810BF4CC305F/2153BAD5-A11E-4668-8CAE-FF514183C171.0)

如果你看到这个教程的过程稿，会发现大多数篇章在相当长的时间都是乱七八糟的状况，由一个个毫无关联的卡片堆在一起，这些想到什么写什么的卡片可能是：

* 卡片写作的节奏；
* 纳博科夫的采访；
* Hum 的醉了写；
* 找一个卡片的例子；
* 每周不同的侧重；
* 版本管理和 GTD；
* 等等。

有些是我在一开始想到的，有些是在地铁里补充的，有些是睡觉之前的突发奇想，这是一段持续的「头脑风暴」，几乎没有任何限制的无压输出方式，反而能激发你最大的创造力。

特别需要提醒的一点是：做好「写作素材卡片」和「纯知识类卡片」的区隔。我自己就吃了不少亏，起初有任何想法都会优先记录到印象笔记，感觉这样能在信息调取的时候更容易被搜索到，提供信息利用率。

想法很好，但实践中发现如果卡片更多融入到知识库中而失去写作素材的属性，会导致要写的作品迟迟积累不到成稿状态，甚至发展成：为了信息管理而信息管理。所以，我建议将「写作素材卡片」直接记录到 Ulysses 的文稿，相对于纯知识卡片，成为作品构成单位要更有价值。

简单来说就：要适度区分以日常学习为目的形成的知识卡片和专门为信息输出积累的素材卡片，否则这两者的混淆会大大拖慢写作进度。

第三步：资料搜集
--------

我习惯在积累到一定数量的卡片之后，补充进行相应的资料收集工作，下面主要介绍三个注意事项。

### 3.1 补充确认模糊信息

写作是一个需要不断补充和确认信息的过程，大脑不会记住所有内容，尤其是一些精准的数据和表达，这时候你就需要进一步去搜索核查。在写作的过程中遇到不确定的内容，不要卡壳去搜索，而是先用 【 】 标记，类似【图片：xxxx】 、【纳博科夫说过的一段关于灵感的那句话】 的占位符，在资料搜集环境统一补充。

### 3.2 在个人数据库和互联网搜索增量信息

由于日常在自己的个人数据库中存储大量高质量信息，这些信息有很多跟我们在写的主题高度契合，去笔记工具里搜一波，通常都能找到有价值的资料，这也是对咱们在前几章建立的信息收集管理系统的一种自信。

搜索的过程也是整理个人信息数据库的一次重要契机，我们在第三章提到对印象笔记、DEVONthink 等工具的整理应该尽量克制，接受部分信息以一种无秩序甚至凌乱的状态存放，原因也正是如此。

平时只需最低成本的维护其基本秩序，即便一些文章没读完直接归档存储，也有机会在这种时候被调取使用。

![](.evernote-content/0A85C7AC-2114-472C-8302-810BF4CC305F/570EBE00-8969-4815-ACB1-9C91AF8CB194.0)

从图片中能够看出来，这时候之前的分类又有什么用处呢？所以，别再纠结分类和秩序了，读不完先归档也没什么大不了，在输出这个环节，它们都还有好多次被翻牌的机会。

### 3.3 如何保存搜索素材

关于写作素材的保存，我的习惯是直接存储在 Ulysses 的正文，一些写作过程中实在不知道如何安放的内容会先放到注释中。

![](.evernote-content/0A85C7AC-2114-472C-8302-810BF4CC305F/E3BA0448-E197-42CC-9209-C02F917C4B7D.0)

大型的写作项目会建立专门的资料库。你可能会担心每个项目都有一个资料库，会不会导致一些资料要复制很多份，这如果是在纸笔的时代，的确如此，但是在数字化时代，有了超链接或者 Replicate 功能 2，我们就不需要这么做了，只需要做好链接索引就好，所有资料的原始版本只需要保存一份。

只需要维护一个版本的资料，所有索引位置的文件会同步更新到最新版本，而不会发生版本错乱。

第四步：撰写初稿
--------

在卡片积累到一定程度之后，就可以开始撰写初稿了。

撰写初稿和写卡片不同，你要表达的观点已经明确，相关素材和案例已经写好，但真要把这些零散的内容串联起来依然富有挑战，需要投入更多的专注和耐心。

首先要注意把握这一步开始的时机，写卡片和查资料自然越扎实越好，但需提前设置截止时间，否则永远觉得「准备得不充分」。另一方面，如果这两个步骤准备得确实不够扎实，撰写初稿的前提不成立，硬写起来也会颇为痛苦。

其次是找大块时间。我自己一般是在周六下午跑到咖啡馆，把写作工具全屏，一边听古典音乐，一边把零散的段落拼接成完整的文章。这样形成的文章会更完整通顺，风格趋于一致，读者看起来更流畅。

最后是不要边写边改，要刻意避免回头看，试图修改刚写下的东西。一旦你启动编辑模式，便一直会有一个声音说「这很无聊、这糟透了、这没有任何价值」，而这些对我们的写作过程除了打扰其实没有任何帮助。

为了避免边写边改，有些作者甚至用毛巾盖住电脑屏幕，抵抗这个写作的头号坏习惯。美国著名编辑 Grant 描述这个过程类说「试图在吃饭的时候收拾桌子一样不合理」，所以把写作和修改分开吧。

![](.evernote-content/0A85C7AC-2114-472C-8302-810BF4CC305F/FAD1804B-AFE8-47F0-AB76-E3B5F0991752.0)

第五步：修改
------

好的文章都是修改出来了，在写东西上花的时间越长，对这句话的感受就越深，正如纳博科夫所说：

> 风格和结构是一篇文章的精华，而伟大的思想不过是空洞的废话。

要想获得你自己的风格和结构，别无他法，只有用一次次修改才能淬炼获取。通过学习专业的写作方法，阅读优秀的作品，慢慢积累模式、形成意识、摸清规律，拥有更敏锐的文化自觉，然后在每一次写作中实践，抓住修改的机会，获得最有价值的反馈。所以，不要担心修改，多改，一遍一遍地改到 Deadline 那一刻。

现在流行一个词叫「品控手册」，我理解特别像我们当年上学那会用的「错题本」。在写作中也是如此，把你犯过的所有错误记录下来，每次修改的时候对照着这个「错题本」针对性的调整，而不是凭感觉。比如开头的部分，有没有开门见山，有没有生动案例、有没有口语化表达，比如是不是可以把「大家」都改成「你」，那些被动的表述是否可以调整为主动态等等。这不能保证你创造出旷世之作，但能帮助你高效稳定地提升水准。

第六步：校对排版
--------

排版也是信息输出非常重要的部分，好的排版可以让读者的阅读成本变低，让文章的阅读体验更好，因此大家也更愿意阅读你的文字，了解你的思想。

现在排版的成本越来越低，我自己在用 Ulysses 写作，写完之后预览成 HTML 格式，全选复制，直接粘贴到微信后台，格式、图片都能完美承继，完成不需要为排版付出任何额外成本。

![](.evernote-content/0A85C7AC-2114-472C-8302-810BF4CC305F/CF9ECE9D-EFF9-4F23-8EA0-CFDF6A4609A6.0)

如果你使用的 Markdown 工具没有导出 HTML 的功能，强烈建议你试试 [阿禅](https://zh.wikipedia.org/wiki/%E4%BC%8D%E5%98%89%E8%B4%A4) 提供的 [自动排版服务](https://knb.im/mp/)，只要把 Markdown 格式内容粘贴进去，就能获得非常精美的排版。

如果你用的是 Chrome 或 Firefox 浏览器，也可以使用 [Markdown-Here](https://markdown-here.com/) 这个插件在微信后台一键搞定排版。

如果你实在懒得折腾，其实还有一种简单的方式：保存一篇其他公众号排版精美的文章，保存下来，每次排版之前先将其复制到后台，再把自己写的内容粘进去，用格式刷刷成一样的就好，简单快捷。

最后的话
----

这个流程是我完成这 10 多万字，近 30 篇专栏文章的实践总结，但是并不意味这就是唯一正确的流程，像是大名鼎鼎的《华尔街日报教你如何讲故事》就建议首先扎实充分地占有资料，而不是积累卡片。所以写作流程无定式，还是要结合自身情况。

销售行业有句老话：站在屋子里打不着鹿，你得拿起装备走出门去。胡适先生说：

> 这个世界聪明人太多，肯下笨功夫的人太少，所以成功者只是少数人。

再好的方法也不如拿起笔来写起来，从这个礼拜开始，一边码字一边优化自己的信息输出流程吧。
