把 Chrome 当作工作专用浏览器 | 工作日志
=========================

iOS 和 macOS 曾是应用设计的乐园，同样的功能和需求，会有一大票设计各异的应用争先来实现。比如曾经喷涌出现的 [Twitter 客户端](https://daringfireball.net/2009/04/twitter_clients_playground) 和 [天气应用](https://artandlogic.com/2013/07/the-new-ui-design-playground/)，它们都追求细腻的界面、交互方式、和动画效果，以达到体验上的差异化。

我一度以为，进入了苹果生态圈的我，会越来越少地使用网页和网页 App，转而依赖更流畅、更美观的原生应用（Native App）。是否基于 iOS 和 macOS 原生框架进行开发，也曾是我判断应用的标准之一。

但是，这股原生应用的风潮慢慢下去了，尤其是在 macOS 上。很多应用基于跨平台的需求，以及开发成本的原因，越来越不愿意在 macOS 上开发原生应用。就连苹果，也将推出代号为「[Marzipan](https://www.imore.com/marzipan)」的跨平台 App 项目来吸引开发者以比较低的成本将 App 迁移至 macOS。

如今不少我们常用的 App，都是基于网页技术进行开发的，比如 Todoist、Slack、Spotify、Notion 等，它们都不是纯正的 macOS 原生应用。但即便如此，我列举的这几个应用还算体验比较好的，完全在接受范围之内。而有些应用，要么是桌面客户端体验不如网页版，甚至功能不全，更新也慢，比如石墨、Teambition；要么是完全没有桌面客户端，只有网页版，比如 Google Sheets 和 Google Docs。

恰好这些工具是我在工作中常常会用到的，所以用网页访问在线工具，又返回到了我的视野之内。此外，我的本职工作是网站编辑，我们的内容编排、分发等操作，都是在网页上进行的。

所以我从曾经不喜欢网页应用的一个人，变成了开始接受一些基于网页开发的应用，再慢慢变成了天天和网页打交道的一个人。

天天和网页打交道带来的问题
-------------

浏览器是我们访问网页的窗口，在电影《无敌破坏王 2：大闹互联网》里，浏览器甚至被形象地比喻为交通工具（飞船），搭载我们在网络的世界里遨游。

但正如交通工具会有公共交通的和私人使用之分，我自己在使用浏览器时，也发现有工作和生活上的区别。

工作就是我前面提到的，主要用于网页工具和少数派网站，需要和网页进行频繁的交互。而生活，主要就是消费内容，比如查资料、读文章、看视频等，大多不需要和网页进行频繁的交互。

两者一碰撞，就带来了几个问题：

### 问题一：多账号需要频繁登录登出

我在生活上是一套账号，比如我有自己的 [少数派账号](https://sspai.com/user/136/updates)、[微博](https://weibo.com/wendaohansan)、和 [Twitter](https://twitter.com/eamesliu)。

在工作上，我又有另外一套账号，比如我有时候会用 [付费内容组的少数派账号](https://sspai.com/user/860797/updates) 发文章，偶尔也会用 [少数派的官方微博](https://weibo.com/sspaime) 和 [少数派官方 Twitter](https://twitter.com/sspai_com) 发布付费栏目的消息。

所以常常需要在两套账号里来回输密码、登录、登出。

### 问题二：浏览器书签越来越多

工作内容一旦挤到浏览器里，你就会发现自己的浏览器书签越来越多。比如在我的 Safari 浏览器里，为了快速访问少数派的各个页面，因此我把首页、Matrix、专题、付费栏目、Power+ 等页面，都单独保存为书签。

除了直接访问的页面，还有一些用于「操作」的页面，比如点一下就在后台新建一篇文章，或者新建一条短链。

此外，还有一些工作相关的外部网站，比如七牛云、石墨、金数据等。

于是，我的浏览器书签里就挤满了工作书签，再和生活书签混一起，非常杂乱。

![](.evernote-content/2C76D648-D50F-4D8A-9B3F-4F4475A2FD13/3575AF4D-4C42-4C0E-BD43-306015DF369E.jpg)

越来越多的工作书签

书签虽然可以用文件夹来分类，但如果都收进文件夹里，反而会让书签操作起来变得不方便。因为不少书签都是需要频繁访问的，放到文件夹就意味着每次都需要多点一下。

此外由于我的电脑型号较老，是 13 年秋季款的 MacBook Pro，如果常开太多网页，性能会吃不消。

### 问题三：浏览器的使用体验不统一

我日常主力使用的浏览器是 Safari。因为 iOS 上的默认浏览器没得选，只能是 Safari。所以为了和 iOS 保持一致的体验，在 Mac 上也使用 Safari 是最好的选择。相对来说，Safari 在性能优化、省电等角度，都要比 Chrome 做得好。此外阅读模式功能、手势操作、iCloud 标签页同步、原生的设计风格，这些都是我选择 Safari 的原因。

但是偏偏，我前面提到的部分网页工具，在 Safari 上体验没那么好。比如石墨和 Google Sheets，在 Safari 上访问时页面总是会有些卡顿，而在 Chrome 上访问则相对流畅。

而更巧的是，少数派网站后台的编辑器，也是在 Chrome 上体验更好。比如支持使用快捷键、图片调节宽度时不会卡顿等。

这三个问题，都算不上特别大的麻烦。但是因为它们发生得很**频繁**，每一个问题都会给我造成一点点的不方便。这些一点点的不方便每天累积起来，就非常不舒服。

我的解决方案：两个浏览器
------------

面对这几个问题，我的解决方案是使用两个浏览器，也就是**日常使用一个浏览器，工作时使用另外一个浏览器**。

目前我的选择是日常使用 Safari，所有跟工作相关的内容，都迁移到 Chrome。

![](.evernote-content/2C76D648-D50F-4D8A-9B3F-4F4475A2FD13/E339CFB5-0BF8-4198-8E1A-CC8FE0D06D28.jpg)

Chrome 里都是工作相关的书签

它可以很好地解决前面提到的三个问题：

1. **多账号不用频繁登录登出：**Safari 里都登录我的个人账号，Chrome 则登录我的工作账号；
2. **可以把更多书签放到一级目录：**对于那些频繁访问，或者是能执行功能的书签，可以更多地放到一级目录出来，便于快速访问；
3. **让合适的浏览器干合适的事情：**Safari 日常消费内容的体验更好，那么就用它来消费内容。Chrome 适合网页应用，那么就用它来处理工作相关的内容。

那么，你可能会问，两个浏览器切换着用会不会不方便？比如别人发给我一条少数派的链接，我想用 Chrome 打开，那么是不是多几步复制粘贴的操作？

不需要。接下来，我将提供一些方法和技巧，帮助你在两个浏览器之间快速切换。

快速切换浏览器的方法和技巧
-------------

**首先是解决前面提到问题，如果别人发给你一条工作相关的链接，如何用工作浏览器（Chrome）直接打开？**

这里我想推荐曾经在 Power+ 1.0 《[App 奏折特别期](https://sspai.com/article/45326?series_id=9)》推荐过的 [BrowserOpener](https://itunes.apple.com/cn/app/browseropener/id1387480496?mt=12)，它可以让你自定义规则，并用特定的浏览器打开链接。比如我前面提到的少数派网页、石墨、Google Sheets，都是需要用 Chrome 打开的网页，那么就可以把它们的域名添加到 BrowserOpener 里。

![](.evernote-content/2C76D648-D50F-4D8A-9B3F-4F4475A2FD13/01710ADA-4B30-4BA2-A243-81AB11986EF1.jpg)

BrowserOpener 里的规则

当我点开这些链接时，就会自动使用 Chrome 打开。而规则之外的链接，就会采用默认浏览器 Safari 打开。

**那如果我已经用 Safari 打开了一个工作链接，这时候想把它发送到 Chrome 里呢，应该怎么做？**

可以借助自动化工具 [Keyboard Maestro](https://sspai.com/post/36442)，将当前的 Safari 网页发送到 Chrome 打开，省去了复制粘贴等繁琐步骤。当你没有打开 Safari 时，这组快捷键也可以用来快速启动 Chrome。

⚙️ **Keyboard Maestro 动作下载：** [用 Chrome 打开当前网页](https://cdn.sspai.com/Open%20in%20Google%20Chrome.kmmacros.zip)

![](.evernote-content/2C76D648-D50F-4D8A-9B3F-4F4475A2FD13/28242AA1-E1A2-4929-96F0-DCD7F902975A.jpg)

用 Chrome 打开当前网页的 Keyboard Maestro 动作

**另一种情况，如果是选中一段无法点击的纯文本链接，那么如何用特定浏览器打开呢？**

有两个方法，一个是借助光标拓展小工具 [PopClip](https://sspai.com/post/25483)。它可以让你选中一段文本后，对文本执行各种自动化操作。比如用 Google 搜索文本，统计选中的文本字符，或者添加到 OmniFocus 等任务管理工具中。

![](.evernote-content/2C76D648-D50F-4D8A-9B3F-4F4475A2FD13/083E8DCA-0835-494A-AC99-5E2FCF3C4B5B.jpg)

PopClip

这里我们可以借助 PopClip 的浏览器插件，将文本链接发送到特定浏览器打开：

* [用 Safari 打开链接](https://pilotmoon.com/popclip/extensions/ext/OpenInSafari.popclipextz)
* [用 Chrome 打开链接](https://pilotmoon.com/popclip/extensions/ext/OpenInChrome.popclipextz)

第二个方法是使用启动器应用 [LaunchBar](https://sspai.com/post/38526) 的 Instant Send 功能，它同样可以对选中的文本执行特定操作。我们只需要给 Instant Send 设定一组快捷键（比如我设定的是按两下 `⇧Shift`），然后选中文本按下快捷键，接着按一下 `Tab` 键，再输入 `Chrome`，最后按下回车，就能将链接发送至 Chrome 打开。

用 LaunchBar 的 Instant Send 把链接发送到 Chrome

结语
--

用两个浏览器分别处理日常需求和工作需求，确实是一个听起来蛮奇怪的用法。

我第一次听说这种用法，是在一个叫 [Sayz Lim](https://sayzlim.net/browser-workspace/) 的博客上。他对于这种用法的出发点是在于「专注」，他觉得浏览器是一个触发器，当你打开工作浏览器的那一霎那，仿佛就获得了一种仪式感，能让自己立刻进入工作状态。

但我觉得，作者所强调的「专注」，对我来说并没有任何作用。不过他这种奇特用法倒是给我留下了很深刻的印象，也给我埋下了一颗种子，原来浏览器还可以分场景使用。渐渐地，当我遇到了类似的需求，这个奇特的想法也在我这儿生根发芽，变成了我每天的习惯。
