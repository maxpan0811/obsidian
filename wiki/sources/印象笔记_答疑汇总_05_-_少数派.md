答疑汇总 05
=======

历经两个月的时间，教程终于全部更新完毕。在这次答疑之前，先感谢各位在教程更新过程中的耐心，以及一路的陪伴！

教程的第八章，可谓是教程中最「极客」的一章，特别是对一些之前没有接触过命令行工具和代码的朋友来说。纵观整个教程，后四章的内容都不是那种一学就会的「技巧分享」，而是真正学习成本比较高的「CSS 和 USS 入门教程」。可以想象，这对一部分读者来说有一定的阅读难度。

故而读者 @Alanku 提出了自己的疑问：

> 请恕我完全不懂 CSS，看了及查了也不太明白，它是进阶的网页语法如 HTML 进化版？还是一种文档格式输出如 PDF？还是一种类似 Markdown 的文章排版格式？我不是写网页的人，只是常用 PPT 讲课或用微信简单发一些说明文章给朋友或客户，这用得着吗？
>
> 前文中提及 Ulysses 的杀手功能是自定义样式，我不太熟悉，所以体会未深，目前用 Ulysses 的内置主题切换已非常满意。

我想 @Alanku 反映了相当一部分读者的感受，对于 CSS 和 USS 这类学习成本比较高，但暂时对自己的学习工作没有直接帮助的内容，可能目前还提不起兴趣去花时间研究它。

首先需要明确，CSS 是辅助 HTML 文档进行样式设定的文件，它自身与文章内容无关，只与排版有关 1。通常在写作工具中，这类文件都是被图形化的界面所替代，也就是 Ulysses 的内置主题，或者 Word 中的主题。

如果像 @Alanku 的情况一样，内置的主题已经能完全地满足工作需要，那自定义样式确实暂时不用着急上手。但我为什么要说「Ulysses 的杀手功能是自定义样式」这样的话呢？

一旦某一天你需要自己去修改文章的样式时，就会发现 Ulysses 用起来**省力又省时**。

省力是因为 Ulysses 对 CSS 和 USS 的支持非常完备，使得它有潜力去实现自己的需求。就以增加图片说明来说，本来应用本身是不提供这个功能，但是一旦要求文档中必须有图片说明时，我们还是能自己通过修改现有的样式文件，快速地实现这个效果。如果不会自定义样式，就只能期待开发者早日加上这个功能。

或许你会说，「如果 Ulysses 不支持图片说明，那我就再找个应用来完成就行了」。但且不说另一个应用可能会有额外的花销，就是「找应用」、「了解应用」、「将文档导出再导入」这几步，就足够耗神。与之相比，花十几分钟稍微研究下 Ulysses 的自定义样式，就能帮助你完成工作中的需求。

但话说回来，CSS 和 USS 的学习曲线陡度是显而易见的。如同各位一样，我也不是像在学校上课一样，对着官方文档从头到尾一字一句的阅读。而是由「**需求驱动**」，当我需要实现某个样式效果时，再去文档中找到相关的内容进行学习。如果一时啃不动后面四章的内容，我也建议各位先不妨放一放，等到有真正需求时，再来仔细阅读相关内容。

不过即使是有 USS 和 CSS 的辅助，文章排版，或者说样式设计始终是一件复杂地任务，这也就是为什么在学术写作中，我们还是需要借助 Pandoc 等工具来辅助。相比一站式的 LaTeX ，这种使用流程似乎有些繁琐。读者 @Ahdw 就提出了自己的看法：

> 这么麻烦，我为什么不用免费的编辑器和 LaTex 编译环境呢？引用、排版和公式都解决了。

先为各位不了解 LaTeX 的读者简单介绍下：LaTeX 和 Markdown 类似，也是一门纯文本的标记语言。它的使用方式和 Markdown 也类似，在写作内容时可以使用一些标记和命令来完成排版。但它的排版能力非常强大，可以完成高度专业的排版需求，因此成为了学术界普遍使用的排版工具。

那么是不是如 @Ahdw 所说，一个免费的编辑器配合 LaTeX 会体验更好呢？我的本科毕业论文就是使用 VS Code 作为编辑器，通过 LaTeX 完成排版的，可以说是有相当深度的体验。对比 Ulysses，我个人仍然倾向于：**Ulysses 的写作体验更佳**。

虽然 LaTeX 的功能极其强大，但代价是它的标记和命令的复杂程度远超 Markdown。对于初学者来说，光是熟悉基本的标记和简单的排版命令就需要相当一段长的时间。如果需要用到某个宏包，或者自己调试编译中出现的问题，则需要对 LaTeX 有相当程度的了解。

即使你经过一段时间的摸索，对 LaTeX 已经熟悉起来，再加上有自动补全等功能，在写作的过程中写错标记的可能性仍然不小。可能只是少了一个花括号或者斜线，在编译时才会发现，又得在文章中进行修改。这些错误肯定会随着你对 LaTeX 的熟悉越来越少，但是偶有发生时仍是非常打断写作的体验。

而对于自己完全熟悉 LaTeX 的用户，我相信前两条对他们来说不是苦恼。不过 Ulysses 不只是一个编辑器，你可以用不同的文件夹管理你的素材，用关键词整理好不同的内容，设定写作目标安排进度。所有与「写作」相关的事情，都可以在 Ulysses 中完成。而 LaTeX 只是一个纯粹的写作排版语言，在此之外的事项，都需要我们再找别的工具完成。

当然 LaTeX 也有其不可取代的应用场景。如果你希望在一个工具中完成所有的写作和排版任务，而且**更习惯使用代码**，而非 Word 等富文本工具，LaTeX 绝对是一个值得花时间好好学习的工具。
