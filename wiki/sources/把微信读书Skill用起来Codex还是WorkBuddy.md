一周多前了解到微信读书开放了Skill，觉得新鲜，很快在自己目前常用的 Codex 和 WorkBuddy 上安装了。

就最近的阅读习惯来看，我其实不算微信读书的重度用户，Kindle 输入还是会多一些。因此，如何同时管理好微信读书和其他平台的数据似乎是个值得谈谈的话题。

经过一番探索，我大概有了一个简易的方案，这篇文章就会分享给大家。

# 安装微信读书Skill

首先我在两个平台分别都安装了微信读书 Skill，这个获取方法：

打开微信读书App - 我 - 右上角设置按钮 - 微信读书Skill - 复制安装指令

![](attachments/d7c103ff38f9c0b9.jpg)

这里有个重点，只是有 Skill 还不够，你还得有微信读书的 API，所以你在把安装指令给到 Agent 后，还需要把这个 API 也复制给他们，这样才能真正解锁微信读书 Skill，为你服务。

![](attachments/afd6dbec1abba33a.jpg)![](attachments/27a53879392aa527.jpg)

# 微信读书Skill 能做什么？

# 在这块上，WorkBuddy 说明得很清楚，你可以通过它检索书籍、了解自己的书架状态、看书的进度、阅读记录甚至推荐书什么的都不在话下。

![](attachments/191af9a0f4ed1ce9.jpg)

我对比了 Codex 和 WorkBuddy 的解读，我发现 Codex 更能分析出我阅读读物之间的联系，也能解读出更多东西，而 WorkBuddy 则在数据展示这块清晰很多。

![](attachments/9f4cac3f9d51393b.jpg)

个人觉得，在微信读书Skill 的整体体验中，WorkBuddy 甚至会更好用，毕竟是都是腾讯家的产品，这种内在的强联系也能理解。

![](attachments/8f2e864de7cb18b4.jpg)

那对于跨平台阅读者，我们还能做什么？

# 不止微信读书skill

# 如果你像我一样，多平台阅读，那么怎么去统一管理这些数据呢？

1.其他平台的标注笔记信息导出

拿 Kindle 来说，你只需要在根目录导出 Clippings。

如果你过去和我一样喜欢用第三方工具比如 Knotes、klib 之类工具去管理内容的话，你也可以从这些第三方平台把这些内容导出到云笔记工具然后再传输到本地

2.将阅读记录喂给 Agent

注意这个 Agent 一定是已经安装上了微信读书Skill 的那个哦，你还可以给他提你的需求，比如去重、整合等等，我相信有些人一不小心会同时在多平台读一本书。

![](attachments/fbc4965983b0dcf3.jpg)

3.输出一个阅读报告

这样的话，你就可以对自己数字阅读记录有一个相对全面对感知。至于纸质书的记录，大家也可以根据需求自行补录。

以 WorkBuddy 为例，经过这三步骤后，很快你就会得到一些数据感很强的分析。
