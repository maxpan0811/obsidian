在线阅读系列（二）：用快捷指令和 Evernote 做更漂亮的网页摘抄
===================================

| 本文为付费栏目文章，您已订阅，可阅读全文 |

摘抄是每个人都有的需求，差别只在于工具是否一样、使用勤不勤快。

在快捷指令教程的[第 16 期](https://sspai.com/article/51945?series_id=68)中 Hum 就介绍了一个网页摘抄的快捷指令，可以把所选内容带上样式摘抄到 Evernote 中 1 。末了文章还留个一个题目，没成想这个「避免重复出现标题」的小要求难倒了不少教程读者。

![](.evernote-content/B940B7E6-4DE0-470C-A3B3-B0E9E75BE932/E3CADE4E-4689-434A-AF8E-A70659B78105.png)

快捷指令教程的练习题

快捷指令本身缺少相关功能的模块，Hum 自己在研究过程中又不巧碰上账号出错（可能是 Evernote 团队拆分的缘故），这事儿一耽搁就是半年。最近 Hum 和我分享书摘工作流时旧事重提，我也发现自己需要一个更完善的摘抄快捷指令，于是有了这篇「交作业」的文章。

![](.evernote-content/B940B7E6-4DE0-470C-A3B3-B0E9E75BE932/A6BC7CA2-816B-4992-8B93-2EBF44E45903.png)

网页摘抄效果

动作使用
----

这一动作针对网页的摘录而设计，需要从 Safari 浏览器内容中抽取标题和链接。如果需要摘录不带链接的书本、电影台词等文本，可以参考我以前写过的《[用 Drafts 和 Evernote 做更漂亮的摘抄](https://sspai.com/post/55047)》。网页摘抄的操作步骤是：

1. 在页面上选中文本，然后在整个网页的分享菜单中运行本快捷指令动作（需要接收整个 Safari 页面，不然获取不到标题和链接）
2. 动作将自动运行，根据情况进行不同形式的摘抄：
   1. 如果当前文章以前没有读过，直接在原笔记最下方追加新摘抄。格式是「带超链接的标题 + 摘抄内容 + 日期」
   2. 如果文章以前读过，新的摘抄会插到之前所摘录内容下方，格式为「摘抄内容 + 日期」，避免重复出现标题。

![](.evernote-content/B940B7E6-4DE0-470C-A3B3-B0E9E75BE932/3A4F7D50-4B23-4B62-92E4-2C21D57C427B.gif)

用捷径做摘抄到 Evernote

上图所示就是直接追加新摘抄的过程。插入摘抄的过程中需要多点几次，因为 Evernote 模块不支持直接插内容，快捷指令需要删掉原先的整条笔记、再加一条插好摘抄的新笔记进去，中途需要一次删除操作确认。

![](.evernote-content/B940B7E6-4DE0-470C-A3B3-B0E9E75BE932/301CB231-D604-44D5-AD65-E41FB44479EE.jpg)

插入内容时需要你确认删除原笔记

追加好的摘录统一为标题、内容和日期的组合样式：

* **标题**：居左、字体蓝色加粗并附带超链接。
* **摘录**：居左、字体灰色，带引用样式。
* **日期**：靠右、字体黑色。

这一格式直接从 Hum 的动作里继承过来，保证了网页摘录的**可回溯性**，点击标题就能找回原先的页面（只要它还没失效），也能在记忆模糊的时候通过日期来辅助查找。另外，按这个格式做了一定数量的摘抄之后，整条笔记看上去就是一块一块的，每一块都是一条或几条带**来源**的摘录，视觉上的区分效果也很到位。

此外有个细节需要注意，由于快捷指令不能很好识别非系统语言的变量名，需要把开头**「输入快捷指令的信息」**变量名字改成和系统语言一致（其他自定义变量倒不受影响），比如中文系统就使用 `页面 URL`，英文系统中则是 `Page URL`。如果你登同一个账号的几台设备语言不一致，那只能为每种语言的设备各准备一个动作。

![](.evernote-content/B940B7E6-4DE0-470C-A3B3-B0E9E75BE932/4804DD58-F7AD-4142-9ED6-EAEE486CE744.png)

不同系统下需要修改变量名称

这一快捷指令的使用就这么简单，但是它的实现过程复杂到超出我和 Hum 的预期 —— 因为快捷指令毕竟不是一个正经的编程工具，实现时用到了非常多曲线救国的技巧，有折腾意愿的读者应该不会错过下面的原理部分。

原理解析
----

根据以前是否摘录过当前网页，这个快捷指令会分两种情况处理。在两种摘录方式的分流之前，我们先从简单的追加操作切入，掌握带样式文本的实现，之后再转到较为复杂的插入场景。

### 基础的摘录追加

基础部分最后添加的摘抄效果和 Hum 的原快捷指令一致，但是为了后续插入摘录内容做准备，我的步骤会有所不同 2 。

![](.evernote-content/B940B7E6-4DE0-470C-A3B3-B0E9E75BE932/96A2E595-7624-4F51-B448-D736992BD0CB.png)

追加摘抄 示意图

先来制作带样式的摘录笔记。在动作使用部分我们了解过摘录的样式要求，这一主题已经用 HTML 写好了模版（如下代码所示），`<blockquote>` 标签和 `<p>` 标签里的内容各自对应了摘录内容和日期的格式（为方便插入，标题另行设置），接下来要做的只是用快捷指令获取具体内容然后添加进去。

```
<blockquote style="margin: 1em; padding: 0px 1em; border-left-style: solid; border-left-color: rgb(255, 126, 121);">
<div style="margin-top: 1em;margin-bottom: 1em;-en-paragraph:true;"><span style="color: rgb(136, 136, 136);">快捷指令输入</span></div>
</blockquote>
<p style="margin-top: 1em;margin-bottom: 1em;-en-paragraph:true;" align="right">当前日期</p>
```

![](.evernote-content/B940B7E6-4DE0-470C-A3B3-B0E9E75BE932/B13FFBF7-0035-448B-BBFA-3B54D80CBAFF.jpg)

新笔记内容

这个快捷指令默认的日期格式是数字加短横的简单组合，可以点击「当前日期」变量来另行设置。此外，想要换用其他样式的读者也可以自己找找 HTML 模版改写进去。

标题部分的数据来源也是 Safari 页面，快捷指令从中自动获取了标题、链接并组合起来 —— 这甚至比直接使用 Evernote 还要方便，一切都是自动完成的。后面有一些对所生成标题源码的零星修剪步骤（包括文本替换），之后插入摘录时这些修整步骤才会派上用场。

![](.evernote-content/B940B7E6-4DE0-470C-A3B3-B0E9E75BE932/D6200877-67DA-45DE-88A2-9005FF99D7A5.png)

基础的追加步骤

最后把 HTML 再转回富文本（多信息文本，即我们最终看到的效果），添加到 Evernote 笔记末尾即完成了基础的摘抄动作。

### 避免重复出现文章标题

实现了简单的追加后，我们来解决摘抄标题的重复问题。最终理想的效果是在检测到读过某篇文章时就到原摘抄下面插入新摘抄内容，毕竟我们总希望**同一篇文章的摘抄能够保存在一起**，而不是分散在笔记的各个角落。

![](.evernote-content/B940B7E6-4DE0-470C-A3B3-B0E9E75BE932/B3E0FB0B-F1EA-48B5-BAF9-DFCF37150135.png)

新内容会插在原笔记下面

优化的瓶颈在于「插入到原摘录下方」这一项 —— 如何精确定位到出现重复的那一条摘录？我的思路是**分而治之**，把原笔记拆成小块后再处理。不过在此之前，我们要在整个快捷指令前段步骤里加个**判断步骤**，区分是 / 否有重复标题两种情况。

这道判断并不难，获取原笔记后检查一下是否包含标题就行。确认有重复的话，就要获取笔记的源码并开始单独的处理工序。

![](.evernote-content/B940B7E6-4DE0-470C-A3B3-B0E9E75BE932/0C72B1C3-09E1-4662-AFA5-D62D08258301.png)

判断以前有没有这篇文章

这里的核心思路是**拆开再处理**，先找到含重复标题的那一项再把文本插到后面去。听上去似乎难度不大，但是快捷指令的相关模块都有点缺胳膊少腿，对于正则表达式的支持不是那么到位，只能用上一些旁敲侧击的技巧。

![](.evernote-content/B940B7E6-4DE0-470C-A3B3-B0E9E75BE932/9D5A1DA3-B40B-4029-88F6-9C23A439C66E.png)

拆分笔记 示意图

1. **拆开源码**：按照 `</p>` 标签把源码拆成一块一块，每块都是一篇文章及其下属所有摘抄。上一节中我们特意修整了摘抄的源码，那一步非常关键，可以在不改变摘录最终外观的情况下设置一个隐形的「锚点」，为拆分工作做好准备。
2. **找到重复**：在拆开后的摘录片段中找到含有当前页面标题的那一项。这一步和最初的判断一样，检查是否包含标题即可。
3. **补上标签**：第一步中为了拆分文本，我们把 `</p>` 暂时消耗掉过，找出标题重复的那一段源码后就可以把标签补回去（HTML 标签一般要成对出现，不然没法正常显示）。  

   ![](.evernote-content/B940B7E6-4DE0-470C-A3B3-B0E9E75BE932/9292B0F6-0BF1-4AF9-893E-3DCD2D6766A4.png)

   拆分 HTML 源码的步骤

**知识点：如何在快捷指令中使用 / 禁用正则表达式**

快捷指令有很多模块与正则表达式有一定关系，其实设计得比较中规中矩，和常见编程语言也能够对应起来，但是到底什么时候能用正则表达式、 什么时候不能、怎样绕过限制…… 估计都很让人头痛。下面是我整理的一些 tips，希望帮到需要正则表达式的读者：

* **替换模块**，可选是否使用正则，启用时替换前后都可以使用正则。
* **匹配模块**，仅支持正则表达式，普通文本需要转译以避免歧义。
* **判断模块**，不支持正则表达式。不过可以先使用一次**匹配**模块、再接上**计数**和**判断**，组合在一起形成正则判断（例子可参考《[在 iOS 上找回桌面级重命名体验](https://sspai.com/article/52371?series_id=70)》）

找到重复项后就可以开始组合新笔记。到这里问题已经不大，剩下的只是一些组合替换操作。这个快捷指令动作前前后后一直在避免使用正则表达式，因为很多文章标题中会含有正则意义的特殊符号（比如标题刚和栏目的分隔符 `|` 就会被快捷指令的匹配模块误解为「或者」的意思），比较幸运，成品中也没有用到正则。

![](.evernote-content/B940B7E6-4DE0-470C-A3B3-B0E9E75BE932/3A1A0995-60F2-4EA7-AF9D-A8A22097F44C.png)

插入摘抄 示意图

有了之前的素材准备后，可以把引用和新日期追加到含重复标题的那一块笔记下面，然后替换掉原先重复项；末了，把替换好的源码转回富文本放回 Evernote，插入操作终于告一段落。

![](.evernote-content/B940B7E6-4DE0-470C-A3B3-B0E9E75BE932/A2E1DC39-551C-4DD4-87CA-4825BDE5553B.png)

小结
--

继之前聚焦手动摘录场景的《[用 Drafts 和 Evernote 做更漂亮的摘抄](https://sspai.com/post/55047)》之后，这次交上来的「作业」完善了网页摘录这一环，整个 Evernote 的摘抄工作流变得更加完整，一些典型场景下的手动录入和自动抓取都得以完成。

比起动作本身，实话实说，还是实现过程带来的成就感更大。无论摘抄样式还是添加方式，单看起来似乎都不难，但既要照顾 Evernote 的笔记格式、又要考虑快捷指令的功能限制，两边一交叉后能用的方法就寥寥无几。不过使用快捷指令的有趣之处也正是在此，用有限模块搭建出新功能的乐趣也只有自己动手过的人才能体会。
