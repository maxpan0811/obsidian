---

Original特大妹 特大号

最近和一位做集成的朋友聊天

他说刚帮一家企业做了件事

把一个SOP，做成了Skill

SOP：企业标准操作流程

Skill：AI可以调用的能力模块

这个SKill给企业自建的Agent平台用

他兴奋地说

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/10B06C87-FE4E-4A74-9773-94EE2BDB3626.jpg)

我听了有点诧异

最近龙虾火，把Skills也带火了

但还真是第一次听到

身边的人开始拿它赚钱了

为了掩饰心中的震惊，我故作随意问一句

↓

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/A9A038B8-6AB1-4A22-B601-8655C47E21DD.png)

他立马激动起来

化身AI老司机，开始给我一统科普

↓

我先问你个文题啊

一个企业最值钱的东西是啥？

那肯定是多年来积累下来的

*业务流程、操作经验、数据处理规则、分析逻辑、制度和方法论*

但是，它们往往是隐形的

存在于各个软件、系统，甚至是老员工的脑袋里

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/2E30B743-8B87-422E-9881-A2A62BE9DA44.jpg)

这些「最值钱」的东西，很难复用

*比如：某个老员工离职后，原来很多熟练的流程、分析方法，新人就不得不重新摸索一遍*

而Skill的价值就在于

把企业经验软件化

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/EA3855D3-502A-4C26-BEC2-25D571B6AF6C.png)

这些东西变成Skill后

所有人都能用，AI可以调，经验不会丢失

所以做【企业级Skill】才是王道

才能卖上价钱

↓

他最后还补了一句

AI时代缺的不是工具，而是想法

就是对客户真实需求的深刻理解

然后提出解决问题的想法

现在工具平权了

只要有想法，落地都不是问题

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/30C20B3A-3DF0-405B-A5F4-074DA32F4F72.png)

聊到这里，我追问了一句

↓

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/547F196C-B7A7-4EC2-B3BC-7461224357A7.jpg)

他说，这是一家大型能源企业

今年计划在内部构建

一个Agent平台+N个Skill体系

正在尝试着把

企业能力Skill化，供各种Agent调用

软件功能 → Skill

业务流程 → Skill

员工经验 → Skill

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/11B76E29-0426-4386-8FA3-196D0C0FB8FA.png)

客户比较认同一个趋势

未来的企业级软件形态

不再是由N个独立软件系统组成

而是一整套由Skills组成的能力网络

*CRM Skill、ERP Skill、Finance Skill、HR Skill、BI Skill*

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/DEA6D25E-54D5-45DB-8648-D3ACCE83A7FD.png)

Agent只需要组合这些Skills

就能完成复杂工作

所以，他们打算从现在开始，零开始

一步步构建 【内部Skill 体系】

*Skill Marketplace：技能市场，用来分发和交易SkillsSkill Library：技能库，用来沉淀和管理企业能力Skill SDK：技能开发工具包，用来开发和扩展Skills*

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/7276B367-9FBB-49E3-AEC0-C23D06DD5BB0.png)![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/278AACE4-5211-48C3-A05A-485FA91A0E9D.png)

聊完后

我觉得【B端Skills】是个好话题

①大趋势，企业软件演变方向

②离钱近，企业级客户买单意愿强

但是，现在市面上的新词太多

很多用户搞不明白之间的关系

下面我来简单科普下几个重要概念

↓

01 /什么是 SOP ?

SOP（Standard Operating Procedure）

就是企业里的标准操作流程

举例：一个投诉SOP

登录系统→搜索数据→筛选字段导→导出Excel→人工分析→生成报告

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/58574DE8-0A4B-4B0D-9F64-FFE92303A23F.png)

02 /什么是 Skill ?

Skill可以理解为

一个AI可以调用的能力模块

例如：查询数据库、分析 Excel、获取客户信息、生成报告、发送邮件...

这些能力都可以被封装成一个Skill

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/297F3D33-50F1-4997-AADC-33DE9F43BD93.png)

当AI需要完成任务时

就会调用对应的Skill

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/BB2256A5-C290-4DEC-BBE6-A9EA1C3C3DBF.png)

比如

用户说：帮我查一下这个客户的订单情况

AI可能执行这样的流程

①调用CRM Skill＞②查询订单数据＞③ 整理信息＞④返回结果

所以从结构上看，Skill就像AI的技能包

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/CF546C09-D124-4A20-8322-91A11AF50B49.png)

03 /Skill、大模型、Agent三者关系

看一张图，你就理解了

↓

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/44617298-442E-4A99-9D9B-346917BEE2DE.gif)

龙虾其实就是Agent

装龙虾，你得先装个脑子

脑子就是LLM大模型大模型

然后，你还得给龙虾装各种Skills

这就是给龙虾装技能，让它能干各种活

↓

我们平时用的DeepSeek、GPT们

是LLM大模型，不是Agent

LLM大模型本身有明显的能力边界

它擅长两件事：①理解语言 ②进行推理

但它并不会真正执行任务

比如：查数据库、登录企业系统、调用 API..

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/3AA29792-F52E-435D-9C16-25C949FABC1F.png)

Skill 之所以重要

是因为它给大模型提供执行能力，好比手脚

因此在实际系统中，常见结构是

***AI Agent = LLM + Skill***

Agent接到任务后，一般会经历三个步骤

①理解任务②制定执行计划③调用Skills工具

举个简单例子

用户说：“帮我做一份本月销售分析。”

Agent 的执行流程可能是

⏷

调用 CRM Skill 获取销售数据

调用 Data Skill 做统计分析

调用 Chart Skill 生成图表

调用 Report Skill 输出报告

整个过程自动完成

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/C40115E6-8952-47D8-B823-555A1614B317.png)

04 /Skill和MCP啥关系？

最近另一个很热的词是MCP

MCP是一种协议

好比AI世界里的USB

让AI可以用统一方式访问外部工具和系统

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/2A538AA2-F29D-4C59-B73A-5A6D8F5A2DFB.png)

Agent、Skill 和 MCP 啥关系？

看个简单结构，就一目了然了

↓

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/2A0E7F54-AC73-43A2-9C51-D27A33E8A9A2.png)

例如用户问

客户张三的订单什么时候发货？

执行流程可能是

Agent 识别任务

↓

调用 order\_query\_skill

↓

Skill 通过 MCP 调 CRM

↓

返回订单状态

![](.evernote-content/7AE85F1B-6B5D-4F8B-AD40-FA7217C3BF79/8886EE6F-8251-47F7-9C5A-18788A7C5B87.png)
