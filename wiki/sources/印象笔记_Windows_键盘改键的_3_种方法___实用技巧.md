最近我的朋友买了一个蓝牙机械键盘，想同时在 Mac 和 Windows 电脑上切换使用。但是键盘的键帽无法选择，只有适合 Windows 的键盘布局。我们都知道，Windows 和 Mac 键盘最明显的区别之一就是左下角的 `Windows` 键 和 `Alt` 键的位置不同，习惯了 macOS 键位的朋友会在使用快捷键时比较别扭，此时就需要对按键进行互换，让它们更符合使用习惯。

![](.evernote-content/EAAD3C78-7CB7-4810-A646-81E3C9890C76/49378757-22A8-4500-9A9E-E908E07A8822.png)

macOS 键盘和 Windows 键盘布局对比

还有一些键盘会提供单独的可定制快捷键，但如果手上只有普通的键盘该怎么办呢？没关系，AutoHotkey 也能帮你将闲置的菜单键等按键变成功能丰富的 Hyper 键。所以本文就来讲一讲在 Windows 中为键盘改键的 3 种方法。

注：macOS 用户可以移步 Power+ 1.0 中的[《macOS 改键利器：Karabiner-Elements 使用详解》](https://sspai.com/post/46184) 一文，查看如何通过 Karabiner-Elements 来进行改键。

用 MapKeyboard 互换按键位置
--------------------

将两个按键交换的情况，最常见的有两种情况：

* **键盘布局**不同，比如 macOS 用户使用 Windows 时，可能会希望让 `Windows` 键和 `Alt` 键进行互换，保持外接键盘和 Mac 键盘布局保持一致。
* **个人按键习惯**不同，比如让 `ESC` 键和 `CapsLock` 键、让 `FN` 键和 `Ctrl` 键进行互换来改善手感和提高效率。

这种情况是比较容易调整的，我们只需要安装一款名为 MapKeyboard 的软件。这款工具提供了可视化的键位改变方法，不需要手动编写配置文件，非常易于使用。

![](.evernote-content/EAAD3C78-7CB7-4810-A646-81E3C9890C76/03A2E473-4C2B-49D5-9F14-74B2E2807547.png)

MapKeyboard

目前 MapKeyboard 的最后一个免费版本是 2.1 版，如今开发者已经将它作为一项功能加入了他新的收费软件 Hotkeycontrol 中。你可以[在这里下载](https://www.jianguoyun.com/p/DWGQFJoQ7qW5Bxj4keMB)到 MapKeyboard 最后的免费版。

尽管 MapKeyboard 2.1 不再单独更新，窗口布局也显得有些混乱，但它的功能丝毫没有受到影响。我们以互换 `ESC` 键和 `CapsLock` 键为例，讲讲如何操作。

**使用方法**：

1. 首先打开 MapKeyboard，在主界面中选择 `ESC`，这一步是选择需要修改的按键。
2. 在左下角的「Remap selected key to」的方框中，选择 `Caps`。
3. 在主界面中选择 `Caps`。
4. 在左下角的「Remap selected key to」的方框中，选择 `ESC`。
5. 最后 Save layout，弹出提示需要注销一次电脑，选择确认注销。

![](.evernote-content/EAAD3C78-7CB7-4810-A646-81E3C9890C76/F3B5A963-CC6D-4F35-AB98-592CA15D3820.png)

MapKeyboard 改键后

注销之后，按键映射就会被修改为新的布局。

如果觉得不习惯，希望换回来也不用担心。MapKeyboard 会记住你的修改，重新打开 MapKeyboard 时，仍然会是这个界面。你可以很容易知道自己的哪些按键被修改过了，直接使用 Reset keyboard layout 可以让整个键盘映射还原。

通过这样的方法，你可以对 `Windows`、`Alt` 键进行交换，也可以让一些键盘布局比较奇怪的笔记本键盘、外接键盘获得新生。

![](.evernote-content/EAAD3C78-7CB7-4810-A646-81E3C9890C76/4C495ACE-CC58-4961-937E-C726C05F7EE7.jpg)

比较奇怪的键盘布局

这类工具在 Windows 下非常多，还有 [KeyTweak](https://keytweak.softonic.cn/)、Keybmap 等工具均能实现类似的效果。它们的原理通常是修改系统注册表，因此改键只需要执行一次，修改之后也不会运行在后台，不会占用系统资源。由于能够自动识别已修改的按钮，所以也不会造成无法还原的情况，相比手动修改更稳妥一些。这些小工具的缺点是年久失修，很多是五六年前的产物。不过只要可以正常使用，这也不算什么大问题。

用 AutoHotkey 修改 F1~12 按键功能
--------------------------

第二种需要改键的情况是利用键盘上的空闲按钮来做更多的事情，一些按键本身的使用频率不高，可以通过一些方式将其修改为特定的功能。

![](.evernote-content/EAAD3C78-7CB7-4810-A646-81E3C9890C76/694E31EC-7E12-4F0A-9606-E1D8368ADBAB.png)

键盘按键 图 / Wikipedia

在键盘上，顶部的 `F1~F12` 功能按键使用频率比较低。它们本身会承担一些功能，但总的来说使用频率远不如 `Ctrl、Alt` 等修饰键那么高，也不如 `Page Up` 这些编辑键那么好用，还经常会被占用导致运行失败。最常见的做法也是带上 `FN`（Function）键，将它们变成音量调整、亮度调整一类的功能键。

那么如何为 `F1~F12` 添加更丰富的功能呢？我们可以先了解一下这些功能按键本身有哪些作用，再决定对哪几个按键开刀。

* `F1`：Help 键，打开当前窗口的帮助文档。
* `F2`：重命名键，重命名文件或文件夹。
* `F3`：寻找键，通常会跳转到搜索框。
* `F4`：地址栏表单，通常会跳转到地址栏，多次按下可以查看地址栏下拉框。
* `F5`：刷新，刷新当前页面。
* `F6`：菜单栏轮换，通常会跳转到菜单栏。
* `F7`：特别功能，在 Windows 中默认没有作用。
* `F8`：特别功能，在启动时可以进入安全模式。
* `F9`：特殊功能，在 Windows 中默认没有作用。
* `F10`：功能列表，配合 Shift 可以打开右键菜单，单独按下没有作用。
* `F11`：全屏，把当前窗口全屏化显示。
* `F12`：特殊功能，在 Windows 中默认没有作用，在 Word 中可以另存为。

我们可以发现，其中 `F7`、`F8`、`F9`、`F12`，这几个按键几乎是闲置的，如果你常用的软件中没有和这几个键直接相关的操作（不含修饰键的操作），那就可以对它们进行自定义。如果你按下这些功能键，但没有产生应有的效果，说明这些按键很有可能已经被其他软件占用了。另外 `F1~F6` 使用频率较高，也建议不要对 `F1~F6` 做修改。

如果你想让 F 系列的功能键变成一个键盘上已有的按键，你依然可以通过 MapKeyboard 来进行修改，比如修改为大写锁定的 `CapsLock` 键、用于截图的 `PrintScreen` 键等等。

但如果想要实现特定的功能，而不是利用它实现现有的按键，那可以使用 AutoHotkey 来重新自定义它。在 AutoHotkey 中修改键盘映射也非常简单，只需要 3 行代码就能实现。我们以 `F7` 为例 ——

**操作步骤**：

1. 新建一个 .ahk 文件，首先输入 `F7::` ，意思是自定义 `F7` 键。
2. 再输入 `return`，意思是不执行任何操作，直接结束。
3. 最后在两者之间加入你想要的功能，比如让它变成一颗 Chrome 启动键，那就输入 `Run Chrome`。

```
F7::
Run Chrome
return
```

然后保存为 .ahk 文件，双击运行。这样，`F7` 就可以成为任意一个软件的专用启动按钮，这样的方法可以用在大多数按键上。

![](.evernote-content/EAAD3C78-7CB7-4810-A646-81E3C9890C76/C4259734-724E-45E3-90A6-B14D83F26462.png)

AutoHotkey

一些电竞键盘还会提供不少供玩家自定义的按键，这些按键通常可以用厂商自带的驱动软件进行修改。如果需要用 AutoHotkey 操作，这些按键通常是 `F13~F24`，最多 12 个按键，方法与上面类似。

**小知识：为什么不能修改 Fn 键**

还有人可能会希望修改 Fn + F1~F12 的功能，但修改起来非常复杂。因为 Fn 键通常是通过硬件手段实现的 1 。当然也有一些厂商是通过软件实现的，比如 Sony 的笔记本，但主流的用法是**混合式支持，即一部分通过 BIOS 支持在任何环境下直接使用，另一部分通过在系统中安装驱动、软件来实现**。

正因如此，Fn 键通常不依赖于操作系统而是依赖更底层的 BIOS 支持，操作系统甚至无法检测你是否按下了 Fn 键，因此几乎不可能通过软件的方法进行修改。这也是为什么大部分机型的 Fn 按键都不支持在软件中重新映射 2 。

不过这也并非完全不可能，虽然有关重新映射 Fn 按键的资料相当少见，但「通过软件方法对 Fn 组合键修改键盘映射」确实可以实现，曾经有一篇文章题为《[Mystery of the Fn key](https://archive.is/20120630223625/http://www.autohotkey.net/~daonlyfreez/tutorials/3p/Veil/fnkey.htm)》的文章提到过如何映射 Fn 键，原理和解决方法都讲的非常完整，利用了 AutoHotkey 和 HID 的 DLL 文件证明这并非不可能。但是由于年代久远，文中提到的工具 AutoHotkeyRemoteControlDLL.ahk 已经很难找到，而且原文也提到这款工具在新系统下可能会出现崩溃，因此也无法验证这一方法能否成功。

为菜单键修改功能
--------

除了 `F1~F12` 的功能键，另一个可以被使用的按键是键盘上的菜单键（`Menu` 键），菜单键的作用是模拟鼠标右键。所以在默认情况下无法与其他按钮产生组合键，算是整个键盘上比较鸡肋的一个按键。

![](.evernote-content/EAAD3C78-7CB7-4810-A646-81E3C9890C76/F5A727F7-E42E-469C-9381-A7A9662738E0.jpg)

Menu 键 图 / Wikipedia

除了上面的两种用法，你还可以将一个按键作为「修饰键的组合」，菜单键就很适合做这件事。

系统快捷键之间的关系，Windows 和 macOS 之间修饰键的关系，相关的内容在今年年初的 《[快捷键终极使用指南 | 效率思维](https://sspai.com/post/52239)》 中有提到过。由于使用键盘快捷键的软件众多，随手设置快捷键往往会出现冲突。macOS 中有一种 Hyper 键，它的作用是直接代替 `⌃Control + ⌥Option + ⇧Shift + ⌘Command` 的组合，这样就不会出现快捷键冲突的问题，通过 [Karabiner-Elements](https://sspai.com/post/46184) 可以设置这样的 Hyper 键。

而 Windows 中的软件通常无法将 `Win` 键作为组合键之一，所以我们可以让某个按键实现 `Ctrl + Alt + Shift` 的组合，起到类似 Hyper 键的作用。这个按键的重任就落在了使用率不高的菜单键上。

由于作为修饰键需要留下足够的时间去按第二个键，所以不能直接将其映射为特定的按钮（严格地讲，这不算映射），和上面的单键映射相比，需要换一种方法实现。

**操作步骤**：

1. 新建一个 .ahk 文件，输入 `AppsKey::` 和 `return`，为菜单键定义动作。
2. 利用 Send 指令，模拟按下 Ctrl、Alt、Shift 的操作，就像下面这样 ——

```
AppsKey::
Send {Ctrl}
Send {Alt}
Send {Shift}
return
```

看上去完全没有问题，但我们按下菜单键时，其实只有一瞬间执行了「按下」操作，导致我们没有时间去按下其他的组合键。所以我们还需要做一些改进。

```
AppsKey::
Send {Ctrl down}
Send {Alt down}
Send {Shift down}
return

AppsKey Up::
Send {Ctrl up}
Send {Alt up}
Send {Shift up}
return
```

这才是完整的动作，意思是「当我们按下菜单键时，模拟按住三个键」，这样就能让菜单键实现「同时按住三个键」。如果只有上面这部分，按键将会一直被按住，所以还需要「当我们抬起菜单键时，模拟松开三个键」才能正常运行。

![](.evernote-content/EAAD3C78-7CB7-4810-A646-81E3C9890C76/BA3D8C7E-98D1-4249-B0FE-5FC96C41B275.png)

正确的配置

菜单键的位置通常在键盘的右下角，如果你觉得不太方便按到，你可以将开头的 `AppsKey::` 改为 `CapsLock::`，将 CapsLock 键作为 Hyper 键使用。再用 `AppsKey::CapsLock` 将菜单键映射为 CapsLock 键。这样就相当于让两个按键交换了位置，可以用左手小拇指去操作 Hyper 键。

![](.evernote-content/EAAD3C78-7CB7-4810-A646-81E3C9890C76/F44110DB-248E-44F4-B2C0-12B93F7D9C3A.png)

将两个按键互换后应该这样写

修改完之后，你就可以放心大胆地为各种软件添加 `Ctrl + Shift + Alt + J` 之类的自定义快捷键，只需要按菜单键 + 快捷键就能实现，不过在 Photoshop 这类本身就带有大量快捷键的大型软件可能依然会产生冲突（比如默认的 `Ctrl + Shift + Alt + S` 就是导出为 Web 格式）。但如果拿来作为截图等功能的修饰键，日常使用还是很够用的。

**Tips：Shift + F10 也可以起到菜单键的作用**

如果你的键盘上没有菜单键，但是你又想用键盘打开右键菜单。其实 Windows 也提供了一种方法来实现这个功能，`Shift + F10` 的组合按键和菜单键基本相同。如果组合键不起作用，还可以尝试一下 `Ctrl + Shift + F10`。

其他改键方案
------

此外，你还可以通过 AutoHotkey 自定义快捷键组合。在 AutoHotkey 中，四个修饰键分别可以用这些字符代替：

1. `Shift` 对应 `+`。
2. `Ctrl` 对应 `^`。
3. `Alt` 对应 `!`。
4. `Win` 对应 `#`。

比如当你需要对 `Ctrl + J` 进行自定义时，就可以以 `^J::` 作为开头，对它进行功能的定义。

具体的每个键叫什么名字，你可以在 [AutoHotkey「按键」部分的中文文档](http://ahkcn.sourceforge.net/docs/KeyList.htm) 中找到更多按键的符号和说明。

总结
--

本文提供了三种不同的改键方案：

* 通过现有的改键软件将键盘按键映射为其他按键，实现互换。
* 通过 AutoHotkey 对 `F1~F12` 等功能键进行自定义。
* 通过 AutoHotkey 对菜单键等闲置按键添加 Hyper 键功能。

这几种方案可以解决因键盘布局、使用习惯等情况造成的问题，也能提高特定按键的使用率。具体使用时也可以结合自己的情况做一些修改，希望本文能够为大家提供一些改键的思路。
