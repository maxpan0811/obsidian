不知从什么时候起，越来越多的在线服务不再专门开发桌面版客户端，而选择了开发网页版，然后在网页版的基础上加一个本地客户端的外壳，摇身一变就成为 macOS、Windows 客户端，这就是我们常说的「套壳应用」。

网页套壳应用带来的好处是 **兼容多个平台、使用体验一致**，而且 **更新迅速**。但同时也带来了许多缺点，一旦涉及到 **离线使用**、**本地保存**、**导入导出**这类本地相关的功能，显得很羸弱。它的优缺点都很明显，一时也难以评判这种现象是好是坏。

不过「套壳应用」并非是粗制滥造的代名词，也是有优劣之分的，一些「制作精良的套壳应用」的可以做到桌面版客户端的体验，也有一些较差的应用直接生成一个客户端，连加载页面都不愿意做，直接显示一个白屏。为了统一这些应用的体验，我选择使用网页聚合工具来整合它们。

今天要介绍的这款网页聚合工具 Station，能帮我们把工作中繁多的网页应用整合到一起，它为网页应用做出的种种优化能够极大地改善使用体验。

如何面对越来越多的网页应用？
--------------

目前 Power+ 组内使用最为频繁的在线服务有 Slack、Notion、Todoist、iCloud 日历，分别完成交流、记录、任务、日历等工作。我自己的阅读流程则是通过 Inoreader、Pocket、Evernote，社交方面则会用到微信、Telegram、Twitter…… 每天都会在这些服务之间来回转。

作为一个 Windows 用户，这些应用之中只有微信、Telegram、Evernote 有完善的本地客户端。其他应用都是网页版 + 本地套壳的组合，有些甚至干脆没有官方本地客户端（iCloud 日历、Pocket）。

即便是那些做了个壳的客户端，体验也并算不好，Todoist Preview 有一个 UWP 版本的本地客户端，却总出一些排序混乱的低级错误。Slack 客户端整合良好，但初次进入总是要拖拖拉拉加载很久，实在是让人心累。

![](.evernote-content/F494C318-CA23-425F-8549-019AFC0E7DA8/447E4F85-E861-440E-AFDC-E9C3C8A187BC.png)

UWP 和 Slack 客户端

每当我遇到这些情况时，就不得不中断手中的工作，去搜一搜解决方案，改一改配置，甚至重启电脑才能解决，很影响心情不说，注意力也被分散了。

说到这里，就会有人提问了：「那为什么不用浏览器呢？」其实浏览器的问题就更多了，我随便就能列举出**许多使用浏览器的缺点** ——

**登录流程多，重复操作多**：面对那些没有客户端的在线工具，大多数人的选择通常是「打开一个浏览器页面，输入网址、通过导航页进入工具首页，然后输入密码登录，等待网页加载再进入工作」。如果只要进行一次这个流程倒还好说，当我们需要第二个服务时，就要重复上面这个操作：**打开页面、输入密码、等待登录**。重启电脑后，一些会自动下线的服务（iCloud、微博）都需要重新登录，令人不厌其烦。

![](.evernote-content/F494C318-CA23-425F-8549-019AFC0E7DA8/B2FFFB7A-CF97-4AFC-AE6C-EDD88106819C.png)

很容易「忘记」我的 Inoreader

**难以定位应用页面**：浏览器不会把应用和普通网页区分开。我们在工作时不可能只打开这几个工具页面，还会上网搜索、查找、阅读…… 不一会儿就打开了十几个标签页和好些个窗口。这时候想**回到最初的应用页面**，就要在拥挤的标签页角标中四处搜寻这个工具了。

![](.evernote-content/F494C318-CA23-425F-8549-019AFC0E7DA8/77383492-A6A7-4523-9118-14129F85BFBB.png)

我的 Slack 去哪儿了？

好在 Chrome 现在推出了创建桌面快捷方式功能，允许我们直接创建一个窗口来显示这些应用，这对这种情况有所缓解。

![](.evernote-content/F494C318-CA23-425F-8549-019AFC0E7DA8/29094461-9436-4047-A5CA-5E38136D8CC3.png)

Chrome 允许创建快捷方式

**没有基础的快捷键**：快捷键是本地客户端的一大特点，使用网页客户端的一个重要问题就是快捷键的缺失。比如我习惯用 Ctrl + Alt + W 来快速查看微信信息，但在网页版中就做不到这一点，其他的网页应用也是同理。

**缺乏提醒功能**：许多网页版应用的提醒功能做的很差，加之不同浏览器的提醒方式各异，无法确保用户及时收到信息。在浏览器中管理应用的通知也不方便，想在工作时间关闭 YouTube 的推送只能手动进入设置。Inoreader 这类更新密集的应用也只能完全禁用掉，不能选择在特定时间段内停用。

![](.evernote-content/F494C318-CA23-425F-8549-019AFC0E7DA8/183E10AE-521F-4C15-B8F4-8B5ABC59AF4D.png)

Chrome 的通知管理藏得很深

**内存管理一般**：这又是属于 Chrome 的一大难题，一次性打开大量的网页会占用许多内存。如果要确保能够持续接收应用提醒，就需要连同扩展插件让 Chrome 一直运行在后台。由于 Chrome 给每个标签页、插件都分配了进程，这样做的代价就是持续消耗内存和电量。标签页越多，内存占用就越高。

![](.evernote-content/F494C318-CA23-425F-8549-019AFC0E7DA8/1FB232EB-6B29-4F8C-A5BC-5AAB6267F3D5.png)

内存怪兽 Chrome

看看，这分散的纯网页应用给我们带来了多少麻烦事。

合格的网页聚合软件需要这些功能
---------------

为了提升网页应用体验，人们会想办法将多种工具聚集在一起统一操作，来实现更好的体验。这类软件我们统称为「网页聚合软件」。

其实，早在两年前已经有开发商抓住了商机，推出了对应的「网页聚合软件」来解决这个问题。少数派于 2016 年就曾在首页陆续推荐过 [Station](https://sspai.com/post/41572)、[Franz](https://sspai.com/post/34004)、[Rambox](https://sspai.com/post/39525) 这三款网页聚合软件，但由于当时网页应用并不十分丰富，大家对这类应用的需求并不突出，未能引起关注。

就在这短短两年间，各类纯网页应用如雨后春笋般冒出来。其中不仅有 Google Docs、Office Online、iCloud 等大厂领衔的产品，Slack、Trello、Notion 等新秀跟进，也不乏腾讯文档、Teambition、石墨、幕布等国内工具的身影。

![](.evernote-content/F494C318-CA23-425F-8549-019AFC0E7DA8/0F187F24-34BD-4A4F-96CF-AB21206C9277.png)

Station 支持的部分应用列表

同时，越来越多的开发商开始注重手机客户端的开发，设备之间的同步也变得愈发重要。为了避免同步、网络之类的麻烦，很多时候也只是推出网页版而不做 PC 客户端。

可以断言，**随着时间的推移，专心做本地软件的服务类应用只会越来越少**。是时候挑选一个好用的网页聚合软件了。

再把上面提到的问题总结一下，我们需要一款可以承载这些需求的软件：

1. 登录体验便捷
2. 窗口管理良好
3. 提醒功能完善
4. 内存占用率低

近期我在多款网页聚合软件之间做了对比，得出了一个结论：**Station 是网页聚合软件中的最佳选择**。

Station 是如何提升本地体验的
------------------

Station 的最基础功能，就是在一个界面中聚合多种常用的服务类工具，这并不少见，而且浏览器的普适性更高。

但 Station 的优势就在于它能够提供许多新鲜的本地化功能，包括但不限于 1Password 支持、通知推送管理、快速切换、快捷键、多页面等功能。**这在许多网页套壳应用，甚至是本地客户端中都是罕见的**。

我们这就来一一解决上面提到的问题。

### 登录体验便捷

使用这些软件的第一步都是注册账号，然后**添加网页应用**，再**登录网页应用**。把这个操作重复多遍，才能完成所有应用的添加。在这一点上，大部分网页聚合软件都是一样的。

接下来就出现了一些差异。Station 的两位竞争对手 Rambox 和 Franz 号称支持**多设备同步**。似乎换一台电脑就可以免去这麻烦的步骤了。

结果换了台电脑才发现完全不是这么回事，因为它们的同步，竟然只是**同步应用列表**而不同步密码，这就意味着更换一台设备后**仍然需要全部重新登录一遍**…… 完全没有简化任何步骤，这就是典型的「为了功能而功能」的表现。

尽管我理解这是出于安全的考虑，但这种体验实在是很差。

相比之下，Station 采用了内置 1Password 的方法，如果你订阅了 1Password，在初次连接时输入域名、邮箱、密钥、主密码，就可以享受所有服务自动填充密码的快感了。

![](.evernote-content/F494C318-CA23-425F-8549-019AFC0E7DA8/C4357891-9B66-46D6-AACD-75BDC01E11B1.png)

登录 1Password

### 窗口管理良好

在网页工具中经常需要进行页面跳转、窗口切换，这些都是考验本地优化的地方。

Station 在这方面做了四个主要改进：窗口集成、快捷键切换、下载、多窗口。

#### 窗口集成

同样 Notion 中点开一个新窗口链接，在 Franz 和 Rambox 中就会启动外部浏览器。而 Station 会做一个判断，如果链接仍然在该域名下，应用就会被分成两个标签页，标签页之间互不干扰。也可以直接 `Add a new page` 来添加一个新的窗口。

上面我们提到「浏览器不会把应用和普通网页区分开」「一个工具开好多窗口」，而在 Station 的这种嵌套结构下就不再是问题了。所有的应用只占一个位置，无论现在处于哪个页面、相同的工具无论有多少窗口，你都可以快速找到你想要的工具。

![](.evernote-content/F494C318-CA23-425F-8549-019AFC0E7DA8/DB5B5681-B627-43DA-BC39-B5D837FAECAE.png)

可以打开多个窗口

#### 快捷键切换

在实际切换应用时，Station 提供了一些额外的快捷键，有 Quick-Switch、Recent、指定应用、前后切换这 4 种方式来切换。这也是它在本地化方面的主要改进。

* 按下 `Ctrl + T` 进入 Quick-Switch 界面。
* 按下 `Ctrl + Alt + ← →` 方向键可以在左侧所有应用中上下切换。
* 按下 `Ctrl + 1-9` 可以直接进入左侧的 1 到 9 号应用。
* 按下 `Ctrl + Tab` 可以回到前一个应用，按住 `Ctrl` 后多次按下 `Tab` 可以像系统切换应用一样在最近使用的几个视图之间切换。

Quick-Switch 的界面有点类似 Spotlight，不仅支持快捷搜索，支持在应用之间切换，部分应用还可以精确到具体的页面，比如图中的 Notion 排期、会议记录，开头视频中的 Google Docs 等。

![](.evernote-content/F494C318-CA23-425F-8549-019AFC0E7DA8/682B49C6-1E21-427A-80F9-A31D53234FF3.png)

Quick-Switch

Station 中的其他快捷键可以按下 Ctrl + H 查看，包括页面缩放、刷新、窗口的操作，与 Chrome 的快捷键近似。有效降低了快捷键的学习成本。

![](.evernote-content/F494C318-CA23-425F-8549-019AFC0E7DA8/832642AA-B217-4C1B-962E-AC0EEEF5122C.png)

Station 支持的快捷键

#### 下载和多窗口支持

当你想同时查看两个工具时，也可以点击多窗口按钮让两个窗口并排。把特定工具分离出来，和正常浏览器一样显示多个页面。

![](.evernote-content/F494C318-CA23-425F-8549-019AFC0E7DA8/683D23F4-7819-4EE0-A6FA-E3B734FF3243.png)

多窗口支持

此外，Station 集成了下载管理，比如在 Slack 中下载多个文件，进度条会直接显示在右下角。目录可以在设置中自己定义，这些都是细节上的优化。

![](.evernote-content/F494C318-CA23-425F-8549-019AFC0E7DA8/E801C753-A225-4EDB-9392-A574A2A9108B.png)

下载进度条和提示

在对比过多款软件的窗口布局、操作逻辑之后，我发现 Station 不仅操作逻辑完整清晰，切换应用的快捷键也是最多的，应用的整体感很强。

### 提醒功能完善

看到这里，可能仍然有人认为「浏览器也能做到自动填充密码、多窗口、下载等功能，算不上 Station 的独特优势。」

这里就不得不提一下 Station 优化上做的另一大改进：提醒功能。

初次提醒时会请求是否同意接收提醒，后续使用时可以为每一个应用设置是否打开提醒推送。如果你正在应用中，还会出现一个悬浮窗状的提醒。

![](.evernote-content/F494C318-CA23-425F-8549-019AFC0E7DA8/154D9AE1-EC37-4823-9C3C-0AE20AC0DD70.png)

消息悬浮窗提醒

Station 采用了和 iOS 中通知中心的设计相似的提醒逻辑。应用内有一个专用的聚合通知中心，有「一段时间内勿打扰」功能，还能禁止特定应用发送通知、启用和关闭全局勿扰模式、通知一键已读…… 这些功能看似简单，但在浏览器中却很难做到这些。

![](.evernote-content/F494C318-CA23-425F-8549-019AFC0E7DA8/38AF782A-66B5-4396-8A00-563C21151CA7.png)

通知中心

不仅如此，Station 还同时接入了 Windows 和 macOS 中的系统提醒。我们终于可以像本地软件一样接收到来自网页应用的提醒了。而且提醒的完善程度简直让 Windows 汗颜（Windows 至今不支持定时勿扰）。

### 内存占用率低

这里还要提一下 Station 的内存占用问题。尽管 Station 采用了 Chromium 内核，但它不会一次打开所有的应用，而是在初次进入时对常用的应用进行预载。它也没有那些乱七八糟的插件来拖慢运行速度，所以能够有效降低网页应用的内存占用。

Station、Rambox、Franz 横向对比
-------------------------

前面讲的都是 Station 的独特优势，我们再从 **本地优化、应用数量、收费策略、外观对比** 这几个角度来看看 Station、Rambox、Franz 这三个应用之间的差距。

### 本地优化

本地优化对于一款网页聚合工具而言算是最重要的部分，**毕竟将应用从浏览器搬到聚合软件中的最大意义，就是为了从根本上解决浏览器体验差的问题**。

这三款软件的思路有些不同，分别在这些本地功能上做了特别优化 ——

[Station](https://getstation.com/) 支持通知管理、快速切换应用、关联 1Password 管理密码、查看内存占用等独特功能。

[Rambox](https://rambox.pro/#home) 免费版支持主密码保护、应用同步、允许自定义网页。Rambox 付费版提供了 Work Hours（特定时间提醒）、主题切换等功能，允许对网页做一些额外配置。

[Franz](https://meetfranz.com/) 免费版支持应用同步、社区插件。Franz 专业版提供拼写检查、自定义代理、内部部署等功能。

三款软件均推出了 Windows、macOS、Linux 客户端，在本地优化上它们有各自的特点，没有绝对的好坏。比如 Rambox 支持密码保护的特性更适合安全性要求高的用户。

![](.evernote-content/F494C318-CA23-425F-8549-019AFC0E7DA8/898EDE7E-F8DE-4DF2-A445-F99966D01BB5.png)

Station、Franz、Rambox 的功能和付费策略

### 应用数量

再看看支持的应用数量方面。

[Station 支持 500 多款网页应用](https://apps.getstation.com/)，不支持自定义，支持的应用数量是最多的，目前还在不断添加。特定网页的配置（比如 Google 类的服务只需登录一次）、通知功能都是最完善的。**可以通过提交申请来让官方添加应用适配**。

**手动添加任意网页的方法**：在 Station 中添加一个 Trello 页面，按下 `Ctrl + Alt + I` 打开页面调试工具，切换到 Console 控制台页面，输入 `this.document.location = 'https://www.weibo.com'`，可以让 Trello 页面变成微博页面，且每次点开时都会进入微博。此方法适用于大部分 Station 不提供的网页。（于 2019 年 2 月 23 日补充）

[Rambox 支持 150 多款应用](https://rambox.pro/#apps)，神奇的是，这些应用大多非常冷门…… 但支持自定义添加网页，理论上限可以超过 Station，添加自定义网页时的体验不够友好，需要自行添加图标，自定义网页的提醒抓取比较差。**好处是自定义的上限很高**，可以添加一些额外的配置代码。

[Franz 支持 85 款应用](https://meetfranz.com/services)，不支持自定义，自带应用的通知都能抓取的很好。

如果你经常需要使用国内的一些网页工具，这些工具中只有 Rambox 支持直接添加自定义网页。但自定义太多反而会模糊和浏览器之间的区别。像 Station 这样尽可能**主动提供并适配更多支持在线服务的工具**才是这类软件的本职工作。

注：这三款软件都支持微信网页版，但由于微信网页版自身的限制，不会同步历史信息。像 QQ、企业微信…… 这些连官方的网页端服务都没有做的软件就不用想了，老老实实用客户端吧。要注意的是，这并不是网页聚合软件本身的错。

### 收费策略

收费策略方面，[Station](https://getstation.com/)  **对个人用户完全免费**，将来可能会对企业用户收费。而 [Rambox](https://rambox.pro/#pricing) 分为社区版和 Pro 版（4 美元 / 月）。[Franz](http://www.meetfranz.com/) 分为免费版和 Premium 版（4 欧元 / 月），Franz 的免费版有一个隐藏限制，每次重新打开软件需要等待 10 秒。虽然是小事，但就像是视频前的 15 秒广告一样，总让人不太舒服。Station 再次占了上风。

### 外观对比

Station 的颜值也是这三款应用中最棒的，侧边栏的图标是统一风格的两色图标，设置界面、通知中心等界面 UI 都很现代风格。Franz 其次，支持 Dark Mode，整体画风也不错。而 Rambox 在这方面做的是最差的，初次进入差点以为是 UI 错位了，页面的布局也有些奇怪。

总结
--

[Station](https://getstation.com/) 是一款 Electron 应用，采用的是 Chromium 前端配合 Node.js 后端框架，是基于 Chromium 的深度优化应用。尽管这类应用的主体都是浏览器，但唯独 Station 在通知、切换、窗口管理等诸多功能上改进了使用体验，而不是简单的把一些工具放在一个「壳」里。确实称得上是一款「网页聚合工作站」。

Station 给我们展示了什么才能算是「制作精良的套壳应用」，它在同类应用中的优势称得上是碾压级的，是我心目中最理想的网页聚合软件。
