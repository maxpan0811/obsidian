Mac 上高质量的开源应用，你可以在这里一次找到

有过几次，我想把 Mac 上的所有第三方应用都换成开源的，似乎开源的东西就是我可控的。其实不然。多数人真正喜欢开源软件的原因，其实倒不在于可以随意修改它们的代码，而是这类软件在其他方面提供给我们的自由与便利：

* 花费较少：多数是免费使用的。
* 持续更新：即使开发者不管了，社区也能继续提供维护软件。
* 安装方便：可以通过 HomeBrew 统一管理，甚至能实现实现 [一键装机](https://sspai.com/post/43265)。

正是这些人人都可以享受到的便利，才让开源软件日益受到普通用户的欢迎。恰好最近接触到一个 [GitHub 项目](https://github.com/serhii-londar/open-source-mac-os-apps#system) ，它收集了 macOS 上大量好用的开源应用，所以在这篇文章里，我也从中挑一些和你分享，同时谈谈自己对于开源软件的看法。

注：为尊重开源社区的习惯，本文使用「软件（software）」的称呼。

如何获得开源软件
--------

很多时候，我们想，都不知道有没有开源软件来解决。好在有人帮忙整理了优质开源软件列表，@scomper 的《[GitHub 中那些不错的免费软件](https://sspai.com/post/41325)》中就有不少好软件，而 GitHub 还有人整理了专门的 macOS 开源软件项目 [serhii-londar/open-source-mac-os-apps:](https://github.com/serhii-londar/open-source-mac-os-apps#system) 。

![](.evernote-content/3E5B37FC-7926-4B05-A3BB-FDF1AB059D87/CC473035-F7E4-45AF-A8C8-58C8AD432AB2)

macOS 开源项目

除了这些比较系统的文章和项目，不少论坛里也有人在做收集和分享的工作。在看准了一款软件后，我们也用不着去它们的官网上下载，可以直接通过 [HomeBrew](https://brew.sh/) 来安装，以后管理、升级和卸载起来也方便（天知道这些软件会把配置文件藏在哪个文件夹，还是把这些烦人的事情交给 HomeBrew 吧）。

关于 HomeBrew 的使用，可以关注少数派的文章和专题：

* 文章：[9 条进阶命令，把 HomeBrew 打造成管理第三方应用的 App Store](https://sspai.com/post/43451)
* 专题：[Homebrew，Mac 应用管家](https://sspai.com/topic/181)

推荐一些 Mac 上的开源软件
---------------

其实，我现在的电脑上，大半的第三方软件都是开源的，尤其是一些用不着「云服务」的工具类软件，我更愿意用开源作品代替其它的商业产品。下面，我就推荐一些 Mac 上好用的开源软件。

### Mac 上最好的视频播放器：IINA

这几年，Mac 上的老牌开源播放器 Mplayer、Mplayer X 都慢慢跟不上系统的更新，好在后起之秀 [IINA](https://github.com/lhc70000/iina) 很快填补上了播放器领域的空缺。

![](.evernote-content/3E5B37FC-7926-4B05-A3BB-FDF1AB059D87/C7FEBA4F-3F23-46F0-9F0A-75DC46A3008C)

IINA 的播放界面

IINA 灵活运用模糊、透明元素打造了现代风格的界面，还支持双指缩放调整窗口大小、滑动调整进度等手势操作，加上在线字幕、自定义快捷键等功能，让观影多了一分享受。

拓展阅读：[macOS 上最好的现代视频播放器：IINA](https://github.com/HandBrake/HandBrake)

### 轻量笔记工具：Simplenote

[Simplenote](https://github.com/Automattic/simplenote-macos) 给人的印象似乎只有一个词：极简。

![](.evernote-content/3E5B37FC-7926-4B05-A3BB-FDF1AB059D87/E96E5A7F-A606-4FDE-ABC4-5B0391FA3204)

采用黑暗模式的 Simplenote

保持简单界面的同时，Simplenote 支持轻量的标记语法，也有时下流行的黑暗模式，还可以和朋友分享笔记，对于记文字笔记为主的用户来说还能在一定程度上代替印象笔记。

这个看起来像出自「小作坊」的笔记工具，和世界上最大的建站工具 [WordPress](https://wordpress.com/create/?sgmt=gb&utm_source=adwords&utm_campaign=Google_WPcom_Search_Brand_Desktop_JP_en&utm_medium=cpc&keyword=wordpress&creative=263011219574&campaignid=654740795&adgroupid=54980746720&matchtype=e&device=c&network=g&targetid=kwd-313411415&locationid=2392&gclid=EAIaIQobChMIrcDll_yJ3AIVwjUrCh1-Bg3BEAAYASAAEgK_v_D_BwE) 师出同门，所以倒不用担心哪天它倒闭、自己的数据就没了。

### 极限视频压缩工具：HandBrake

辛辛苦苦剪辑好的小视频，上传时发现体积过大，或者格式不支持，真是一件令人崩溃的事情。好在你用不着购买价格高昂的专业工具，也不用学习复杂的命令行，直接用 [HandBrake](https://github.com/HandBrake/HandBrake) 就能事先视频文件的压缩，在尽可能保持画质的情况下把 iPhone 视频、Mac 录屏压缩到原来的十分之一还不到。

![](.evernote-content/3E5B37FC-7926-4B05-A3BB-FDF1AB059D87/A286D6FA-2987-4337-A20B-EF23025556B9)

视频压缩界面

值得高兴的是，最新版本的 HandBrake 还支持 4K 视频的压缩了，并且覆盖了 H.265 格式，iPhone 拍的视频不需要事先转换格式就能直接处理。

### Photoshop 免费替代品：GIMP

虽说市面上 Photoshop 的替代品不少，但许多人可能用不上那么多的功能，甚至，不少人告诉我他们只需要一个魔棒来扣图。对于没有专业需求人来说，免费的 [GIMP](https://github.com/GNOME/gimp) 是更合适的选择。

![](.evernote-content/3E5B37FC-7926-4B05-A3BB-FDF1AB059D87/6541DE73-B47B-4137-A409-07BA139DE645)

GIMP 界面

如果你恰好比较喜欢折腾，也可以玩玩看 GIMP 的插件（用 Python 写的，有基础的话还可以自己开发），来实现个性化滤镜等功能。

### 压缩

平时我经常和压缩文件打交道，所以也做了不少相关的 [自动化工具](https://sspai.com/post/40506)，但如果要我推荐一个解压缩工具，我还是更推荐 [The Unarchiver](https://bitbucket.org/kosovan/theunarchiver)，毕竟省心，一步到位。

除了支持格式全、速度快这些硬本事，和其他解压工具相比，The Unarchiver 最独特的地方就在于其**存在感低**，设置为默认解压工具后，双击压缩包、不打开 The Unarchiver 主程序就能进行解压。

![](.evernote-content/3E5B37FC-7926-4B05-A3BB-FDF1AB059D87/EC5E22EB-4A7F-40D0-874C-94D918BA204C)

双击解压

### 改键大师：Karabiner-Elements

什么人需要修改键盘按键的功能？相信我，不单单是那些电脑极客，每个人都可以通过定制自己的键盘来获得更好的输入体验——因为许多键盘的设计实在太糟糕了，除了被程序员和写作者吐槽已久的 `⇪Caps lock` 键，MacBook 上那颗总是失灵的 `⎋Esc`（蝶式键盘的锅）键也让人抓狂。

所幸，我们可以用 [Karabiner-Elements](https://github.com/tekezo/Karabiner-Elements) 来改变按键的功能。除了笔记本电脑的内置键盘，外接键盘 Karabiner-Elements 同样能定制，你可以把它的键位调整得和 Mac 更接近，让设备来适应你，而不一定要改变原先用的舒舒服服的输入习惯。

![](.evernote-content/3E5B37FC-7926-4B05-A3BB-FDF1AB059D87/7BFE6BBA-CCEC-4116-A991-97057EB87FA1)

把外接键盘的键位修改得和 MacBook 内置键盘一样

拓展阅读：[让键盘变成你想要的样子：改键利器 Karabiner-Elements](https://sspai.com/post/42921)

结语
--

开源软件因免费，但是它多少有点个人英雄主义的色彩，不少知名项目的关键成员退出后，软件就陷入停滞状态；经历 macOS 几个系统迭代，这些老项目往往就死亡了。

在享受开源软件所带来便利的同时，我也鼓励大家参与一些开源项目，它们不一定都深奥难懂，有的只是简单的脚本（比如我的 [自动化脚本库](https://github.com/BlackwinMin/sspai-sample-script)），稍有折腾意愿的人都可以参与维护，让这些开源项目生长得更健康长久。
