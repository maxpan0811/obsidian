Windows 10 也有「官方」取色工具了：PowerToys 0.20 更新详解
==========================================

![](.evernote-content/C12B4D12-D569-4691-87F3-3FF05D3ACF29/E1C7020D-1B48-4704-A43A-2C562CA96533)

Windows 10 也有「官方」取色工具了：PowerToys 0.20 更新详解

去年微软针对 Windows 10 系统重启了 PowerToys 工具，并且针对 Windows 10 系统特性推出了一系列的小工具来改善 Windows 10 的易用性。

时隔一年后的今天，这款工具终于推出了 0.20 版，带来一系列实用新特性的同时，也向我们展示了不少未来的开发计划和愿景。

![](.evernote-content/C12B4D12-D569-4691-87F3-3FF05D3ACF29/436E20AE-725E-4238-BCA7-1D29791BCF24)

0.20 版本主界面

**首先聊聊新加入的取色工具**。

虽然在此之前也有类似的工具提供了取色工具，但这些工具要么需要单独启动，要么需要和其他软件配合使用，过程繁琐而且流程很长。比如我此前的取色工作流是：打开 Adobe XD，使用屏幕截图工具截去需要取色的区域，然后将截图贴到 adobe XD 的画布上，新建一个矩形框后再用取色器取色，过程繁琐而且效率极低。

PowerToys 在 v0.20 版本中加入的 Color Picker，则极大地缩短了整个取色需求的操作流程。

在 Color Picker 中，我们可以设置取色后的数值是标准的 HEX 或直接转换后的 RGB 数值，调用取色工具的方式和其他 PowerToys 工具类似，按下键盘快捷键 `Win+Shift+C` 后屏幕中就会出现取色坐标，移动鼠标找到需要取色的位置，鼠标左键点击之后就可以一键将提取到的色彩数值拷贝到剪贴板中。

![](.evernote-content/C12B4D12-D569-4691-87F3-3FF05D3ACF29/B710F11A-6D06-405F-B6D8-1D76D049A165.gif)

值得一提的是，为了方便一些比较细致的取色场景，在使用 PowerToys 取色工具进行去色的过程中，我们还可以像上图那样使用滚动操作来对指针区域进行放大。

**针对资源管理器的更新也值得一提。**

PowerToys 在此前的某次 [更新](https://sspai.com/post/59884) 中为 Windows 用户带来了原生的 SVG 图形和 Markdown 文件预览支持，但彼时我们还只能通过资源管理器的「预览窗格」进行查看，对于喜欢简洁资源管理器布局的用户来说不太友好。

所以 PowerToys 0.20 版本进一步为 SVG 图形带来了更为直观易用的缩略图预览，安装更新后重启资源管理器，然后我们就能像浏览常见图片文件那样以图标缩略图的形式预览 SVG 的图形内容。这个功能非常适合互联网从业者，在需要对大量的 SVG 图形进行管理的时候非常方便。

![](.evernote-content/C12B4D12-D569-4691-87F3-3FF05D3ACF29/F81ADEC5-BC5A-4E94-AECA-B5293BBE3DC9)

有了图标缩略图预览，预览窗格就显得没那么必要了

**此外，一直以来都称得上是 PowerToys 特色功能的 FancyZones 也带来了一项非常实用的改进**。

在 Windows 平台上，熟悉快捷键操作的用户对按住 Ctrl 进行「多选」这个操作应该也并不陌生——而在使用 FancyZones 的过程中，我们也可以在原有的窗口吸附操作 `Shift+窗口拖拽` 的基础之上同时按住 Ctrl 键进行多选了，即 `Ctrl+Shift+窗口拖拽`，来让被拖拽的窗口同时扩充至两个甚至多个窗口区域大小。

![](.evernote-content/C12B4D12-D569-4691-87F3-3FF05D3ACF29/3003E4A7-D3AF-4911-B180-FAD713B56B3E)

以上图展示的窗口为例，在设定好窗口管理模板后，以往我们只能通过按住 Shift 键将图中的命令行窗口吸附至 1、2、3 号窗口中的任意一个；更新至 0.20 后，我们就可以根据实际需求，同时按住 Ctrl 键来将其直接吸附至 1、3 窗口所在的整个区域大小，免去了手动切换窗口布局的麻烦。

其他小工具方面，快速启动器 PowerToy Run 在本次更新后拥有了更好的键盘交互表现，更重要的是终于可以实时检测到新安装的应用了。

![](.evernote-content/C12B4D12-D569-4691-87F3-3FF05D3ACF29/DA85AEAA-1608-4115-B2F9-BFA53C4F1A61)

快捷键管理器则加入了应用级快捷键支持，现在可以针对某一款软件单独进行快捷键重映射了。

![](.evernote-content/C12B4D12-D569-4691-87F3-3FF05D3ACF29/E6B28E56-4026-4C36-A0E5-3092CA9D29DA)

虽然复活这一年时间里，PowerToys 的版本号依旧停留在 0.x，软件名称则一直挂着 Preview 后缀，但事实上它的功能越来越多也越来越完善，距离 1.0 正式版发布已经近在咫尺。

从目前开发社区中的讨论来看，接下来 PowerToys 除了功能稳定性改进之外，可能会牵涉到的主要调整包括：

首先是 PowerToy Run 目前占用的 `Alt+Space` 的快捷键可能未来会有所调整，比如调整到 `Win+R` 用来替代系统原有的「启动」功能，或使用 `Win+Space` 来唤醒——后者虽然与输入法开关冲突，考虑到 PowerToys 的官方背景，直接替代系统功能而非选择 `Alt+Space` 这种可能会被第三方系统快捷键占用的快捷按键更加合适。

![](.evernote-content/C12B4D12-D569-4691-87F3-3FF05D3ACF29/C5EE22DE-3767-4246-8411-A69C624233C3)

未来（可能的）PowerToys Run 设计

另一个讨论到的、可能的更新内容则和界面有关，包括为目前还有些简陋的 Shortcut Guide 界面套上 Fluent 设计，新设计预计还将进一步与当前应用本身的功能挂钩，在应用的下方显示相关的快捷键提示，比起之前的 Shortcut Guide 提示无论是视觉上还是实用度上都会更高。

以上两个更新计划目前都处于讨论过程中，未来这款来自官方的实用小工具还将为我们带来哪些惊喜，让我们拭目以待吧。
