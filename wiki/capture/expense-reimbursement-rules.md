## 目的地城市归因规则

每张大交通票据（机票/高铁）定义一个目的地城市。滴滴按实际发生地归因：

1. **大交通** → 到达城市 = 目的地
2. **滴滴"去站点"**（出发地到机场/高铁站）→ 归**出发城市**（实际发生地）
3. **滴滴"出站点"**（机场/高铁站到目的地）→ 归**到达城市**（实际发生地）
4. **非出行日的滴滴** → 归入当时所在城市
5. **住宿费** → 归入停留城市

## 滴滴发票地点提取

滴滴电子发票 PDF 的文本结构固定（PyMuPDF 提取）：
- 行 ~40-46：表头（出行人、有效身份证件号、出行日期、出发地、到达地、等级、交通工具类型）
- 行 ~47-52：数据行
- 行 49：`YYYY-MM-DD` 格式的出行日期（唯一匹配 `^\d{4}-\d{2}-\d{2}$` 的行）
- 行 50：出发地
- 行 51：到达地
- 行 55：金额（`（小写）` 标签后的行，匹配 `^\d+\.\d{2}$`）

城市判断 key：工体/马家堡→北京，三林/虹桥/浦东→上海，双流/武侯/吉泰→成都，高崎→厦门

## 发票分类

| 一级 | 发件人/文件名特征 |
|------|-----------------|
| 大交通-机票 | 携程 `机票预订出行订单尾号XXXX XXXX-电子普通发票.pdf` |
| 大交通-高铁 | `2611\d+.pdf` / `2631\d+.pdf`（铁路电子客票号） |
| 市内交通 | `滴滴电子发票*.pdf` |
| 住宿费 | 酒店名+晚数 `成都亚朵酒店X晚住宿发票.pdf` |

## 排除规则

- **滴滴出行行程报销单** ≠ 发票，只取滴滴电子发票
- `_1.pdf` 后缀的携程机票发票 = 重复下载，跳过
- 同订单的 .pdf/.ofd/.xml 三格式只取 PDF

## 邮件下载（QQ邮箱）

```bash

```

之前用 `flag set <ID> +seen` 会替换所有 flags 导致失败，应用 `flag add`。

## 输出文件

- 按城市分类：`~/Downloads/YYYY年M月出差报销明细_按城市分类.xlsx`
- 两个 Sheet：按目的地城市（发票级明细）+ 行程时间线（汇总）
- 备注列 = 真实文件名

## 往返地址标注格式

`城市·起点 → 终点（用途）`，如：
- `北京·工体→首都机场（去机场）`
- `上海·虹桥T2→三林（机场到家）`
- `成都·双流T2→武侯区高新（机场到酒店）`
