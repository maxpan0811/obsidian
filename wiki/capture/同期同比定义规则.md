# 同期同比定义规则（2026-07-18 确认）

## 定义
**同期同比** = 26年同期 / 25年同期 - 1

其中：
- **26年同期**：26年7月16号之前预定的 → 26年全年出发的产量
- **25年同期**：25年7月16号之前预定的 → 25年全年出发的产量

## 关键区别
- **同期同比**：26年同期 vs 25年同期（都是截至7月16号预定的全年产量）
- **不是**：26年同期 vs 25年全年（这样会高估或低估增长）

## 应用场景
- 携程 vs 非携程渠道对比表格中的"同比"列
- 分公司明细表格中的"同比"列
- 全国参考值中的"同比"
- 所有需要计算同比增长的场景

## 计算公式
```
同期同比 = (26年同期产量 - 25年同期产量) / 25年同期产量
```

## 代码实现
在export_desktop.py中：
- `prod26`：26年同期产量（截至7月16号预定的26年全年产量）
- `prod25`：25年同期产量（截至7月16号预定的25年全年产量）
- `prod26_full`：26年全年产量（不用于同期同比计算）
- `prod25_full`：25年全年产量（不用于同期同比计算）

```python
# 同期同比计算
yoy = (prod26 - prod25) / prod25
```

## 常见错误
❌ 错误：用26年同期除以25年全年
✅ 正确：用26年同期除以25年同期

## 为什么重要
因为7月16号是一个重要的预定截止点，在这个时间点之前预定的订单更能反映市场趋势和竞争态势。使用同期同比可以消除季节性因素的影响，更准确地比较不同年份的业务表现。
