# 数据核验协议：逐字段逐值列出

## 触发条件

用户说"数据有出入"时，不要先辩解或猜口径，直接按以下步骤走。

## 步骤

1. **列字段清单** — 每列写清 Excel 列号 + 字段名 + 筛选条件 + 值
2. **让用户确认** — 用户可能会说"这个列不用""那个条件换一下"
3. **修正后重跑** — 差一个条件可能差几千万
4. **定位差值** — 目标值对不上时，逐省/逐维度减法定位差额来源

## 反例

❌ 用户说 3.93 亿不对，AI 回复"看板是 4.45 亿，口径可能不同"——没列字段，没排查就下结论
❌ 用户说"选考核区域=欧洲，不是归属板块"，AI 重跑发现 Col1=Col2 没有差异就以为没问题——实际是用户漏了天津

## 正确示范

✅ 用户说 3.93 亿/2.04 亿跟看板对不上 → 列出 5 字段 × 条件的矩阵 → 用户纠正"考核区域=欧洲"→ 重跑还是 4.45 → 逐省减法定位天津 0.52 亿 → 完全吻合

**Why**：差异核对的瓶颈从来不是计算能力，是口径对齐。跳过字段清单直接算数 ≈ 用错误口径证明自己正确。

**How to apply**：任何数据看板类任务，用户质疑数据时，第一反应不是"我再算一遍"，而是"我把用到的字段列给你看"。
