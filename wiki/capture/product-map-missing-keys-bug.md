# PRODUCT_MAP 缺键 bug

## 问题
`~/.claude/skills/非携程渠道达成率分析工具/scripts/export_desktop.py` 的 `PRODUCT_MAP` 缺少：
- `'澳新': '澳新'` — 实际数据的板块值就是 `澳新`，不是 `大洋洲`
- `'中东非': '中东非'` — 实际数据的板块值就是 `中东非`，不是 `中东` 或 `非洲`

**后果**：所有澳新和中东非的订单在分公司维度被跳过，华东非携程少了 ~1亿。

## 正确映射
```python
PRODUCT_MAP = {
    '欧洲': '欧洲',
    '美洲': '美洲', '中南美': '美洲', '北美洲': '美洲',
    '澳新': '澳新', '大洋洲': '澳新',
    '中东非': '中东非', '中东': '中东非', '非洲': '中东非',
    '中亚+高加索+蒙古': '中亚高加索', '高加索': '中亚高加索',
    '亚洲': '中亚高加索', '日韩': '中亚高加索', '东南亚': '中亚高加索', '海岛': '中亚高加索',
}
```

## 验证方法
用 `Counter` 统计实际文件中 `目的地归属业务板块` 的所有值，确保每个值都在 PRODUCT_MAP 中有映射。

**How to apply**：任何新增涉及目的地归属业务板块的脚本，必须先 `Counter` 枚举实际值再写映射。不要猜测板块名。
