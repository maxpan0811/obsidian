## v2 修正：两阶段匹配

### 问题发现（2026-07-12 Hy 指出）
旧匹配仅做关键词交集，忽略品牌体系。67条匹配中35条（52%）是错配：
- `渝之旅九龙坡区步行街二门市部` → 错配到`重庆去哪儿九龙坡区杨家坪正街门市部门市`（仅靠"九龙坡区""杨家坪"地点关键词撞上）
- `海外五小区门市` → 错配到`百事通`（不含携程系品牌）
- 联盟名中大量非携程系门店（渝之旅/海外/乐游/中旅/全球通/乐畅等）被误归入联盟营收

### 修正方案：两阶段匹配

```
阶段1 — 品牌体系门禁（新增）
  联盟条目中扫描携程系品牌关键词 → 判断是否放行
  携程系品牌 = [携程, 百事通, 旅游百事通, 去哪儿]
  
  ✅ 含任一 → 放行，进入阶段2
  ❌ 不含 → 标记「未匹配（非携程系XX）」，终止
  注：海外/海外旅业 NOT mapped to 百事通（Hy 2026-07-12确认严格匹配）

阶段2 — 地点关键词交集匹配（沿用现有逻辑）
  提取双方独有地点关键词（3-6字，剔除通用词）
  → 取交集 → 按长度×2 + 数量×10 + 长词×15 加权评分
  → score ≥ 20 → Top 60% → 去重输出 Top 3
```

## How to apply:
类似联盟门店→订单系统匹配的分析，先做品牌门禁判定，再做地点关键词匹配。品牌门禁的关键词列表根据订单系统的实际品牌种类配置，不可映射的品牌（如渝之旅→百事通）需要明确决策。

**Why:** 订单系统仅包含携程系门店（携程/百事通/去哪儿），联盟名单包含大量非携程系品牌。地点关键词交集在品牌兼容的前提下可准确匹配，但品牌冲突时即使地点名相同也不能算同一个门店。
