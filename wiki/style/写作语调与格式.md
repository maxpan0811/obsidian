# 写作语调与格式

## 语调

- **技术文档**：客观、准确、简洁。不写"你可以看到"、"你会发现"
- **分析文档**：结论先行（KPI Summary → 趋势 → 具体数据 → 建议）
- **操作文档**：祈使句（"打开"→"点击"→"保存"），每步附带预期结果

## 格式规则

| 场景 | 格式 |
|------|------|
| 第一次出现的术语 | 加粗 + 链接到 glossary |
| 系统字段名 | `代码块` |
| 界面菜单路径 | → 分隔（设置 → 产品管理 → 编辑） |
| 警告/注意 | > **注意**: ... |
| 命令示例 | ```bash 代码块 |
| 版本号 | vX.Y.Z 格式 |

## 对比

| 书面语 | 口语 |
|--------|------|
| 该系统支持三种模式 | 这个系统有三种模式可以用 |
| 建议定期执行 Lint | 记得隔段时间跑一下 Lint |
| 该参数默认值为 10 | 这个参数默认是 10 |
